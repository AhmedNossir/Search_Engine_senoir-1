        <h1 class="border-bottom-0">Managing allowed IP addresses for your organization</h1> 
        <div class="d-none d-lg-block ml-2"> <button class="btn-link Link--secondary js-print tooltipped tooltipped-n" aria-label="Print this article"> <!-- From https://heroicons.dev/ --> 
          <svg fill="none" height="18" width="18" stroke-linecap="round" stroke-linejoin="round" stroke-width="2" viewbox="0 0 24 24" stroke="currentColor">
           <path d="M17 17h2a2 2 0 002-2v-4a2 2 0 00-2-2H5a2 2 0 00-2 2v4a2 2 0 002 2h2m2 4h6a2 2 0 002-2v-4a2 2 0 00-2-2H9a2 2 0 00-2 2v4a2 2 0 002 2zm8-12V5a2 2 0 00-2-2H9a2 2 0 00-2 2v4h10z"></path>
          </svg> </button>

       </div> 
       <div class="lead-mktg">
        <p>You can restrict access to your organization's assets by configuring a list of IP addresses that are allowed to connect.</p>
       </div> 
       <div class="product-callout border rounded-1 mb-4 p-3 color-border-success color-bg-success"> 
        <p>Allowed IP addresses are available with GitHub Enterprise Cloud and GitHub AE. For more information, see "<a href="/en/articles/github-s-products">GitHub's products</a>."</p>

        <h2 id="in-this-article" class="f5 mb-2"><a class="Link--primary" href="#in-this-article">In this article</a></h2> 
        <ul class="list-style-none pl-0 f5 mb-0"> 
         <li class="ml-0  mb-2 lh-condensed"><a href="#about-allowed-ip-addresses">About allowed IP addresses</a></li> 
         <li class="ml-0  mb-2 lh-condensed"><a href="#adding-an-allowed-ip-address">Adding an allowed IP address</a></li>

         <li class="ml-0  mb-2 lh-condensed"><a href="#deleting-an-allowed-ip-address">Deleting an allowed IP address</a></li>

      <div id="article-contents" class="article-grid-body"> 
       <p>Organization owners can manage allowed IP addresses for an organization.</p> 
       <h3 id="about-allowed-ip-addresses"><a href="#about-allowed-ip-addresses">About allowed IP addresses</a></h3> 
       <p>You can restrict access to organization assets by configuring an allow list for specific IP addresses. For example, you can allow access from only the IP address of your office network. The allow list for IP addresses will block access via the web, API, and Git from any IP addresses that are not on the allow list.</p> 
       <p>You can approve access for a single IP address, or a range of addresses, using CIDR notation. For more information, see "<a href="https://en.wikipedia.org/wiki/Classless_Inter-Domain_Routing#CIDR_notation">CIDR notation</a>" on Wikipedia.</p> 
       <p>To enforce the IP allow list, you must first add IP addresses to the list, then enable the IP allow list. You must add your current IP address, or a matching range, before you enable the IP allow list.</p> 
       <p>You can also configure allowed IP addresses for the organizations in an enterprise account. For more information, see "<a href="/en/github/setting-up-and-managing-your-enterprise/enforcing-security-settings-in-your-enterprise-account#managing-allowed-ip-addresses-for-organizations-in-your-enterprise-account">Enforcing security settings in your enterprise account</a>."</p> 
       <h3 id="adding-an-allowed-ip-address"><a href="#adding-an-allowed-ip-address">Adding an allowed IP address</a></h3> 
       <ol> 
        <li> <p>In the top right corner of GitHub, click your profile photo, then click <strong>Your organizations</strong>. <span class="procedural-image-wrapper"><img src="/assets/images/help/profile/your-organizations.png" alt="Your organizations in the profile menu"></span></p> </li> 
        <li> <p>Click <strong>Settings</strong> next to the organization. <span class="procedural-image-wrapper"><img src="/assets/images/help/organizations/settings-button.png" alt="The settings button"></span></p> </li> 
        <li> <p>In the left sidebar, click <strong>Organization security</strong>.</p> <p><span class="procedural-image-wrapper"><img src="/assets/images/help/organizations/org-security-settings-tab.png" alt="Organization security settings"></span></p> </li> 
        <li> <p>Under "IP Address", type an IP address, or range of addresses, in CIDR notation. <span class="procedural-image-wrapper"><img src="/assets/images/help/security/ip-address-field.png" alt="Key field to add IP address"></span></p> </li> 
        <li> <p>Under "Description", type a description of the allowed IP address or range. <span class="procedural-image-wrapper"><img src="/assets/images/help/security/ip-address-name-field.png" alt="Key field to add name for IP address"></span></p> </li> 
        <li> <p>Click 
          <svg version="1.1" width="16" height="16" viewbox="0 0 16 16" class="octicon octicon-plus" aria-label="The plus icon" role="img">
           <path fill-rule="evenodd" d="M7.75 2a.75.75 0 01.75.75V7h4.25a.75.75 0 110 1.5H8.5v4.25a.75.75 0 11-1.5 0V8.5H2.75a.75.75 0 010-1.5H7V2.75A.75.75 0 017.75 2z"></path>
          </svg> <strong>Add</strong>. <span class="procedural-image-wrapper"><img src="/assets/images/help/security/new-allowlist-entry-button.png" alt="Add allowed ip address button"></span></p> </li> 
       </ol> 
       <h3 id="enabling-allowed-ip-addresses"><a href="#enabling-allowed-ip-addresses">Enabling allowed IP addresses</a></h3> 
       <ol> 
        <li> <p>In the top right corner of GitHub, click your profile photo, then click <strong>Your organizations</strong>. <span class="procedural-image-wrapper"><img src="/assets/images/help/profile/your-organizations.png" alt="Your organizations in the profile menu"></span></p> </li> 
        <li> <p>Click <strong>Settings</strong> next to the organization. <span class="procedural-image-wrapper"><img src="/assets/images/help/organizations/settings-button.png" alt="The settings button"></span></p> </li> 
        <li> <p>In the left sidebar, click <strong>Organization security</strong>.</p> <p><span class="procedural-image-wrapper"><img src="/assets/images/help/organizations/org-security-settings-tab.png" alt="Organization security settings"></span></p> </li> 
        <li> <p>Under "IP allow list", select <strong>Enable IP allow list</strong>. <span class="procedural-image-wrapper"><img src="/assets/images/help/security/enable-ip-allowlist-organization-checkbox.png" alt="Checkbox to allow IP addresses"></span></p> </li> 
        <li> <p>Click <strong>Save</strong>.</p> </li> 
       </ol> 
       <h3 id="editing-an-allowed-ip-address"><a href="#editing-an-allowed-ip-address">Editing an allowed IP address</a></h3> 
       <ol> 
        <li> <p>In the top right corner of GitHub, click your profile photo, then click <strong>Your organizations</strong>. <span class="procedural-image-wrapper"><img src="/assets/images/help/profile/your-organizations.png" alt="Your organizations in the profile menu"></span></p> </li> 
        <li> <p>Click <strong>Settings</strong> next to the organization. <span class="procedural-image-wrapper"><img src="/assets/images/help/organizations/settings-button.png" alt="The settings button"></span></p> </li> 
        <li> <p>In the left sidebar, click <strong>Organization security</strong>.</p> <p><span class="procedural-image-wrapper"><img src="/assets/images/help/organizations/org-security-settings-tab.png" alt="Organization security settings"></span></p> </li> 
        <li> <p>Under "IP allow list", to the right of the entry you want to edit, click <strong>Edit</strong>. <span class="procedural-image-wrapper"><img src="/assets/images/help/security/ip-address-edit-button.png" alt="Edit allowed IP address button"></span></p> </li> 
        <li> <p>Type an IP address, or range of addresses, in CIDR notation. <span class="procedural-image-wrapper"><img src="/assets/images/help/security/ip-address-edit-field.png" alt="Key field to add IP address"></span></p> </li> 
        <li> <p>Type a description of the allowed IP address or range. <span class="procedural-image-wrapper"><img src="/assets/images/help/security/ip-address-edit-name-field.png" alt="Key field to add name for IP address"></span></p> </li> 
        <li> <p>Click <strong>Update</strong>.</p> </li> 
       </ol> 
       <h3 id="deleting-an-allowed-ip-address"><a href="#deleting-an-allowed-ip-address">Deleting an allowed IP address</a></h3> 
       <ol> 
        <li> <p>In the top right corner of GitHub, click your profile photo, then click <strong>Your organizations</strong>. <span class="procedural-image-wrapper"><img src="/assets/images/help/profile/your-organizations.png" alt="Your organizations in the profile menu"></span></p> </li> 
        <li> <p>Click <strong>Settings</strong> next to the organization. <span class="procedural-image-wrapper"><img src="/assets/images/help/organizations/settings-button.png" alt="The settings button"></span></p> </li> 
        <li> <p>In the left sidebar, click <strong>Organization security</strong>.</p> <p><span class="procedural-image-wrapper"><img src="/assets/images/help/organizations/org-security-settings-tab.png" alt="Organization security settings"></span></p> </li> 
        <li> <p>Under "IP allow list", to the right of the entry you want to delete, click <strong>Delete</strong>. <span class="procedural-image-wrapper"><img src="/assets/images/help/security/ip-address-delete-button.png" alt="Delete allowed IP address button"></span></p> </li> 
        <li> <p>To permanently delete the entry, click <strong>Yes, delete this IP allow list entry</strong>. <span class="procedural-image-wrapper"><img src="/assets/images/help/security/perm-delete-allowlist-entry-button.png" alt="Permanently delete IP allow list entry button"></span></p> </li> 
       </ol> 
       <h3 id="using-github-actions-with-an-ip-allow-list"><a href="#using-github-actions-with-an-ip-allow-list">Using GitHub Actions with an IP allow list</a></h3> 
       <div class="extended-markdown warning border rounded-1 mb-4 p-3 color-border-danger color-bg-danger f5"> 
        <p><strong>Warning</strong>: If you use an IP allow list and would also like to use GitHub Actions, you must use self-hosted runners. For more information, see "<a href="/en/actions/automating-your-workflow-with-github-actions/about-self-hosted-runners">Hosting your own runners</a>."</p> 
       </div> 
       <p>To allow your self-hosted runners to communicate with GitHub, add the IP address or IP address range of your self-hosted runners to the IP allow list. For more information, see "<a href="#adding-an-allowed-ip-address">Adding an allowed IP address</a>."</p>
