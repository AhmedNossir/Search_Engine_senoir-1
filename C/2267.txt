       <h3 id="about-management-of-security-and-analysis-settings"><a href="#about-management-of-security-and-analysis-settings">About management of security and analysis settings</a></h3> 
       <p>GitHub can help secure the repositories in your organization. You can manage the security and analysis features for all existing or new repositories that members create in your organization. If you have a license for GitHub Advanced Security then you can also manage access to these features. For more information, see "<a href="/en/github/getting-started-with-github/about-github-advanced-security">About GitHub Advanced Security</a>."</p> 
       <div class="extended-markdown note border rounded-1 mb-4 p-3 color-border-info color-bg-info f5"> 
        <p><strong>Note</strong>: You can't disable some security and analysis features that are enabled by default for public repositories.</p> 
       </div> 
       <p>If you enable security and analysis features, GitHub performs read-only analysis on your repository. For more information, see "<a href="/en/github/understanding-how-github-uses-and-protects-your-data/about-githubs-use-of-your-data">About GitHub's use of your data</a>."</p> 
       <h3 id="displaying-the-security-and-analysis-settings"><a href="#displaying-the-security-and-analysis-settings">Displaying the security and analysis settings</a></h3> 
       <ol> 
        <li>In the top right corner of GitHub, click your profile photo, then click <strong>Your organizations</strong>. <span class="procedural-image-wrapper"><img src="/assets/images/help/profile/your-organizations.png" alt="Your organizations in the profile menu"></span></li> 
        <li>Click <strong>Settings</strong> next to the organization. <span class="procedural-image-wrapper"><img src="/assets/images/help/organizations/settings-button.png" alt="The settings button"></span></li> 
        <li>In the left sidebar, click <strong>Security &amp; analysis</strong>. <span class="procedural-image-wrapper"><img src="/assets/images/help/organizations/org-settings-security-and-analysis.png" alt="&quot;Security &amp; analysis&quot; tab in organization settings"></span></li> 
       </ol> 
       <p>The page that's displayed allows you to enable or disable all security and analysis features for the repositories in your organization.</p> 
       <p>If your organization belongs to an enterprise with a license for GitHub Advanced Security, the page will also contain options to enable and disable Advanced Security features. Any repositories that use GitHub Advanced Security are listed at the bottom of the page.</p> 
       <h3 id="enabling-or-disabling-a-feature-for-all-existing-repositories"><a href="#enabling-or-disabling-a-feature-for-all-existing-repositories">Enabling or disabling a feature for all existing repositories</a></h3> 
       <p>You can enable or disable features for all repositories. The impact of your changes on repositories in your organization is determined by their visibility:</p> 
       <ul> 
        <li><strong>Dependency graph</strong> - Your changes affect only private repositories because the feature is always enabled for public repositories.</li> 
        <li><strong>Dependabot alerts</strong> - Your changes affect all repositories.</li> 
        <li><strong>Dependabot security updates</strong> - Your changes affect all repositories.</li> 
        <li><strong>GitHub Advanced Security</strong> - Your changes affect only private repositories because GitHub Advanced Security and the related features are always enabled for public repositories.</li> 
        <li><strong>Secret scanning</strong> - Your changes affect only private repositories where GitHub Advanced Security is also enabled. Secret scanning is always enabled for public repositories.</li> 
       </ul> 
       <div class="extended-markdown note border rounded-1 mb-4 p-3 color-border-info color-bg-info f5"> 
        <p><strong>Note:</strong> If you enable GitHub Advanced Security, committers to these repositories will use seats on your GitHub Advanced Security license. This option is disabled if you have exceeded your license capacity. For more information, see "<a href="/en/billing/managing-licensing-for-github-advanced-security/about-licensing-for-github-advanced-security">About licensing for GitHub Advanced Security</a>."</p> 
       </div> 
       <ol> 
        <li> <p>Go to the security and analysis settings for your organization. For more information, see "<a href="#displaying-the-security-and-analysis-settings">Displaying the security and analysis settings</a>."</p> </li> 
        <li> <p>Under "Configure security and analysis features", to the right of the feature, click <strong>Disable all</strong> or <strong>Enable all</strong>. The control for "GitHub Advanced Security" is disabled if you have no available seats in your GitHub Advanced Security license.</p> <p><span class="procedural-image-wrapper"><img src="/assets/images/help/organizations/security-and-analysis-disable-or-enable-all-ghas-dotcom.png" alt="&quot;Enable all&quot; or &quot;Disable all&quot; button for &quot;Configure security and analysis&quot; features"></span></p> </li> 
        <li> <p>Optionally, enable the feature by default for new repositories in your organization.</p> <p><span class="procedural-image-wrapper"><img src="/assets/images/help/organizations/security-and-analysis-enable-by-default-in-modal.png" alt="&quot;Enable by default&quot; option for new repositories"></span></p> </li> 
        <li> <p>Click <strong>Disable FEATURE</strong> or <strong>Enable FEATURE</strong> to disable or enable the feature for all the repositories in your organization.</p> <p><span class="procedural-image-wrapper"><img src="/assets/images/help/organizations/security-and-analysis-enable-dependency-graph.png" alt="Button to disable or enable feature"></span></p> </li> 
       </ol> 
       <p> When you enable one or more security and analysis features for existing repositories, you will see any results displayed on GitHub within minutes:</p> 
       <ul> 
        <li>All the existing repositories will have the selected configuration.</li> 
        <li>New repositories will follow the selected configuration if you've enabled the checkbox for new repositories.</li> 
        <li>We use the permissions to scan for manifest files to apply the relevant services.</li> 
        <li>You'll see information on your dependency graph.</li> 
        <li>GitHub will generate Dependabot alerts and create pull requests.</li> 
       </ul> 
       <h3 id="enabling-or-disabling-a-feature-automatically-when-new-repositories-are-added"><a href="#enabling-or-disabling-a-feature-automatically-when-new-repositories-are-added">Enabling or disabling a feature automatically when new repositories are added</a></h3> 
       <ol> 
        <li> <p>Go to the security and analysis settings for your organization. For more information, see "<a href="#displaying-the-security-and-analysis-settings">Displaying the security and analysis settings</a>."</p> </li> 
        <li> <p>Under "Configure security and analysis features", to the right of the feature, enable or disable the feature by default for new repositories, or all new private repositories, in your organization.</p> <p><span class="procedural-image-wrapper"><img src="/assets/images/help/organizations/security-and-analysis-enable-or-disable-feature-checkbox-dotcom.png" alt="Checkbox for enabling or disabling a feature for new repositories"></span></p> </li> 
       </ol> 
       <h3 id="allowing-dependabot-to-access-private-dependencies"><a href="#allowing-dependabot-to-access-private-dependencies">Allowing Dependabot to access private dependencies</a></h3> 
       <p>Dependabot can check for outdated dependency references in a project and automatically generate a pull request to update them. To do this, Dependabot must have access to all of the targeted dependency files. Typically, version updates will fail if one or more dependencies are inaccessible. For more information, see "<a href="/en/github/administering-a-repository/about-dependabot-version-updates">About Dependabot version updates</a>."</p> 
       <p>By default, Dependabot can't update dependencies that are located in private repositories or private package registries. However, if a dependency is in a private GitHub repository within the same organization as the project that uses that dependency, you can allow Dependabot to update the version successfully by giving it access to the host repository.</p> 
       <p>If your code depends on packages in a private registry, you can allow Dependabot to update the versions of these dependencies by configuring this at the repository level. You do this by adding authentication details to the <em>dependabot.yml</em> file for the repository. For more information, see "<a href="/en/github/administering-a-repository/configuration-options-for-dependency-updates#configuration-options-for-private-registries">Configuration options for dependency updates</a>."</p> 
       <p>To allow Dependabot to access a private GitHub repository:</p> 
       <ol> 
        <li> <p>Go to the security and analysis settings for your organization. For more information, see "<a href="#displaying-the-security-and-analysis-settings">Displaying the security and analysis settings</a>."</p> </li> 
        <li> <p>Under "Dependabot private repository access", click <strong>Add private repositories</strong> or <strong>Add internal and private repositories</strong>. <span class="procedural-image-wrapper"><img src="/assets/images/help/organizations/dependabot-private-repository-access.png" alt="Add repositories button"></span></p> </li> 
        <li> <p>Start typing the name of the repository you want to allow. <span class="procedural-image-wrapper"><img src="/assets/images/help/organizations/dependabot-private-repo-choose.png" alt="Add repositories button"></span></p> </li> 
        <li> <p>Click the repository you want to allow.</p> </li> 
        <li> <p>Optionally, to remove a repository from the list, to the right of the repository, click 
          <svg version="1.1" width="16" height="16" viewbox="0 0 16 16" class="octicon octicon-x" aria-label="The X icon" role="img">
           <path fill-rule="evenodd" d="M3.72 3.72a.75.75 0 011.06 0L8 6.94l3.22-3.22a.75.75 0 111.06 1.06L9.06 8l3.22 3.22a.75.75 0 11-1.06 1.06L8 9.06l-3.22 3.22a.75.75 0 01-1.06-1.06L6.94 8 3.72 4.78a.75.75 0 010-1.06z"></path>
          </svg>. <span class="procedural-image-wrapper"><img src="/assets/images/help/organizations/dependabot-private-repository-list.png" alt="&quot;X&quot; button to remove a repository"></span></p> </li> 
       </ol> 
       <h3 id="removing-access-to-github-advanced-security-from-individual-repositories-in-an-organization"><a href="#removing-access-to-github-advanced-security-from-individual-repositories-in-an-organization">Removing access to GitHub Advanced Security from individual repositories in an organization</a></h3> 
       <p>You can manage access to GitHub Advanced Security features for a repository from its "Settings" tab. For more information, see "<a href="/en/github/administering-a-repository/managing-security-and-analysis-settings-for-your-repository">Managing security and analysis settings for your repository</a>." However, you can also disable GitHub Advanced Security features for a repository from the "Settings" tab for the organization.</p> 
       <ol> 
        <li>Go to the security and analysis settings for your organization.
