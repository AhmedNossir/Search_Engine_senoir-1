           <p><em>Apprenez rapidement à configurer votre compte Cloudflare et à activer votre domaine. Comprendre les bonnes pratiques pour les domaines proxysés par Cloudflare.</em></p>
           <p><em>Cet article a précédemment publié sous le titre <strong>Étape 1&nbsp;: Comment Cloudflare fonctionne&nbsp;?</strong></em></p>
           <hr>
           <h2>Présentation</h2>
           <p>Le réseau mondial Cloudflare améliore la sécurité des sites web et leur performance. Lisez notre article sur <a href="https://support.cloudflare.com/hc/articles/205177068">comment Cloudflare fonctionne</a> pour plus de détails.</p>
           <p>Activez votre domaine sur Cloudflare en deux étapes&nbsp;:</p>
           <ol start="1">
            <li><a href="https://support.cloudflare.com/hc/articles/201720164">Créez un compte Cloudflare et ajoutez un site web</a></li>
            <li><a href="https://support.cloudflare.com/hc/articles/205195708">Utilisez les serveurs de noms Cloudflare à votre registrar de domaine</a></li>
           </ol>
           <div class="cf-sidebar-2 cf-note">
            L’étape 2 n’est pas nécessaire pour les domaines Business et Enterprise sur les <a href="https://support.cloudflare.com/hc/en-us/articles/360020348832">configurations CNAME</a>.
           </div>
           <p>Par ailleurs, Cloudflare recommande plusieurs <a href="#h_87dac948-9e12-47e0-a32c-2144aadd0a2a" target="_self">bonnes pratiques</a> pour les domaines nouvellement activés.&nbsp; Si<span> vous rencontrez des problèmes de configuration de votre premier domaine Cloudflare,&nbsp;reportez-vous à notre&nbsp;</span><a href="https://support.cloudflare.com/hc/articles/360027710672" target="_self">FAQ de dépannage pour les nouveaux clients Cloudflare</a>.</p>
           <hr>
           <h2 id="h_87dac948-9e12-47e0-a32c-2144aadd0a2a">Bonnes pratiques pour activer des domaines Cloudflare</h2>
           <p>Cloud recommande les actions et améliorations suivantes pour chaque nouveau domaine ajouté à Cloudflare&nbsp;:</p>
           <h3><strong>Actions recommandées&nbsp;:</strong></h3>
           <p>Assurez une interaction efficace entre votre serveur web d’origine, vos services tiers et Cloudflare&nbsp;:</p>
           <ul>
            <li><a href="https://support.cloudflare.com/hc/articles/201897700" target="_self">Mettez en liste blanche les adresses IP Cloudflare</a>&nbsp;au niveau de votre serveur web d’origine et</li>
            <li><a href="https://support.cloudflare.com/hc/articles/217074967">Mettez en liste blanche des services tiers</a>&nbsp;utilisés par votre site via les <a href="https://support.cloudflare.com/hc/articles/217074967" target="_self"><strong>règles d’accès des adresses IP</strong> Cloudflare</a>.</li>
           </ul>
           <div class="cf-sidebar-2 cf-note">
            Les IP de votre serveur d'origine sont automatiquement mis sur la liste blanche pour votre domaine quand le domaine est ajouté à Cloudflare.
           </div>
           <h3><strong>Améliorations recommandées</strong></h3>
           <p>Découvrez quelles fonctionnalités Cloudflare augmentent la vitesse et optimisent la sécurité de votre site&nbsp;:</p>
           <ul>
            <li><a href="https://support.cloudflare.com/hc/articles/200172856#h_c2f90417-ee71-4b65-b6c7-3636ee2b3c63">Améliore la performance du site</a></li>
            <li><a href="https://support.cloudflare.com/hc/articles/115002059131">Optimise la sécurité</a></li>
           </ul>
           <h3><strong>Lectures recommandées</strong></h3>
           <p>Trafic sécurisé sur HTTPS tant pour les visiteurs de votre site que pour le trafic entre Cloudflare et votre serveur web d’origine.</p>
