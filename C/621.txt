       <h2><span class="mw-headline" id="Quick_guide">Quick guide</span></h2> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>1. How to find the appropriate categories </p> 
       <dl>
        <dd>
         <ul>
          <li>Find categories with the search engine (see <a href="#Categorization_tips">#Categorization tips</a>)</li> 
          <li>or check how similar files are categorized (some may not be categorized though)</li> 
          <li>or start from the main topical category (<a href="/wiki/Category:Topics" title="Category:Topics">Category:Topics</a>)</li> 
          <li>Starting from these categories, check their parent or sub-categories to find an appropriate category. Avoid picking too general categories.</li>
         </ul>
        </dd>
       </dl> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>2. Add the categories to the file </p> 
       <dl>
        <dd>
         <ul>
          <li>Select them on the upload form</li> 
          <li>or: add them manually by adding, e.g. <code>[[Category:Foobar]]</code> to the file description page (see <a href="#Categorizing_pages">#Categorizing pages</a>)</li> 
          <li>or: use HotCat (see <a href="/wiki/Special:MyLanguage/Help:Gadget-HotCat" title="Special:MyLanguage/Help:Gadget-HotCat">Help:Gadget-HotCat</a>)</li> 
          <li>or: use Cat-a-lot (see <a href="/wiki/Special:MyLanguage/Help:Gadget-Cat-a-lot" title="Special:MyLanguage/Help:Gadget-Cat-a-lot">Help:Gadget-Cat-a-lot</a>)</li>
         </ul>
        </dd>
       </dl>

       <p>The main principles are: </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <h4><span class="mw-headline" id="Hierarchic_principle">Hierarchic principle</span></h4> 
      </div> 
      <p>Structura de categorii este (ideal) o multiierarhie cu o singur? categorie r?d?cin?, <a href="/wiki/Category:CommonsRoot" title="Category:CommonsRoot">Category:CommonsRoot</a>. Toate categoriile (cu excep?ia <i>CommonsRoot</i>) trebuie s? fie incluse în cel pu?in o alt? categorie ?i nu trebuie s? existe cicluri (adic? o categorie nu trebuie s? se con?in? pe ea îns??i, direct sau indirect). Structura de categorii trebuie s? reflecte o ierarhie de concepte, de la cel mai generic pân? la cel mai specific. </p> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <h4><span class="mw-headline" id="Modularity_principle">Modularity principle</span></h4> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>The page (file, category) should be put in the most specific category/categories that fit(s) the page (not directly to its parent categories). A category can have more parent categories. A category can combine two (or more) different criteria; such categories are called "compound categories" or "intersection categories". E.g. the root category <a href="/wiki/Category:Churches" title="Category:Churches">Category:Churches</a> and the root category <a href="/wiki/Category:Russia" title="Category:Russia">Category:Russia</a> have a common subcategory <a href="/wiki/Category:Churches_in_Russia" title="Category:Churches in Russia">Churches in Russia</a>. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <h4><span class="mw-headline" id="Simplicity_principle">Simplicity principle</span></h4> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>This principle suggests not to combine too many different criteria. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <h4><span class="mw-headline" id="Selectivity_principle">Selectivity principle</span></h4> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>We should not classify items which are related to different subjects in the same category. The category name should be unambiguous and not homonymous. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <h4><span class="mw-headline" id="Universality_principle">Universality principle</span></h4> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>Identical items should have identical names for all countries and at all levels of categorization. Categorization structure should be as systematical and unified as possible, local dialects and terminology should be supressed in favour of universality if possible. Analogic categorization branches should have analogic structure. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <h3><span class="mw-headline" id="Types_of_reflected_relations">Types of reflected relations</span></h3> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>The category structure should reflect a hierarchy of concepts, from the most generic one down to the very specific. The structure uses and combines more types of relation, e. g. </p> 
       <ul>
        <li><a href="https://en.wikipedia.org/wiki/en:Hyponymy_and_hypernymy" class="extiw" title="w:en:Hyponymy and hypernymy">Hyponymy</a>: a sort/kind/type of… (typically in biological taxonomy)</li> 
        <li><a href="https://en.wikipedia.org/wiki/en:Meronymy_and_holonymy" class="extiw" title="w:en:Meronymy and holonymy">Meronymy</a>: a part of…, a member of… (typically for geographical division, building/room, device/component etc.)</li>
       </ul> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <ul>
        <li>Attributes: 
         <ul>
          <li>Qualitative and general attributes (color, shape, size, ability or disability, nationality, technique, quality, awards…)</li> 
          <li>Location: where, in…, from… (place/event, place/building, place/exhibit, place/people, country/language, source/work, factory or country/product etc.)</li> 
          <li>Timing: when (time/event, time/depicted situation, time of birth, inception or construction, time of death, demolition or termination etc.)</li>
         </ul></li>
       </ul> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <ul>
        <li>Agentive and influence relations: (creator/work, device/product, company/product, discipline or profession/their subjects and terms, parent/children, subordination, owner/property, initiator/follower, subject/other subjects dedicated to it or named after it, subject/its duplicate, imitiation, depiction or symbol etc etc.)</li> 
        <li>Modification: original/modified or modified/original (avoid cyclic structure) – renamed, rebuilt, repurposed or transformed subjects.</li>
       </ul> 
      </div>

       <h2><span class="mw-headline" id="For_more_appropriate_categorization">For more appropriate categorization</span></h2> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>Pages (including category pages) are categorized according to their subject, and not to their contents, because the contents are generally not a permanent feature of the category page; in particular, you can momentarily find inappropriate contents in a category page. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>Example: Assume that <a href="/wiki/Category:Spheres" title="Category:Spheres">Category:Spheres</a> contains only pictures of crystal balls. You must not add <a href="/wiki/Category:Glass" title="Category:Glass">Category:Glass</a> in the category page, according to the current contents, because you can have spheres made with a great variety of materials. Normally, any picture showing a glass object would be already categorized in <a href="/wiki/Category:Glass" title="Category:Glass">Category:Glass</a> (or in a category of its substructure). So, if the <a href="/wiki/Category:Spheres" title="Category:Spheres">Category:Spheres</a> is really crowded with crystal balls pictures, it would be a better idea to create a new category page, like Category:Glass spheres or Category:Crystal balls, categorized in <a href="/wiki/Category:Spheres" title="Category:Spheres">Category:Spheres</a> and <a href="/wiki/Category:Glass" title="Category:Glass">Category:Glass</a>. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>Generally files should only be in the most specific category that exists for certain topic. For example files in <a href="/wiki/Category:Looking_up_the_center_of_the_Eiffel_Tower" title="Category:Looking up the center of the Eiffel Tower">Category:Looking up the center of the Eiffel Tower</a> should not also be in <a href="/wiki/Category:Paris" title="Category:Paris">Category:Paris</a> <i>(see <a href="#Over-categorization">over-categorization</a> below)</i>. If you do not find a category that fits your purpose, you can create it — but carefully read the section about <a href="#How_to_use_categories">using categories</a> first. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>This does not mean that an image only belongs in one category; it just means that images should not be in <i>redundant</i> or non-specific categories. For instance, an image of a Polar Bear being rescued from an iceberg by a helicopter should be in <a href="/wiki/Category:Ursus_maritimus" title="Category:Ursus maritimus">Category:Ursus maritimus</a>, <a href="/wiki/Category:Icebergs" title="Category:Icebergs">Category:Icebergs</a> and <a href="/wiki/Category:Rescue_helicopters" title="Category:Rescue helicopters">Category:Rescue helicopters</a>. It should not, however, be in <a href="/wiki/Category:Ursidae" title="Category:Ursidae">Category:Ursidae</a>, <a href="/wiki/Category:Sea_ice" title="Category:Sea ice">Category:Sea ice</a> or <a href="/wiki/Category:Aircraft" title="Category:Aircraft">Category:Aircraft</a>. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <h3><span class="mw-headline" id="Categorization_tips">Categorization tips</span></h3> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>The categories (or galleries) you choose for your uploads should answer as many as possible of the following questions: </p> 
       <ul>
        <li><b>what? / whom?:</b> what or whom does the file show? What is the main subject? What are the noteworthy features of the image? For instance <a href="/wiki/Category:Automobiles" title="Category:Automobiles">Category:Automobiles</a>, <a href="/wiki/Category:Felis_silvestris_catus" title="Category:Felis silvestris catus">Category:Felis silvestris catus</a> or <a href="/wiki/Category:Jimmy_Wales" title="Category:Jimmy Wales">Category:Jimmy Wales</a></li> 
        <li><b>where?:</b> where was the image taken? What is the location of the subject? What is the location of the camera? This is especially important for pictures of places. E.g. <a href="/wiki/Category:Basin_Street,_New_Orleans" title="Category:Basin Street, New Orleans">Category:Basin Street, New Orleans</a> 
         <dl>
          <dd>
           also use <a href="/wiki/Template:Location" title="Template:Location"><span style="font-family:monospace,monospace;">{{Location}}</span></a>
          </dd>
         </dl></li> 
        <li><b>when?:</b> when did the depicted events happen, or when was the image created? When was the image taken? This is especially important for historical images. An example would be <a href="/wiki/Category:Warsaw_in_September_1939" title="Category:Warsaw in September 1939">Category:Warsaw in September 1939</a>, <a href="/wiki/Category:April_2010_in_Northern_Ireland" title="Category:April 2010 in Northern Ireland">Category:April 2010 in Northern Ireland</a></li>
       </ul> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <ul>
        <li><b>who?:</b> who is the author? This is especially important for works of well known artists and for historical images, for example <a href="/wiki/Category:Paintings_by_Rembrandt" title="Category:Paintings by Rembrandt">Category:Paintings by Rembrandt</a>. You can also use the pages from the <a href="/wiki/Category:Creator_templates" title="Category:Creator templates"><i>Creator</i> namespace</a> as templates to achieve this.</li> 
        <li><b>how?:</b> how does the file (or the image) do that? Specifically: 
         <ul>
          <li><b>what view?:</b> what type of view does the image show? e.g. <a href="/wiki/Category:Plan_views" title="Category:Plan views">Category:Plan views</a>, <a href="/wiki/Category:Panoramics" title="Category:Panoramics">Category:Panoramics</a>.</li> 
          <li><b>what color?:</b> which is the general color? e.g. <a href="/wiki/Category:Yellow_animals" title="Category:Yellow animals">Category:Yellow animals</a></li> 
          <li><b>what photography technique?:</b>. If the image uses a specific <a href="/wiki/Category:Photographic_techniques" title="Category:Photographic techniques">technique</a> or <a href="/wiki/Category:Photographic_effects" title="Category:Photographic effects">effect</a>, apply the corresponding category: e.g. <a href="/wiki/Category:Tone-mapped_HDR_images" title="Category:Tone-mapped HDR images">Category:Tone-mapped HDR images</a>, <a href="/wiki/Category:Black_and_white_photographs" title="Category:Black and white photographs">Category:Black and white photographs</a>, <a href="/wiki/Category:Double_exposure" title="Category:Double exposure">Category:Double exposure</a></li>
         </ul></li>
       </ul> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>The above questions cover the main aspects of the image to be categorized. For some images it makes sense to use all, for other images only one or two are reasonable. In addition there are several other aspects of the images that can be used to categorize the image: </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <ul>
        <li><b>what source?:</b> information about where the image came from. For example, <a href="/wiki/Category:Images_from_the_German_Federal_Archive" title="Category:Images from the German Federal Archive">Category:Images from the German Federal Archive</a></li> 
        <li><b>what format?:</b> information about the media type if it is unusual. For example, <a href="/wiki/Category:Audio" title="Category:Audio">Category:Audio</a> or <a href="/wiki/Category:Animated_GIF" title="Category:Animated GIF">Category:Animated GIF</a>, <a href="/wiki/Category:SVG" class="mw-disambig" title="Category:SVG">Category:SVG</a></li> 
        <li><b>what software?:</b> information about software used to create the image. For example, <a href="/wiki/Category:Created_with_Hugin" title="Category:Created with Hugin">Category:Created with Hugin</a></li> 
        <li><b>what camera?:</b> information about the camera. For example, <a href="/wiki/Category:Taken_with_Nikon_D80" title="Category:Taken with Nikon D80">Category:Taken with Nikon D80</a></li>
       </ul> 
       <p>This last set is useful and important but should always be done in addition of the main set of criteria. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>Categorization in Wikimedia Commons is more detailed and deep than categorization in Wikipedia projects. Compared to them, Commons has more categories for individual subjects – places, people, organizations, events, terms, etc. Almost every article on a Wikipedia can have a corresponding category on Commons. However, even if there exist more images of an ordinary person or incidental event, it is practical to group them into a special category and categorize that category instead of categorizing all similar images individually to an identical set of parent categories. </p> 
      </div>

       <h3><span class="mw-headline" id="Find_an_appropriate_category">Find an appropriate category</span></h3> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>To find appropriate categories for your uploads, you should navigate the category structure starting from a generic category. Narrow your search down to subcategories until you find the most specific category that fits the file you uploaded. You can navigate the category structure by following links to subcategories, or expanding the tree of subcategories by clicking on the little + symbols on subcategory names. The <i><a href="#Major_categories">Major categories</a></i> section above provides a starting point, and the <i><a href="#How_to_categorize:_guidance_by_topic">How to categorize: guidance by topic</a></i> covers some topics more. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <h3><span class="mw-headline" id="Over-categorization">Over-categorization</span></h3> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p><i>for the inclusion criteria (the equivalent of <a href="https://en.wikipedia.org/wiki/WP:OVERCAT" class="extiw" title="w:WP:OVERCAT">w:WP:OVERCAT</a>) see <a href="/wiki/Commons:Category_inclusion_criteria" title="Commons:Category inclusion criteria">Commons:Category inclusion criteria</a></i> </p> 
       <div class="wpImageAnnotatorControl wpImageAnnotatorOff">
        <div class="thumb tright">
         <div class="thumbinner" style="width:222px;">
          <a href="/wiki/File:Over-categorization.svg" class="image"><img alt="" src="https://upload.wikimedia.org/wikipedia/commons/thumb/4/44/Over-categorization.svg/220px-Over-categorization.svg.png" decoding="async" width="220" height="310" class="thumbimage" srcset="https://upload.wikimedia.org/wikipedia/commons/thumb/4/44/Over-categorization.svg/330px-Over-categorization.svg.png 1.5x, https://upload.wikimedia.org/wikipedia/commons/thumb/4/44/Over-categorization.svg/440px-Over-categorization.svg.png 2x" data-file-width="220" data-file-height="310"></a> 
          <div class="thumbcaption">
           <div class="magnify">
            <a href="/wiki/File:Over-categorization.svg" class="internal" title="Enlarge"></a>
           </div>Don't place an item into a category <i>and</i> its parent. For example, a black and white photo of the Eiffel Tower should be placed in <a href="/wiki/Category:Black_and_white_photographs_of_the_Eiffel_Tower" title="Category:Black and white photographs of the Eiffel Tower">Black and white photographs of the Eiffel Tower</a>. It should <i>not</i> be placed in both that category and the <a href="/wiki/Category:Paris" title="Category:Paris">Paris</a> category at the same time.
          </div>
         </div>
        </div>
       </div>
       <div class="shortcut2" style="">
        <style data-mw-deduplicate="TemplateStyles:r433650343">.mw-parser-output .shortcut2{float:right;margin:0.4em;border:1px solid #ccc;background-color:white;padding:0 1em;text-align:center;font-size:x-small;border-radius:4px}</style><a href="/wiki/Commons:Shortcuts" title="Commons:Shortcuts">Shortcut</a>
        <br><a href="/wiki/Commons:OVERCAT" class="mw-redirect" title="Commons:OVERCAT">COM:OVERCAT</a>
       </div> 
       <p>Over-categorization is placing a file, category or other page in several levels of the same branch in the category tree. The general rule is <i>always place an image in the most specific categories, and not in the levels above those.</i> Exceptions to this rule are explained in <a href="#OvercatException">the section below</a>. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>Example: An image needing to be categorized shows a yellow circle. This image should be placed in <a href="/wiki/Category:Yellow_circles" title="Category:Yellow circles">Category:Yellow circles</a>. If it is also placed in <a href="/wiki/Category:Circles" title="Category:Circles">Category:Circles</a>, it is <i>over-categorized.</i> We already know that it's a circle, because all yellow circles are circles. Therefore, <a href="/wiki/Category:Circles" title="Category:Circles">Category:Circles</a> is redundant. <a href="/wiki/Template:Uw-overcat" title="Template:Uw-overcat">Template:Uw-overcat</a> can be used to advise users of this. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>This applies to most files: As mentioned under the adjacent illustration, files in <a href="/wiki/Category:Black_and_white_photographs_of_the_Eiffel_Tower" title="Category:Black and white photographs of the Eiffel Tower">Category:Black and white photographs of the Eiffel Tower</a> should not also be in <a href="/wiki/Category:Paris" title="Category:Paris">Category:Paris</a>, files in <a href="/wiki/Category:Albert_Einstein" title="Category:Albert Einstein">Category:Albert Einstein</a> should not be in <a href="/wiki/Category:Physicists_from_Germany" title="Category:Physicists from Germany">Category:Physicists from Germany</a> and so on. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <h4><span class="mw-headline" id="Why_over-categorization_is_a_problem">Why over-categorization is a problem</span></h4> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>It's often assumed that the more categories an image is in, the easier it will be to find it. Another example: By that logic, every image showing a man should be in <a href="/wiki/Category:Men" title="Category:Men">Category:Men</a>, because even if you know nothing more about the person you're looking for than that he is a man, you'll be able to find it. The result is that the top category fills up, making it necessary to go through hundreds, or in this case more likely thousands of images to find the one you want. You probably won't find what you're looking for, and what's more, those who are looking for a generic picture of a man to illustrate an article like <a href="https://en.wikipedia.org/wiki/Man" class="extiw" title="en:Man">en:Man</a> will find that they've drowned out among the movie stars, scientists and politicians. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>On lower levels, the problem becomes less acute, since the number of images will be smaller — they can still easily reach into the hundreds, though. But there is still a problem: Let's go back to Einstein. I know that he's a physicist, so I'll look in the <a href="/wiki/Category:Physicists" title="Category:Physicists">Category:Physicists</a> category. I find an image of Einstein among the hundreds of images of other physicists, which I'm not too happy with, but it's the only one there. Since there was an image there, I assume that there are no more hidden elsewhere, rather than look further in <a href="/wiki/Category:Physicists_from_Germany" title="Category:Physicists from Germany">Category:Physicists from Germany</a> and thus find <a href="/wiki/Category:Albert_Einstein" title="Category:Albert Einstein">Category:Albert Einstein</a> where there might be a better one. So over-categorization has led to two problems: The top category is cluttered, and users will stop looking for the most relevant category since they've reached one that has a relevant image. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <h4><span class="mw-headline" id="Improper_categorization_of_categories_is_a_cause_of_over-categorization">Improper categorization of categories is a cause of over-categorization</span></h4> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>Strange as it may sound, under-categorization can be a cause of over-categorization. When a category itself is not properly categorized, it can lead users to over-categorize files belonging in that category. An example of this: <a href="/wiki/Category:Eiv%C3%B8r_P%C3%A1lsd%C3%B3ttir" title="Category:Eivør Pálsdóttir">Category:Eivør Pálsdóttir</a> was categorized only in <a href="/wiki/Category:People_by_name" title="Category:People by name">Category:People by name</a>. A user categorizing an image of her might then be tempted to also place the image in <a href="/wiki/Category:Female_vocalists_from_the_Faroe_Islands" title="Category:Female vocalists from the Faroe Islands">Category:Female vocalists from the Faroe Islands</a>. The correct solution is to place the image only in <a href="/wiki/Category:Eiv%C3%B8r_P%C3%A1lsd%C3%B3ttir" title="Category:Eivør Pálsdóttir">Category:Eivør Pálsdóttir</a> and to make that category a subcategory of <a href="/wiki/Category:Female_vocalists_from_the_Faroe_Islands" title="Category:Female vocalists from the Faroe Islands">Category:Female vocalists from the Faroe Islands</a>. At that point, however, any images that were already placed into both categories become overcategorized and need to be manually removed from the parent category. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>A related problem is erroneous categorization. Notting Hill is a district within the borough of Kensington and Chelsea in London. When it was created, <a href="/wiki/Category:Notting_Hill" title="Category:Notting Hill">Category:Notting Hill</a> was placed directly in <a href="/wiki/Category:London" title="Category:London">Category:London</a> instead of in the <a href="/wiki/Category:Royal_Borough_of_Kensington_and_Chelsea" title="Category:Royal Borough of Kensington and Chelsea">Category:Royal Borough of Kensington and Chelsea</a> subcategory, where it should have been placed. A user categorizing an image of Notting Hill might then be tempted to place it both in <a href="/wiki/Category:Notting_Hill" title="Category:Notting Hill">Category:Notting Hill</a> and in <a href="/wiki/Category:Royal_Borough_of_Kensington_and_Chelsea" title="Category:Royal Borough of Kensington and Chelsea">Category:Royal Borough of Kensington and Chelsea</a>. Instead, each image should be placed only in the most specific categories, and those categories should in turn be placed in their most specific categories. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>When you encounter improperly categorized categories, please place them in the appropriate parent categories if you are able to do so. That will not only help avoid over-categorization, but it will also make it easier to move through the category tree. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <h4><span class="mw-headline" id="Exception_for_images_with_more_categorized_subjects"><span id="OvercatException"></span> Exception for images with more categorized subjects</span></h4> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>A file that depicts only one relevant subject should not be over-categorized. Where a file depicts additional relevant subjects, and the additional subjects do not have their own subcategories, consideration can be given to temporarily categorizing the image in both the subcategory and the parent category. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>For example, this situation might arise in the case of a photograph of three politicians, one of whom is <a href="/wiki/Category:Angela_Merkel" title="Category:Angela Merkel">Angela Merkel</a> (who has her own Commons category), with two other politicians who do not yet have their own categories. While the image would undoubtedly be categorized in <a href="/wiki/Category:Angela_Merkel" title="Category:Angela Merkel">Category:Angela Merkel</a> or one of its subcategories, it would typically be considered to be over-categorization to also include it in <a href="/wiki/Category:Politicians_of_Germany" title="Category:Politicians of Germany">Category:Politicians of Germany</a>. Users would, however, be unlikely to search for the two other politicians in the Merkel category. Ideally, we would create specific subcategories for the two other politicians (where warranted), or find other relevant subcategories (e.g. <a href="/wiki/Category:Politicians_of_Bavaria" title="Category:Politicians of Bavaria">Category:Politicians of Bavaria</a> or <a href="/wiki/Category:Members_of_the_FDP" title="Category:Members of the FDP">Category:Members of the FDP</a>, etc.), that would enable us to avoid over-categorization. In some circumstances, however, we may need to temporarily categorize the image in <a href="/wiki/Category:Politicians_of_Germany" title="Category:Politicians of Germany">Category:Politicians of Germany</a> where other appropriate subcategories do not yet exist. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>Countries may be categorized as part of multiple overlapping categories. For example, <a href="/wiki/Category:India" title="Category:India">Category:India</a> is in <a href="/wiki/Category:Countries_of_South_Asia" title="Category:Countries of South Asia">Category:Countries of South Asia</a> as well as <a href="/wiki/Category:Countries_of_Asia" title="Category:Countries of Asia">Category:Countries of Asia</a>. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>Also user categories are exempted of over-categorization as those are not visible to most viewers, and project users include them for many different purposes like sorting, stats, filling values for userboxes, etc. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <h2><span class="mw-headline" id="How_to_categorize:_guidance_by_topic">How to categorize: guidance by topic</span></h2> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>For some categories, there is special guidance on how best to sort content within that category. This guidance can be found in a <a href="/wiki/Category:Commons_category_schemes" title="Category:Commons category schemes">category scheme</a> or a <a href="/wiki/Category:Commons_projects" title="Category:Commons projects">commons project</a> for your topic. There is also some categorizing information in this section and sometimes there is guidance at the top of the category's page, in the Category <a href="https://meta.wikimedia.org/wiki/Namespace" class="extiw" title="meta:Namespace">namespace</a>. So, for instance, some guidance on categorizing content depicting people is at the top of <a href="/wiki/Category:People" title="Category:People">Category:People</a>, and some is in the section <a href="#People">People</a> below. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <h3><span class="mw-headline" id="People">People</span></h3> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>Content depicting people should be put in categories which describe them, such as <a href="/wiki/Category:Economists_from_the_United_States" title="Category:Economists from the United States">Category:Economists from the United States</a>. Start exploring at <a href="/wiki/Category:People" title="Category:People">Category:People</a>. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>Please see <a href="/wiki/Special:MyLanguage/Commons:Category_scheme_People" title="Special:MyLanguage/Commons:Category scheme People">Commons:Category scheme People</a> for details on how to name and organize these categories. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <h3><span id="Landscapes.2C_outdoor_views"></span><span class="mw-headline" id="Landscapes,_outdoor_views">Landscapes, outdoor views</span></h3> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>Content depicting a given subject from a common vantage point are grouped in <i>Views of Subject from Viewpoint</i> categories such as <a href="/wiki/Category:Views_of_Cathedral_of_Seville_from_the_Giralda" title="Category:Views of Cathedral of Seville from the Giralda">Views of Cathedral of Seville from the Giralda</a>. Such categories should be subcategories of both the subject's category (<a href="/wiki/Category:Cathedral_of_Seville" title="Category:Cathedral of Seville">Cathedral of Seville</a> in this example) and the viewpoint's category (<a href="/wiki/Category:Giralda" title="Category:Giralda">Giralda</a> in this example). </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>In this example, the <a href="/wiki/Category:Views_of_Cathedral_of_Seville_from_the_Giralda" title="Category:Views of Cathedral of Seville from the Giralda">Views of Cathedral of Seville from the Giralda</a> category is not placed directly in the subject and viewpoint categories, but in <a href="/wiki/Category:Views_of_the_Cathedral_of_Seville" title="Category:Views of the Cathedral of Seville">Views of the Cathedral of Seville</a> and <a href="/wiki/Category:Views_from_Giralda" title="Category:Views from Giralda">Views from Giralda</a>. Such intermediate categories are often necessary to create structure and avoid <a href="#over-categorization">over-categorization</a>, particularly for views of a city from a vantage point located within the city. For example, <a href="/wiki/Category:Views_of_Rome_from_the_Pincio" title="Category:Views of Rome from the Pincio">Views of Rome from the Pincio</a> needs the intermediate category <a href="/wiki/Category:Views_of_Rome" title="Category:Views of Rome">Views of Rome</a> to avoid placing it directly in <a href="/wiki/Category:Rome" title="Category:Rome">Rome</a>, which would constitute over-categorization. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <h3><span class="mw-headline" id="Texts">Texts</span></h3> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>Texts, such as scans of books, should normally have a category for each version of the scan and each edition of the text. Thus a book published in three separate editions would have a parent category for the book, three subcategories for each text, and further subcategories for the text as a jpeg, a DjVu, etc., assuming each version had actually been uploaded. (Categories would not be created for editions not held on Commons.) This is particularly important for files in formats other than DjVu and PDF, where the category is the only practical means of keeping the scans together; see eg. <a href="/wiki/Category:The_Chronicles_of_England,_Scotland_and_Ireland,_Holinshed,_1587" title="Category:The Chronicles of England, Scotland and Ireland, Holinshed, 1587">Category:The Chronicles of England, Scotland and Ireland, Holinshed, 1587</a> which contains 2857 jpeg images of page scans. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <h3><span class="mw-headline" id="GLAMs">GLAMs</span></h3> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>For categorization issues related to mass content donations from GLAMs (Galleries, Libraries, Archives &amp; Museums), please see <a href="/wiki/Special:MyLanguage/Commons:Guide_to_batch_uploading#Categories" title="Special:MyLanguage/Commons:Guide to batch uploading">Commons:Guide to batch uploading#Categories</a>. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <h2><span class="mw-headline" id="Categorization_workflow">Categorization workflow</span></h2> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>Currently, a bot checks if newly uploaded files are categorized in topical categories and attempts to categorize files that are not. Before 17 June 2015, <a href="/wiki/User:CategorizationBot" title="User:CategorizationBot">CategorizationBot</a> was responsible for this job. As of June 2019, <a href="/wiki/User:SteinsplitterBot" title="User:SteinsplitterBot">SteinsplitterBot</a> occasionally checks for uncategorized files. The workflow is the following: </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <ol>
        <li>User uploads a new file and adds categories (or not).</li> 
        <li>A bot checks if the file is categorized. 
         <ul>
          <li>File is already categorized =&gt; ok</li> 
          <li>File is not categorized =&gt; the bot tags the file for <a href="/wiki/Category:Media_needing_categories" title="Category:Media needing categories">Category:Media needing categories</a>. Today's files are in <a href="/wiki/Category:Media_needing_categories_as_of_6_June_2021" title="Category:Media needing categories as of 6 June 2021">Category:Media needing categories as of 6 June 2021</a></li>
         </ul></li> 
        <li>Users categorize files further (e.g. category diffusion below)</li>
       </ol> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p><i>See also:</i> <a href="/wiki/User:CategorizationBot#Process" title="User:CategorizationBot">User:CategorizationBot#Process</a>, <a href="/wiki/User:Multichill/Categorization_stats" title="User:Multichill/Categorization stats">categorization statistics</a> </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>Other, if manual, categorization workflows are possible&nbsp;: </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <ul>
        <li>Category filling: Use appropriate keywords in the search engine to find the files that should be in a given category, and put them there.</li> 
        <li>Category diffusing: Go to <a href="/wiki/Category:Categories_requiring_diffusion" title="Category:Categories requiring diffusion">Category:Categories requiring diffusion</a>, select a crowded category, create appropriate subcategories if needed, and move the files to the subcategories. Gadgets like Cat-a-lot and Hotcats can help.</li>
       </ul> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <h2><span id="Categories_marked_with_.22HIDDENCAT.22"></span><span class="mw-headline" id="Categories_marked_with_&quot;HIDDENCAT&quot;">Categories marked with "HIDDENCAT"</span></h2> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <div class="shortcut2" style="">
        <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r433650343"><a href="/wiki/Commons:Shortcuts" title="Commons:Shortcuts">Shortcut</a>
        <br><a href="/wiki/Commons:HIDDENCAT" class="mw-redirect" title="Commons:HIDDENCAT">COM:HIDDENCAT</a>
       </div> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>Many non-topical categories are marked with <code>__HIDDENCAT__</code> or {{<a href="/wiki/Template:Hiddencat" title="Template:Hiddencat">Hiddencat</a>}} on the category page. For example, see <a class="external text" href="https://commons.wikimedia.org/w/index.php?title=Category:PD_NASA&amp;action=edit">Category:PD NASA in edit mode</a>. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>While categories are generally visible on every page, categories marked <code>__HIDDENCAT__</code> are only visible: </p> 
       <ul>
        <li>on the edit screen: at the end of the screen, below the edit box</li> 
        <li>on category pages: 
         <ul>
          <li>on subcategories to the hidden category: in the normal location, but on a separate line with a smaller typeface and the label "Hidden categories."</li> 
          <li>on parent categories: in the same way as other categories</li>
         </ul></li> 
        <li>on file description pages and gallery pages: for logged-in users who have selected to "Show hidden categories" in their <a href="/wiki/Special:Preferences#mw-prefsection-rendering" title="Special:Preferences">appearance preferences</a>. This is activated for all newly registered users.</li>
       </ul> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>This feature is generally used for template-based categories, such as license tag based categories. For example, placing <a href="/wiki/Template:PD-old-100" title="Template:PD-old-100"><span style="font-family:monospace,monospace;">{{PD-old-100}}</span></a> on a file description page adds the file to <a href="/wiki/Category:Author_died_more_than_100_years_ago_public_domain_images" title="Category:Author died more than 100 years ago public domain images">Category:Author died more than 100 years ago public domain images</a>, which is marked with <code>__HIDDENCAT__</code>. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>For more details, see the <a href="https://meta.wikimedia.org/wiki/Help:Category#Hidden_categories" class="extiw" title="m:Help:Category">help section on hidden categories</a> for Mediawiki (the software that Commons uses). </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <h2><span class="mw-headline" id="Templates_for_categories">Templates for categories</span></h2> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>Some templates are designed for use on category pages - see <a href="/wiki/Category:Category_namespace_templates" title="Category:Category namespace templates">Category:Category namespace templates</a>. If the category is linked to a Wikidata entry, then you can use: </p> 
       <ul>
        <li><a href="/wiki/Template:Wikidata_Infobox" title="Template:Wikidata Infobox"><span style="font-family:monospace,monospace;">{{Wikidata Infobox}}</span></a></li>
       </ul> 
       <p>which displays a summary of the topic's information that is available on Wikidata, and also auto-adds birth/death/name/monument ID categories. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>Some of the more commonly used ones are <a href="/wiki/Category:Category_header_templates" title="Category:Category header templates">Category:Category header templates</a> such as: </p> 
       <ul>
        <li><a href="/wiki/Template:Category_TOC" class="mw-redirect" title="Template:Category TOC"><span style="font-family:monospace,monospace;">{{Category TOC}}</span></a> and <a href="/wiki/Template:LargeCategoryTOC" title="Template:LargeCategoryTOC"><span style="font-family:monospace,monospace;">{{LargeCategoryTOC}}</span></a> - to provide a table of contents for very large categories with many subcategories</li> 
        <li><a href="/wiki/Template:ImageTOC" title="Template:ImageTOC"><span style="font-family:monospace,monospace;">{{ImageTOC}}</span></a> and <a href="/wiki/Template:FileCategoryTOC" title="Template:FileCategoryTOC"><span style="font-family:monospace,monospace;">{{FileCategoryTOC}}</span></a> - to provide a table of contents for large categories with many files and few subcategories</li> 
        <li><a href="/wiki/Template:Image_template_notice" title="Template:Image template notice"><span style="font-family:monospace,monospace;">{{Image template notice}}</span></a> and <a href="/wiki/Template:Autocat" title="Template:Autocat"><span style="font-family:monospace,monospace;">{{Autocat}}</span></a> - for maintenance categories populated by a specific template.</li>

       <h2><span class="mw-headline" id="Tools">Tools</span></h2> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <style data-mw-deduplicate="TemplateStyles:r428405392">.mw-parser-output .hatnote{font-style:italic}.mw-parser-output .hatnote i,.mw-parser-output .hatnote.noitalic{font-style:normal}.mw-parser-output div.hatnote{padding-left:1.6em;margin-bottom:0.5em}.mw-parser-output div.hatnote+div.hatnote{margin-top:-0.5em}@media print{.mw-parser-output .hatnote{display:none}}</style> 
       <div role="note" class="hatnote navigation-not-searchable">
        Further information: <a href="/wiki/Special:MyLanguage/Commons:Tools" title="Special:MyLanguage/Commons:Tools">Commons:Tools</a> 
       </div> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <ul>
        <li>Gadgets enabled through the <a href="/wiki/Special:Preferences#mw-prefsection-gadgets" title="Special:Preferences">user preferences</a> 
         <ul>
          <li><i>Cat-a-lot</i>: A tool that helps with moving multiple files between categories or adding categories to search results. <small>[<a href="/wiki/Help:Gadget-Cat-a-lot" title="Help:Gadget-Cat-a-lot">documentation</a>&nbsp;/&nbsp;<a href="/wiki/MediaWiki_talk:Gadget-Cat-a-lot.js" title="MediaWiki talk:Gadget-Cat-a-lot.js">talk</a>]</small>&nbsp;</li> 
          <li><i>HotCat</i>:&nbsp;<sup><abbr title="default gadget – enabled by default">d</abbr></sup> Easily add / remove / change a category on a page, with name suggestions. <small>[<a href="/wiki/Help:Gadget-HotCat" title="Help:Gadget-HotCat">documentation</a>&nbsp;/&nbsp;<a href="/wiki/File:HotCat.png" title="File:HotCat.png">example</a>&nbsp;/&nbsp;<a href="/wiki/MediaWiki_talk:Gadget-HotCat.js" title="MediaWiki talk:Gadget-HotCat.js">talk</a>]</small>&nbsp;</li> 
          <li><i>Gallery Details</i>: Adds a link in the toolbox to display galleries and categories (and <a href="/wiki/Special:NewFiles" title="Special:NewFiles">Newimages</a> and <a href="/wiki/Special:Search" title="Special:Search">Search</a> result pages) with extensive details from file description pages and links to easily mark an image without source, etc. If <i>Pretty log</i> is activated, it also works on <a href="/wiki/Special:Log" title="Special:Log">Log</a> pages. <small>[<a href="/wiki/Help:Gadget-GalleryDetails" title="Help:Gadget-GalleryDetails">documentation</a>&nbsp;/&nbsp;<a href="/wiki/MediaWiki_talk:Gadget-GalleryDetails.js" title="MediaWiki talk:Gadget-GalleryDetails.js">talk</a>]</small>&nbsp;</li> 
          <li>Place categories above content, but below image on file description pages. &nbsp;Modifies the placement of categories on the user interface.</li> 
          <li>Add a link to category pages to search for the category name with the option "-incategory". This excludes files already in the category (doesn't work if the category was added by a template). <small>[<a href="/wiki/MediaWiki_talk:Gadget-Searchnotincat.js" title="MediaWiki talk:Gadget-Searchnotincat.js">talk</a>]</small>&nbsp;</li>
         </ul></li>
       </ul> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <ul>
        <li>Toollabs / toolserver tools

          <li><a href="https://meta.wikimedia.org/wiki/User:Dapete/vCat#English" class="extiw" title="m:User:Dapete/vCat">English vCat</a> – plots categories trees. For example <a class="external free" href="https://tools.wmflabs.org/vcat/render?wiki=commonswiki&amp;category=Water%20wells">https://tools.wmflabs.org/vcat/render?wiki=commonswiki&amp;category=Water%20wells</a> generates graph for parent categories of <a href="/wiki/Category:Water_wells" title="Category:Water wells">Category:Water wells</a></li>
