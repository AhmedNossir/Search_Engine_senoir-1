        <h2 class="post-title"><a href="http://andreasn.se/going-to-guadec-2016/" rel="bookmark" title="Going to GUADEC 2016">Going to GUADEC 2016</a></h2> 
        <div class="post-meta"> <span class="post-date"><a href="http://andreasn.se/going-to-guadec-2016/" title="Going to GUADEC 2016">August 10, 2016</a></span> <span class="date-sep"> / </span> <span class="post-author"><a href="http://andreasn.se/author/andreasn/" title="Posts by andreasn" rel="author">andreasn</a></span> <span class="date-sep"> / </span> <span>Comments Off<span class="screen-reader-text"> on Going to GUADEC 2016</span></span> 
        </div> 
       </div> <!-- /post-header --> 
       <div class="post-content"> 
        <p>Tomorrow I’ll be traveling to Karlsruhe, Germany to attend <a href="https://2016.guadec.org/">GUADEC 2016</a><br> It will be a great time to catch up with old friends and meeting new GNOME contributors.</p> 
        <p>On Sunday at 16:30 I’ll be giving a talk about how I used user research, in the form of interviews, to design public transportation routing in GNOME Maps.<br> I’ve just started dipping my toes into user research, and want to learn a lot more, so if you have experience about doing user research, or if you’re just generally interested in the subject, grab me in the hall at the conference. I would love to chat!<br> <a href="https://2016.guadec.org/schedule/">There is a ton of other great talks to see too.</a></p> 
        <p>On Monday morning there will be a <a href="https://wiki.gnome.org/GUADEC/2016/BOFs/Cockpit">Cockpit BOF</a> in room 131.<br> Even though Cockpit is a server UI, it builds upon a lot of technologies that originated in or around GNOME.<br> It’s also a great opportunity to see if there are more server-oriented-things that currently live in GNOME, but would make more sense to present in Cockpit.<br> One recent example of this is how support for advanced network setups, such as bonding, bridging and teaming was deprecated in GNOME Settings in favor of having it in Cockpit. All these technologies require at least two network cards, a setup that is much more common on server hardware than it is on your regular laptop or desktop.<br> It would be interesting to see if there are more things like this out there that can be migrated in a similar manner, to make sure that the right functionality lives in the right place.</p> 
        <p><img class="alignnone size-full wp-image-21" src="http://i2.wp.com/andreasn.se/wp-content/uploads/2016/08/badge-goingto.png?resize=256%2C256" alt="badge-goingto" srcset="http://i2.wp.com/andreasn.se/wp-content/uploads/2016/08/badge-goingto.png?w=256 256w, http://i2.wp.com/andreasn.se/wp-content/uploads/2016/08/badge-goingto.png?resize=150%2C150 150w" sizes="(max-width: 256px) 100vw, 256px" data-recalc-dims="1"></p>

        <h2 class="post-title"><a href="http://andreasn.se/a-tale-of-menus/" rel="bookmark" title="A tale of menus">A tale of menus</a></h2> 
        <div class="post-meta"> <span class="post-date"><a href="http://andreasn.se/a-tale-of-menus/" title="A tale of menus">July 23, 2009</a></span> <span class="date-sep"> / </span> <span class="post-author"><a href="http://andreasn.se/author/andreasn/" title="Posts by andreasn" rel="author">andreasn</a></span> <span class="date-sep"> / </span> <span>Comments Off<span class="screen-reader-text"> on A tale of menus</span></span> 
        </div> 
       </div> <!-- /post-header --> 
       <div class="post-content"> 
        <p>Had a great meeting with the rest of the GNOME Art Team at <a href="http://www.grancanariadesktopsummit.org/">GCDS</a>!<br> Together we came up with some points on where we would like to take GNOME visually in the coming 9 months.<br> One of the things we all agreed on is that a new widget theme is not going to be enough to create a visually stunning desktop.</p> 
        <p><img src="http://i0.wp.com/andreasn.se/wp-content/uploads/2009/07/massive-icons.png?resize=400%2C400" alt="" class="alignnone size-full wp-image-38" srcset="http://i0.wp.com/andreasn.se/wp-content/uploads/2009/07/massive-icons.png?w=400 400w, http://i0.wp.com/andreasn.se/wp-content/uploads/2009/07/massive-icons.png?resize=150%2C150 150w, http://i0.wp.com/andreasn.se/wp-content/uploads/2009/07/massive-icons.png?resize=300%2C300 300w" sizes="(max-width: 400px) 100vw, 400px" data-recalc-dims="1"></p> 
        <p><strong>Fewer but better</strong><br> At the same time as we’re introducing massive 256×256 icons for places that require 64×64 and up, we also want to take the opportunity to <a href="http://bugzilla.gnome.org/show_bug.cgi?id=557469">cut down a bit</a> on the massive amount of icons <a href="http://gnomefx.mozdev.org/blue-file.png">currently used in menus</a>. At the same time, we also want to introduce some guidelines on when to properly use them to enrich your interfaces.<br> The current approach is that some items have them, and some don’t, and this is because no artist had time to draw it, or because the action is too complex to convey in a small icon, or both. And hand to heart, that’s not a really good guideline.</p> 
        <p>Getting rid of things (or changing defaults for that matter) is always tricky, as the initial reaction from people used to the old behavior is that nothing of value gets added. However, we believe this is a visually more attractive default and that it will result in a cleaner and more efficient interface (and you can always change it back).</p> 
        <p><strong>What are the exceptions?</strong><br> A menu item shall have a icon if it represents a dynamic object such as a:</p> 
        <ul> 
         <li>Application</li> 
         <li>File or bookmark</li> 
         <li>Device</li> 
        </ul> 
        <p>How do I make sure the exceptions show in the menus?<br> Just patch your application to use <a href="http://library.gnome.org/devel/gtk/2.17/GtkImageMenuItem.html#gtk-image-menu-item-set-always-show-image">gtk-image-menu-item-set-always-show-image</a></p> 
        <p>Won’t this slow me down, as icons are so quick to spot?<br> While it’s true that the eye recognize color very quickly, having both text and image also means more information for the brain to process. It’s also worth to note that <a href="http://en.wikipedia.org/wiki/Reading_%28process%29#Reading_rate">text skimming speed for adults is around 400-700 wpm</a>.</p>
