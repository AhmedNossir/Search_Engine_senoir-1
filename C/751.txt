       <h3 id="notification-delivery-options"><a href="#notification-delivery-options">Notification delivery options</a></h3> 
       <p>You can receive notifications for activity on GitHub in the following locations.</p> 
       <ul> 
        <li>The notifications inbox in the GitHub web interface</li> 
        <li>The notifications inbox on GitHub for mobile, which syncs with the inbox on GitHub</li> 
        <li>An email client that uses a verified email address, which can also sync with the notifications inbox on GitHub and GitHub for mobile</li> 
       </ul> 
       <p>To use the notifications inbox on GitHub and GitHub for mobile, you must enable web and mobile notifications in your notification settings. For more information, see "<a href="#choosing-your-notification-settings">Choosing your notification settings</a>."</p> 
       <div class="extended-markdown tip border rounded-1 mb-4 p-3 color-border-info color-bg-info f5"> 
        <p><strong>Tip:</strong> If you receive both web and email notifications, you can automatically sync the read or unread status of the notification so that web notifications are automatically marked as read once you've read the corresponding email notification. To enable this sync, your email client must be able to view images from <code>notifications@github.com</code>.</p> 
       </div> 
       <h4 id="benefits-of-the-notifications-inbox"><a href="#benefits-of-the-notifications-inbox">Benefits of the notifications inbox</a></h4> 
       <p>The notifications inbox on GitHub and GitHub for mobile includes triaging options designed specifically for your GitHub notifications flow, including options to:</p> 
       <ul> 
        <li>Triage multiple notifications at once.</li> 
        <li>Mark completed notifications as <strong>Done</strong> and remove them from your inbox. To view all of your notifications marked as <strong>Done</strong>, use the <code>is:done</code> query.</li> 
        <li>Save a notification to review later. Saved notifications are flagged in your inbox and kept indefinitely. To view all of your saved notifications, use the <code>is:saved</code> query.</li> 
        <li>Unsubscribe and remove a notification from your inbox.</li> 
        <li>Preview the issue, pull request, or team discussion where the notification originates on GitHub from within the notifications inbox.</li> 
        <li>See one of the latest reasons you're receiving a notification from your inbox with a <code>reasons</code> label.</li> 
        <li>Create custom filters to focus on different notifications when you want.</li> 
        <li>Group notifications in your inbox by repository or date to get a quick overview with less context switching</li> 
       </ul> 
       <p>In addition, you can receive and triage notifications on your mobile device with GitHub for mobile. For more information, see "<a href="#managing-your-notification-settings-with-github-for-mobile">Managing your notification settings with GitHub for mobile</a>" or "<a href="/en/github/getting-started-with-github/github-for-mobile">GitHub for mobile</a>."</p> 
       <h4 id="benefits-of-using-an-email-client-for-notifications"><a href="#benefits-of-using-an-email-client-for-notifications">Benefits of using an email client for notifications</a></h4> 
       <p>One benefit of using an email client is that all of your notifications can be kept indefinitely depending on your email client's storage capacity. Your inbox notifications are only kept for 5 months on GitHub unless you've marked them as <strong>Saved</strong>. <strong>Saved</strong> notifications are kept indefinitely. For more information about your inbox's retention policy, see "<a href="/en/github/managing-subscriptions-and-notifications-on-github/about-notifications#notification-retention-policy">About notifications</a>."</p> 
       <p>Sending notifications to your email client also allows you to customize your inbox according to your email client's settings, which can include custom or color-coded labels.</p> 
       <p>Email notifications also allow flexibility with the types of notifications you receive and allow you to choose different email addresses for updates. For example, you can send certain notifications for a repository to a verified personal email address. For more information, about your email customization options, see "<a href="#customizing-your-email-notifications">Customizing your email notifications</a>."</p> 
       <h3 id="about-participating-and-watching-notifications"><a href="#about-participating-and-watching-notifications">About participating and watching notifications</a></h3> 
       <p>When you watch a repository, you're subscribing to updates for activity in that repository. Similarly, when you watch a specific team's discussions, you're subscribing to all conversation updates on that team's page. For more information, see "<a href="/en/organizations/collaborating-with-your-team/about-team-discussions">About team discussions</a>."</p> 
       <p>To see repositories that you're watching, go to your <a href="https://github.com/watching">watching page</a>. For more information, see "<a href="/en/github/managing-subscriptions-and-notifications-on-github/managing-subscriptions-for-activity-on-github">Managing subscriptions and notifications on GitHub</a>."</p> 
       <p>You can configure notifications for a repository on the repository page, or on your watching page.</p> 
       <h4 id="about-custom-notifications"><a href="#about-custom-notifications">About custom notifications</a></h4> 
       <p>You can customize notifications for a repository. For example, you can choose to only be notified when updates to one or more types of events (issues, pulls requests, releases, security alerts, or discussions) happen within a repository, or ignore all notifications for a repository. For more information, see "<a href="#configuring-your-watch-settings-for-an-individual-repository">Configuring your watch settings for an individual repository</a>" below.</p> 
       <h4 id="participating-in-conversations"><a href="#participating-in-conversations">Participating in conversations</a></h4> 
       <p>Anytime you comment in a conversation or when someone @mentions your username, you are <em>participating</em> in a conversation. By default, you are automatically subscribed to a conversation when you participate in it. You can unsubscribe from a conversation you've participated in manually by clicking <strong>Unsubscribe</strong> on the issue or pull request or through the <strong>Unsubscribe</strong> option in the notifications inbox.</p> 
       <p>For conversations you're watching or participating in, you can choose whether you want to receive notifications by email or through the notifications inbox on GitHub and GitHub for mobile.</p> 
       <p><img src="/assets/images/help/notifications-v2/participating-and-watching-options.png" alt="Participating and watching notifications options"></p> 
       <p>For example:</p> 
       <ul> 
        <li>If you don't want notifications to be sent to your email, unselect <strong>email</strong> for participating and watching notifications.</li> 
        <li>If you want to receive notifications by email when you've participated in a conversation, then you can select <strong>email</strong> under "Participating".</li> 
       </ul> 
       <p>If you do not enable watching or participating notifications for web and mobile, then your notifications inbox will not have any updates.</p> 
       <h3 id="customizing-your-email-notifications"><a href="#customizing-your-email-notifications">Customizing your email notifications</a></h3> 
       <p>After enabling email notifications, GitHub will send notifications to you as multipart emails that contain both HTML and plain text copies of the content. Email notification content includes any Markdown, @mentions, emojis, hash-links, and more, that appear in the original content on GitHub. If you only want to see the text in the email, you can configure your email client to display the plain text copy only.</p> 
       <div class="extended-markdown tip border rounded-1 mb-4 p-3 color-border-info color-bg-info f5"> 
        <p><strong>Tip:</strong> If you receive both web and email notifications, you can automatically sync the read or unread status of the notification so that web notifications are automatically marked as read once you've read the corresponding email notification. To enable this sync, your email client must be able to view images from <code>notifications@github.com</code>.</p> 
       </div> 
       <p>If you're using Gmail, you can click a button beside the notification email to visit the original issue or pull request that generated the notification.</p> 
       <p><img src="/assets/images/help/notifications/gmail-buttons.png" alt="Buttons in Gmail"></p> 
       <p>Choose a default email address where you want to send updates for conversations you're participating in or watching. You can also specify which activity on GitHub you want to receive updates for using your default email address. For example, choose whether you want updates to your default email from:</p> 
       <ul> 
        <li>Comments on issues and pull requests.</li> 
        <li>Pull request reviews.</li> 
        <li>Pull request pushes.</li> 
        <li>Your own updates, such as when you open, comment on, or close an issue or pull request.</li> 
       </ul> 
       <p>Depending on the organization that owns the repository, you can also send notifications to different email addresses. Your organization may require the email address to be verified for a specific domain. For more information, see "<a href="/en/github/managing-subscriptions-and-notifications-on-github/configuring-notifications#choosing-where-your-organizations-email-notifications-are-sent">Choosing where your organization’s email notifications are sent</a>."</p> 
       <p>You can also send notifications for a specific repository to an email address. For more information, see "<a href="/en/github/administering-a-repository/about-email-notifications-for-pushes-to-your-repository">About email notifications for pushes to your repository</a>."</p> 
       <p>You'll only receive notification emails if you've chosen to receive email notifications in your notification settings. </p> 
       <p>If an organization you're a member of restricts email notifications to an approved email domain, you'll need to verify an email address in that domain to receive email notifications about activity in the organization. For more information, see "<a href="/en/articles/restricting-email-notifications-to-an-approved-domain">Restricting email notifications to an approved domain</a>."</p> 
       <h3 id="filtering-email-notifications"><a href="#filtering-email-notifications">Filtering email notifications</a></h3> 
       <p>Each email notification that GitHub sends contains header information. The header information in every email is consistent, so you can use it in your email client to filter or forward all GitHub notifications, or certain types of GitHub notifications.</p> 
       <p>If you believe you're receiving notifications that don't belong to you, examine the <code>X-GitHub-Recipient</code> and <code>X-GitHub-Recipient-Address</code> headers. These headers show who the intended recipient is. Depending on your email setup, you may receive notifications intended for another user.</p> 
       <p>Email notifications from GitHub contain the following header information:</p> 
       <table>
        <thead>
         <tr>
          <th>Header</th>
          <th>Information</th>
         </tr>
        </thead>
        <tbody>
         <tr>
          <td><code>From</code> address</td>
          <td>This address will always be '<code>notifications@github.com</code>'.</td>

       <h3 id="automatic-watching"><a href="#automatic-watching">Automatic watching</a></h3> 
       <p>By default, anytime you gain access to a new repository, you will automatically begin watching that repository. Anytime you join a new team, you will automatically be subscribed to updates and receive notifications when that team is @mentioned. If you don't want to automatically be subscribed, you can unselect the automatic watching options.</p> 
       <p> <img src="/assets/images/help/notifications-v2/automatic-watching-options.png" alt="Automatic watching options"></p> 
       <p>If "Automatically watch repositories" is disabled, then you will not automatically watch your own repositories. You must navigate to your repository page and choose the watch option.</p>

       <h3 id="dependabot-alerts-notification-options"><a href="#dependabot-alerts-notification-options">Dependabot alerts notification options</a></h3> 
       <p>You can choose the delivery method and frequency of notifications about Dependabot alerts on repositories that you are watching or where you have subscribed to notifications for security alerts.</p> 
       <p>By default, you will receive notifications:</p> 
       <ul> 
        <li>by email, an email is sent when Dependabot is enabled for a repository, when a new manifest file is committed to the repository, and when a new vulnerability with a critical or high severity is found (<strong>Email each time a vulnerability is found</strong> option).</li> 
        <li>in the user interface, a warning is shown in your repository's file and code views if there are any vulnerable dependencies (<strong>UI alerts</strong> option).</li> 
        <li>on the command line, warnings are displayed as callbacks when you push to repositories with any vulnerable dependencies (<strong>Command Line</strong> option).</li> 
        <li>in your inbox, as web notifications. A web notification is sent when Dependabot is enabled for a repository, when a new manifest file is committed to the repository, and when a new vulnerability with a critical or high severity is found (<strong>Web</strong> option).</li> 
        <li>on GitHub for mobile, as web notifications. For more information, see "<a href="/en/github/managing-subscriptions-and-notifications-on-github/configuring-notifications#enabling-push-notifications-with-github-for-mobile">Enabling push notifications with GitHub for mobile</a>."</li> 
       </ul> 
       <div class="extended-markdown note border rounded-1 mb-4 p-3 color-border-info color-bg-info f5"> 
        <p><strong>Note:</strong> The email and web/GitHub for mobile notifications are:</p> 
        <ul> 
         <li> <p><em>per repository</em> when Dependabot is enabled on the repository, or when a new manifest file is committed to the repository.</p> </li> 
         <li> <p><em>per organization</em> when a new vulnerability is discovered.</p> </li> 
        </ul> 
       </div>

For example, you can receive a weekly digest email summarizing alerts for up to 10 of your repositories using the <strong>Email a digest summary of vulnerabilities</strong> and <strong>Weekly security email digest</strong> options.</p>
