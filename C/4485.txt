      <p><a href="https://en.wikipedia.org/wiki/geocoding" class="extiw" title="w:geocoding">Co?rafi Kodlama</a>, Wikimedia Commons'un medyan?n co?rafi konumunu belirlemek için kullan?lan uygulamas?d?r. When we geocode media on commons we identify the latitude and longitude from <b>where the media was recorded</b> and attach it to the description page using a <a href="/wiki/Template:Location" title="Template:Location"><span style="font-family:monospace,monospace;">{{Location}}</span></a> or <a href="/wiki/Template:Location_dec" class="mw-redirect" title="Template:Location dec"><span style="font-family:monospace,monospace;">{{Location dec}}</span></a> template. For jpeg images, geodata can also be stored in the image <a href="/wiki/Commons:Manipulating_meta_data#Purpose_for_using_EXIF_at_Commons" class="mw-redirect" title="Commons:Manipulating meta data">exif</a>. </p>

      <h2><span class="mw-headline" id="Geocoding_media_on_Commons">Geocoding media on Commons</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Commons:Co%C4%9Frafi_Kodlama&amp;action=edit&amp;section=1" title="Edit section: Geocoding media on Commons">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <p>Geocoding is easy and only takes a couple of steps. </p>
      <p>On Commons we record <b>the location of the camera</b> as accurately as we can. The locations of subjects should be geocoded on their Wikipedia articles, and the Commons description should link to those Wikipedia articles. </p> 
      <h2><span class="mw-headline" id="Automatic_procedures">Automatic procedures</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Commons:Co%C4%9Frafi_Kodlama&amp;action=edit&amp;section=2" title="Edit section: Automatic procedures">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <p>GPS or other automated methods can be especially useful in rural locations without landmarks which can be easily recognized in the aerial images. If your camera has a built-in <i>GPS receiver</i> (as most smartphone camera phones do) then your work is already done. Since November 2011, the Upload Wizard adds the proper Geocoding Templates from the GPS data stored in your images. For pictures not uploaded through the Wizard, <a href="/wiki/Template:GPS_EXIF" title="Template:GPS EXIF"><span style="font-family:monospace,monospace;">{{GPS EXIF}}</span></a> can be added to summon a bot to code them. </p>
      <p>You can also use a separate GPS receiver to record the location while you are recording the media, and use computer software to merge the locations into the image files. Such software uses the camera time tag in the image file to look up the coordinates from the GPS. For accurate results, the camera time must be correct or the time error entered manually. The time error can be monitored by photographing the time displayed by the GPS. Errors caused by spotty GPS coverage can be adjusted by the same tools that are used for manual geocoding, though more easily since you already have an approximate position. </p> 
      <h3><span class="mw-headline" id="Adding_Geotags_before_uploading">Adding Geotags before uploading</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Commons:Co%C4%9Frafi_Kodlama&amp;action=edit&amp;section=3" title="Edit section: Adding Geotags before uploading">edit</a><span class="mw-editsection-bracket">]</span></span></h3> 
      <p>Some software, such as <a rel="nofollow" class="external text" href="http://www.microsoft.com/downloads/en/details.aspx?FamilyID=184075d2-40b5-4172-88ae-878f81896d4d">Microsoft Pro Photo Tools</a> or <a rel="nofollow" class="external text" href="http://picasa.google.com/">Picasa</a> *, allow you to add geographical information to images from a map. The upload wizard will also recognize these tags. You cannot add camera directions by this procedure, but programs such as <a rel="nofollow" class="external text" href="http://www.geosetter.de/en">Geosetter</a> can insert metadata including&nbsp;: geographic coordinates of the location shooting, camera direction, and the name of the exact location. </p>
      <p>If your camera has accidentally tagged your images with completely wrong geodata, you can remove them on a batch of pictures by using <a rel="nofollow" class="external text" href="http://freeweb.siol.net/hrastni3/foto/exif/exiftoolgui.htm">ExifToolGUI</a>. It uses ExifTool that is also installed by Geosetter mentioned above. </p>
      <p>* <a rel="nofollow" class="external text" href="http://picasa.google.com/features.html">Picasa web albums</a> also allow addition of geographical information from a map. But this information does not appear to be embeded in the image files and cannot be pulled back to ones own computer. </p> 
      <h2><span class="mw-headline" id="Fully_manual_procedure">Fully manual procedure</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Commons:Co%C4%9Frafi_Kodlama&amp;action=edit&amp;section=4" title="Edit section: Fully manual procedure">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <p>The easiest way to identify the location in urban areas is to use an online mapping site with aerial imaging such as: </p> 
      <h3><span class="mw-headline" id="GeoLocator_tool">GeoLocator tool</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Commons:Co%C4%9Frafi_Kodlama&amp;action=edit&amp;section=5" title="Edit section: GeoLocator tool">edit</a><span class="mw-editsection-bracket">]</span></span></h3> 
      <ul>
       <li><a rel="nofollow" class="external free" href="http://tools.freeside.sk/geolocator/geolocator.html">http://tools.freeside.sk/geolocator/geolocator.html</a> (<a href="https://en.wikipedia.org/wiki/User:Teslaton/Tools/GeoLocator" class="extiw" title="w:User:Teslaton/Tools/GeoLocator">project page</a>)</li>
      </ul> 
      <p>Steps to obtain coordinates and build appropriate location tag: </p> 
      <ol>
       <li>find your location on embedded map (move/zoom/search it as needed)</li> 
       <li>place location marker on most exact location ([Alt+click] over your location, or by dragging a marker)</li> 
       <li>define <a href="#Parameters">heading</a> (camera view direction) from that location ([Shift+mouse move]/[Shift+click])</li> 
       <li>copy resulting <code>{{location}}</code> or <code>{{location dec}}</code> markup</li>
      </ol> 
      <p>Steps to update an existing location tag (to improve location or to add heading): </p> 
      <ol>
       <li>paste existing location tag into query box and press Apply. Tag gets parsed and location will appear on a map.</li> 
       <li>update location marker to more exact location (by dragging it or by [Alt+click] over new location)</li> 
       <li>define <a href="#Parameters">heading</a> (camera view direction) from that location ([Shift+mouse move]/[Shift+click])</li> 
       <li>copy resulting <code>{{location}}</code> or <code>{{location dec}}</code> markup back</li>
      </ol>

      <ul>
       <li><a rel="nofollow" class="external free" href="http://www.giswiki.org/hjl_geocoding.htm">http://www.giswiki.org/hjl_geocoding.htm</a></li>

       <li>click on <i>location d</i> or <i>location dms</i> button</li> 
       <li>copy text</li>

      <h4><span class="mw-headline" id="Manual_method">Manual method</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Commons:Co%C4%9Frafi_Kodlama&amp;action=edit&amp;section=8" title="Edit section: Manual method">edit</a><span class="mw-editsection-bracket">]</span></span></h4> 
      <ul>
       <li>Go to <a rel="nofollow" class="external text" href="http://www.wikimapia.org">WikiMapia.org</a></li> 
       <li>(Optional) Set View-&gt;Hybrid in the menu, so you can see the roads.</li>

      <h4><span class="mw-headline" id="Method_1">Method 1</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Commons:Co%C4%9Frafi_Kodlama&amp;action=edit&amp;section=10" title="Edit section: Method 1">edit</a><span class="mw-editsection-bracket">]</span></span></h4> 
      <ol>
       <li>Go to <a rel="nofollow" class="external text" href="http://maps.google.com/maps?f=q&amp;q=http:%2F%2Ftools.wikimedia.de%2F~kolossos%2Fwp-world%2FCoordsForWikipedia.kml&amp;ie=UTF8&amp;ll=52.48278,14.051514&amp;spn=38.87701,82.089844&amp;t=k&amp;z=4&amp;om=1">Google Maps with crosshair</a></li> 
       <li>Zoom-in to the lowest resolution over the place you want to geotag by double-clicking on the map</li> 
       <li>Place the cross hair exactly over the place and click on it.</li> 
       <li>Two templates will be shown one in <a href="/wiki/Template:Coor_dms" class="mw-redirect" title="Template:Coor dms"><span style="font-family:monospace,monospace;">{{Coor dms}}</span></a> and one in <a href="/wiki/Template:Location" title="Template:Location"><span style="font-family:monospace,monospace;">{{Location}}</span></a> format.</li>

          </dl>
         </dd>
        </dl>

        </ol>
       </dd>
      </dl> 
      <ul>
       <li>For each new location:</li>
      </ul> 
      <dl>
       <dd>

         <li>Click on "Satellite" if it helps to find the location more easily, and zoom in.</li> 
         <li>Click on the exact location the media was recorded to center the map.</li> 
         <li>Click on your bookmark and you'll get a prompt with the geocode tag to add to the image.</li>
        </ol>
       </dd>
      </dl>

      <h2><span class="mw-headline" id="Geodetic_system">Geodetic system</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Commons:Co%C4%9Frafi_Kodlama&amp;action=edit&amp;section=15" title="Edit section: Geodetic system">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <p>All coordinates should be referenced to the <a href="https://en.wikipedia.org/wiki/WGS84" class="extiw" title="en:WGS84">en:WGS84</a> datum, the one supported by GPS systems and Google Maps. References taken from printed maps are unlikely to use this datum, and lead to inaccuracies in the order of hundreds of metres. National Grid reference systems use a local datum so will provide the wrong result. </p> 
      <h2><span class="mw-headline" id="Adding_a_location_template">Adding a location template</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Commons:Co%C4%9Frafi_Kodlama&amp;action=edit&amp;section=16" title="Edit section: Adding a location template">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <p>The use of the following templates is encouraged: </p> 
      <ul>
       <li><a href="/wiki/Template:Location" title="Template:Location"><span style="font-family:monospace,monospace;">{{Location}}</span></a> for degree, minute, second coordinates</li> 
       <li><a href="/wiki/Template:Location_dec" class="mw-redirect" title="Template:Location dec"><span style="font-family:monospace,monospace;">{{Location dec}}</span></a> for decimal degree coordinates 
        <ul>
         <li>Example for southern hemisphere, decimal: location dec|-7.58|-14.24</li>
        </ul></li>
      </ul>

Commons geocoding needs are also not the same as those of other projects.</b> Note that you also should not create your own wrappers around <a href="/wiki/Template:Location" title="Template:Location"><span style="font-family:monospace,monospace;">{{Location}}</span></a> or <a href="/wiki/Template:Location_dec" class="mw-redirect" title="Template:Location dec"><span style="font-family:monospace,monospace;">{{Location dec}}</span></a>; this will break the automation as well. Just call one of the templates directly from the image description page. </p>

      <h3><span class="mw-headline" id="Parameters">Parameters</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Commons:Co%C4%9Frafi_Kodlama&amp;action=edit&amp;section=17" title="Edit section: Parameters">edit</a><span class="mw-editsection-bracket">]</span></span></h3> 
      <p>Wikipedia geocoding often uses <a href="https://en.wikipedia.org/wiki/Wikipedia:WikiProject_Geographical_coordinates#Parameters" class="extiw" title="w:Wikipedia:WikiProject Geographical coordinates">parameters</a> after the coordinates to give more information about the location, but some of them are not useful here. </p> 
      <ul>
       <li><i>type</i> and <i>scale</i> - Redundant on Commons, as most images are at lowest scale. The defaults are <i>type:landmark</i> and <i>scale:5000</i>. These parameters should be given only if values different from the defaults are desired.</li> 
       <li><i>heading</i> - Used to indicate the direction the camera was pointing at. It is given as degree values <i>0-360</i> (clockwise with north as 0) or a compass point abbreviation as defined in <a href="https://en.wikipedia.org/wiki/Boxing_the_compass" class="extiw" title="w:Boxing the compass">w:Boxing the compass</a>. (Example: <i>heading:NW</i>)</li>
      </ul> 
      <h4><span class="mw-headline" id="Heading_parameter_related_tools">Heading parameter related tools</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Commons:Co%C4%9Frafi_Kodlama&amp;action=edit&amp;section=18" title="Edit section: Heading parameter related tools">edit</a><span class="mw-editsection-bracket">]</span></span></h4> 
      <ul>
       <li>Compass overlay can be displayed on Google Maps or in Google Earth to help find the correct heading: 
        <ul>
         <li>Google Maps: <a rel="nofollow" class="external text" href="http://maps.google.com/?q=http:%2F%2Ftoolserver.org%2F~dschwen%2Fkml.php%3Fpage%3DCommons:Geocoding%2FCompass_overlay.kml">map with fullscreen compass overlay</a></li> 
         <li>Google Earth: <a rel="nofollow" class="external text" href="http://toolserver.org/~dschwen/kml.php?page=Commons:Geocoding/Compass_overlay.kml">Compass_overlay.kml</a> can be added to "My Places" and displayed when necessary. The overlay does not change when the view is rotated, so remember to click on the top right compass to reset the viewpoint orientation.</li>
        </ul></li> 
       <li>Google Earth 5.0 shows the heading in degrees with the "Ruler" tool, when a line is drawn from the camera location to the object in the center of the photo. The heading does not depend on viewpoint orientation.</li> 
       <li><a rel="nofollow" class="external text" href="http://tools.freeside.sk/geolocator/geolocator.html">GeoLocator</a> tool allows interactive picking of correct heading on a map as part of metadata authoring.</li>
      </ul> 
      <h2><span class="mw-headline" id="Use_of_the_information">Use of the information</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Commons:Co%C4%9Frafi_Kodlama&amp;action=edit&amp;section=19" title="Edit section: Use of the information">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <p>Clicking on the coordinates on an Image page will lead to a list of various services that can show the location. It is also possible to browse Commons images on a map: </p> 
      <ul>
       <li>A world map with thumbnails of geocoded images can be found from <a href="//www.mediawiki.org/wiki/Toolserver:WikiMiniAtlas" class="extiw" title="tswiki:WikiMiniAtlas">WikiMiniAtlas on Commons</a>.</li> 
       <li>Some Geocoded images from Commons plotted in Google Earth with <a rel="nofollow" class="external text" href="http://toolserver.org/~para/GeoCommons/">GeoCommons</a> (images tagged with <a href="/wiki/Template:Object_location" title="Template:Object location">{{Object location}}</a> are not displayed).</li> 
       <li>Some geocoded images from Commons plotted in <a rel="nofollow" class="external text" href="http://maps.google.com/maps?q=http:%2F%2Ftoolserver.org%2F~para%2FGeoCommons%2FGeoCommons-simple.kml&amp;ie=UTF8&amp;om=1&amp;ll=35.46067,15.46875&amp;spn=90.926037,111.445313&amp;z=3">Google Maps</a> (images tagged with <a href="/wiki/Template:Object_location" title="Template:Object location">{{Object location}}</a> are not displayed).</li> 
       <li>In the "gadget" tab of <a href="/wiki/Special:Preferences" title="Special:Preferences">Special:Preferences</a>, there is an option "View category on Google maps". If enabled, this adds a link "map" to all categories. This shows the location of geocoded images in the category on Google maps.</li> 
       <li>Adding <a href="/wiki/Template:GeoGroupTemplate" class="mw-redirect" title="Template:GeoGroupTemplate"><span style="font-family:monospace,monospace;">{{GeoGroupTemplate}}</span></a> to a category page adds links to view the images in the category with various maps.</li>
      </ul> 
      <h2><span class="mw-headline" id="Things_to_do">Things to do</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Commons:Co%C4%9Frafi_Kodlama&amp;action=edit&amp;section=20" title="Edit section: Things to do">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <ul>
       <li>Add geocoding to any location related media on Commons. Areas you know personally are easiest, but geocoding images from other areas based on satellite data and other photos is also possible. For help in finding material to work on, choose an interesting category and enter the name in <a href="//toolserver.org/~para/GeoCommons/geocodingtodo.php" class="extiw" title="tools:~para/GeoCommons/geocodingtodo.php">this tool</a> to see a gallery of images that need to be geocoded.</li> 
       <li>Find coordinates for untaged pictures tagged with <a href="/wiki/Template:Location_possible" title="Template:Location possible"><span style="font-family:monospace,monospace;">{{Location possible}}</span></a> in: <a href="/wiki/Category:Location_possible" title="Category:Location possible">Category:Location possible</a>.</li> 
       <li>Refine inaccurate geocoding on images that are off the correct camera location, or have been positioned in exactly the same place as a number of other images (see <a href="//toolserver.org/~para/GeoCommons/inaccurates.php" class="extiw" title="tools:~para/GeoCommons/inaccurates.php">list</a>)</li> 
       <li>Monitor and fix tags of images with incorrect geocoding in <a href="/wiki/Category:Media_with_erroneous_locations" title="Category:Media with erroneous locations">Category:Media with erroneous locations</a> and recently geocoded images (<a href="//toolserver.org/~para/GeoCommons/recentchanges.php" class="extiw" title="tools:~para/GeoCommons/recentchanges.php">gallery</a>).</li> 
       <li>Find coordinates with unknown headings and try to fix them (see <a rel="nofollow" class="external text" href="http://toolserver.org/~dispenser/cgi-bin/geosearch.py?page=heading:\?&amp;site=commons">list</a>).</li> 
       <li>Verify or fix coordinates of images imported from external sources where geocoding may be less conscientious (see for example <a rel="nofollow" class="external text" href="http://toolserver.org/~dispenser/cgi-bin/geosearch.py?regexp=source%3A%5BFf%5Dlickr&amp;type=regex&amp;site=commons">Flickr images</a>)</li> 
       <li>Replace most usages of <a href="/wiki/Special:WhatLinksHere/Template:Coor_d" title="Special:WhatLinksHere/Template:Coor d">{{coor d}}</a>, <a href="/wiki/Special:WhatLinksHere/Template:Coor_dm" title="Special:WhatLinksHere/Template:Coor dm">{{coor dm}}</a> and <a href="/wiki/Special:WhatLinksHere/Template:Coor_dms" title="Special:WhatLinksHere/Template:Coor dms">{{coor dms}}</a> with {{<a href="/wiki/Template:Location" title="Template:Location">Location</a>}} or {{<a href="/wiki/Template:Location_dec" class="mw-redirect" title="Template:Location dec">Location dec</a>}} in "File:" namespace pages.</li>
      </ul> 
      <h2><span class="mw-headline" id="Special:Preferences"><a href="/wiki/Special:Preferences" title="Special:Preferences">Special:Preferences</a></span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Commons:Co%C4%9Frafi_Kodlama&amp;action=edit&amp;section=21" title="Edit section: Special:Preferences">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <p>Some of the tools can be activate in the "gadget" tab of <a href="/wiki/Special:Preferences" title="Special:Preferences">Special:Preferences</a>: </p> 
      <ul>
       <li><i>Locate images on Google Maps</i>: On Category pages, adds a "Map" link to locate <a href="/wiki/Commons:Geocoding" title="Commons:Geocoding">geocoded</a> images from that category on Google Maps. <small>[<a href="/wiki/MediaWiki_talk:Gadget-Gmaps.js" title="MediaWiki talk:Gadget-Gmaps.js">talk</a>]</small>&nbsp;</li> 
       <li><i>Geocoding tools</i>: Adds three <a href="/wiki/Commons:Geocoding" title="Commons:Geocoding">geocoding</a> links to the toolbox: <a href="//toolserver.org/~para/GeoCommons/recentchanges.php" class="extiw" title="tools:~para/GeoCommons/recentchanges.php">recent changes</a>, <a href="//toolserver.org/~dispenser/view/File_viewer#log:coord-commonswiki.log" class="extiw" title="tools:~dispenser/view/File viewer">daily error log</a>, and <a href="//toolserver.org/~dispenser/cgi-bin/geosearch.py%3Fsite%3Dcommons" class="extiw" title="tools:~dispenser/cgi-bin/geosearch.py?site=commons">search</a>. <small>[<a href="/wiki/MediaWiki_talk:Gadget-Geotoolbox.js" title="MediaWiki talk:Gadget-Geotoolbox.js">talk</a>]</small>&nbsp;</li> 
       <li><i>Geocoding To Do</i>: Add a tab to the category linking to the <a href="//toolserver.org/~para/GeoCommons/geocodingtodo.php" class="extiw" title="tools:~para/GeoCommons/geocodingtodo.php">tool</a> allowing to view images that are not <a href="/wiki/Commons:Geocoding" title="Commons:Geocoding">geocoded</a>. <small>[<a href="/wiki/MediaWiki_talk:Gadget-Geocodecattodo.js" title="MediaWiki talk:Gadget-Geocodecattodo.js">talk</a>]</small>&nbsp;</li>
      </ul>
