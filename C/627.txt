      <p>En <b>kategori</b> är en funktion i <a href="https://sv.wikipedia.org/wiki/Mediawiki" class="extiw" title="sv:Mediawiki">wikimjukvaran</a> som Commons använder. Funktionen gör det möjligt att gruppera sidor och mediafiler som har med varandra att göra. Varje enskilt ämne kan ha en egen kategori. Kategorisidorna har namn som anger vilket ämne som samlas i respektive kategori. Vid behov kan dessa namn kompletteras med ytterligare beskrivande text. </p>
      <p>Kategoristrukturen är det viktigaste verktyget för att sortera och hitta filer på Commons. Alla filer måste kunna hittas med hjälp av kategorisystemet. För att möjliggöra detta måste alla filer placeras i minst en kategori. Kategorierna ska också placeras i mer generella kategorier, så att de bildar en hierarki. </p>

      <h2><span id=".C3.96versikt"></span><span class="mw-headline" id="Översikt">Översikt</span></h2> 
      <p>1. Hur man hittar lämpliga kategorier </p> 
      <dl>
       <dd>
        <ul>
         <li>Använd <a class="external text" href="https://commons.wikimedia.org/w/index.php?title=Special:Search&amp;ns14">sökfunktionen</a> (se avsnittet <a href="#Tips_vid_kategorisering">Tips vid kategorisering</a>)</li> 
         <li>eller se efter hur liknande filer är kategoriserade</li> 
         <li>eller pröva verktyget <a href="//toolserver.org/~daniel/WikiSense/CommonSense.php" class="extiw" title="tools:~daniel/WikiSense/CommonSense.php">Common Sense</a></li> 
         <li>eller börja på <a href="/wiki/Category:Topics" title="Category:Topics">toppnivån</a> i kategoristrukturen och leta dig ner till mer specifika kategorier</li> 
         <li>Undersök alltid vilka underkategorier som finns och använd så specifika kategorier som möjligt</li>
        </ul>
       </dd>
      </dl> 
      <p>2. Hur man lägger till kategorier till en bild </p> 
      <dl>
       <dd>
        <ul>
         <li>Välj lämpliga kategorier på uppladdningsformuläret</li> 
         <li>eller lägg till dem manuellt genom att redigera filbeskrivningssidan och skriva in <code>[[Category:Kategorinamn]]</code> (se avsnittet <a href="#Att_kategorisera_sidor">Att kategorisera sidor</a>)</li> 
         <li>eller aktivera och använd <a href="/wiki/Special:Preferences#mw-prefsection-gadgets" title="Special:Preferences">finessen</a> <a href="/wiki/Help:Gadget-HotCat" title="Help:Gadget-HotCat">Hotcat</a></li> 
         <li>eller aktivera och använd <a href="/wiki/Special:Preferences#mw-prefsection-gadgets" title="Special:Preferences">finessen</a> <a href="/wiki/Help:Gadget-Cat-a-lot" title="Help:Gadget-Cat-a-lot">Cat-a-lot</a></li>
        </ul>
       </dd>
      </dl> 
      <h2><span class="mw-headline" id="Kategoristruktur">Kategoristruktur</span></h2> 
      <h3><span class="mw-headline" id="Principer">Principer</span></h3> 
      <p>Huvudprinciperna är: </p> 
      <h4><span id="Den_hierarkiska_principen_.3D"></span><span class="mw-headline" id="Den_hierarkiska_principen_=">Den hierarkiska principen =</span></h4> 
      <p>Den idealiska kategoristrukturen kan definieras som en acyklisk <a href="https://sv.wikipedia.org/wiki/riktad_graf" class="extiw" title="sv:riktad graf">orienterad graf</a> som utgår ifrån exakt en nod, <a href="/wiki/Category:CommonsRoot" title="Category:CommonsRoot">CommonsRoot</a>. Det innebär att: </p> 
      <ul>
       <li>alla kategorier (utom <a href="/wiki/Category:CommonsRoot" title="Category:CommonsRoot">CommonsRoot</a>) ska placeras i minst en överordnad kategori</li> 
       <li>ingen cirkulär kategorisering ska förekomma (det vill säga att ingen kategori ska vara en direkt eller indirekt underkategori till sig själv)</li>
      </ul> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <h4><span class="mw-headline" id="Modularity_principle">Modularity principle</span></h4> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>The page (file, category) should be put in the most specific category/categories that fit(s) the page (not directly to its parent categories). A category can have more parent categories. A category can combine two (or more) different criteria; such categories are called "compound categories" or "intersection categories". E.g. the root category <a href="/wiki/Category:Churches" title="Category:Churches">Category:Churches</a> and the root category <a href="/wiki/Category:Russia" title="Category:Russia">Category:Russia</a> have a common subcategory <a href="/wiki/Category:Churches_in_Russia" title="Category:Churches in Russia">Churches in Russia</a>. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <h4><span class="mw-headline" id="Simplicity_principle">Simplicity principle</span></h4> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>This principle suggests not to combine too many different criteria. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <h4><span class="mw-headline" id="Selectivity_principle">Selectivity principle</span></h4> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>We should not classify items which are related to different subjects in the same category. The category name should be unambiguous and not homonymous. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <h4><span class="mw-headline" id="Universality_principle">Universality principle</span></h4> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>Identical items should have identical names for all countries and at all levels of categorization. Categorization structure should be as systematical and unified as possible, local dialects and terminology should be supressed in favour of universality if possible. Analogic categorization branches should have analogic structure. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <h3><span class="mw-headline" id="Types_of_reflected_relations">Types of reflected relations</span></h3> 
      </div> 
      <p>Kategoristrukturen ska utgöra en hierarki bestående av generella koncept som bryts ner i mer specifika koncept. Strukturen använder och kombinerar flera sorters relationer, till exempel: </p> 
      <ul>
       <li><a href="https://sv.wikipedia.org/wiki/hyponymi" class="extiw" title="sv:hyponymi">hyponymi</a>: en sorts… (typiskt inom <a href="https://sv.wikipedia.org/wiki/taxonomi" class="extiw" title="sv:taxonomi">taxonomi</a>)</li> 
       <li><a href="https://sv.wikipedia.org/wiki/meronymi" class="extiw" title="sv:meronymi">meronymi</a>: en del av… (typiskt för geografiska indelningar, byggnader/rum, enhet/komponent och liknande)</li> 
       <li>attribut: färg, form, storlek, förmåga, nationalitet, teknik, kvalité, utmärkelser och liknande 
        <ul>
         <li>placering</li> 
         <li>tidpunkt (avbildad händelse, födelsedatum, dödsdatum, tillkomst, rivningsdatum, nedläggning och liknande)</li> 
         <li>påverkan</li> 
         <li>tillhörighet</li>
        </ul></li>
      </ul> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <ul>
        <li>Agentive and influence relations: (creator/work, device/product, company/product, discipline or profession/their subjects and terms, parent/children, subordination, owner/property, initiator/follower, subject/other subjects dedicated to it or named after it, subject/its duplicate, imitiation, depiction or symbol etc etc.)</li> 
        <li>Modification: original/modified or modified/original (avoid cyclic structure) – renamed, rebuilt, repurposed or transformed subjects.</li>
       </ul> 
      </div> 
      <h3><span id="Toppniv.C3.A5kategorier"></span><span class="mw-headline" id="Toppnivåkategorier">Toppnivåkategorier</span></h3> 
      <p>De kategorier som är placerade direkt i <a href="/wiki/Category:CommonsRoot" title="Category:CommonsRoot">CommonsRoot</a> delar in kategoristrukturen i ett antal övergripande syften: </p> 
      <ul>
       <li><b><a href="/wiki/Category:Topics" title="Category:Topics">Ämnen</a></b> – Kategoriserar mediafiler baserat på de ämnen som de illustrerar. Alla mediafiler ska placeras i minst en kategori i denna del av kategoristrukturen. Ämneskategorier ska anges direkt och inte via <a href="/wiki/Special:MyLanguage/Commons:Templates" title="Special:MyLanguage/Commons:Templates">mallar</a>.</li> 
       <li><b><a href="/wiki/Category:Copyright_statuses" title="Category:Copyright statuses">Upphovsrättsstatus</a></b> – Kategoriserar mediafiler baserat på licenser eller andra upphovsrättsliga förhållanden. Alla mediafiler ska placeras i minst en kategori i denna del av kategoristrukturen. Upphovsrättskategorier tilldelas via <a href="/wiki/Commons:Copyright_tags/sv" title="Commons:Copyright tags/sv">upphovsrättsmallar</a>.</li> 
       <li><b><a href="/wiki/Category:Image_sources" title="Category:Image sources">Källor</a></b> – Kategoriserar mediafiler baserat på var de kommer ifrån (arkivsamlingar, böcker, webbplatser och så vidare). Källkategorier tilldelas vanligtvis via <a href="/wiki/Special:MyLanguage/Commons:Templates" title="Special:MyLanguage/Commons:Templates">mallar</a> på filbeskrivningssidan.</li> 
       <li><b><a href="/wiki/Category:Media_types" title="Category:Media types">Mediatyper</a></b> – Kategoriserar mediafiler baserat på vilken typ av media de innehåller (animationer, ljud, video och så vidare). Bildfiler placeras inte alltid i någon kategori i denna del av kategoristrukturen, eftersom de allra flesta mediafilerna på Commons är bilder.</li>
      </ul> 
      <div class="mw-translate-fuzzy"> 
       <ul>
        <li><b><a href="/wiki/Category:Commons" title="Category:Commons">Commons</a></b> – Kategoriserar innehåll som rör arbetet på Wikimedia Commons (hjälpsidor, projekt, riktlinjer och så vidare). Strukturen i <a href="/wiki/Category:Commons-en" title="Category:Commons-en">Commons-en</a> utgör referens för innehåll på andra språk, inklusive <a href="/wiki/Category:Commons-sv" title="Category:Commons-sv">svenska</a>.</li> 
        <li><b><a href="/wiki/Category:User_categories" title="Category:User categories">Användare</a></b> – Kategoriserar användarsidor efter deltagarnas egenskaper (språk, kunskaper, särskilda rättigheter och så vidare) samt användares egna gallerisidor och kategorier.</li>
       </ul> 
      </div> 
      <h2><span id="Att_anv.C3.A4nda_kategorier"></span><span class="mw-headline" id="Att_använda_kategorier">Att använda kategorier</span></h2> 
      <p>Placera alltid filer som du laddar upp i ämneskategorier så att andra kan hitta och använda dina bidrag. </p>
      <p>I de flesta fall finns redan de kategorier som behövs. Om du behöver skapa en ny kategori, bekanta dig med den existerande kategoristrukturen och försäkra dig om att det verkligen inte finns någon likvärdig kategori samt att du följer den praxis och de regler som tillämpas bland existerande kategorier. Om det finns en <a href="/wiki/Category:Commons_category_schemes" title="Category:Commons category schemes">kategoristruktur</a> eller ett <a href="/wiki/Category:Commons_projects" title="Category:Commons projects">projekt</a> som beskriver hur ämnet ifråga bör kategoriseras, följ den beskrivningen. </p>
      <p><span id="Category_names"></span> </p> 
      <h3><span class="mw-headline" id="Kategorinamn">Kategorinamn</span></h3> 
      <p>Kategorinamn ska i allmänhet vara på engelska i enlighet med Commons <a href="/wiki/Special:MyLanguage/Commons:Language_policy" title="Special:MyLanguage/Commons:Language policy">språkpolicy</a>, men det finns undantag, såsom vissa egennamn, biologisk systematik och uttryck som saknar motsvarighet på engelska. För närvarande <a href="https://bugzilla.wikimedia.org/show_bug.cgi?id=29928" class="extiw" title="bugzilla:29928">saknas tekniskt stöd för flerspråkiga kategorinamn</a>, och parallella kategorisystem för olika språk skulle snabbt bli oöverskådliga. </p>
      <p>Namn på kategorier som rör typer av föremål eller grupper av människor skrivs i allmänhet i pluralform, till exempel <a href="/wiki/Category:Tools" title="Category:Tools">Category:Tools</a> (verktyg), <a href="/wiki/Category:Artists" title="Category:Artists">Category:Artists</a> (konstnärer), <a href="/wiki/Category:Lakes" title="Category:Lakes">Category:Lakes</a> (sjöar), <a href="/wiki/Category:Paintings" title="Category:Paintings">Category:Paintings</a> (målningar) och <a href="/wiki/Category:Sculptures" title="Category:Sculptures">Category:Sculptures</a> (skulpturer). Detta gäller dock inte generella koncept och aktiviteter, till exempel <a href="/wiki/Category:History" title="Category:History">Category:History</a> (historia), <a href="/wiki/Category:Weather" title="Category:Weather">Category:Weather</a> (väder), <a href="/wiki/Category:Music" title="Category:Music">Category:Music</a> (musik), <a href="/wiki/Category:Painting" title="Category:Painting">Category:Painting</a> (målning som aktivitet). Det gäller heller inte kategorier som rör en enskild person eller ett enskilt föremål, till exempel ett specifikt byggnadsverk, monument eller konstverk. </p> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>There is also a related <a href="/wiki/Special:MyLanguage/Commons:Naming_categories" title="Special:MyLanguage/Commons:Naming categories">failed proposal</a>. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>Categories grouping subcategories by name should generally be named "by name" rather than "by alphabet" (e.g. <a href="/wiki/Category:Ships_by_name" title="Category:Ships by name">Category:Ships by name</a>). </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>We still lack internationalization for category names, but this issue should be resolved with appropriate changes to the MediaWiki software (see <a href="https://phabricator.wikimedia.org/T31928" class="extiw" title="phab:T31928">T31928</a>: <i>Show translated titles per user language in categories too</i>). Creating intermingled category structures in different languages would only make things worse. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>For a general discussion of MediaWiki's category feature, see the <a href="https://meta.wikimedia.org/wiki/Help:Categories" class="extiw" title="m:Help:Categories">manual page on categories</a>. </p> 
      </div> 
      <h3><span class="mw-headline" id="Att_kategorisera_sidor">Att kategorisera sidor</span></h3> 
      <p>För att placera en sida i en kategori (oavsett om det är en filbeskrivningssida, gallerisida eller en annan kategorisida) redigerar man sidan och lägger till följande: </p>
      <p><code>[[Category:<i>Kategorinamn</i>]]</code> </p>
      <p>Om du till exempel laddat upp ett diagram som visar kometers omloppsbana kan du lägga till följande på filbeskrivningssidan: <code> [[Category:Astronomical diagrams]]<br> [[Category:Comets]] </code> </p>
      <p>Diagrammet kommer då att synas i kategorierna <a href="/wiki/Category:Astronomical_diagrams" title="Category:Astronomical diagrams">Astronomical diagrams</a> (astronomiska diagram) och <a href="/wiki/Category:Comets" title="Category:Comets">Comets</a> (kometer). </p>
      <p>För information om att hitta lämpliga kategorier att lägga till, se avsnittet <a href="#Att_hitta_lämpliga_kategorier">Att hitta lämpliga kategorier</a>. </p> 
      <h3><span class="mw-headline" id="Att_skapa_en_kategori">Att skapa en kategori</span></h3> 
      <p>För att skapa en ny kategori: </p> 
      <ol>
       <li>Gör en noggrann sökning för att säkerställa att någon motsvarande kategori inte redan finns.</li> 
       <li>Sök efter mediafiler, gallerisidor eller andra sidor som passar in i kategorin, placera dessa i den tilltänkta kategorin genom att redigera dem, lägga till <code>[[Category:<i>Kategorinamn</i>]]</code> och spara. Kategorin syns som en röd länk (en länk till en icke existerande sida) längst ner på respektive sida.</li> 
       <li>Klicka på den röda länken. Du kan nu redigera kategorisidan som vilken annan wikisida som helst.</li>
      </ol> 
      <p>En kategorisida bör innehålla följande information: </p> 
      <ul>
       <li>Kategorilänkar som placerar kategorin i minst en överordnad kategori. Lägg till en eller flera rader enligt formen <code>[[Category:<i>Kategorinamn</i>]]</code> i slutet av sidan (men ovanför eventuella interwikilänkar).</li> 
       <li>En kort beskrivning av vad kategorin avser (om kategorinamnet inte är tillräckligt tydligt på egen hand). Beskrivningar på olika språk kan anges med hjälp av <a href="/wiki/Special:MyLanguage/Commons:Language_templates" title="Special:MyLanguage/Commons:Language templates">språkmallar</a> som <code>{{<a href="/wiki/Template:En" title="Template:En">en</a>|<i>Engelskspråkig beskrivning</i>}}</code>, <code>{{<a href="/wiki/Template:Sv" title="Template:Sv">sv</a>|<i>Svenskspråkig beskrivning</i>}}</code> och liknande eller med mallen <a href="/wiki/Template:Multilingual_description" title="Template:Multilingual description"><span style="font-family:monospace,monospace;">{{Multilingual description}}</span></a>.</li>
      </ul> 
      <div class="mw-translate-fuzzy"> 
       <ul>
        <li>Interwikilänkar till motsvarande sida på Wikipedia, till exempel <code>[[en:Category:<i>Kategorinamn på engelskspråkiga Wikipedia</i>]]</code> och <code>[[sv:Kategori:<i>Kategorinamn på svenskspråkiga Wikipedia</i>]]</code>. Placera dessa sist på sidan.</li> 
        <li>En sorteringssträng om kategorin bör sorteras baserat på något annat än kategorinamnet. Till exempel bör man lägga till en rad enligt formen <code>{{DEFAULTSORT:Efternamn, Förnamn}}</code> ovanför kategorilänkarna i kategorier namngivna efter personer.</li>
       </ul> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>See also <a href="#How_to_categorize:_guidance_by_topic">#How to categorize: guidance by topic</a> for guidance on specific classes of category, e.g. categories about <a href="#People">#People</a>. </p> 
      </div> 
      <h3><span id="Att_flytta_eller_byta_namn_p.C3.A5_en_kategori"></span><span class="mw-headline" id="Att_flytta_eller_byta_namn_på_en_kategori">Att flytta eller byta namn på en kategori</span></h3> 
      <p>Se <a href="/wiki/Commons:Rename_a_category/sv" title="Commons:Rename a category/sv">Commons:Rename a category/sv</a>. </p> 
      <h2><span id="F.C3.B6r_l.C3.A4mpligare_kategorisering"></span><span class="mw-headline" id="För_lämpligare_kategorisering">För lämpligare kategorisering</span></h2> 
      <p>Gallerier och kategorier placeras inte i kategorier baserat på vad de för tillfället innehåller, utan baserat på det <i>ämne</i> som de behandlar. </p>
      <p>Anta till exempel att <a href="/wiki/Category:Spheres" title="Category:Spheres">Category:Spheres</a> (sfärer) endast innehåller bilder av glaskulor. Eftersom det finns många andra material som en sfär kan bestå av vore det olämligt att placera <a href="/wiki/Category:Spheres" title="Category:Spheres">Category:Spheres</a> i <a href="/wiki/Category:Glass" title="Category:Glass">Category:Glass</a> (glas) baserat på detta innehåll. Bilder som föreställer sfärer av just glas ska normalt redan vara placerade i <a href="/wiki/Category:Glass" title="Category:Glass">Category:Glass</a> eller någon av dess underkategorier. Så om <a href="/wiki/Category:Spheres" title="Category:Spheres">Category:Spheres</a> är fylld med bilder på glaskulor är det lämpligare att skapa <a href="/wiki/Category:Glass_spheres" title="Category:Glass spheres">Category:Glass spheres</a> (glassfärer), placera den i <a href="/wiki/Category:Spheres" title="Category:Spheres">Category:Spheres</a> samt <a href="/wiki/Category:Glass" title="Category:Glass">Category:Glass</a> och flytta ner innehåll till den nya underkategorin. </p>
      <p>Normalt ska filer endast placeras i den mest specifika kategorin för ett särskilt ämne. Till exempel ska filer som hör hemma i <a href="/wiki/Category:Looking_up_the_center_of_the_Eiffel_Tower" title="Category:Looking up the center of the Eiffel Tower">Category:Looking up the center of the Eiffel Tower</a> (Eiffeltornet sett rakt underifrån) inte samtidigt placeras i <a href="/wiki/Category:Paris" title="Category:Paris">Category:Paris</a>. Se avsnittet <a href="#Överkategorisering">Överkategorisering</a>. Om du inte hittar en lämplig kategori kan du skapa en, men du bör först läsa avsnittet <a href="#Att_använda_kategorier">Att använda kategorier</a>. </p>
      <p>Detta betyder inte att filer ska placeras i en enda kategori. Det betyder bara att de inte ska placeras i <i>överflödiga</i> eller onödigt generella kategorier. En bild av en isbjörn som räddas från ett isberg av en helikopter bör till exempel placeras i <a href="/wiki/Category:Ursus_maritimus" title="Category:Ursus maritimus">Category:Ursus maritimus</a> (isbjörnar), <a href="/wiki/Category:Icebergs" title="Category:Icebergs">Category:Icebergs</a> (isberg) och <a href="/wiki/Category:Rescue_helicopters" title="Category:Rescue helicopters">Category:Rescue helicopters</a> (räddningshelikoptrar), men den bör inte placeras i <a href="/wiki/Category:Ursidae" title="Category:Ursidae">Category:Ursidae</a> (björnar) eller <a href="/wiki/Category:Aircraft" title="Category:Aircraft">Category:Aircraft</a> (flygmaskiner). </p> 
      <h3><span class="mw-headline" id="Tips_vid_kategorisering">Tips vid kategorisering</span></h3> 
      <p>De kategorier som används för en fil ska besvara så många som möjligt av följande frågor: </p> 
      <ul>
       <li><b>Vem eller vad</b> föreställer filen? Till exempel <a href="/wiki/Category:Automobiles" title="Category:Automobiles">Category:Automobiles</a> (bilar), <a href="/wiki/Category:Felis_silvestris_catus" title="Category:Felis silvestris catus">Category:Felis silvestris catus</a> (katter), <a href="/wiki/Category:Yellow_animals" title="Category:Yellow animals">Category:Yellow animals</a> (gula djur) eller <a href="/wiki/Category:Jimmy_Wales" title="Category:Jimmy Wales">Category:Jimmy Wales</a>.</li> 
       <li><b>Var</b> är filen skapad? Var är det avbildade föremålet placerat? Var är kameran placerad? Till exempel <a href="/wiki/Category:Basin_Street,_New_Orleans" title="Category:Basin Street, New Orleans">Category:Basin Street, New Orleans</a>. (Använd även <a href="/wiki/Template:Location" title="Template:Location"><span style="font-family:monospace,monospace;">{{Location}}</span></a>-mallen för att ange var bilden är tagen.)</li> 
       <li><b>När</b> skapades filen? När inträffade de avbildade händelserna? Detta är särskilt viktigt för historiska bilder. Till exempel <a href="/wiki/Category:Warsaw_in_September_1939" title="Category:Warsaw in September 1939">Category:Warsaw in September 1939</a> (Warszawa i september 1939) och <a href="/wiki/Category:April_2010_in_Northern_Ireland" title="Category:April 2010 in Northern Ireland">Category:April 2010 in Northern Ireland</a> (april 2010 i Nordirland).</li> 
       <li><b>Vem</b> har skapat innehållet? Detta är särskilt viktigt för verk av kända konstnärer. Till exempel <a href="/wiki/Category:Paintings_by_Rembrandt" title="Category:Paintings by Rembrandt">Category:Paintings by Rembrandt</a> (målningar av Rembrandt). Du kan även använda sidor från <a href="/wiki/Category:Creator_templates" title="Category:Creator templates"><i>Creator</i>-namnrymden</a> som mallar för att åstadkomma den här kategoriseringen.</li> 
       <li><b>Hur</b> presenteras innehållet? Till exempel särskilda vyer som <a href="/wiki/Category:Plan_views" title="Category:Plan views">Category:Plan views</a> (planvyer) och <a href="/wiki/Category:Panoramics" title="Category:Panoramics">Category:Panoramics</a> (panoramavyer) eller särskilda <a href="/wiki/Category:Photographic_techniques" title="Category:Photographic techniques">fotografiska tekniker</a> eller <a href="/wiki/Category:Photographic_effects" title="Category:Photographic effects">effekter</a> såsom <a href="/wiki/Category:Tone-mapped_HDR_images" title="Category:Tone-mapped HDR images">Category:Tone-mapped HDR images</a> (tonmappade högintensitetsomfångsfotografier), <a href="/wiki/Category:Black_and_white_photographs" title="Category:Black and white photographs">Category:Black and white photographs</a> (svartvita fotografier) och <a href="/wiki/Category:Double_exposure" title="Category:Double exposure">Category:Double exposure</a> (dubbelexponeringar).</li>
      </ul> 
      <p>Ovanstående frågeställningar omfattar de viktigaste kategoriseringsaspekterna. För vissa filer är det lämpligt att tillämpa samtliga. För andra filer är kanske endast någon eller några relevanta. Det finns även andra aspekter som kan användas för kategorisering utöver den grundläggande ämneskategoriseringen: </p> 
      <ul>
       <li><b>Källa:</b> Var kommer filen ifrån? Till exempel <a href="/wiki/Category:Images_from_the_German_Federal_Archive" title="Category:Images from the German Federal Archive">Category:Images from the German Federal Archive</a> (filer från tyska riksarkivet).</li> 
       <li><b>Format:</b> För ovanligare filtyper. Till exempel <a href="/wiki/Category:Audio" title="Category:Audio">Category:Audio</a> (ljudfiler), <a href="/wiki/Category:Animated_GIF" title="Category:Animated GIF">Category:Animated GIF</a> (animerade GIF-bilder) och <a href="/wiki/Category:SVG" class="mw-disambig" title="Category:SVG">Category:SVG</a> (skalbar vektorgrafik).</li> 
       <li><b>Programvara:</b> Information om datorprogram som använts för att skapa filen. Till exempel <a href="/wiki/Category:Created_with_Hugin" title="Category:Created with Hugin">Category:Created with Hugin</a> (panoramabilder sammanfogade med hjälp av programmet Hugin).</li> 
       <li><b>Kamera:</b> Till exempel <a href="/wiki/Category:Taken_with_Nikon_D80" title="Category:Taken with Nikon D80">Category:Taken with Nikon D80</a> (fotografier tagna med kameran Nikon D80).</li>
      </ul> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>Categorization in Wikimedia Commons is more detailed and deep than categorization in Wikipedia projects. Compared to them, Commons has more categories for individual subjects – places, people, organizations, events, terms, etc. Almost every article on a Wikipedia can have a corresponding category on Commons. However, even if there exist more images of an ordinary person or incidental event, it is practical to group them into a special category and categorize that category instead of categorizing all similar images individually to an identical set of parent categories. </p> 
      </div> 
      <h3><span id="Hitta_en_l.C3.A4mplig_kategori"></span><span class="mw-headline" id="Hitta_en_lämplig_kategori">Hitta en lämplig kategori</span></h3> 
      <p>För att hitta en lämplig kategori att placera en fil i kan man börja med att gå till en övergripande kategori och leta sig ner till en underkategori som är så specifik som möjligt, antingen genom att klicka på de länkade namnen på underkategorierna eller genom att klicka på plustecknet bredvid kategorinamnet för att flika ut underkategorierna. Listan i avsnittet <a href="#Toppnivåkategorier">Toppnivåkategorier</a> kan användas som utgångspunkt. Avsnittet <a href="#Kategorisering_av_specifika_ämnen">Kategorisering av specifika ämnen</a> ger ytterligare vägledning. Du kan även pröva <a href="//toolserver.org/~daniel/WikiSense/CommonSense.php" class="extiw" title="tools:~daniel/WikiSense/CommonSense.php">Common Sense</a>, ett verktyg för att hitta kategorier med hjälp av nyckelord. </p> 
      <h3><span id=".C3.96verkategorisering"></span><span class="mw-headline" id="Överkategorisering">Överkategorisering</span></h3> 
      <div class="mw-translate-fuzzy"> 
       <div class="wpImageAnnotatorControl wpImageAnnotatorOff">
        <div class="thumb tright">
         <div class="thumbinner" style="width:222px;">
          <a href="/wiki/File:Over-categorization.svg" class="image"><img alt="" src="https://upload.wikimedia.org/wikipedia/commons/thumb/4/44/Over-categorization.svg/220px-Over-categorization.svg.png" decoding="async" width="220" height="310" class="thumbimage" srcset="https://upload.wikimedia.org/wikipedia/commons/thumb/4/44/Over-categorization.svg/330px-Over-categorization.svg.png 1.5x, https://upload.wikimedia.org/wikipedia/commons/thumb/4/44/Over-categorization.svg/440px-Over-categorization.svg.png 2x" data-file-width="220" data-file-height="310"></a> 
          <div class="thumbcaption">
           <div class="magnify">
            <a href="/wiki/File:Over-categorization.svg" class="internal" title="Enlarge"></a>
           </div>Placera inte saker både i en överordnad och en underordnad kategori. Till exempel bör ett svartvitt foto av Eiffeltornet placeras i kategorin för <a href="/wiki/Category:Black_and_white_photographs_of_the_Eiffel_Tower" title="Category:Black and white photographs of the Eiffel Tower">svartvita fotografier av Eiffeltornet</a>. Det bör <i>inte</i> placeras i den kategorin och samtidigt i <a href="/wiki/Category:Paris" title="Category:Paris">Pariskategorin</a>.
          </div>
         </div>
        </div>
       </div> 
       <p>Överkategorisering är när en fil, kategori eller annan sida placeras i flera nivåer i samma gren av kategoriträdet. Filer ska i regel alltid placeras i så specifika kategorier som möjligt och inte i därtill överordnade kategorier. </p> 
      </div> 
      <p><b>Exempel:</b> En bild som behöver kategoriseras föreställer en gul cirkel. Bilden bör placeras i <a href="/wiki/Category:Yellow_circles" title="Category:Yellow circles">Category:Yellow circles</a> (gula cirklar). Om den dessutom placeras i <a href="/wiki/Category:Circles" title="Category:Circles">Category:Circles</a> (cirklar) är den <i>överkategoriserad.</i> Vi vet redan att det är en cirkel eftersom alla gula cirklar är cirklar. Därför är <a href="/wiki/Category:Circles" title="Category:Circles">Category:Circles</a> överflödig. </p>
      <p>Detta gäller de flesta filer: Som det står i bildtexten intill bör filer i <a href="/wiki/Category:Black_and_white_photographs_of_the_Eiffel_Tower" title="Category:Black and white photographs of the Eiffel Tower">Category:Black and white photographs of the Eiffel Tower</a> (svartvita fotografier av Eiffeltornet) inte också placeras i <a href="/wiki/Category:Paris" title="Category:Paris">Category:Paris</a>, filer i <a href="/wiki/Category:Albert_Einstein" title="Category:Albert Einstein">Category:Albert Einstein</a> bör inte placeras i <a href="/wiki/Category:Physicists_from_Germany" title="Category:Physicists from Germany">Category:Physicists from Germany</a> (fysiker från Tyskland) och så vidare. </p> 
      <h4><span id="D.C3.A4rf.C3.B6r_.C3.A4r_.C3.B6verkategorisering_ett_problem"></span><span class="mw-headline" id="Därför_är_överkategorisering_ett_problem">Därför är överkategorisering ett problem</span></h4> 
      <p>Det är ett vanligt (men felaktigt) antagande att ju fler kategorier en fil är placerad i, desto lättare är det att hitta den. Med den logiken borde alla bilder som föreställer män placeras i kategorin <a href="/wiki/Category:Men" title="Category:Men">Category:Men</a> (män). Resultatet blir att kategorin fylls av tiotusentals filer, vilket knappast gör det lättare att hitta en specifik fil. Den som letar en bild för att illustrera en Wikipediaartikel om <a href="https://sv.wikipedia.org/wiki/man" class="extiw" title="sv:man">män</a> får svårt att hitta en lämpligt generell bild bland alla filmstjärnor, vetenskapsmän och politiker. </p>
      <p>Längre ner i kategoriträdet uppstår andra problem. Om man letar efter bild av Einstein och vet att han var fysiker kanske man börjar leta i <a href="/wiki/Category:Physicists" title="Category:Physicists">Category:Physicists</a> (fysiker). Om man då hittar en halvbra bild där kan man förledas att tro att det är den enda bilden som finns istället för att leta i <a href="/wiki/Category:Physicists_from_Germany" title="Category:Physicists from Germany">Category:Physicists from Germany</a> och därmed hitta <a href="/wiki/Category:Albert_Einstein" title="Category:Albert Einstein">Category:Albert Einstein</a>, där det kan finnas bättre bilder. Alltså leder överkategorisering til två problem: att överordnade kategorier blir oöverskådliga samt att användare förleds att sluta leta efter en mer specifik kategori med fler relevanta filer när de hittat en relevant fil i en överordnad kategori. </p> 
      <h4><span id="D.C3.A5lig_kategorisering_av_kategorier_orsakar_.C3.B6verkategorisering"></span><span class="mw-headline" id="Dålig_kategorisering_av_kategorier_orsakar_överkategorisering">Dålig kategorisering av kategorier orsakar överkategorisering</span></h4> 
      <p>Det kan låta konstigt, men underkategorisering kan orsaka överkategorisering. När en kategori i sig inte är korrekt placerad i kategoriträdet kan det leda till att användare överkategoriserar filer som hör hemma i den kategorin. Ett exempel på detta: <a href="/wiki/Category:Eiv%C3%B8r_P%C3%A1lsd%C3%B3ttir" title="Category:Eivør Pálsdóttir">Category:Eivør Pálsdóttir</a> var ursprungligen endast placerad i <a href="/wiki/Category:People_by_name" title="Category:People by name">Category:People by name</a> (en övergripande samlingskategori för att lista personer i alfabetisk ordning). En användare som kategoriserar en bild av henne kan frestas att placera bilden i <a href="/wiki/Category:Female_vocalists_from_the_Faroe_Islands" title="Category:Female vocalists from the Faroe Islands">Category:Female vocalists from the Faroe Islands</a> (kvinnliga vokalister från Färöarna). Den korrekta lösningen är att endast placera bilden i <a href="/wiki/Category:Eiv%C3%B8r_P%C3%A1lsd%C3%B3ttir" title="Category:Eivør Pálsdóttir">Category:Eivør Pálsdóttir</a> och att göra den kategorin till en underkategori till i <a href="/wiki/Category:Female_vocalists_from_the_Faroe_Islands" title="Category:Female vocalists from the Faroe Islands">Category:Female vocalists from the Faroe Islands</a>. Då blir dock alla bilder som redan placerats i båda kategorierna överkategoriserade och måste manuellt tas bort ifrån den överordnade kategorin. </p>
      <p>Ett relaterat problem är felkategoriserade kategorier. Notting Hill är en stadsdel i stadskommunen Kensington och Chelsea i London. När kategorin <a href="/wiki/Category:Notting_Hill" title="Category:Notting Hill">Category:Notting Hill</a> skapades placerades den direkt i <a href="/wiki/Category:London" title="Category:London">Category:London</a> istället för i underkategorin <a href="/wiki/Category:Royal_Borough_of_Kensington_and_Chelsea" title="Category:Royal Borough of Kensington and Chelsea">Category:Royal Borough of Kensington and Chelsea</a>, där den borde ha placerats. En användare som kategoriserar en bild av Notting Hill kan då frestas att placera bilden både i <a href="/wiki/Category:Notting_Hill" title="Category:Notting Hill">Category:Notting Hill</a> och i <a href="/wiki/Category:Royal_Borough_of_Kensington_and_Chelsea" title="Category:Royal Borough of Kensington and Chelsea">Category:Royal Borough of Kensington and Chelsea</a>. Istället bör alla filer placeras i så specifika kategorier som möjligt, och de kategorierna bör i sin tur placeras i så specifika kategorier som möjligt. </p>
      <p>När du påträffar dåligt kategoriserade kategorier, placera dem i lämpliga överordnade kategorier om du kan. Det hjälper inte bara till att undvika överkategorisering; det gör det också lättare att navigera i kategoriträdet. </p> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <h4><span class="mw-headline" id="Exception_for_images_with_more_categorized_subjects"><span id="OvercatException"></span> Exception for images with more categorized subjects</span></h4> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>A file that depicts only one relevant subject should not be over-categorized. Where a file depicts additional relevant subjects, and the additional subjects do not have their own subcategories, consideration can be given to temporarily categorizing the image in both the subcategory and the parent category. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>For example, this situation might arise in the case of a photograph of three politicians, one of whom is <a href="/wiki/Category:Angela_Merkel" title="Category:Angela Merkel">Angela Merkel</a> (who has her own Commons category), with two other politicians who do not yet have their own categories. While the image would undoubtedly be categorized in <a href="/wiki/Category:Angela_Merkel" title="Category:Angela Merkel">Category:Angela Merkel</a> or one of its subcategories, it would typically be considered to be over-categorization to also include it in <a href="/wiki/Category:Politicians_of_Germany" title="Category:Politicians of Germany">Category:Politicians of Germany</a>. Users would, however, be unlikely to search for the two other politicians in the Merkel category. Ideally, we would create specific subcategories for the two other politicians (where warranted), or find other relevant subcategories (e.g. <a href="/wiki/Category:Politicians_of_Bavaria" title="Category:Politicians of Bavaria">Category:Politicians of Bavaria</a> or <a href="/wiki/Category:Members_of_the_FDP" title="Category:Members of the FDP">Category:Members of the FDP</a>, etc.), that would enable us to avoid over-categorization. In some circumstances, however, we may need to temporarily categorize the image in <a href="/wiki/Category:Politicians_of_Germany" title="Category:Politicians of Germany">Category:Politicians of Germany</a> where other appropriate subcategories do not yet exist. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>Countries may be categorized as part of multiple overlapping categories. For example, <a href="/wiki/Category:India" title="Category:India">Category:India</a> is in <a href="/wiki/Category:Countries_of_South_Asia" title="Category:Countries of South Asia">Category:Countries of South Asia</a> as well as <a href="/wiki/Category:Countries_of_Asia" title="Category:Countries of Asia">Category:Countries of Asia</a>. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>Also user categories are exempted of over-categorization as those are not visible to most viewers, and project users include them for many different purposes like sorting, stats, filling values for userboxes, etc. </p> 
      </div> 
      <h2><span id="Kategorisering_av_specifika_.C3.A4mnen"></span><span class="mw-headline" id="Kategorisering_av_specifika_ämnen">Kategorisering av specifika ämnen</span></h2> 
      <p>För vissa ämnen finns särskild vägledning i form av <a href="/wiki/Category:Commons_category_schemes" title="Category:Commons category schemes">kategoriplaner</a> eller <a href="/wiki/Category:Commons_projects" title="Category:Commons projects">projekt</a>. Ytterligare ämnesspecifik vägledning finns i det här avsnittet, och ibland finns även rekommendationer högst upp på kategorisidorna. Till exempel finns det vägledning både på kategorisidan <a href="/wiki/Category:People" title="Category:People">Category:People</a> (personer) och i avsnittet om <a href="#Personer">personer</a> som följer. </p> 
      <h3><span class="mw-headline" id="Personer">Personer</span></h3> 
      <p>Innehåll som föreställer personer bör placeras i kategorier som beskriver dem, såsom <a href="/wiki/Category:Economists_from_the_United_States" title="Category:Economists from the United States">Category:Economists from the United States</a> (ekonomer från USA). Börja utforska kategoriträdet med <a href="/wiki/Category:People" title="Category:People">Category:People</a> (personer) som utgångspunkt. </p>
      <p>Se kategoriplanen <a href="/wiki/Special:MyLanguage/Commons:Category_scheme_People" title="Special:MyLanguage/Commons:Category scheme People">Commons:Category scheme People</a> för detaljerad information om hur dessa kategorier namnges och organiseras. </p> 
      <h3><span id="Landskap_och_utomhusmilj.C3.B6er"></span><span class="mw-headline" id="Landskap_och_utomhusmiljöer">Landskap och utomhusmiljöer</span></h3> 
      <p>Innehåll som visar ett motiv från en samma betrakningspunkt grupperas i kategorier som namnges enligt formen <i>Views of Subject from Viewpoint</i> (vyer av Motiv från Betraktningspunkt) såsom <a href="/wiki/Category:Views_of_Cathedral_of_Seville_from_the_Giralda" title="Category:Views of Cathedral of Seville from the Giralda">Category:Views of Cathedral of Seville from the Giralda</a> (vyer över Katedralen i Sevilla från Giralda). Sådana kategorier bör vara underkategorier till såväl motivets kategori (<a href="/wiki/Category:Cathedral_of_Seville" title="Category:Cathedral of Seville">Cathedral of Seville</a> i det här exemplet) som betraktningspunktens kategori (<a href="/wiki/Category:Giralda" title="Category:Giralda">Giralda</a> i det här exemplet). </p>
      <p>I det här exemplet är kategorin <a href="/wiki/Category:Views_of_Cathedral_of_Seville_from_the_Giralda" title="Category:Views of Cathedral of Seville from the Giralda">Views of Cathedral of Seville from the Giralda</a> inte placerad direkt i kategorierna för motivet och betraktningspunkten, utan i <a href="/wiki/Category:Views_of_the_Cathedral_of_Seville" title="Category:Views of the Cathedral of Seville">Views of the Cathedral of Seville</a> och <a href="/wiki/Category:Views_from_Giralda" title="Category:Views from Giralda">Views from Giralda</a>. Sådana mellankategorier är ofta nödvändiga för att skapa struktur och undvika <a href="#Överkategorisering">överkategorisering</a>, särskilt för vyer över städer från en utsiktsplats belägen i staden. Till exempel behöver <a href="/wiki/Category:Views_of_Rome_from_the_Pincio" title="Category:Views of Rome from the Pincio">Views of Rome from the Pincio</a> (vyer över Rom från Pincio) mellankategorin <a href="/wiki/Category:Views_of_Rome" title="Category:Views of Rome">Views of Rome</a> för att undvika att placera den direkt i <a href="/wiki/Category:Rome" title="Category:Rome">Rome</a>, vilket skulle utgöra överkategorisering. </p> 
      <h3><span class="mw-headline" id="Texter">Texter</span></h3> 
      <p>Texter, såsom inskannade böcker, bör normalt ha en kategori för varje inläsning och utgåva av texten. En bok som givits ut i tre upplagor har då en överordnad kategori för boken, tre underkategorier för varje utgåva samt ytterligare underkategorier för JPEG-versioner, DjVu-versioner och så vidare, förutsatt att alla versionerna har laddats upp till Commons. (Kategorier för versioner som inte finns på Commons behöver inte skapas.) Detta är särskilt viktigt för filer i andra format än DjVu och PDF, eftersom kategorier är det enda praktiska sättet att gruppera dessa; se till exempel <a href="/wiki/Category:The_Chronicles_of_England,_Scotland_and_Ireland,_Holinshed,_1587" title="Category:The Chronicles of England, Scotland and Ireland, Holinshed, 1587">Category:The Chronicles of England, Scotland and Ireland, Holinshed, 1587</a>, som innehåller 2&nbsp;857 inlästa sidor sparade som enskilda JPEG-bilder. </p>

       <h3><span class="mw-headline" id="GLAMs">GLAMs</span></h3> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>For categorization issues related to mass content donations from GLAMs (Galleries, Libraries, Archives &amp; Museums), please see <a href="/wiki/Special:MyLanguage/Commons:Guide_to_batch_uploading#Categories" title="Special:MyLanguage/Commons:Guide to batch uploading">Commons:Guide to batch uploading#Categories</a>. </p>

      <h2><span id="Arbetsfl.C3.B6de"></span><span class="mw-headline" id="Arbetsflöde">Arbetsflöde</span></h2> 
      <p>För närvarande finns en robot som kontrollerar att filer som nyligen laddats upp är placerade i ämneskategorier. Om så inte är fallet försöker den kategorisera filen automatiskt. Arbetsflödet är som följer: </p> 
      <div class="mw-translate-fuzzy"> 
       <ol>
        <li>En användare laddar upp en fil (och kategoriserar den eventuellt)</li> 
        <li><a href="/wiki/User:CategorizationBot" title="User:CategorizationBot">CategorizationBot</a> kontrollerar om filen är kategoriserad 
         <ul>
          <li>Filen är redan kategoriserad ? OK</li> 
          <li>Filen är inte kategoriserad ? Roboten medelar den som laddat upp filen och försöker kategorisera filen baserat på dess användning i gallerier och på Wikipedia 
           <ul>
            <li>Roboten lägger till kategorier och märker filen för granskning</li> 
            <li>Roboten kan inte kategorisera filen baserat på användning ? Roboten placerar filen i en underkategori till <a href="/wiki/Category:Media_needing_categories" title="Category:Media needing categories">Category:Media needing categories</a> (filer i behov av kategorisering). Dagens okategoriserade filer finns i <a href="/w/index.php?title=Category:Media_needing_categories_as_of_6_juni_2021&amp;action=edit&amp;redlink=1" class="new" title="Category:Media needing categories as of 6 juni 2021 (page does not exist)">Category:Media needing categories as of 6 juni 2021</a>.</li>
           </ul></li>
         </ul></li> 
        <li>Användare kategoriserar filen vidare (till exempel genom att flytta filer till underkategorier)</li>
       </ol> 
      </div> 
      <p>Se även <a href="/wiki/User:CategorizationBot#Process" title="User:CategorizationBot">processbeskrivningen</a> och <a href="/wiki/User:Multichill/Categorization_stats" title="User:Multichill/Categorization stats">statistik</a> för CategorizationBot. </p>
      <p>Andra, manuella arbetsflöden är möjliga: </p> 
      <ul>
       <li><b>Hitta filer att placera i en kategori:</b> Välj ut en kategori, använd lämpliga sökord för att hitta filer som hör hemma i den och flytta filerna dit eller till en lämplig underkategori.</li> 
       <li><b>Flytta filer till underkategorier:</b> Välj ut en <a href="/wiki/Category:Categories_requiring_diffusion" title="Category:Categories requiring diffusion">kategori med filer som borde flyttas till underkategorier</a> och flytta dem till lämpliga underkategorier. <a href="/wiki/Special:Preferences#mw-prefsection-gadgets" title="Special:Preferences">Finesser</a> som <a href="/wiki/Help:Gadget-Cat-a-lot" title="Help:Gadget-Cat-a-lot">Cat-a-lot</a> och <a href="/wiki/Help:Gadget-HotCat" title="Help:Gadget-HotCat">Hotcat</a> kan vara till hjälp.</li>
      </ul> 
      <h2><span id="Kategorier_m.C3.A4rkta_med_.22HIDDENCAT.22"></span><span class="mw-headline" id="Kategorier_märkta_med_&quot;HIDDENCAT&quot;">Kategorier märkta med "HIDDENCAT"</span></h2> 
      <div class="shortcut2" style="">
       <style data-mw-deduplicate="TemplateStyles:r433650343">.mw-parser-output .shortcut2{float:right;margin:0.4em;border:1px solid #ccc;background-color:white;padding:0 1em;text-align:center;font-size:x-small;border-radius:4px}</style><a href="/wiki/Commons:Shortcuts" title="Commons:Shortcuts">Shortcut</a>
       <br><a href="/wiki/Commons:HIDDENCAT" class="mw-redirect" title="Commons:HIDDENCAT">COM:HIDDENCAT</a>
      </div> 
      <p>Många kategorier som inte är ämneskategorier är märkta med __HIDDENCAT__ eller {{<a href="/wiki/Template:Hiddencat" title="Template:Hiddencat">Hiddencat</a>}} ("dold kategori"). Se till exempel <a class="external text" href="https://commons.wikimedia.org/w/index.php?title=Category:PD_NASA&amp;action=edit">Category:PD NASA i redigeringsläge</a>. </p>
      <p>Kategorier är normalt synliga överallt, men kategorier märkta med __HIDDENCAT__ är bara synliga: </p> 
      <ul>
       <li>på redigeringssidan: mot slutet av sidan, under textrutan</li> 
       <li>på kategorisidor: 
        <ul>
         <li>i underkategorier till den dolda kategorin: på dess vanliga plats, men på en egen rad med mindre teckensnitt märkt "Hidden categories"</li> 
         <li>i överordnade kategorier: på samma sätt som alla andra kategorier</li> 
         <li>på filbeskrivningssidor och gallerisidor: för inloggade användare som valt att "Visa dolda kategorier" i sina <a href="/wiki/Special:Preferences#mw-prefsection-rendering" title="Special:Preferences">utseendeinställningar</a>. Detta är aktiverat för alla nyregistrerade användare.</li>
        </ul></li>
      </ul> 
      <p>Den här funktionen används ofta för mallbaserade kategorier, såsom kategorier som läggs till genom upphovsrättsmallar. Om man till exempel lägger till <a href="/wiki/Template:PD-old-100" title="Template:PD-old-100"><span style="font-family:monospace,monospace;">{{PD-old-100}}</span></a> på en filbeskrivningssida, läggs filen till i <a href="/wiki/Category:Author_died_more_than_100_years_ago_public_domain_images" title="Category:Author died more than 100 years ago public domain images">Category:Author died more than 100 years ago public domain images</a> (bilder som är i allmän ägo eftersom upphovsmannen avlidit för mer än 100 år sedan), som är märkt med __HIDDENCAT__. </p>
      <p>För ytterligare detaljer, se <a href="https://meta.wikimedia.org/wiki/Help:Category#Hidden_categories" class="extiw" title="m:Help:Category">hjälpavsnittet om dolda kategorier</a> för Mediawiki (programvaran som Commons använder). </p> 
      <h2><span id="Mallar_f.C3.B6r_kategorier"></span><span class="mw-headline" id="Mallar_för_kategorier">Mallar för kategorier</span></h2> 
      <p>Det finns flera mallar som är avsedda för användning på kategorisidor; se <a href="/wiki/Category:Category_namespace_templates" title="Category:Category namespace templates">Category:Category namespace templates</a> (mallar för kategorinamnrymden). </p>
      <p>En del av de vanligast förekommande är <a href="/wiki/Category:Category_header_templates" title="Category:Category header templates">mallar för sidhuvuden på kategorisidor</a> såsom: </p>

       <li><a href="/wiki/Template:Category_TOC" class="mw-redirect" title="Template:Category TOC"><span style="font-family:monospace,monospace;">{{Category TOC}}</span></a> och <a href="/wiki/Template:LargeCategoryTOC" title="Template:LargeCategoryTOC"><span style="font-family:monospace,monospace;">{{LargeCategoryTOC}}</span></a> för att visa en innehållsförteckning i väldigt stora kategorier med många underkategorier</li> 
       <li><a href="/wiki/Template:ImageTOC" title="Template:ImageTOC"><span style="font-family:monospace,monospace;">{{ImageTOC}}</span></a> och <a href="/wiki/Template:FileCategoryTOC" title="Template:FileCategoryTOC"><span style="font-family:monospace,monospace;">{{FileCategoryTOC}}</span></a> för att visa en innehållsförteckning i stora kategorier med många filer och få underkategorier</li> 
       <li><a href="/wiki/Template:Image_template_notice" title="Template:Image template notice"><span style="font-family:monospace,monospace;">{{Image template notice}}</span></a> och <a href="/wiki/Template:Autocat" title="Template:Autocat"><span style="font-family:monospace,monospace;">{{Autocat}}</span></a> för administrativa kategorier som tilldelas med hjälp av en viss mall</li> 
       <li><a href="/wiki/Template:Global_maintenance_category" title="Template:Global maintenance category"><span style="font-family:monospace,monospace;">{{Global maintenance category}}</span></a> och <a href="/wiki/Template:Local_maintenance_category" title="Template:Local maintenance category"><span style="font-family:monospace,monospace;">{{Local maintenance category}}</span></a></li>

         <li><a href="/wiki/Template:By_country_category" title="Template:By country category"><span style="font-family:monospace,monospace;">{{By country category}}</span></a>, <a href="/wiki/Template:By_language_category" title="Template:By language category"><span style="font-family:monospace,monospace;">{{By language category}}</span></a> och <a href="/wiki/Template:By_century_category" title="Template:By century category"><span style="font-family:monospace,monospace;">{{By century category}}</span></a> för specifika typer av metakategorier</li>

This excludes files already in the category (doesn't work if the category was added by a template). <small>[<a href="/wiki/MediaWiki_talk:Gadget-Searchnotincat.js" title="MediaWiki talk:Gadget-Searchnotincat.js">talk</a>]</small>&nbsp;</li>
