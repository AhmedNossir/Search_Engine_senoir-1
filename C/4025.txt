      <p>Categories, a software feature of MediaWiki, provide automatic indexes that are useful as tables of contents. </p>
      <p>You can categorize pages and files by adding one or more <b>Category</b> tags to the content text. These tags create links at the bottom of the page that take you to the list of all pages in that category, which makes it easy to browse related articles. </p>

      <h2><span class="mw-headline" id="Summary">Summary</span></h2> 
      <p>Each of the pages in the <b>Category</b> <a href="/wiki/Special:MyLanguage/Help:Namespaces" title="Special:MyLanguage/Help:Namespaces">namespace</a> represents a so-called <b>category</b>, a grouping of related pages, and contains an index for the pages of its category. For example, this page belongs to "<a href="/wiki/Special:MyLanguage/Category:Help" title="Special:MyLanguage/Category:Help">Category:Help</a><span style="display:none"><a href="/wiki/Category:Help" title="Category:Help"> </a></span>". If you open the category page, you will see a link to this page there. </p>
      <p>When a page belongs to one or more categories, these categories appear at the bottom of the page (or in the upper-right corner, depending on the <a href="/wiki/Special:MyLanguage/Help:Preferences" title="Special:MyLanguage/Help:Preferences">skin</a> being used). </p>
      <p>The category pages themselves contain 2 parts: </p> 
      <ul>
       <li>at their beginning, an optional part may contain text that can be edited, like any other page,</li> 
       <li>at their end, an ever present, automatically generated, alphabetical list of all pages in that category, in the form of links. In fact, in the Unicode sort order.</li>
      </ul> 
      <p>To assign a category to a page, simply add the link "[[Category:<i>Category name</i>]]" to the page's wikitext. The usual place to add it is at the bottom of the page. </p>
      <p>To link a category page within a page as a normal wiki link (without adding the page to the category), prefix the link name with a colon. For example: [[:Category:Not in this category]] </p>
      <p>A new category can be <a href="/wiki/Special:MyLanguage/Help:Starting_a_new_page" title="Special:MyLanguage/Help:Starting a new page">created</a> before assigning any page to it, in the same way as any other regular page. </p>
      <p>Individual wikis may have their own top-level categories, such as <a href="https://en.wikipedia.org/wiki/Category:Contents" class="extiw" title="wikipedia:Category:Contents">Category:Contents</a> in Wikipedia. </p>
      <p>For a complete list of all categories which have at least one page, see <a href="/wiki/Special:Categories" title="Special:Categories">Special:Categories</a>. </p>
      <p>For a complete list of all created/edited categories, including the ones that don't have any pages, see <a href="/wiki/Special:AllPages/Category:" title="Special:AllPages/Category:">Special:Allpages/Category:</a> (note the colon at the end). </p> 
      <h2><span class="mw-headline" id="Adding_a_page_to_a_category">Adding a page to a category</span></h2> 
      <p>To add a page or uploaded file to a category, simply edit the page and add the following text (where <i><b>Name</b></i> is the name of the category you want to add it to). </p> 
      <pre>[[Category:<i><b>Name</b></i>]]
</pre> 
      <p>Any number of Category tags may be added to the page and the page will be listed in all of them. Category tags, along with <a href="/wiki/Special:MyLanguage/Manual:Interwiki#Interwiki_links_to_other_languages" title="Special:MyLanguage/Manual:Interwiki">interwiki language links</a><span style="display:none"><a href="/wiki/Manual:Interwiki#Interwiki_links_to_other_languages" title="Manual:Interwiki"> </a></span> placed in the sidebar, are usually added at the very bottom of the page for the convenience of other editors. </p>
      <p>On a categorized page, categories are displayed in the Categories: box strictly in the order they appear in the wikitext. </p>
      <p>If <a href="/wiki/Special:MyLanguage/Manual:$wgUseCategoryBrowser" title="Special:MyLanguage/Manual:$wgUseCategoryBrowser">$wgUseCategoryBrowser</a><span style="display:none"><a href="/wiki/Manual:$wgUseCategoryBrowser" title="Manual:$wgUseCategoryBrowser"> </a></span> is set to true, under the first list each category is listed again, <a href="https://en.wikipedia.org/wiki/breadcrumb_(navigation)" class="extiw" title="wikipedia:breadcrumb (navigation)">breadcrumb</a>-style, with all its parent categories, sorted alphabetically. (At least in MediaWiki 1.18.2) if a category is a subcategory of more than one parent, both hierarchies will be listed, but the tagged category will be stripped off all but one of these. This creates the potential for what appear to be duplicate entries if a category with multiple parents <i>and</i> one of its parents are both tagged on a page. For example suppose Maryanne is a subcategory of both Mary and Anne. If a page tags categories Maryanne and Anne then the Category breadcrumbs will show </p> 
      <pre>Anne 
Anne
Mary -&gt; Maryanne
</pre> 
      <p>"Anne" appears to be duplicated, but what is meant is </p> 
      <pre>Anne 
Anne -&gt; Maryanne
Mary -&gt; Maryanne
</pre> 
      <p>This is a bug, and has been reported in <a href="https://phabricator.wikimedia.org/T35614" class="extiw" title="phab:T35614">phab:T35614</a>. </p> 
      <h3><span class="mw-headline" id="Sort_key">Sort key</span></h3> 
      <p>By default, a page is sorted within a category under the first letter of its name — <i>without the namespace</i>. Also, MediaWiki groups accented characters separately from their unaccented version, so pages starting by À, Á, Ä, will be listed under separate headings, instead of under heading A. (If you have shell access, these defaults can be changed using <code><a href="/wiki/Special:MyLanguage/Manual:$wgCategoryCollation" title="Special:MyLanguage/Manual:$wgCategoryCollation">$wgCategoryCollation</a><span style="display:none"><a href="/wiki/Manual:$wgCategoryCollation" title="Manual:$wgCategoryCollation"> </a></span></code>.) </p>
      <p>A <i>sort key</i> specifies under which letter heading, and where in the category list, the page will appear. You can add a sort key by placing it inside the tag after a pipe character. For example, the tag below will add the page under heading "S". </p> 
      <pre>[[Category:<i><b>Name</b></i>|<i><b>Sort</b></i>]]
</pre> 
      <p>Whether or not sort keys are case-sensitive <a href="/wiki/Special:MyLanguage/Help:Sorting#Categories" title="Special:MyLanguage/Help:Sorting">depends on your configuration settings</a>. The order of the sections within a category follows the Unicode sort order. The sort key does not change the page title displayed in the category. </p>
      <p>Spaces and other characters are perfectly valid. </p>
      <p>See <a href="/wiki/Special:MyLanguage/Help:Sorting" title="Special:MyLanguage/Help:Sorting">Help:Sorting</a><span style="display:none"><a href="/wiki/Help:Sorting" title="Help:Sorting"> </a></span> for further information on category sorting. </p> 
      <h2><span class="mw-headline" id="Creating_a_category_page">Creating a category page</span></h2> 
      <p>Categories exist even if their page has not been created, but these categories are isolated from others and serve little purpose for organization or navigation. </p>
      <p>A category is created by creating a page in the <i><b>Category:</b></i> namespace. A category page can be created the same way as other wiki pages (see <a href="/wiki/Special:MyLanguage/Help:Starting_a_new_page" title="Special:MyLanguage/Help:Starting a new page">Help:Starting a new page</a><span style="display:none"><a href="/wiki/Help:Starting_a_new_page" title="Help:Starting a new page"> </a></span>); just add "<code>Category:</code>" before the page title. </p>
      <p>To avoid extra work, try searching within your wiki before creating a new category. The list of all categories can be found in "Special pages" in the "tools" box of the <a href="/wiki/Special:MyLanguage/Help:Navigation#Sidebar" title="Special:MyLanguage/Help:Navigation">sidebar</a>. </p> 
      <h2><span class="mw-headline" id="Managing_the_category_hierarchy">Managing the category hierarchy</span></h2> 
      <p>Categories may belong to other categories in a hierarchy. Since category pages are much like any other page, a Category tag may be added to the bottom of a category page. </p>
      <p>It is a good idea to organize all categories into a hierarchy with a single top level category. The category structure can take the form of a tree with separate branches, but more often will have a graph structure. Generally, there should be a contiguous chain of parent-child links between each category and the top level category. </p> 
      <h3><span class="mw-headline" id="Category_tree">Category tree</span></h3> 
      <style data-mw-deduplicate="TemplateStyles:r4624193">.mw-parser-output .hatnote{font-style:italic;padding-bottom:0.4em;margin-bottom:0.4em;color:#555;border-bottom:1px solid #ccc}.mw-parser-output .hatnote.no-border{border-bottom:none}</style>
      <div role="note" class="hatnote relarticle mainarticle">
       Main page: <a href="/wiki/Special:MyLanguage/Extension:CategoryTree#Usage" title="Special:MyLanguage/Extension:CategoryTree">CategoryTree extension manual</a>
      </div> 
      <p>Since MediaWiki&nbsp;1.31, <a href="/wiki/Special:MyLanguage/Extension:CategoryTree" title="Special:MyLanguage/Extension:CategoryTree">CategoryTree</a><span style="display:none"><a href="/wiki/Extension:CategoryTree" title="Extension:CategoryTree"> </a></span> extension is packaged by default, allowing <a href="/wiki/Special:CategoryTree" title="Special:CategoryTree">Special:CategoryTree</a> and <code class="mw-highlight" style="white-space:nowrap"><span class="p">&lt;</span><span class="nt">categorytree</span><span class="p">&gt;</span></code> to provide a tree-like structure display for the categories. </p> 
      <h2><span class="mw-headline" id="Hidden_categories">Hidden categories</span></h2> 
      <p>The categories that a page is in are normally listed at the bottom of the page. In MediaWiki 1.13+, a category can be hidden from this list by adding the <a href="/wiki/Special:MyLanguage/Help:Magic_words" title="Special:MyLanguage/Help:Magic words">magic word</a> "<code>__HIDDENCAT__</code>" to the category page. </p>
      <p>Users can choose to see hidden categories in a separate "Hidden categories" list, by checking "Show hidden categories" in the "Appearance" section of <a href="/wiki/Special:Preferences" title="Special:Preferences">Special:Preferences</a>. </p>
      <p>Hidden categories are automatically added to <a href="/wiki/Category:Hidden_categories" title="Category:Hidden categories">Category:Hidden categories</a>. This category is specified in the system message <a href="/wiki/MediaWiki:Hidden-category-category" title="MediaWiki:Hidden-category-category">MediaWiki:Hidden-category-category</a>. </p> 
      <h2><span class="mw-headline" id="Linking_to_a_category">Linking to a category</span></h2> 
      <p>To create a link to a category, use a leading colon before the category name (without this colon, the current page would be added to the category): </p> 
      <dl>
       <dd>
        <code>[[:Category:Help]]</code> ? <b><a href="/wiki/Category:Help" title="Category:Help">Category:Help</a></b>
       </dd>
      </dl> 
      <p>To change the link text, write the text inside the link tag after a pipe: </p> 
      <dl>
       <dd>
        <code>[[:Category:Help|Help category]]</code> ? <b><a href="/wiki/Category:Help" title="Category:Help">Help category</a></b>
       </dd>
      </dl> 
      <p><a href="/wiki/Special:MyLanguage/Help:Redirects" title="Special:MyLanguage/Help:Redirects">Redirect pages</a> that redirect to categories must also use the colon, otherwise they will be added to the category instead of redirecting. </p> 
      <h2><span id="Categories_and_templates_.28transcluded_pages.29"></span><span class="mw-headline" id="Categories_and_templates_(transcluded_pages)">Categories and templates (transcluded pages)</span></h2> 
      <p>If you add <code class="mw-highlight mw-highlight-lang-moin mw-content-ltr" dir="ltr"><span class="k">[[Category:Cats]]</span></code> inside a template (a transcluded page), <i>both</i> the template and pages which transclude this template will be categorized as Cats. </p> 
      <ul>
       <li>If you want only the template to be categorized, you should ensure to write category marker inside <code><a href="/wiki/Special:MyLanguage/Transclusion#Partial_transclusion" title="Special:MyLanguage/Transclusion">&lt;noinclude&gt;</a><span style="display:none"><a href="/wiki/Transclusion#Partial_transclusion" title="Transclusion"> </a></span></code> tags.</li> 
       <li>If you want only the pages which uses the template to be categorized, but not the template itself, you should ensure to write category marker inside <code><a href="/wiki/Special:MyLanguage/Transclusion#Partial_transclusion" title="Special:MyLanguage/Transclusion">&lt;includeonly&gt;</a><span style="display:none"><a href="/wiki/Transclusion#Partial_transclusion" title="Transclusion"> </a></span></code> tags.</li>
      </ul> 
      <p>Due to caching issues, editing categories applied through templates may take a while to be visible everywhere. You can perform a <a href="/wiki/Special:MyLanguage/Manual:Null_edit" title="Special:MyLanguage/Manual:Null edit">null edit</a> on a page to update immediately its categories. </p>
      <p><span id="Redirecting_a_category"></span> </p> 
      <h2><span class="mw-headline" id="Redirecting_a_category">Redirecting a category</span></h2> 
      <p>Like normal wiki pages, category pages can be redirected to other normal or category pages. <i>However</i>, this is not recommended, as pages categorized in redirected categories do <i>not</i> get categorized in the target category (<a href="https://phabricator.wikimedia.org/T5311" class="extiw" title="phab:T5311">phab:T5311</a>). Some Wikimedia sites use a "category redirect" template to mark redirected categories, allowing manual or automated cleanup of pages categorized there. </p> 
      <h2><span class="mw-headline" id="Tracking_categories">Tracking categories</span></h2> 
      <p>Certain syntax issues, such as a broken image link, or too many expensive parser functions, can cause a page to be added to a tracking category. See <a href="/wiki/Special:MyLanguage/Help:Tracking_categories" title="Special:MyLanguage/Help:Tracking categories">Help:Tracking categories</a><span style="display:none"><a href="/wiki/Help:Tracking_categories" title="Help:Tracking categories"> </a></span> for details. </p> 
      <h2><span class="mw-headline" id="Renaming_a_category">Renaming a category</span></h2> 
      <p>If you move a category, since <a href="#Redirecting_a_category">redirect will not work for categorizing</a>, all category tags will be wrong, so you have to fix them. </p>
