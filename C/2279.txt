       <h3 id="accessing-the-audit-log"><a href="#accessing-the-audit-log">Accessing the audit log</a></h3> 
       <p>The audit log lists events triggered by activities that affect your organization within the last 90 days. Only owners can access an organization's audit log.</p> 
       <ol> 
        <li>In the top right corner of GitHub, click your profile photo, then click <strong>Your organizations</strong>. <span class="procedural-image-wrapper"><img src="/assets/images/help/profile/your-organizations.png" alt="Your organizations in the profile menu"></span></li> 
        <li>Click <strong>Settings</strong> next to the organization. <span class="procedural-image-wrapper"><img src="/assets/images/help/organizations/settings-button.png" alt="The settings button"></span></li> 
        <li>In the Settings sidebar, click <strong>Audit log</strong>. <span class="procedural-image-wrapper"><img src="/assets/images/help/organizations/org-settings-audit-log.png" alt="Org audit log settings in sidebar"></span></li> 
       </ol> 
       <h3 id="searching-the-audit-log"><a href="#searching-the-audit-log">Searching the audit log</a></h3> 
       <p>The log lists the following information about each action:</p> 
       <ul> 
        <li>Which repository an action was performed in</li> 
        <li>The user that performed the action</li> 
        <li>The action that was performed</li> 
        <li>Which country the action took place in</li> 
        <li>The date and time the action occurred</li> 
       </ul> 
       <p>Note that you cannot search for entries using text. You can, however, construct search queries using a variety of filters. Many operators used when querying the log, such as <code>-</code>, <code>&gt;</code>, or <code>&lt;</code>, match the same format as searching across GitHub. For more information, see "<a href="/en/github/searching-for-information-on-github/about-searching-on-github">Searching on GitHub</a>."</p> 
       <h4 id="search-based-on-operation"><a href="#search-based-on-operation">Search based on operation</a></h4> 
       <p>Use the <code>operation</code> qualifier to limit actions to specific types of operations. For example:</p> 
       <ul> 
        <li><code>operation:access</code> finds all events where a resource was accessed.</li> 
        <li><code>operation:authentication</code> finds all events where an authentication event was performed.</li> 
        <li><code>operation:create</code> finds all events where a resource was created.</li> 
        <li><code>operation:modify</code> finds all events where an existing resource was modified.</li> 
        <li><code>operation:remove</code> finds all events where an existing resource was removed.</li> 
        <li><code>operation:restore</code> finds all events where an existing resource was restored.</li> 
        <li><code>operation:transfer</code> finds all events where an existing resource was transferred.</li> 
       </ul> 
       <h4 id="search-based-on-repository"><a href="#search-based-on-repository">Search based on repository</a></h4> 
       <p>Use the <code>repo</code> qualifier to limit actions to a specific repository. For example:</p> 
       <ul> 
        <li><code>repo:my-org/our-repo</code> finds all events that occurred for the <code>our-repo</code> repository in the <code>my-org</code> organization.</li> 
        <li><code>repo:my-org/our-repo repo:my-org/another-repo</code> finds all events that occurred for both the <code>our-repo</code> and <code>another-repo</code> repositories in the <code>my-org</code> organization.</li> 
        <li><code>-repo:my-org/not-this-repo</code> excludes all events that occurred for the <code>not-this-repo</code> repository in the <code>my-org</code> organization.</li> 
       </ul>

       <h4 id="search-based-on-the-user"><a href="#search-based-on-the-user">Search based on the user</a></h4> 
       <p>The <code>actor</code> qualifier can scope events based on who performed the action. For example:</p> 
       <ul> 
        <li><code>actor:octocat</code> finds all events performed by <code>octocat</code>.</li> 
        <li><code>actor:octocat actor:hubot</code> finds all events performed by both <code>octocat</code> and <code>hubot</code>.</li> 
        <li><code>-actor:hubot</code> excludes all events performed by <code>hubot</code>.</li> 
       </ul> 
       <p>Note that you can only use a GitHub username, not an individual's real name.</p>

       <h4 id="search-based-on-time-of-action"><a href="#search-based-on-time-of-action">Search based on time of action</a></h4> 
       <p>Use the <code>created</code> qualifier to filter events in the audit log based on when they occurred. Date formatting must follow the <a href="http://en.wikipedia.org/wiki/ISO_8601">ISO8601</a> standard, which is <code>YYYY-MM-DD</code> (year-month-day). You can also add optional time information <code>THH:MM:SS+00:00</code> after the date, to search by the hour, minute, and second. That's <code>T</code>, followed by <code>HH:MM:SS</code> (hour-minutes-seconds), and a UTC offset (<code>+00:00</code>).</p> 
       <p>When you search for a date, you can use greater than, less than, and range qualifiers to further filter results. For more information, see "<a href="/en/github/searching-for-information-on-github/understanding-the-search-syntax">Understanding the search syntax</a>."</p> 
       <p>For example:</p> 
       <ul> 
        <li><code>created:2014-07-08</code> finds all events that occurred on July 8th, 2014.</li> 
        <li><code>created:&gt;=2014-07-08</code> finds all events that occurred on or after July 8th, 2014.</li> 
        <li><code>created:&lt;=2014-07-08</code> finds all events that occurred on or before July 8th, 2014.</li> 
        <li><code>created:2014-07-01..2014-07-31</code> finds all events that occurred in the month of July 2014.</li> 
       </ul> 
       <p>The audit log contains data for the past 90 days, but you can use the <code>created</code> qualifier to search for events earlier than that.</p> 
       <h4 id="search-based-on-location"><a href="#search-based-on-location">Search based on location</a></h4> 
       <p>Using the qualifier <code>country</code>, you can filter events in the audit log based on the originating country. You can use a country's two-letter short code or its full name. Keep in mind that countries with spaces in their name will need to be wrapped in quotation marks. For example:</p> 
       <ul> 
        <li><code>country:de</code> finds all events that occurred in Germany.</li> 
        <li><code>country:Mexico</code> finds all events that occurred in Mexico.</li> 
        <li><code>country:"United States"</code> all finds events that occurred in the United States.</li> 
       </ul>

       <h3 id="using-the-audit-log-api"><a href="#using-the-audit-log-api">Using the audit log API</a></h3> 
       <p>You can interact with the audit log using the GraphQL API or the REST API.</p> 
       <h4 id="using-the-graphql-api"><a href="#using-the-graphql-api">Using the GraphQL API</a></h4> 
       <div class="extended-markdown note border rounded-1 mb-4 p-3 color-border-info color-bg-info f5"> 
        <p><strong>Note</strong>: The audit log GraphQL API is available for organizations using GitHub Enterprise. For more information, see "<a href="/en/articles/github-s-products">GitHub's products</a>."</p> 
       </div> 
       <p>To ensure a secure IP and maintain compliance for your organization, you can use the audit log GraphQL API to keep copies of your audit log data and monitor:</p> 
       <ul> 
        <li>Access to your organization or repository settings</li> 
        <li>Changes in permissions</li> 
        <li>Added or removed users in an organization, repository, or team</li> 
        <li>Users being promoted to admin</li> 
        <li>Changes to permissions of a GitHub App</li> 
       </ul> 
       <p>Note that you can't retrieve Git events using the GraphQL API. To retrieve Git events, use the REST API instead. For more information, see "<a href="#git-category-actions"><code>git</code> category actions</a>."</p> 
       <p>The GraphQL response can include data for up to 90 to 120 days.</p> 
       <p>For example, you can make a GraphQL request to see all the new organization members added to your organization. For more information, see the "<a href="/en/graphql/reference/interfaces#auditentry">GraphQL API Audit Log</a>."</p> 
       <h4 id="using-the-rest-api"><a href="#using-the-rest-api">Using the REST API</a></h4> 
       <div class="extended-markdown note border rounded-1 mb-4 p-3 color-border-info color-bg-info f5"> 
        <p><strong>Note:</strong> The audit log REST API is available for users of GitHub Enterprise Cloud only.</p> 
       </div> 
       <p>To ensure a secure IP and maintain compliance for your organization, you can use the audit log REST API to keep copies of your audit log data and monitor:</p> 
       <ul> 
        <li>Access to your organization or repository settings</li> 
        <li>Changes in permissions</li> 
        <li>Added or removed users in an organization, repository, or team</li> 
        <li>Users being promoted to admin</li> 
        <li>Changes to permissions of a GitHub App</li> 
        <li>Git events, such as cloning, fetching, and pushing</li> 
       </ul> 
       <p>The audit log retains Git events for 7 days. This is shorter than other audit log events, which can be retained for 90 days.</p> 
       <p>For more information about the audit log REST API, see "<a href="/en/rest/reference/orgs#get-the-audit-log-for-an-organization">Organizations</a>."</p>
