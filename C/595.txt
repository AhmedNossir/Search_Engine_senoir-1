      <p>Una <b>categoria</b> (<i>category</i> in inglese) è una feature del software <a href="https://it.wikipedia.org/wiki/MediaWiki" class="extiw" title="it:MediaWiki">MediaWiki</a>, una pagina speciale la cui finalità è il raggruppamento di pagine e file multimediali correlati. In pratica, verrà associato un singolo soggetto ad una data categoria. Il nome della categoria dovrebbe essere il più possibile identificativo del soggetto, ma testo extra può essere utile per definirlo precisamente. </p>
      <p>La struttura della <a href="/wiki/Commons:Guide_to_layout/Category_pages" title="Commons:Guide to layout/Category pages">categoria</a> è la via primaria per organizzare e trovare file su Commons. È essenziale che ogni file possa essere trovato sfogliando la struttura delle categorie. Per far sì che ciò sia possibile, ogni file deve essere inserito in una categoria, e/o in una pagina <a href="/wiki/Commons:Guide_to_layout/Gallery_pages" title="Commons:Guide to layout/Gallery pages">galleria</a> che è categorizzata. Le categorie stesse dovrebbero essere inserite in categorie più generali, formando una struttura gerarchica. </p>

       <h2><span class="mw-headline" id="Quick_guide">Quick guide</span></h2> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>1. How to find the appropriate categories </p> 
       <dl>
        <dd>
         <ul>
          <li>Find categories with the search engine (see <a href="#Categorization_tips">#Categorization tips</a>)</li> 
          <li>or check how similar files are categorized (some may not be categorized though)</li> 
          <li>or start from the main topical category (<a href="/wiki/Category:Topics" title="Category:Topics">Category:Topics</a>)</li> 
          <li>Starting from these categories, check their parent or sub-categories to find an appropriate category. Avoid picking too general categories.</li>
         </ul>
        </dd>
       </dl> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>2. Add the categories to the file </p> 
       <dl>
        <dd>
         <ul>
          <li>Select them on the upload form</li> 
          <li>or: add them manually by adding, e.g. <code>[[Category:Foobar]]</code> to the file description page (see <a href="#Categorizing_pages">#Categorizing pages</a>)</li> 
          <li>or: use HotCat (see <a href="/wiki/Special:MyLanguage/Help:Gadget-HotCat" title="Special:MyLanguage/Help:Gadget-HotCat">Help:Gadget-HotCat</a>)</li> 
          <li>or: use Cat-a-lot (see <a href="/wiki/Help:Gadget-Cat-a-lot/it" title="Help:Gadget-Cat-a-lot/it">Aiuto:Gadget-Cat-a-lot</a>)</li>
         </ul>
        </dd>
       </dl>

       <h3><span class="mw-headline" id="Principles">Principles</span></h3> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>The main principles are: </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <h4><span class="mw-headline" id="Hierarchic_principle">Hierarchic principle</span></h4> 
      </div> 
      <p>La struttura delle categorie è (idealmente) una multi-gerachia con una singola categoria-radice, <a href="/wiki/Category:CommonsRoot" title="Category:CommonsRoot">Category:CommonsRoot</a>. Tutte le categorie (eccetto <i>CommonsRoot</i>) dovrebbero essere contenute in almeno un'altra categoria, e non ci dovrebbero esseri cicli (ad esempio, una categoria non dovrebbe contenere se stessa, direttamente o indirettamente). La struttura delle categorie dovrebbe riflettere una gerarchia di concetti, dal più generico, in alto, al più specifico, in basso. </p> 
      <dl>
       <dd>
        Per maggiori dettagli, vedi <a href="https://meta.wikimedia.org/wiki/Help:Category#Category_structure" class="extiw" title="meta:Help:Category">Meta:Help:Category#Category structure</a>
       </dd>
      </dl> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <h4><span class="mw-headline" id="Modularity_principle">Modularity principle</span></h4> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>The page (file, category) should be put in the most specific category/categories that fit(s) the page (not directly to its parent categories). A category can have more parent categories. A category can combine two (or more) different criteria; such categories are called "compound categories" or "intersection categories". E.g. the root category <a href="/wiki/Category:Churches" title="Category:Churches">Category:Churches</a> and the root category <a href="/wiki/Category:Russia" title="Category:Russia">Category:Russia</a> have a common subcategory <a href="/wiki/Category:Churches_in_Russia" title="Category:Churches in Russia">Churches in Russia</a>. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <h4><span class="mw-headline" id="Simplicity_principle">Simplicity principle</span></h4> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>This principle suggests not to combine too many different criteria. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <h4><span class="mw-headline" id="Selectivity_principle">Selectivity principle</span></h4> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>We should not classify items which are related to different subjects in the same category. The category name should be unambiguous and not homonymous. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <h4><span class="mw-headline" id="Universality_principle">Universality principle</span></h4> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>Identical items should have identical names for all countries and at all levels of categorization. Categorization structure should be as systematical and unified as possible, local dialects and terminology should be supressed in favour of universality if possible. Analogic categorization branches should have analogic structure. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <h3><span class="mw-headline" id="Types_of_reflected_relations">Types of reflected relations</span></h3> 
      </div> 
      <p>La struttura della <a href="/wiki/Special:MyLanguage/Commons:Guide_to_layout/Category_pages" title="Special:MyLanguage/Commons:Guide to layout/Category pages">categoria</a> è la via primaria per organizzare e trovare file su Commons. È essenziale che ogni file possa essere trovato sfogliando la struttura delle categorie. Per far sì che ciò sia possibile, ogni file deve essere inserito in una categoria, e/o in una pagina <a href="/wiki/Special:MyLanguage/Commons:Guide_to_layout/Gallery_pages" title="Special:MyLanguage/Commons:Guide to layout/Gallery pages">galleria</a> che è categorizzata. Le categorie stesse dovrebbero essere inserite in categorie più generali, formando una struttura gerarchica. </p> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <ul>
        <li>Attributes: 
         <ul>
          <li>Qualitative and general attributes (color, shape, size, ability or disability, nationality, technique, quality, awards…)</li> 
          <li>Location: where, in…, from… (place/event, place/building, place/exhibit, place/people, country/language, source/work, factory or country/product etc.)</li> 
          <li>Timing: when (time/event, time/depicted situation, time of birth, inception or construction, time of death, demolition or termination etc.)</li>
         </ul></li>
       </ul> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <ul>
        <li>Agentive and influence relations: (creator/work, device/product, company/product, discipline or profession/their subjects and terms, parent/children, subordination, owner/property, initiator/follower, subject/other subjects dedicated to it or named after it, subject/its duplicate, imitiation, depiction or symbol etc etc.)</li> 
        <li>Modification: original/modified or modified/original (avoid cyclic structure) – renamed, rebuilt, repurposed or transformed subjects.</li>
       </ul> 
      </div> 
      <h3><span class="mw-headline" id="Categorie_principali">Categorie principali</span></h3> 
      <p>Le categorie più alte (quelle contenute direttamente in CommonsRoot) dividono la struttura delle categorie per scopo delle categorie lì contenute: </p> 
      <ul>
       <li><b><a href="/wiki/Category:Topics" title="Category:Topics">Category:Topics</a></b> - Questa categoria è la radice globale e comune dei file multimediali categorizzati per argomento. Tutti i file multimediali dovrebbero essere categorizzati in questa categoria per consentire agli altri di trovarli per argomento.</li> 
       <li><b><a href="/wiki/Category:Copyright_statuses" title="Category:Copyright statuses">Category:Copyright statuses</a></b> - Questa categoria è la radice globale e comune dei file multimediali categorizzati per licenza. Tutti i file multimediali dovrebbero essere categorizzati in questa categoria con l'appropriato <a href="/wiki/Commons:Tag_per_l%27attribuzione_del_copyright" class="mw-redirect" title="Commons:Tag per l'attribuzione del copyright">tag per l'attribuzione del copyright</a>.</li> 
       <li><b><a href="/wiki/Category:Image_sources" title="Category:Image sources">Category:Image sources</a></b> - Questa categoria è la radice <b>globale e comune</b> dei file multimediali categorizzati per <b>fonte</b>, ovvero da dove provengono (libri, collezioni, siti ecc.)</li> 
       <li><b><a href="/wiki/Category:Media_types" title="Category:Media types">Category:Media types</a></b> - Questa categoria è la radice globale e comune dei file multimediali categorizzati per tipo di file. Per favore nota che questo tipo di categorizzazione è a volte omessa per le immagini, poiché la maggior parte dei file di Commons sono immagini di qualche tipo.</li> 
       <li><b><a href="/wiki/Category:Commons" title="Category:Commons">Category:Commons</a></b> - Questa è la radice globale e comune della categorizzazione delle pagine di Commons riguardo operazioni di manutenzione e pagine (Commons:- ed Help:-), eccetto i file multimediali. Le pagine tradotte in altre lingue dovrebbero essere categorizzate in ogni categoria di lingua con lo stile "Category:Commons-<a href="https://en.wikipedia.org/wiki/ISO_639" class="extiw" title="w:ISO 639">ISO-LANGUAGE-CODE</a>". La struttura di <b><a href="/wiki/Category:Commons-en" title="Category:Commons-en">Category:Commons-en</a></b> è la gerarchia esempio per ogni altra sottocategoria di un'altra lingua.</li>
      </ul> 
      <dl>
       <dd>
        <dl>
         <dd>
          C'è una sottocategoria, <a href="/wiki/Category:Commons_maintenance_content" title="Category:Commons maintenance content">Category:Commons maintenance content</a>, che è per la manutenzione speciale dei <b>contenuti comuni globali</b> di Wikimedia Commons e non dovrebbe essere tradotta.?<b>Tutti i file multimediali</b> dovrebbero essere categorizzati in una delle seguenti 4 categorie, ma <b>solo se</b> ci sono file che hanno problemi e necessitano di manutenzione, dovrebbero essere presenti anche in una delle sottocategorie di <a href="/wiki/Category:Commons_maintenance_content" title="Category:Commons maintenance content">Category:Commons maintenance content</a>.
         </dd>
        </dl>
       </dd>
      </dl> 
      <ul>
       <li><b><a href="/wiki/Category:User" title="Category:User">Category:User</a></b> - Questa categoria è per le categorie che contengono le pagine utenti degli utenti di Commons, ordinate per lingua parlata. Contiene inoltre <a href="/wiki/Category:User_galleries" title="Category:User galleries">Category:User galleries</a>, che è una categoria specifica per le gallerie degli utenti (o per le immagini che in qualche modo non sono destinate a Commons in quanto non hanno un contenuto educativo e documentaristico).</li>
      </ul> 
      <h2><span class="mw-headline" id="Come_usare_le_categorie">Come usare le categorie</span></h2> 
      <p>Dovresti sempre inserire i tuoi caricamenti in categorie e/o gallerie a seconda dell'argomento, in modo tale da consentire agli altri utenti di trovare ed usare i file. </p>
      <p>Solitamente è raro dover creare una nuova categoria (ma per un'eccezione, vedi la sezione <a href="#Persone">Persone</a>, più sotto). Prima di farlo, assicurati di avere familiarità con la struttura delle categorie esistenti, e con le politiche di Commons. Per favore controlla se esiste uno <a href="/wiki/Category:Commons_category_schemes" title="Category:Commons category schemes">schema di categoria</a> o un <a href="/wiki/Category:Commons_projects" title="Category:Commons projects">progetto</a> per il tuo argomento, e segui le convenzioni lì descritte. </p>
      <p>Per favore fornisci il maggior numero possibile di dettagli circa i tuoi caricamenti — vedi il <a href="/wiki/Template:Information" title="Template:Information">Template:Information</a> per suggerimenti su cosa inserire. Devi anche fornire valide informazioni sulla licenza dei tuoi caricamenti — vedi <a href="/wiki/Commons:Licenze" class="mw-redirect" title="Commons:Licenze">Commons:Licenze</a> per i dettagli. Vedi <a href="/wiki/Commons:Criteri_per_l%27inclusione" class="mw-redirect" title="Commons:Criteri per l'inclusione">Commons:Criteri per l'inclusione</a> per informazioni circa quali file multimediali sono adatti per Commons </p>
      <p><span id="Category_names"></span> </p> 
      <h3><span class="mw-headline" id="Nomi_delle_categorie">Nomi delle categorie</span></h3> 
      <div class="mw-translate-fuzzy"> 
       <p>I nomi delle categorie dovrebbero sempre essere in inglese (eccetto per le forme di vita, per le quali dovrebbe essere usato il nome scientifico, in latino. </p> 
      </div> 
      <p>I nomi delle categorie che si riferiscono a oggetti o gruppi di persone dovrebbero generalmente essere al plurale: <a href="/wiki/Category:Tools" title="Category:Tools">Category:Tools</a>, <a href="/wiki/Category:Artists" title="Category:Artists">Category:Artists</a>, <a href="/wiki/Category:Lakes" title="Category:Lakes">Category:Lakes</a> ecc., in opposto a <a href="/wiki/Category:History" title="Category:History">Category:History</a>, <a href="/wiki/Category:Weather" title="Category:Weather">Category:Weather</a>, <a href="/wiki/Category:Music" title="Category:Music">Category:Music</a>. Vedi <b><a href="/wiki/Commons:Naming_categories" title="Commons:Naming categories">Commons:Naming categories</a></b> per ulteriori informazioni sulle convenzioni di nomenclatura delle categorie. </p> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>There is also a related <a href="/wiki/Special:MyLanguage/Commons:Naming_categories" title="Special:MyLanguage/Commons:Naming categories">failed proposal</a>. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>Categories grouping subcategories by name should generally be named "by name" rather than "by alphabet" (e.g. <a href="/wiki/Category:Ships_by_name" title="Category:Ships by name">Category:Ships by name</a>). </p> 
      </div> 
      <p>Ci manca ancora l'internazionalizzazione dei nomi delle categorie, ma il problema dovrebbe essere risolto con appropriati cambiamenti al software MediaWiki (vedi <a href="https://bugzilla.wikimedia.org/show_bug.cgi?id=5638" class="extiw" title="bugzilla:5638">bugzilla:5638</a>). Creare strutture di categorie in lingue differenti può solo peggiorare le cose. </p>
      <p>Per una discussione generale sulle possibilità di MediaWiki per quanto riguarda le categorie, vedi la <a href="https://meta.wikimedia.org/wiki/Help:Categories" class="extiw" title="m:Help:Categories">pagina di istruzioni sulle categorie</a>. </p> 
      <h2><span class="mw-headline" id="Categorizzare_pagine">Categorizzare pagine</span></h2> 
      <p>Per aggiungere una pagina (che sia un'immagine, una galleria o un'altra categoria) in una categoria, aggiungi il seguente codice alla fine della pagina: </p> 
      <dl>
       <dd>
        <code>[[Category:<i>Nome categoria</i>]]</code>
       </dd>
      </dl> 
      <p>Per esempio, se stai caricando un diagramma che mostra l'orbita di una cometa, potresti aggiungere la seguente sintassi alla pagina di descrizione dell'immagine: </p> 
      <pre>[[Category:Astronomical diagrams]]
[[Category:Comets]]
</pre> 
      <p>Questo farà apparire il diagramma nelle categorie <a href="/wiki/Category:Astronomical_diagrams" title="Category:Astronomical diagrams">Astronomical diagrams</a>, <a href="/wiki/Category:Comets" title="Category:Comets">Comets</a>. </p>
      <p>Per informazioni su come trovare le categorie più appropriate per i tuoi caricamenti e per le gallerie, leggi la sezione <a href="#Trovare_una_categoria_appropriata">Trovare una categoria appropriata</a>, più sotto. </p> 
      <h3><span class="mw-headline" id="Creare_nuove_categorie">Creare nuove categorie</span></h3> 
      <p>Per creare una nuova categoria, per prima cosa aggiungi il collegamento ad essa nelle pagine da categorizzare, come descritto più sopra (<i>[[Category:...]]</i>). Una volta salvata la pagina, clicca sul link rosso a quella categoria — puoi modificare la categoria come fosse una qualsiasi altra pagina wiki. </p>
      <p>Una pagina di categoria dovrebbe contenere le seguenti informazioni (in ordine di importanza): </p> 
      <ul>
       <li>Collegamenti a categorie per inserire la categoria in una o più categorie superiori.</li> 
       <li>Una breve testo descrittivo che spieghi cosa dovrebbe contenere la categoria. L'inglese è la lingua da preferire per la descrizione, ma altre lingue possono essere aggiunte (con il template {{it|...}} per una descrizione in italiano, {{en|...}} per una descrizione in inglese, {{fr|...}} per una descrizione in francese e così via, come specificato in <a href="/wiki/Special:MyLanguage/Commons:Language_templates" title="Special:MyLanguage/Commons:Language templates">Commons:Language templates</a>).</li>
      </ul> 
      <div class="mw-translate-fuzzy"> 
       <ul>
        <li>Interwiki alle categorie o gallerie che hanno un corrispondente argomento in Wikipedia (ad esempio l'interwiki [[it:...]] per la pagina in Wikipedia italiana, [[en:...]] per la pagina in Wikipedia inglese ecc.).</li>
       </ul> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>See also <a href="#How_to_categorize:_guidance_by_topic">#How to categorize: guidance by topic</a> for guidance on specific classes of category, e.g. categories about <a href="#People">#People</a>. </p> 
      </div> 
      <h2><span class="mw-headline" id="Rinominare_o_spostare_categorie">Rinominare o spostare categorie</span></h2> 
      <p>Per favore, vedi <a href="/wiki/Commons:Rename_a_category/it" title="Commons:Rename a category/it">Commons:Rinominare una categoria</a>. </p> 
      <h2><span class="mw-headline" id="Per_una_categorizzazione_appropriata">Per una categorizzazione appropriata</span></h2> 
      <p>Le pagine (incluse le pagine che sono categorie) sono categorizzate secondo il loro <b>soggetto</b>, e non secondo i loro <b>contenuti</b>. Questo perché i contenuti non sono generalmente una caratteristica permanente di una categoria; in particolare, puoi in un certo momento trovare contenuti inappropriati in una categoria. </p>
      <p>Esempio: assumiamo che la <a href="/wiki/Category:Spheres" title="Category:Spheres">Category:Spheres</a> contiene sono immagini di sfere di cristallo. Non devi aggiungere la <a href="/wiki/Category:Glass" title="Category:Glass">Category:Glass</a> nella pagina di categoria, secondo i suoi contenuti correnti, perché puoi avere sfere fatte con una grande varietà di materiali. Normalmente, ogni immagine che mostra un oggetto in vetro dovrebbe essere già categorizzata nella <a href="/wiki/Category:Glass" title="Category:Glass">Category:Glass</a> (o in una categoria della sua sotto-struttura). Così, se la <a href="/wiki/Category:Spheres" title="Category:Spheres">Category:Spheres</a> è veramente piena di immagini di sfere di cristallo, è una buona idea creare una nuova categoria, come <b>Category:Glass spheres</b> o <b>Category:Crystal balls</b>, categorizzate in <a href="/wiki/Category:Spheres" title="Category:Spheres">Category:Spheres</a> e <a href="/wiki/Category:Glass" title="Category:Glass">Category:Glass</a>. </p>
      <p>Generalmente, i file dovrebbero essere inseriti solo nella categoria più specifica che esiste per un certo argomento. Per esempio, i file nella <a href="/wiki/Category:Paris" title="Category:Paris">Category:Paris</a> non dovrebbero essere contenuti anche nella <a href="/wiki/Category:France" title="Category:France">Category:France</a> (vedi la sezione <i><a href="#Sovracategorizzazione">Sovracategorizzazione</a></i>). Se non trovi una categoria che si adatti al tuo scopo, puoi crearla — ma assicurati di aver letto prima la sezione che riguarda l'<a href="#Uso_delle_categorie">uso delle categorie</a>. </p>
      <p>Questo non significa che un'immagine deve appartenere solo ad una categoria; vuol dire solo che le immagini non dovrebbero essere in categorie <i>ridondanti</i> o non specifiche. Per esempio, un'immagine di un orso polare che viene salvato da un iceberg da un elicottero dovrebbe essere inserita in <a href="/wiki/Category:Ursus_maritimus" title="Category:Ursus maritimus">Category:Ursus maritimus</a>, <a href="/wiki/Category:Icebergs" title="Category:Icebergs">Category:Icebergs</a>, <a href="/wiki/Category:Helicopters" title="Category:Helicopters">Category:Helicopters</a> e <a href="/wiki/Category:Search_and_Rescue" title="Category:Search and Rescue">Category:Search and Rescue</a>. Al contrario, non dovrebbe essere inserita in <a href="/wiki/Category:Ursidae" title="Category:Ursidae">Category:Ursidae</a> o <a href="/wiki/Category:Aircraft" title="Category:Aircraft">Category:Aircraft</a>. </p> 
      <h3><span class="mw-headline" id="Consigli_sulla_categorizzazione">Consigli sulla categorizzazione</span></h3> 
      <p>Le categorie (o le gallerie) che scegli per i tuoi caricamenti dovrebbero rispondere ad almeno una delle seguenti domande: </p> 
      <ul>
       <li><b>cosa? / chi?</b> cosa o chi mostra il file? Per esempio <a href="/wiki/Category:Houses" title="Category:Houses">Category:Houses</a> o <a href="/wiki/Category:Jimmy_Wales" title="Category:Jimmy Wales">Category:Jimmy Wales</a></li> 
       <li><b>dove?</b> dov'è stata creata l'immagine? Ciò è importante in particolare per le immagini di luoghi, ovviamente. Ad esempio, <a href="/wiki/Category:Saxony" title="Category:Saxony">Category:Saxony</a>.</li> 
       <li><b>quando?</b> dovrebbe fornire un minimo contesto storico, se rilevante. Un esempio potrebbe essere <a href="/wiki/Category:World_War_II" title="Category:World War II">Category:World War II</a></li> 
       <li><b>chi?</b> se l'artista è ben conosciuto, come <a href="/wiki/Category:Rembrandt" title="Category:Rembrandt">Category:Rembrandt</a>, ha senso categorizzare l'immagine per artista; puoi anche usare le pagine del namespace <i>Creator</i> come template per farlo. Vedi <a href="/wiki/Special:MyLanguage/Commons:Creator" title="Special:MyLanguage/Commons:Creator">Commons:Creator</a> per i dettagli.</li> 
       <li><b>come?</b> dovrebbe dare alcune informazioni sul tipo di file multimediale, come <a href="/wiki/Category:Audio" title="Category:Audio">Category:Audio</a></li>
      </ul> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>The above questions cover the main aspects of the image to be categorized. For some images it makes sense to use all, for other images only one or two are reasonable. In addition there are several other aspects of the images that can be used to categorize the image: </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <ul>
        <li><b>what source?:</b> information about where the image came from. For example, <a href="/wiki/Category:Images_from_the_German_Federal_Archive" title="Category:Images from the German Federal Archive">Category:Images from the German Federal Archive</a></li> 
        <li><b>what format?:</b> information about the media type if it is unusual. For example, <a href="/wiki/Category:Audio" title="Category:Audio">Category:Audio</a> or <a href="/wiki/Category:Animated_GIF" title="Category:Animated GIF">Category:Animated GIF</a>, <a href="/wiki/Category:SVG" class="mw-disambig" title="Category:SVG">Category:SVG</a></li> 
        <li><b>what software?:</b> information about software used to create the image. For example, <a href="/wiki/Category:Created_with_Hugin" title="Category:Created with Hugin">Category:Created with Hugin</a></li> 
        <li><b>what camera?:</b> information about the camera. For example, <a href="/wiki/Category:Taken_with_Nikon_D80" title="Category:Taken with Nikon D80">Category:Taken with Nikon D80</a></li>
       </ul> 
       <p>This last set is useful and important but should always be done in addition of the main set of criteria. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>Categorization in Wikimedia Commons is more detailed and deep than categorization in Wikipedia projects. Compared to them, Commons has more categories for individual subjects – places, people, organizations, events, terms, etc. Almost every article on a Wikipedia can have a corresponding category on Commons. However, even if there exist more images of an ordinary person or incidental event, it is practical to group them into a special category and categorize that category instead of categorizing all similar images individually to an identical set of parent categories. </p> 
      </div> 
      <h3><span class="mw-headline" id="Trovare_una_categoria_appropriata">Trovare una categoria appropriata</span></h3> 
      <p>Per trovare le categorie appropriate per i tuoi caricamenti, dovresti navigare la struttura delle categorie partendo da una categoria generica, per poi affinare la tua ricerca andando "verso il basso" nelle sottocategorie fino a che non trovi la più specifica per il file che hai caricato. Puoi navigare la struttura delle categorie seguendo i collegamenti alle sottocategorie oppure espandendo l'albero delle sottocategorie cliccando sul piccolo simbolo + che si trova a fianco del nome della sottocategorie. La sezione <i><a href="#Struttura_delle_categorie">Struttura delle categorie</a></i>, più sotto, fornisce un punto di partenza, e la sezione <i><a href="#Come_categorizzare:_guida_per_argomento">Come categorizzare: guida per argomento</a></i> copre alcuni argomenti. Puoi anche provare ad usare lo strumento <b><a rel="nofollow" class="external text" href="http://tools.wikimedia.de/~daniel/WikiSense/CommonSense.php">CommonSense</a></b>, il cui scopo è quello di aiutare a categorizzare in base a parole chiave. </p> 
      <h3><span class="mw-headline" id="Sovracategorizzazione">Sovracategorizzazione</span></h3> 
      <div class="mw-translate-fuzzy"> 
       <p>La sovracategorizzazione accade quando un'immagine è inserita in numerose categorie entro lo stesso albero. La regola generale è <i>mettere sempre un'immagine nelle categorie più specifiche, e non nei livelli sopra di esse</i>. </p> 
      </div> 
      <p>Un esempio: Facciamo finta che le sfere gialle sono sfere di colore giallo. Possiamo pensare alle categorie <i>Category:Yellow spheres</i> e <a href="/wiki/Category:Spheres" title="Category:Spheres">Category:Spheres</a>. L'immagine che deve essere categorizzata mostra sfere gialle. Categorizziamo il file in <b>Category:Yellow spheres</b>. Ora, se categorizziamo l'immagine anche in <a href="/wiki/Category:Spheres" title="Category:Spheres">Category:Spheres</a>, avremo fatto una <i>sovracategorizzazione</i>: questo perché sappiamo già che le sfere gialle sono sfere. </p>
      <p>Questo si applica anche alla maggior parte delle immagini: come detto più sopra, i file in <a href="/wiki/Category:Paris" title="Category:Paris">Category:Paris</a> non dovrebbero essere anche in <a href="/wiki/Category:France" title="Category:France">Category:France</a>; i file in <a href="/wiki/Category:Albert_Einstein" title="Category:Albert Einstein">Category:Albert Einstein</a> non dovrebbero essere anche in <a href="/wiki/Category:Physicists_from_Germany" title="Category:Physicists from Germany">Category:Physicists from Germany</a> e così via. Visivamente, è lo stesso problema della freccia rossa mostrato nella sezione più sopra. </p> 
      <h4><span id="Quando_la_sovracategorizzazione_.C3.A8_un_problema"></span><span class="mw-headline" id="Quando_la_sovracategorizzazione_è_un_problema">Quando la sovracategorizzazione è un problema</span></h4> 
      <p>Spesso si pensa che se un'immagine è in più categorie, più facile sia trovarla. Un altro esempio: per logica, ogni immagine che mostra un uomo dovrebbe essere in <a href="/wiki/Category:Men" title="Category:Men">Category:Men</a>, perché, a meno che tu non sappia qualcosa di più sulla persona a parte il fatto che è un uomo, sarà possibile trovarla. Il risultato è che la categoria si riempirà in fretta, rendendo necessario passare attraverso centinaia, e in alcuni casi anche migliaia di immagini per trovare quella voluta. Probabilmente non riuscirai a trovarla anche cercandola, e ancor di più, quelli che cercheranno la generica immagine di un uomo per illustrare una voce come <a href="https://it.wikipedia.org/wiki/Uomo" class="extiw" title="it:Uomo">Uomo</a> troveranno la difficoltà di passare attraverso immagini di celebrità del cinema, di scienziati o politici. </p>
      <p>A bassi livelli, il problema diventa meno acuto, poiché il numero di immagini sarà minore — anche se si potrebbe dover cercare fra centinaia di immagini. Ma c'è ancora un problema: torniamo ad Einstein. Si sa che è un fisico, così guarderemo lì. Troviamo un'immagine tra le centinaia presenti nella categoria, ma non siamo ancora contenti, perché ce n'è solo una. Siccome ne ho trovata una lì, possiamo essere convinti che non ce ne saranno altre, nascoste da qualche parte, anzichè cercare in <a href="/wiki/Category:Physicists_from_Germany" title="Category:Physicists from Germany">Category:Physicists from Germany</a> per poi trovare <a href="/wiki/Category:Albert_Einstein" title="Category:Albert Einstein">Category:Albert Einstein</a>, dove ce ne potrebbe anche essere una migliore. Quindi, la sovracategorizzazione porta due problemi: la categoria superiore è sovraffollata, e gli utenti non cercheranno oltre una categoria più rilevante poiché ne hanno raggiunta una che ha un'immagine rilevante. </p> 
      <h4><span id="Una_categorizzazione_inappropriata_delle_categorie_.C3.A8_la_causa_della_sovracategorizzazione"></span><span class="mw-headline" id="Una_categorizzazione_inappropriata_delle_categorie_è_la_causa_della_sovracategorizzazione">Una categorizzazione inappropriata delle categorie è la causa della sovracategorizzazione</span></h4> 
      <p>Può suonare strano, la sottocategorizzazione può essere attualmente una causa della sovracategorizzazione. Ciò accade quando una categoria non è propriamente categorizzata, portando gli utenti a sovracategorizzare un'immagine anziché inserirla nella categoria più specifica. Un esempio di questo: <a href="/wiki/Category:Eiv%C3%B8r_P%C3%A1lsd%C3%B3ttir" title="Category:Eivør Pálsdóttir">Category:Eivør Pálsdóttir</a> è categorizzata solo in <a href="/w/index.php?title=Category:People_by_alphabet&amp;action=edit&amp;redlink=1" class="new" title="Category:People by alphabet (page does not exist)">Category:People by alphabet</a>. Così, se si aggiunge un'immagine della donna, e si sa solo che è una sua immagine, si inserirà l'immagine anche in <a href="/wiki/Category:People_of_the_Faroe_Islands" title="Category:People of the Faroe Islands">Category:People of the Faroe Islands</a> e <a href="/wiki/Category:Vocalists" title="Category:Vocalists">Category:Vocalists</a>. Questa è una sovracategorizzazione, poichè si causa affollamento nelle categorie superiori aggiungendo immagini direttamente in esse. </p>
      <p>Un problema correlato è la categorizzazione errata: <a href="/wiki/Category:Notting_Hill" title="Category:Notting Hill">Category:Notting Hill</a> è stata per più di un mese inserita in <a href="/wiki/Category:London" title="Category:London">Category:London</a>. Quando vi si aggiungeva un'immagine, si aveva la tentazione di inserire l'immagine anche in <a href="/w/index.php?title=Category:Kensington_and_Chelsea&amp;action=edit&amp;redlink=1" class="new" title="Category:Kensington and Chelsea (page does not exist)">Category:Kensington and Chelsea</a>, che è dove si trova Notting Hill. Invece, ogni immagine dovrebbe essere inserita solo nelle categorie più specifiche, e queste categorie dovrebbero a loro volta essere inserite nelle categorie più specifiche. </p>
      <p>Quando si incontra un caso come questo, per favore categorizza le categorie propriamente, se ne sei in grado. Questo non solo aiuterà ad evitare la sovracategorizzazione, ma renderà più facile muoversi attraverso l'albero delle categorie. </p> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <h4><span class="mw-headline" id="Exception_for_images_with_more_categorized_subjects"><span id="OvercatException"></span> Exception for images with more categorized subjects</span></h4> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>A file that depicts only one relevant subject should not be over-categorized. Where a file depicts additional relevant subjects, and the additional subjects do not have their own subcategories, consideration can be given to temporarily categorizing the image in both the subcategory and the parent category. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>For example, this situation might arise in the case of a photograph of three politicians, one of whom is <a href="/wiki/Category:Angela_Merkel" title="Category:Angela Merkel">Angela Merkel</a> (who has her own Commons category), with two other politicians who do not yet have their own categories. While the image would undoubtedly be categorized in <a href="/wiki/Category:Angela_Merkel" title="Category:Angela Merkel">Category:Angela Merkel</a> or one of its subcategories, it would typically be considered to be over-categorization to also include it in <a href="/wiki/Category:Politicians_of_Germany" title="Category:Politicians of Germany">Category:Politicians of Germany</a>. Users would, however, be unlikely to search for the two other politicians in the Merkel category. Ideally, we would create specific subcategories for the two other politicians (where warranted), or find other relevant subcategories (e.g. <a href="/wiki/Category:Politicians_of_Bavaria" title="Category:Politicians of Bavaria">Category:Politicians of Bavaria</a> or <a href="/wiki/Category:Members_of_the_FDP" title="Category:Members of the FDP">Category:Members of the FDP</a>, etc.), that would enable us to avoid over-categorization. In some circumstances, however, we may need to temporarily categorize the image in <a href="/wiki/Category:Politicians_of_Germany" title="Category:Politicians of Germany">Category:Politicians of Germany</a> where other appropriate subcategories do not yet exist. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>Countries may be categorized as part of multiple overlapping categories. For example, <a href="/wiki/Category:India" title="Category:India">Category:India</a> is in <a href="/wiki/Category:Countries_of_South_Asia" title="Category:Countries of South Asia">Category:Countries of South Asia</a> as well as <a href="/wiki/Category:Countries_of_Asia" title="Category:Countries of Asia">Category:Countries of Asia</a>. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>Also user categories are exempted of over-categorization as those are not visible to most viewers, and project users include them for many different purposes like sorting, stats, filling values for userboxes, etc. </p> 
      </div> 
      <h2><span class="mw-headline" id="Come_categorizzare:_guida_per_argomento">Come categorizzare: guida per argomento</span></h2> 
      <p>Per alcune categorie, esiste una speciale guida su come organizzare i contenuti in quella categoria. Alcune di quelle guide sono in questa sezione. Ce ne sono anche altre in testa alla pagina della categoria, nel <a href="https://meta.wikimedia.org/wiki/Namespace" class="extiw" title="meta:Namespace">namespace</a> Category. Quindi, per esempio, alcune guide su come categorizzare immagini di persone è presente in testa a <a href="/wiki/Category:People" title="Category:People">Category:People</a>, ed una parte è anche nella sezione <b><a href="#Persone">Persone</a></b>, più sotto. </p> 
      <h3><span class="mw-headline" id="Persone">Persone</span></h3> 
      <p>Materiale che ritrae persone può essere inserito in <a href="/wiki/Commons:Guide_to_layout/Category_pages" title="Commons:Guide to layout/Category pages">categorie</a> e/o <a href="/wiki/Commons:Guide_to_layout/Gallery_pages" title="Commons:Guide to layout/Gallery pages">gallerie</a> che le descrivono, come <a href="/wiki/Category:Economists_from_the_United_States" title="Category:Economists from the United States">Category:Economists from the United States</a>. Inizia esplorando <a href="/wiki/Category:People" title="Category:People">Category:People</a>. </p>
      <p>Alcune persone degne di nota possono avere molto materiale che le ritraggono, o nomi che sono molto conosciuti, quindi potrebbe valere la pena creare una categoria o una galleria usando il loro nome, come <a href="/wiki/Category:Albert_Einstein" title="Category:Albert Einstein">Category:Albert Einstein</a> (questo è uno dei casi in cui potresti ritrovarti a dover creare nuove categorie frequentemente, ad esempio ogni volta che carichi la prima immagine di una persona degna di nota). Usa la lingua inglese per il nome della categoria o della galleria; è un peccato forzare l'uso di questa lingua per tutti i nomi del mondo ma, come scritto altrove, usiamo la lingua inglese su Commons per la limitatezza del software. </p>
      <p>Una volta che esiste una categoria o galleria con il nome di una persona, tutte le categorie che descrivono quella persona devono essere applicate a tutte le immagini della persona. Queste categorie dovrebbero però essere applicate alla categoria o galleria della persona, e <b>non</b> alle singole pagine di descrizione delle immagini che mostrano quella persona. La singola pagina di descrizione dovrebbe avere un collegamento alla categoria o alla galleria della persona ritratta, e forse categorie che distinguono l'immagine della persona da immagini dello stesso tipo. Tutte le categorie e le gallerie con il nome di una persona dovrebbero essere inserite nella categoria <a href="/w/index.php?title=Category:People_by_alphabet&amp;action=edit&amp;redlink=1" class="new" title="Category:People by alphabet (page does not exist)">People by alphabet</a> e ordinate per cognome: [[Category:People by alphabet|Cognome, nome]]. Comunque, <b>non</b> inserire immagini direttamente in questa categoria! Esistono così tante immagini di persone degne di nota che includerle tutte direttamente intaserebbe la categoria. Limitare <i>Category:People by alphabet</i> a categorie e gallerie riduce notevolmente il numero dei colegamenti presenti. </p>
      <p>Please see <a href="/wiki/Special:MyLanguage/Commons:Category_scheme_People" title="Special:MyLanguage/Commons:Category scheme People">Commons:Category scheme People</a> for details on how to name and organize these categories. </p> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <h3><span id="Landscapes.2C_outdoor_views"></span><span class="mw-headline" id="Landscapes,_outdoor_views">Landscapes, outdoor views</span></h3> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>Content depicting a given subject from a common vantage point are grouped in <i>Views of Subject from Viewpoint</i> categories such as <a href="/wiki/Category:Views_of_Cathedral_of_Seville_from_the_Giralda" title="Category:Views of Cathedral of Seville from the Giralda">Views of Cathedral of Seville from the Giralda</a>. Such categories should be subcategories of both the subject's category (<a href="/wiki/Category:Cathedral_of_Seville" title="Category:Cathedral of Seville">Cathedral of Seville</a> in this example) and the viewpoint's category (<a href="/wiki/Category:Giralda" title="Category:Giralda">Giralda</a> in this example). </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>In this example, the <a href="/wiki/Category:Views_of_Cathedral_of_Seville_from_the_Giralda" title="Category:Views of Cathedral of Seville from the Giralda">Views of Cathedral of Seville from the Giralda</a> category is not placed directly in the subject and viewpoint categories, but in <a href="/wiki/Category:Views_of_the_Cathedral_of_Seville" title="Category:Views of the Cathedral of Seville">Views of the Cathedral of Seville</a> and <a href="/wiki/Category:Views_from_Giralda" title="Category:Views from Giralda">Views from Giralda</a>. Such intermediate categories are often necessary to create structure and avoid <a href="#over-categorization">over-categorization</a>, particularly for views of a city from a vantage point located within the city. For example, <a href="/wiki/Category:Views_of_Rome_from_the_Pincio" title="Category:Views of Rome from the Pincio">Views of Rome from the Pincio</a> needs the intermediate category <a href="/wiki/Category:Views_of_Rome" title="Category:Views of Rome">Views of Rome</a> to avoid placing it directly in <a href="/wiki/Category:Rome" title="Category:Rome">Rome</a>, which would constitute over-categorization. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <h3><span class="mw-headline" id="Texts">Texts</span></h3> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>Texts, such as scans of books, should normally have a category for each version of the scan and each edition of the text. Thus a book published in three separate editions would have a parent category for the book, three subcategories for each text, and further subcategories for the text as a jpeg, a DjVu, etc., assuming each version had actually been uploaded. (Categories would not be created for editions not held on Commons.) This is particularly important for files in formats other than DjVu and PDF, where the category is the only practical means of keeping the scans together; see eg. <a href="/wiki/Category:The_Chronicles_of_England,_Scotland_and_Ireland,_Holinshed,_1587" title="Category:The Chronicles of England, Scotland and Ireland, Holinshed, 1587">Category:The Chronicles of England, Scotland and Ireland, Holinshed, 1587</a> which contains 2857 jpeg images of page scans. </p> 
      </div>

       <h2><span class="mw-headline" id="Categorization_workflow">Categorization workflow</span></h2> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>Currently, a bot checks if newly uploaded files are categorized in topical categories and attempts to categorize files that are not. Before 17 June 2015, <a href="/wiki/User:CategorizationBot" title="User:CategorizationBot">CategorizationBot</a> was responsible for this job. As of June 2019, <a href="/wiki/User:SteinsplitterBot" title="User:SteinsplitterBot">SteinsplitterBot</a> occasionally checks for uncategorized files. The workflow is the following: </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <ol>
        <li>User uploads a new file and adds categories (or not).</li> 
        <li>A bot checks if the file is categorized. 
         <ul>
          <li>File is already categorized =&gt; ok</li> 
          <li>File is not categorized =&gt; the bot tags the file for <a href="/wiki/Category:Media_needing_categories" title="Category:Media needing categories">Category:Media needing categories</a>. Today's files are in <a href="/wiki/Category:Media_needing_categories_as_of_6_June_2021" title="Category:Media needing categories as of 6 June 2021">Category:Media needing categories as of 6 June 2021</a></li>
         </ul></li> 
        <li>Users categorize files further (e.g. category diffusion below)</li>
       </ol> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p><i>See also:</i> <a href="/wiki/User:CategorizationBot#Process" title="User:CategorizationBot">User:CategorizationBot#Process</a>, <a href="/wiki/User:Multichill/Categorization_stats" title="User:Multichill/Categorization stats">categorization statistics</a> </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>Other, if manual, categorization workflows are possible&nbsp;: </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <ul>
        <li>Category filling: Use appropriate keywords in the search engine to find the files that should be in a given category, and put them there.</li> 
        <li>Category diffusing: Go to <a href="/wiki/Category:Categories_requiring_diffusion" title="Category:Categories requiring diffusion">Category:Categories requiring diffusion</a>, select a crowded category, create appropriate subcategories if needed, and move the files to the subcategories. Gadgets like Cat-a-lot and Hotcats can help.</li>
       </ul> 
      </div>

       <h2><span id="Categories_marked_with_.22HIDDENCAT.22"></span><span class="mw-headline" id="Categories_marked_with_&quot;HIDDENCAT&quot;">Categories marked with "HIDDENCAT"</span></h2> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <div class="shortcut2" style="">
        <style data-mw-deduplicate="TemplateStyles:r433650343">.mw-parser-output .shortcut2{float:right;margin:0.4em;border:1px solid #ccc;background-color:white;padding:0 1em;text-align:center;font-size:x-small;border-radius:4px}</style><a href="/wiki/Commons:Shortcuts" title="Commons:Shortcuts">Shortcut</a>
        <br><a href="/wiki/Commons:HIDDENCAT" class="mw-redirect" title="Commons:HIDDENCAT">COM:HIDDENCAT</a>
       </div> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>Many non-topical categories are marked with <code>__HIDDENCAT__</code> or {{<a href="/wiki/Template:Hiddencat" title="Template:Hiddencat">Hiddencat</a>}} on the category page. For example, see <a class="external text" href="https://commons.wikimedia.org/w/index.php?title=Category:PD_NASA&amp;action=edit">Category:PD NASA in edit mode</a>. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>While categories are generally visible on every page, categories marked <code>__HIDDENCAT__</code> are only visible: </p> 
       <ul>
        <li>on the edit screen: at the end of the screen, below the edit box</li> 
        <li>on category pages: 
         <ul>
          <li>on subcategories to the hidden category: in the normal location, but on a separate line with a smaller typeface and the label "Hidden categories."</li> 
          <li>on parent categories: in the same way as other categories</li>
         </ul></li> 
        <li>on file description pages and gallery pages: for logged-in users who have selected to "Show hidden categories" in their <a href="/wiki/Special:Preferences#mw-prefsection-rendering" title="Special:Preferences">appearance preferences</a>. This is activated for all newly registered users.</li>
       </ul> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>This feature is generally used for template-based categories, such as license tag based categories. For example, placing <a href="/wiki/Template:PD-old-100" title="Template:PD-old-100"><span style="font-family:monospace,monospace;">{{PD-old-100}}</span></a> on a file description page adds the file to <a href="/wiki/Category:Author_died_more_than_100_years_ago_public_domain_images" title="Category:Author died more than 100 years ago public domain images">Category:Author died more than 100 years ago public domain images</a>, which is marked with <code>__HIDDENCAT__</code>. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>For more details, see the <a href="https://meta.wikimedia.org/wiki/Help:Category#Hidden_categories" class="extiw" title="m:Help:Category">help section on hidden categories</a> for Mediawiki (the software that Commons uses). </p> 
      </div>

       <h2><span class="mw-headline" id="Tools">Tools</span></h2>

          <li><i>Gallery Details</i>: Adds a link in the toolbox to display galleries and categories (and <a href="/wiki/Special:NewFiles" title="Special:NewFiles">Newimages</a> and <a href="/wiki/Special:Search" title="Special:Search">Search</a> result pages) with extensive details from file description pages and links to easily mark an image without source, etc. If <i>Pretty log</i> is activated, it also works on <a href="/wiki/Special:Log" title="Special:Log">Log</a> pages. <small>[<a href="/wiki/Help:Gadget-GalleryDetails" title="Help:Gadget-GalleryDetails">documentation</a>&nbsp;/&nbsp;<a href="/wiki/MediaWiki_talk:Gadget-GalleryDetails.js" title="MediaWiki talk:Gadget-GalleryDetails.js">talk</a>]</small>&nbsp;</li> 
          <li>Place categories above content, but below image on file description pages. &nbsp;Modifies the placement of categories on the user interface.</li> 
          <li>Add a link to category pages to search for the category name with the option "-incategory". This excludes files already in the category (doesn't work if the category was added by a template). <small>[<a href="/wiki/MediaWiki_talk:Gadget-Searchnotincat.js" title="MediaWiki talk:Gadget-Searchnotincat.js">talk</a>]</small>&nbsp;</li>
