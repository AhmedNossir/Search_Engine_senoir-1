      <p>Eine <b>Kategorie</b> ist eine Softwarefunktion von <a href="https://de.wikipedia.org/wiki/MediaWiki" class="extiw" title="de:MediaWiki">MediaWiki</a>, einer Spezialseite, verwandte Seiten und Medien zu organisieren und zu gruppieren. Jede einzelne Datei sollte (mindestens) einer vorhandenen Kategorie zugeordnet werden. Der Kategoriename sollte ausreichen, um das Thema der Kategorie zu identifizieren, aber oftmals kann es hilfreich sein, durch zusätzlichen Text den Inhalt bzw. das Thema der Kategorie genauer zu beschreiben. Die Kategoriestruktur ist die wichtigste Methode zum Organisieren, Suchen und Finden von Dateien in Commons. Es ist essentiell, dass jede Datei beim Durchsuchen der Kategorienstruktur an einer Stelle gefunden werden kann. Um dies zu ermöglichen, muss jede Datei direkt einer Kategorie zugeordnet werden. Jede Kategorie sollte sich wiederum in einer allgemeineren Kategorie befinden, gemeinsam sollen sie eine hierarchische Struktur bilden. </p>

      <h2><span class="mw-headline" id="Kurzanleitung">Kurzanleitung</span></h2> 
      <p>1. So findet man die entsprechenden Kategorien </p> 
      <dl>
       <dd>
        <ul>
         <li>Kategorien mit der Suchfunktion finden (siehe <a href="#Categorization_tips">#Categorization tips</a>)</li> 
         <li>oder überprüfen, wie ähnliche Dateien auf Commons bereits kategorisiert werden (das kann jedoch für einige Dateien nicht weiterhelfen, die dennoch nicht im vorhandenen Kategoriensystem kategoriziert werden können)</li> 
         <li>oder man beginnt mit der Hauptkategorie (<a href="/wiki/Category:Topics" title="Category:Topics">Category:Topics</a>)</li> 
         <li>Ausgehend von diesen bereits gegebenen Kategorien überprüft man die übergeordneten Kategorien oder die Unterkategorien, um eine geeignete Kategorie für die jeweilige Dateien zu finden. Dabei sollten zu allgemeine Kategorien vermieden werden.</li>
        </ul>
       </dd>
      </dl> 
      <p>2. Füge Kategorien zu den Dateien </p> 
      <dl>
       <dd>
        <ul>
         <li>wähle sie im Hochladeformular aus</li> 
         <li>oder füge sie manuel hinzu, z. B. durch Hinzufügen des Codes <code>[[Category:Foobar]]</code> zur Dateibeschreibungsseite (siehe <a href="#Categorizing_pages">#Categorizing pages</a>)</li> 
         <li>oder verwende HotCat (siehe <a href="/wiki/Special:MyLanguage/Help:Gadget-HotCat" title="Special:MyLanguage/Help:Gadget-HotCat">Help:Gadget-HotCat</a>)</li> 
         <li>oder verwende Cat-a-lot (siehe <a href="/wiki/Help:Gadget-Cat-a-lot/de" title="Help:Gadget-Cat-a-lot/de">Help:Gadget-Cat-a-lot/de</a>)</li>
        </ul>
       </dd>
      </dl> 
      <h2><span class="mw-headline" id="Kategoriestruktur_in_Wikimedia_Commons">Kategoriestruktur in Wikimedia Commons</span></h2> 
      <h3><span class="mw-headline" id="Prinzipien">Prinzipien</span></h3> 
      <p>Die Grundprinzipien sind: </p> 
      <h4><span class="mw-headline" id="Kategoriestruktur">Kategoriestruktur</span></h4> 
      <p>Die Kategoriestruktur ist (idealerweise) eine Multi-Hierarchie mit einer einzigen Wurzelkategorie, <a href="/wiki/Category:CommonsRoot" title="Category:CommonsRoot">Category:CommonsRoot</a>. Alle Kategorien (außer <i>CommonsRoot</i>) sollten in mindestens einer Oberkategorie enthalten sein. Es sollte keine „Schleifen“ in der Struktur geben (d.&nbsp;h. sollte sich nicht selbst enthalten, weder direkt noch indirekt). </p> 
      <h4><span id="Modularit.C3.A4tsprinzip"></span><span class="mw-headline" id="Modularitätsprinzip">Modularitätsprinzip</span></h4> 
      <p>Die Seite (Datei, , Kategorie) sollte in die bestimmteste Kategorie/Kategorien in die die Seite passt einsortiert werden (nicht in ihre Überkategorien). Eine Kategorie kann mehrere Überkategorien haben. Eine Kategorie kann zwei oder mehr unterschiedliche Kriterien kombinieren; diese Kategorien sind sog. „zusammengesetzte Kategorien“ oder „Schnittmengenkategorien“. Z.B. die Grundkategorie <a href="/wiki/Category:Churches" title="Category:Churches">Category:Churches</a> und die Grundkategory <a href="/wiki/Category:Russia" title="Category:Russia">Category:Russia</a> haben als gemeinsame Kategorie: <a href="/wiki/Category:Churches_in_Russia" title="Category:Churches in Russia">Churches in Russia</a>. </p> 
      <h4><span class="mw-headline" id="Vereinfachungsprinzip">Vereinfachungsprinzip</span></h4> 
      <p>Dieses Prinzip beinhaltet nicht zu viele unterschiedliche Kriterien zu kombinieren. </p> 
      <h4><span class="mw-headline" id="Auswahlprinzip">Auswahlprinzip</span></h4> 
      <p>Wir sollten Gegenstände unterschiedlicher Themen nicht in dieselbe Kategorie gruppieren. Die Kategoriebezeichnung sollte eindeutig und nicht gleichlautend sein. </p> 
      <h4><span id="Universitalit.C3.A4tsprinzip"></span><span class="mw-headline" id="Universitalitätsprinzip">Universitalitätsprinzip</span></h4> 
      <p>Identische Themen sollen identische Bezeichnungen in allen Ländern und allen Kategorisationsebenen haben. Die Kategoriestruktur sollte so systematisch und einheitlich wie möglich sein, lokale Dialekte und Bezeichnungen sollten soweit möglich zugunsten allgemeingültiger vernachlässigt werden. Analoge Kategoriezweige sollen auch analoge Strukturen aufweisen. </p> 
      <h3><span id="Art_der_ber.C3.BCcksichtigten_Verh.C3.A4ltnisse"></span><span class="mw-headline" id="Art_der_berücksichtigten_Verhältnisse">Art der berücksichtigten Verhältnisse</span></h3> 
      <p>Die Kategoriestruktur sollte ein Kategoriekonzept von der allgemeinsten zur ganz bestimmten berücksichtigen. Die Struktur verwendet und kombiniert unterschiedliche Zusammenhänge, z. B. </p> 
      <ul>
       <li><a href="https://en.wikipedia.org/wiki/en:Hyponymy_and_hypernymy" class="extiw" title="w:en:Hyponymy and hypernymy">Hyponymy</a>: eine Art ...(typisches biologisches Taxonom)</li> 
       <li><a href="https://en.wikipedia.org/wiki/de:Meronymie" class="extiw" title="w:de:Meronymie">Meronymy</a>: Teil eines..., Mitglied eines (typische geographische Unterteilung, Gebäude/Raum, Gerät/Komponente)</li> 
       <li>Attribute 
        <ul>
         <li>Qualitäts- und allgemeine Attribute (Farbe, Form, Größe, Fähigkeit oder Unvermögen, Nationalität, Technik, Qualität, Auszeichnungen..)</li> 
         <li>Ort: wo, in, von (Ort/Ereignis, Ort/Gebäude, Ort/Ausstellung, Ort/Leute, Land/Sprache, Quelle/Werk, Werk oder Land/Produkt etc)</li> 
         <li>Zeitwahl: wann (Zeit/Ereignis, Zeit/abgebildete Situation, Entstehungszeitpunkt, Beginn oder Konstruktion, Sterbezeitpunkt, Zerstörung, Beendigung etc.)</li>
        </ul></li> 
       <li>Aktive und beeinflussende Beziehungen: (Schöpfer/Werk, Gerät/Produkt, Firma/Produkt, Fach- oder Berufsrichtung/ihre Themen und Regeln, Eltern/Kinder, Unterordnung, Eigner/Eigentum, Initiator/Folger, Thema/andere Themen die sich darauf beziehen oder danach benannt sind, Thema/sein Duplikat, seine Imitiation, Darstellung oder Symbol, etc.)</li> 
       <li>Abwandlung: Original/Abwandlung oder Abwandlung/Original (Zirkelverweise vermeiden) – umbenannte, erneuerte, umfunktionierte oder transformierte Themen.</li>
      </ul> 
      <h3><span class="mw-headline" id="Hauptkategorien">Hauptkategorien</span></h3> 
      <p>Die Hauptkategorien (also die, die direkt in CommonsRoot enthalten sind) unterteilen die Kategoriestruktur nach dem Zweck der enthaltenen Kategorien: </p> 
      <ul>
       <li><b><a href="/wiki/Category:Topics" title="Category:Topics">Category:Topics</a></b> – Dies ist die Wurzelkategorie für die THEMATISCHE Kategorisierung. ALLE Dateien sollten in mindestens einer Kategorie in diesem Teil der Struktur eingeordnet sein, so dass sie bei einer thematischen Suche gefunden werden können. Thematische Kategorien sollten nicht über Vorlagen eingebunden werden.</li> 
       <li><b><a href="/wiki/Category:Copyright_statuses" title="Category:Copyright statuses">Category:Copyright statuses</a></b> – Dies ist die Wurzelkategorie für die Kategorisierung nach LIZENZ. ALLE Dateien sollen in diesen Teil der Struktur eingeordnet werden, indem man sie mit einem geeigneten <a href="/wiki/Commons:Copyright_tags/de" title="Commons:Copyright tags/de">Lizenzbaustein</a> versieht. Diese Art von Kategorien wird über die Einbindung von Vorlagen hinzugefügt.</li> 
       <li><b><a href="/wiki/Category:Media_by_source" title="Category:Media by source">Category:Media by source</a></b> - Dies ist die Wurzelkategorie für die Kategorisierung nach der QUELLE, aus der sie stammt (Bücher, Sammlungen, Internetseiten, etc.). Diese Art von Kategorien wird gewöhnlich über Vorlagen hinzugefügt.</li> 
       <li><b><a href="/wiki/Category:Media_types" title="Category:Media types">Category:Media types</a></b> – Dies ist die Wurzelkategorie für die Kategorisierung nach dem TYP der Datei. Bitte beachte, dass auf diesen Aspekt der Kategorisierung bei Bildern verzichtet werden kann, da sie den Großteil der Dateien auf Commons ausmachen.</li> 
       <li><b><a href="/wiki/Category:Commons" title="Category:Commons">Category:Commons</a></b> – Dies ist die Wurzelkategorie für Kategorien, die sich mit der Wartung befassen und für Seiten (Commons:- und Help:-), die keine Mediendateien sind. Die übersetzten Seiten sollten in die Kategorie ihrer Sprache einsortiert werden, indem "Category:Commons-<a href="https://en.wikipedia.org/wiki/de:ISO_639" class="extiw" title="w:de:ISO 639">ISO-SPRACHCODE</a>" genutzt wird. Die Struktur von <a href="/wiki/Category:Commons-en" title="Category:Commons-en">Category:Commons-en</a> ist eine einfache Hierarchie, die für jede andere Sprache entsprechende Subkategorien hat. Verwende keine zwei Doppelpunkte in Kategorie- oder Seitennamen. Siehe <a href="/wiki/Category_talk:Commons_batch_uploading" title="Category talk:Commons batch uploading">diese Diskussion</a> und <a href="/wiki/Help:Namespaces/de" title="Help:Namespaces/de">Help:Namensräume</a>.</li>
      </ul> 
      <dl>
       <dd>
        <dl>
         <dd>
          Es gibt eine Subkategorie <a href="/wiki/Category:Commons_maintenance_content" title="Category:Commons maintenance content">Category:Commons maintenance content</a> für die spezielle Wartung globaler Inhalte von Wikimedia Commons, die nicht übersetzt wird. ALLE Mediendateien sollten unter den ersten vier Kategorien kategorisiert werden, NUR Dateien, bei denen es Probleme gibt, die behoben werden müssen, sollten auch in der Subkategorie <a href="/wiki/Category:Commons_maintenance_content" title="Category:Commons maintenance content">Category:Commons maintenance content</a> enthalten sein.
         </dd>
        </dl>
       </dd>
      </dl> 
      <ul>
       <li><b><a href="/wiki/Category:Commons_users" title="Category:Commons users">Category:Commons users</a></b> – Dies ist für Kategorien, die Galerien, Bilder und Texte von Commons-Nutzern enthalten, sortiert beispielsweise nach der Sprache, die sie sprechen. Sie enthält auch die <a href="/wiki/Category:User_galleries" title="Category:User galleries">Category:User galleries</a> für benutzerspezifische (d.h. nicht-thematische) Kategorien, die nicht in Englisch sein müssen.</li>
      </ul> 
      <h2><span class="mw-headline" id="Kategorien_verwenden">Kategorien verwenden</span></h2> 
      <p>Dateien sollten immer thematisch in Kategorien und/oder Galerieseiten eingeteilt werden, damit die Beiträge von anderen gefunden und verwendet werden können. </p>
      <p>Es ist selten notwendig, eine neue Kategorie zu erstellen (es gibt Ausnahmen, wie das Hochladen eines neuen Textes, siehe <i><a href="#Personen">Personen</a></i> unten). Bevor du dies tust, stelle sicher, dass du mit der bestehenden Kategoriestruktur und mit den Gepflogenheiten und Richtlinien von Commons vertraut bist. Schaue bitte nach, ob es für dein Thema ein <a href="/wiki/Category:Commons_category_schemes" title="Category:Commons category schemes">Kategorieschema</a> oder ein <a href="/wiki/Category:Commons_projects" title="Category:Commons projects">Commons-Projekt</a> gibt und folgen den dort beschriebenen Konventionen. </p>
      <p><span id="Kategorienamen"></span> </p> 
      <h3><span class="mw-headline" id="Kategoriebezeichnungen">Kategoriebezeichnungen</span></h3> 
      <p>Kategoriebezeichnungen sollten grundsätzlich in Englisch erfolgen (siehe <a href="/wiki/Commons:Language_policy/de" title="Commons:Language policy/de">Commons:Sprache</a>). Ausnahmen sind z. B. einige Eigennamen, biologische Taxa und Bezeichnungen für die die nicht englische Bezeichnung allgemein in der englischen Sprache verwendet wird. Es wird das <a href="https://en.wikipedia.org/wiki/de:Liste_lateinischer_Alphabete" class="extiw" title="w:de:Liste lateinischer Alphabete">Lateinisches Alphabet</a> verwendet einschliesslich diakritischer Zeichen und abgeleiteter Buchstaben, nicht-lateinische Buchstaben werden Englisch-Latein transkribiert. Englische Buchstaben (<a href="https://en.wikipedia.org/wiki/ISO/IEC_646" class="extiw" title="w:ISO/IEC 646">ISO/IEC 646</a>) werden gegenüber nationalen Varianten bevorzugt () wo möglich. </p> 
      <ul>
       <li>Arten oder Gruppen von Objekten oder Personen sollten grundsätzlich Namen in der Mehrzahl haben: <a href="/wiki/Category:Tools" title="Category:Tools">Category:Tools</a>, <a href="/wiki/Category:Artists" title="Category:Artists">Category:Artists</a>, <a href="/wiki/Category:Lakes" title="Category:Lakes">Category:Lakes</a>, <a href="/wiki/Category:Paintings" title="Category:Paintings">Category:Paintings</a>, <a href="/wiki/Category:Sculptures" title="Category:Sculptures">Category:Sculptures</a>, <a href="/wiki/Category:Popes" title="Category:Popes">Category:Popes</a> usw. und falls möglich auf Englisch.</li> 
       <li>Allgemeine Themen oder Aktivitäten erfordern üblicherweise einen Namen in der Einzahl: <a href="/wiki/Category:History" title="Category:History">Category:History</a>, <a href="/wiki/Category:Weather" title="Category:Weather">Category:Weather</a>, <a href="/wiki/Category:Music" title="Category:Music">Category:Music</a>, <a href="/wiki/Category:Painting" title="Category:Painting">Category:Painting</a>, <a href="/wiki/Category:Sculpture" title="Category:Sculpture">Category:Sculpture</a>, <a href="/wiki/Category:Papacy" title="Category:Papacy">Category:Papacy</a> usw. und falls möglich auf Englisch.</li> 
       <li>Ein bestimmtes individuelles Objekt (ein spezifische Person, Gebäude, Monument, Kunstwerk, Organisation, Ereignis, usw.) verwendet für gewöhnlich die Einzahl (allerdings nicht immer). <a href="https://en.wikipedia.org/wiki/de:Eigenname" class="extiw" title="w:de:Eigenname">Eigennamen</a> die keine gebräuchliche englische Entsprechung haben werden nicht spontan übersetzt, sondern es wird die Ursprungsform verwendet.</li>
      </ul> 
      <p>Es gab zu diesem Thema auch einen <a href="/wiki/Special:MyLanguage/Commons:Naming_categories" title="Special:MyLanguage/Commons:Naming categories">gescheiterten Vorschlag</a>. </p>
      <p>Kategorien die Unterkategorien anhand des Namens zusammenfassen sollten grundsätzlich „by name“ genannt werden, anstatt „by alphabet“ (z.B. <a href="/wiki/Category:Ships_by_name" title="Category:Ships by name">Category:Ships by name</a>). </p>
      <p>Es mangelt uns immer noch an der Internationalisierung der Kategorie-Namen, jedoch sollte dieses Problem durch entsprechende Änderungen an der MediaWiki-Software gelöst werden (siehe <a href="https://phabricator.wikimedia.org/T31928" class="extiw" title="phab:T31928">T31928</a>: <i>Auch bei Kategorien übersetzte Titel entsprechend der Benutzersprache anzeigen</i>). Vermischte Kategoriestrukturen in unterschiedlichen Sprachen anzulegen, würde es nur schlimmer machen. </p>
      <p>Für die allgemeine Diskussion zu MediaWikis Kategorien-Funktion, siehe <a href="https://meta.wikimedia.org/wiki/Help:Categories" class="extiw" title="m:Help:Categories">Handbuchseite über Kategorien</a>. </p> 
      <h3><span class="mw-headline" id="Seiten_einordnen">Seiten einordnen</span></h3> 
      <p>Um eine Seite (sei es ein Bild, eine Galerieseite oder eine Kategorieseite) zu einer Kategorie hinzuzufügen, füge den folgenden Code am Ende der Seite ein. </p> 
      <dl>
       <dd>
        <code>[[Category:<i>Kategoriename</i>]]</code>
       </dd>
      </dl> 
      <p>Beispiel: Für ein Diagramm, das den Orbit des Halleyschen Kometen zeigt, wären die folgenden Kategorien auf der Bildbeschreibungsseite einzutragen: </p> 
      <dl>
       <dd>
        <code>[[Category:Astronomical diagrams]]<br> [[Category:Comets]]</code>
       </dd>
      </dl> 
      <p>Dadurch wird das Diagramm in den Kategorien <a href="/wiki/Category:Astronomical_diagrams" title="Category:Astronomical diagrams">Astronomical diagrams</a> und <a href="/wiki/Category:Comets" title="Category:Comets">Comets</a> auftauchen. </p>
      <p>Trage nur die spezifischste(n) Kategorie(n) ein. Für genauere Informationen darüber, wie man die richtigen Kategorien für Bilder und Galerien findet, siehe den Abschnitt <i><a href="#Kategorisieren_deiner_Beiträge">Kategorisieren deiner Beiträge</a></i> weiter unten. </p> 
      <h3><span class="mw-headline" id="Neue_Kategorien_anlegen">Neue Kategorien anlegen</span></h3> 
      <p>Um eine neue Kategorie anzulegen: </p> 
      <ol>
       <li>Unternimm eine gründliche Suche, um sicherzustellen, dass es keine Kategorie gibt, die diesem Zweck dient.</li> 
       <li>Finde Bilder (oder eine Galerie oder andere Seiten), die in die neue Kategorie einsortiert werden sollten. Bearbeite die Seite und füge am Ende die neue Kategorie ein, z.B. <code>[[Category:Title]]</code>. Speicher die bearbeitete Seite. Die neue Kategorie erscheint als Rotlink unten auf der Seite.</li> 
       <li>Klicke auf den Rotlink. Die neue, leere Kategorie-Seite erscheint zur Bearbeitung. Diese Kategorie kannst du bearbeiten wie jede andere Wiki-Seite.</li>
      </ol> 
      <p>Eine Kategorieseite sollte die folgenden Informationen enthalten (nach Wichtigkeit sortiert): </p> 
      <ul>
       <li>Kategorisierungs-Verweise, die die Kategorie in eine oder mehrere Überkategorien einordnen. Füge dazu am Ende der Seite Zeilen der Form <code>[[Category:Überkategorie]]</code> ein.</li> 
       <li>Ein kurzer Beschreibungstext, der beschreibt, was die Kategorie enthalten soll, sofern der Titel selber nicht eindeutig genug ist. Eine englische Beschreibung sollte immer vorhanden sein, weitere Sprachen sind optional. Mehrere Sprachversionen der Beschreibung können jeweils gekennzeichnet werden, zum Beispiel mit der Vorlage <code>{{<a href="/wiki/Template:Mld" class="mw-redirect" title="Template:Mld">mld</a>}}</code>, um nur die Beschreibung in der jeweils vom Benutzer bevorzugten Sprache anzeigen zu lassen, oder den Vorlagen <code>{{ab|...}}</code> für eine Beschreibung in Abchasisch, <code>{{de|...}}</code> für eine deutschsprachige usw., die auf <a href="/wiki/Commons:Vorlagen_f%C3%BCr_Galerien" class="mw-redirect" title="Commons:Vorlagen für Galerien">Commons:Vorlagen für Galerien</a> aufgelistet sind.</li> 
       <li>Interwikis verweisen auf Kategorien oder Artikel zum gleichen Thema in den Wikipedias, indem sie den Seitenlink zur entsprechenden Wikipedia-Seite angeben. Klicke nach Erstellung einer Kategorie-Seite auf „Links hinzufügen“ unter „In Wikipedia“ am Ende der linken Seitenleiste.</li> 
       <li>Falls die Kategorie in Überkategorien nach einer anderen Zeichenfolge als dem Kategorietitel einsortiert werden soll, füge <code>{{DEFAULTSORT:}}</code> ein. Beispielsweise sollten Personenkategorien nicht nach der üblichen Schreibweise für deren Seitentitel sortiert werden. Füge bei solchen Kategorien eine Zeile mit der richtigen Sortierzeichenfolge nach dem Muster <code>{{DEFAULTSORT:Nachname, Vorname}}</code> ein. Für weitergehende Informationen dazu siehe <a href="https://meta.wikimedia.org/wiki/Categories#Sort_key" class="extiw" title="meta:Categories">meta:Categories#Sort key</a>.</li>
      </ul> 
      <p>Siehe auch <a href="#Wie_Kategorisierung:_Leitfaden_nach_Thema">#Wie Kategorisierung: Leitfaden nach Thema</a> für eine Anleitung für spezielle Arten von Kategorien, z. B. Kategorien für <a href="#Personen">#Personen</a>. </p> 
      <h3><span class="mw-headline" id="Umbenennen_oder_Verschieben_von_Kategorien">Umbenennen oder Verschieben von Kategorien</span></h3> 
      <p>Bitte lies <a href="/wiki/Commons:Rename_a_category/de" title="Commons:Rename a category/de">Commons:Umbenennen einer Kategorie</a> dazu. </p> 
      <h2><span id="Kategorisieren_deiner_Beitr.C3.A4ge"></span><span class="mw-headline" id="Kategorisieren_deiner_Beiträge">Kategorisieren deiner Beiträge</span></h2> 
      <p>Seiten (einschließlich Kategorieseiten) werden nach ihrem Thema und nicht nach ihrem Inhalt kategorisiert, da der Inhalt in der Regel kein fester Bestandteil der Kategorieseite ist; insbesondere kannst du vorübergehend unpassende Inhalte in einer Kategorieseite finden. </p>
      <p>Beispiel: Nehmen wir an, dass <a href="/wiki/Category:Spheres" title="Category:Spheres">Category:Spheres</a> nur Bilder von Kristallkugeln enthält. Nach dem aktuellen Inhalt dürfest du <a href="/wiki/Category:Glass" title="Category:Glass">Category:Glass</a> auf der Kategorieseite nicht hinzufügen, da Kugeln aus den verschiedensten Materialien gefertigt sein können. Normalerweise würde jedes Bild, das ein Glasobjekt zeigt, bereits in <a href="/wiki/Category:Glass" title="Category:Glass">Category:Glass</a> kategorisiert sein. (oder in einer Kategorie seiner Substruktur). Wenn also die <a href="/wiki/Category:Spheres" title="Category:Spheres">Category:Spheres</a> wirklich mit Bildern von Kristallkugeln überfüllt ist, wäre es eine bessere Idee, eine neue Kategorieseite zu erstellen, wie Category:Glass spheres oder Category:Crystal balls, kategorisiert in <a href="/wiki/Category:Spheres" title="Category:Spheres">Category:Spheres</a> und <a href="/wiki/Category:Glass" title="Category:Glass">Category:Glass</a>. </p>
      <p>Generell sollten Dateien nur in der spezifischsten Kategorie, die es für ein bestimmtes Thema gibt, abgelegt werden. Zum Beispiel sollten Dateien in <a href="/wiki/Category:Looking_up_the_center_of_the_Eiffel_Tower" title="Category:Looking up the center of the Eiffel Tower">Category:Looking up the center of the Eiffel Tower</a> nicht auch in <a href="/wiki/Category:Paris" title="Category:Paris">Category:Paris</a> enthalten sein. <i>(siehe <a href="#Overcategorization">Über-Kategorisierung</a> unten)</i>. Wenn du keine Kategorie findest, die deinem Zweck entspricht, kannst du sie erstellen - lies dir aber zuerst sorgfältig den Abschnitt über die <a href="#How_to_use_categories">Verwendung von Kategorien</a> durch. </p>
      <p>Das bedeutet nicht, dass ein Bild nur in eine Kategorie gehört; es bedeutet nur, dass Bilder nicht in <i>redundante</i> oder unspezifische Kategorien gehören. Beispielsweise sollte ein Bild eines Eisbären, der von einem Hubschrauber aus einem Eisberg gerettet wird, in <a href="/wiki/Category:Ursus_maritimus" title="Category:Ursus maritimus">Category:Ursus maritimus</a>, <a href="/wiki/Category:Icebergs" title="Category:Icebergs">Category:Icebergs</a> und <a href="/wiki/Category:Rescue_helicopters" title="Category:Rescue helicopters">Category:Rescue helicopters</a>. enthalten sein. Es sollte sich jedoch nicht in <a href="/wiki/Category:Ursidae" title="Category:Ursidae">Category:Ursidae</a>, <a href="/wiki/Category:Sea_ice" title="Category:Sea ice">Category:Sea ice</a> oder <a href="/wiki/Category:Aircraft" title="Category:Aircraft">Category:Aircraft</a> befinden. </p> 
      <h3><span class="mw-headline" id="Kategorisierungstipps">Kategorisierungstipps</span></h3> 
      <p>Die Kategorien (oder Galerien) sollten mindestens eine der folgenden Fragen beantworten: </p> 
      <ul>
       <li><b>Was? / Wen?:</b> was oder wen zeigt die Datei? Was ist das Hauptmotiv? Was sind die bemerkenswerten Merkmale des Bildes? Zum Beispiel <a href="/wiki/Category:Automobiles" title="Category:Automobiles">Category:Automobiles</a>, <a href="/wiki/Category:Felis_silvestris_catus" title="Category:Felis silvestris catus">Category:Felis silvestris catus</a> oder <a href="/wiki/Category:Jimmy_Wales" title="Category:Jimmy Wales">Category:Jimmy Wales</a></li> 
       <li><b>Wo?:</b> Wo wurde das Bild aufgenommen? Was ist der Standort des Motivs? Was ist der Standort der Kamera? Dies ist besonders wichtig für Bilder von Orten. Z.B. <a href="/wiki/Category:Basin_Street,_New_Orleans" title="Category:Basin Street, New Orleans">Category:Basin Street, New Orleans</a> 
        <dl>
         <dd>
          verwende auch <a href="/wiki/Template:Location" title="Template:Location"><span style="font-family:monospace,monospace;">{{Location}}</span></a>
         </dd>
        </dl></li> 
       <li><b>Wann?:</b> Wann sind die abgebildeten Ereignisse passiert, bzw. wann wurde das Bild erstellt? Wann wurde das Bild aufgenommen? Dies ist besonders bei historischen Bildern wichtig. Ein Beispiel wäre <a href="/wiki/Category:Warsaw_in_September_1939" title="Category:Warsaw in September 1939">Category:Warsaw in September 1939</a>, <a href="/wiki/Category:April_2010_in_Northern_Ireland" title="Category:April 2010 in Northern Ireland">Category:April 2010 in Northern Ireland</a></li> 
       <li><b>wer?:</b> Wer ist der Autor? Dies ist besonders wichtig für Werke bekannter Künstler und für historische Bilder, zum Beispiel <a href="/wiki/Category:Paintings_by_Rembrandt" title="Category:Paintings by Rembrandt">Category:Paintings by Rembrandt</a>. Du kannst dazu auch die Seiten aus dem <a href="/wiki/Category:Creator_templates" title="Category:Creator templates"><i>Creator</i>-Namensraum</a> als Vorlagen verwenden.</li> 
       <li><b>wie?:</b> Wie macht die Datei (oder das Bild) das? Im Einzelnen: 
        <ul>
         <li><b>welche Ansicht?:</b> welche Art von Ansicht zeigt das Bild? z.B. <a href="/wiki/Category:Plan_views" title="Category:Plan views">Category:Plan views</a>, <a href="/wiki/Category:Panoramics" title="Category:Panoramics">Category:Panoramics</a>.</li> 
         <li><b>welche Farbe?:</b> welche ist die allgemeine Farbe? z.B. <a href="/wiki/Category:Yellow_animals" title="Category:Yellow animals">Category:Yellow animals</a></li> 
         <li><b>welche Fototechnik?:</b>. Wenn das Bild eine bestimmte <a href="/wiki/Category:Photographic_techniques" title="Category:Photographic techniques">Technik</a> oder <a href="/wiki/Category:Photographic_effects" title="Category:Photographic effects">Effekt</a> verwendet, wende die entsprechende Kategorie an: z.B. <a href="/wiki/Category:Tone-mapped_HDR_images" title="Category:Tone-mapped HDR images">Category:Tone-mapped HDR images</a>, <a href="/wiki/Category:Black_and_white_photographs" title="Category:Black and white photographs">Category:Black and white photographs</a>, <a href="/wiki/Category:Double_exposure" title="Category:Double exposure">Category:Double exposure</a></li>
        </ul></li>
      </ul> 
      <p>Die obigen Fragen betreffen die Hauptaspekte des zu kategorisierenden Bildes. Bei einigen Bildern ist es sinnvoll, alle zu verwenden, bei anderen sind nur ein oder zwei sinnvoll. Darüber hinaus gibt es verschiedene andere Aspekte der Bilder, die zur Kategorisierung des Bildes verwendet werden können: </p> 
      <ul>
       <li><b>welche Quelle?:</b> Informationen darüber, woher das Bild stammt. Zum Beispiel <a href="/wiki/Category:Images_from_the_German_Federal_Archive" title="Category:Images from the German Federal Archive">Category:Images from the German Federal Archive</a></li> 
       <li><b>welches Format?:</b> Informationen über den Medientyp, wenn er ungewöhnlich ist. Zum Beispiel <a href="/wiki/Category:Audio" title="Category:Audio">Category:Audio</a> oder <a href="/wiki/Category:Animated_GIF" title="Category:Animated GIF">Category:Animated GIF</a>, <a href="/wiki/Category:SVG" class="mw-disambig" title="Category:SVG">Category:SVG</a></li> 
       <li><b>welche Software?:</b> Informationen über die zur Erstellung des Bildes verwendete Software. Zum Beispiel <a href="/wiki/Category:Created_with_Hugin" title="Category:Created with Hugin">Category:Created with Hugin</a></li> 
       <li><b>welche Kamera?:</b> Informationen über die Kamera. Zum Beispiel <a href="/wiki/Category:Taken_with_Nikon_D80" title="Category:Taken with Nikon D80">Category:Taken with Nikon D80</a></li>
      </ul> 
      <p>Diese letzte Gruppe ist nützlich und wichtig, sollte aber immer zusätzlich zu den Hauptkriterien angewandt werden. </p>
      <p>Die Kategorisierung auf Wikimedia Commons ist detaillierter und tiefer als die Kategorisierung von Wikipedia-Projekten. Im Vergleich zu ihnen gibt es bei Commons mehr Kategorien für einzelne Themen - Orte, Personen, Organisationen, Ereignisse, Begriffe usw. Fast jeder Artikel in einer Wikipedia kann eine entsprechende Kategorie zu Commons haben. Doch selbst wenn es mehr Bilder einer gewöhnlichen Person oder eines zufälligen Ereignisses gibt, ist es praktisch, sie in einer speziellen Kategorie zu gruppieren und diese Kategorie zu kategorisieren, anstatt alle ähnlichen Bilder einzeln in identische übergeordnete Kategorien einzuordnen. </p> 
      <h3><span class="mw-headline" id="Eine_passende_Kategorie_finden">Eine passende Kategorie finden</span></h3> 
      <p>Um geeignete Kategorien für deine Dateiens zu finden, solltest du in der Kategoriestruktur ausgehend von einer generischen Kategorie navigieren. Grenzen deine Suche auf Unterkategorien ein, bis du die spezifischste Kategorie findest, die zu der von dir hochgeladenen Datei passt. Du kannst in der Kategoriestruktur navigieren, indem du Links zu Unterkategorien folgst oder den Baum der Unterkategorien erweiterst, indem du auf die kleinen +-Symbole an den Namen der Unterkategorien klickst. Der Abschnitt <i><a href="#_Hauptkategorien">Hauptkategorien</a></i> oben bietet einen Ausgangspunkt, und der Abschnitt <i><a href="#Wie_Kategorisierung:_Leitfaden_nach_Thema">Kategorisieren: Leitfaden nach Thema</a></i> behandelt einige Themen ausführlicher. </p>

      <p>Überkategorisierung bedeutet, dass eine Datei, eine Kategorie oder eine andere Seite auf mehreren Ebenen desselben Zweiges im Kategoriebaum platziert wird. Die allgemeine Regel lautet: <i>Platziere ein Bild immer in den spezifischsten Kategorien und nicht in den darüber liegenden Ebenen.</i> Ausnahmen von dieser Regel werden im <a href="#OvercatException">Abschnitt unten</a> erläutert. </p>
      <p>Beispiel: Ein Bild, das kategorisiert werden soll, zeigt einen gelben Kreis. Dieses Bild soll in <a href="/wiki/Category:Yellow_circles" title="Category:Yellow circles">Category:Yellow circles</a> platziert werden. Wenn es auch in <a href="/wiki/Category:Circles" title="Category:Circles">Category:Circles</a> platziert wird, ist es <i>überkategorisiert</i>. Wir wissen bereits, dass es sich um einen Kreis handelt, denn alle gelben Kreise sind Kreise. Daher ist <a href="/wiki/Category:Circles" title="Category:Circles">Category:Circles</a> redundant. <a href="/wiki/Template:Uw-overcat" title="Template:Uw-overcat">Template:Uw-overcat</a> kann verwendet werden, um die Benutzer darauf hinzuweisen. </p>
      <p>Dies gilt für die meisten Dateien: Wie unter der nebenstehenden Abbildung erwähnt, sollten Dateien in <a href="/wiki/Category:Black_and_white_photographs_of_the_Eiffel_Tower" title="Category:Black and white photographs of the Eiffel Tower">Category:Black and white photographs of the Eiffel Tower</a> nicht auch in <a href="/wiki/Category:Paris" title="Category:Paris">Category:Paris</a>, Dateien in <a href="/wiki/Category:Albert_Einstein" title="Category:Albert Einstein">Category:Albert Einstein</a> nicht auch in <a href="/wiki/Category:Physicists_from_Germany" title="Category:Physicists from Germany">Category:Physicists from Germany</a> enthalten sein und so weiter. </p> 
      <h4><span id="Warum_.C3.9Cberkategorisierung_ein_Problem_ist"></span><span class="mw-headline" id="Warum_Überkategorisierung_ein_Problem_ist">Warum Überkategorisierung ein Problem ist</span></h4> 
      <p>Es wird oft angenommen, dass ein Bild umso leichter zu finden ist, je mehr Kategorien es umfasst. Ein weiteres Beispiel: Nach dieser Logik sollte jedes Bild, das einen Mann zeigt, in <a href="/wiki/Category:Men" title="Category:Men">Category:Men</a> stehen, denn selbst wenn du nicht mehr über die gesuchte Person weisst, als dass es sich um einen Mann handelt, wirst du sie finden können. Das Ergebnis ist, dass sich die oberste Kategorie füllt, so dass man Hunderte, oder in diesem Fall wahrscheinlicher Tausende von Bildern durchgehen muss, um das gewünschte Bild zu finden. Du wirst wahrscheinlich nicht finden, was du suchst, und außerdem werden diejenigen, die ein allgemeines Bild eines Mannes zur Illustration eines Artikels wie <a href="https://en.wikipedia.org/wiki/Man" class="extiw" title="en:Man">en:Man</a> suchen, feststellen, dass sie unter den Filmstars, Wissenschaftlern und Politikern untergegangen sind. </p>
      <p>Auf den unteren Ebenen wird das Problem weniger akut, da die Anzahl der Bilder geringer sein wird - sie können aber immer noch leicht in die Hunderte gehen. Aber es gibt immer noch ein Problem: Gehen wir zurück zu Einstein. Ich weiß, dass er Physiker ist, also werde ich in der Kategorie <a href="/wiki/Category:Physicists" title="Category:Physicists">Category:Physicists</a> suchen. Unter den Hunderten von Bildern anderer Physiker finde ich ein Bild von Einstein, mit dem ich nicht allzu glücklich bin, aber es ist das einzige, das dort zu finden ist. Da es dort ein Bild gab, gehe ich davon aus, dass es nicht mehr anderswo versteckt ist, anstatt weiter in <a href="/wiki/Category:Physicists_from_Germany" title="Category:Physicists from Germany">Category:Physicists from Germany</a> zu suchen und so <a href="/wiki/Category:Albert_Einstein" title="Category:Albert Einstein">Category:Albert Einstein</a> zu finden, wo es vielleicht ein besseres gibt. Die Über-Kategorisierung hat also zu zwei Problemen geführt: Die oberste Kategorie ist unübersichtlich, und die Benutzer werden aufhören, nach der relevantesten Kategorie zu suchen, da sie eine erreicht haben, die ein relevantes Bild hat. </p> 
      <h4><span id="Unpassende_Kategorisierung_von_Kategorien_ist_eine_Ursache_von_.C3.9Cberkategorisierung"></span><span class="mw-headline" id="Unpassende_Kategorisierung_von_Kategorien_ist_eine_Ursache_von_Überkategorisierung">Unpassende Kategorisierung von Kategorien ist eine Ursache von Überkategorisierung</span></h4> 
      <p>So seltsam es klingen mag, aber eine Unterkategorisierung kann eine Ursache für eine Überkategorisierung sein. Wenn eine Kategorie selbst nicht richtig kategorisiert ist, kann dies dazu führen, dass Benutzer Dateien, die zu dieser Kategorie gehören, überkategorisieren. Ein Beispiel hierfür: <a href="/wiki/Category:Eiv%C3%B8r_P%C3%A1lsd%C3%B3ttir" title="Category:Eivør Pálsdóttir">Category:Eivør Pálsdóttir</a> wurde nur in <a href="/wiki/Category:People_by_name" title="Category:People by name">Category:People by name</a> kategorisiert. Ein Benutzer, der ein Bild von ihr kategorisiert, könnte dann versucht sein, das Bild auch in <a href="/wiki/Category:Female_vocalists_from_the_Faroe_Islands" title="Category:Female vocalists from the Faroe Islands">Category:Female vocalists from the Faroe Islands</a> zu platzieren. Die richtige Lösung besteht darin, das Bild nur in <a href="/wiki/Category:Eiv%C3%B8r_P%C3%A1lsd%C3%B3ttir" title="Category:Eivør Pálsdóttir">Category:Eivør Pálsdóttir</a> zu platzieren und diese Kategorie zu einer Unterkategorie von <a href="/wiki/Category:Female_vocalists_from_the_Faroe_Islands" title="Category:Female vocalists from the Faroe Islands">Category:Female vocalists from the Faroe Islands</a> zu machen. Zu diesem Zeitpunkt werden jedoch alle Bilder, die bereits in beiden Kategorien platziert waren, überkategorisiert und müssen manuell aus der übergeordneten Kategorie entfernt werden. </p>
      <p>Ein damit zusammenhängendes Problem ist die fehlerhafte Kategorisierung. Notting Hill ist ein Bezirk innerhalb der Londoner Bezirke Kensington und Chelsea. Bei seiner Schaffung wurde <a href="/wiki/Category:Notting_Hill" title="Category:Notting Hill">Category:Notting Hill</a> direkt in <a href="/wiki/Category:London" title="Category:London">Category:London</a> statt in der Unterkategorie <a href="/wiki/Category:Royal_Borough_of_Kensington_and_Chelsea" title="Category:Royal Borough of Kensington and Chelsea">Category:Royal Borough of Kensington and Chelsea</a> platziert, wo er hätte platziert werden sollen. Ein Benutzer, der ein Bild von Notting Hill kategorisiert, könnte dann versucht sein, es sowohl in der <a href="/wiki/Category:Notting_Hill" title="Category:Notting Hill">Category:Notting Hill</a> als auch in der <a href="/wiki/Category:Royal_Borough_of_Kensington_and_Chelsea" title="Category:Royal Borough of Kensington and Chelsea">Category:Royal Borough of Kensington and Chelsea</a> zu platzieren. Stattdessen sollte jedes Bild nur in den spezifischsten Kategorien platziert werden, und diese Kategorien sollten wiederum in ihren spezifischsten Kategorien platziert werden. </p>
      <p>Wenn du auf unsachgemäß kategorisierte Kategorien stößt, platziere diese bitte in den entsprechenden übergeordneten Kategorien, wenn du dazu in der Lage bist. Das hilft nicht nur, eine Überkategorisierung zu vermeiden, sondern erleichtert auch die Navigation durch den Kategorienbaum. </p> 
      <h4><span id="Ausnahmen_f.C3.BCr_Bilder_mit_Inhalten_mehrerer_Kategorien"></span><span class="mw-headline" id="Ausnahmen_für_Bilder_mit_Inhalten_mehrerer_Kategorien"><span id="OvercatException"></span> Ausnahmen für Bilder mit Inhalten mehrerer Kategorien</span></h4> 
      <p>Eine Datei, die nur ein relevantes Thema abbildet, sollte nicht überkategorisiert werden. Wenn eine Datei zusätzliche relevante Themen abbildet und die zusätzlichen Themen keine eigenen Unterkategorien haben, kann in Erwägung gezogen werden, das Bild vorübergehend sowohl in der Unterkategorie als auch in der übergeordneten Kategorie zu kategorisieren. </p>
      <p>Diese Situation könnte zum Beispiel im Fall eines Fotos von drei Politikern entstehen, von denen eine <a href="/wiki/Category:Angela_Merkel" title="Category:Angela Merkel">Angela Merkel</a> ist (die ihre eigene Commons-Kategorie hat), mit zwei weiteren Politikern, die noch keine eigenen Kategorien haben. Während das Bild zweifellos in die <a href="/wiki/Category:Angela_Merkel" title="Category:Angela Merkel">Category:Angela Merkel</a> oder eine ihrer Unterkategorien eingeordnet würde, wäre es typischerweise eine Überkategorisierung, es auch in die <a href="/wiki/Category:Politicians_of_Germany" title="Category:Politicians of Germany">Category:Politicians of Germany</a> aufzunehmen. Es wäre jedoch unwahrscheinlich, dass die Benutzer nach den beiden anderen Politikern in der Merkel-Kategorie suchen würden. Im Idealfall würden wir spezifische Unterkategorien für die beiden anderen Politiker erstellen (wo dies gerechtfertigt ist) oder andere relevante Unterkategorien finden (z.B. <a href="/wiki/Category:Politicians_of_Bavaria" title="Category:Politicians of Bavaria">Category:Politicians of Bavaria</a> oder <a href="/wiki/Category:Members_of_the_FDP" title="Category:Members of the FDP">Category:Members of the FDP</a>, usw.), die es uns ermöglichen würden, eine Überkategorisierung zu vermeiden. Unter Umständen kann es jedoch notwendig sein, das Bild vorübergehend in <a href="/wiki/Category:Politicians_of_Germany" title="Category:Politicians of Germany">Category:Politicians of Germany</a> einzuordnen, wenn andere geeignete Unterkategorien noch nicht existieren. </p>
      <p>Länder können als Teil mehrerer sich überschneidender Kategorien kategorisiert werden. Beispielsweise gehört <a href="/wiki/Category:India" title="Category:India">Category:India</a> sowohl zu <a href="/wiki/Category:Countries_of_South_Asia" title="Category:Countries of South Asia">Category:Countries of South Asia</a> als auch zu <a href="/wiki/Category:Countries_of_Asia" title="Category:Countries of Asia">Category:Countries of Asia</a>. </p>
      <p>Auch Benutzerkategorien sind von einer Überkategorisierung ausgenommen, da diese für die meisten Betrachter nicht sichtbar sind und die Projektbenutzer beziehen sie für viele verschiedene Zwecke wie Sortierung, Statistiken, Füllwerte für Userboxen usw. ein. </p> 
      <h2><span class="mw-headline" id="Wie_Kategorisierung:_Leitfaden_nach_Thema">Wie Kategorisierung: Leitfaden nach Thema</span></h2> 
      <p>Für einige Kategorien gibt es spezielle Orientierungshilfen, wie Inhalte innerhalb dieser Kategorie am besten zu sortieren sind. Diese Orientierungshilfen findest du in einem <a href="/wiki/Category:Commons_category_schemes" title="Category:Commons category schemes">Kategorienschema</a> oder einem <a href="/wiki/Category:Commons_projects" title="Category:Commons projects">Commons-Project</a> für dein Thema. In diesem Abschnitt findest du auch einige kategorisierende Informationen und manchmal gibt es oben auf der Kategorieseite im Kategorie-<a href="https://meta.wikimedia.org/wiki/Namespace" class="extiw" title="meta:Namespace">Namensraum</a> eine Anleitung. So befinden sich z.B. einige Orientierungshilfen zur Kategorisierung von Inhalten, die Personen darstellen, am Anfang von <a href="/wiki/Category:People" title="Category:People">Category:People</a> und einige im Abschnitt <a href="#People">Personen</a> hier unten. </p> 
      <h3><span class="mw-headline" id="Personen">Personen</span></h3> 
      <p>Inhalte, die Menschen darstellen, sollten in Kategorien eingeordnet werden, die sie beschreiben, wie zum Beispiel <a href="/wiki/Category:Economists_from_the_United_States" title="Category:Economists from the United States">Category:Economists from the United States</a>. Beginne die Erkundung bei <a href="/wiki/Category:People" title="Category:People">Category:People</a>. </p>
      <p>Einzelheiten zur Benennung und Organisation dieser Kategorien findest du unter <a href="/wiki/Special:MyLanguage/Commons:Category_scheme_People" title="Special:MyLanguage/Commons:Category scheme People">Commons:Category scheme People</a>. </p> 
      <h3><span id="Landschaften.2C_Au.C3.9Fenaufnahmen"></span><span class="mw-headline" id="Landschaften,_Außenaufnahmen">Landschaften, Außenaufnahmen</span></h3> 
      <p>Inhalte, die ein bestimmtes Thema aus einem gemeinsamen Blickwinkel darstellen, werden in <i>Ansichten des Themas aus der Sicht</i>-Kategorien wie <a href="/wiki/Category:Views_of_Cathedral_of_Seville_from_the_Giralda" title="Category:Views of Cathedral of Seville from the Giralda">Ansichten der Kathedrale von Sevilla aus der Giralda</a> gruppiert. Solche Kategorien sollten Unterkategorien sowohl der Kategorie des Subjekts (<a href="/wiki/Category:Cathedral_of_Seville" title="Category:Cathedral of Seville">Cathedral of Seville</a> in diesem Beispiel) als auch der Kategorie des Standpunktes (<a href="/wiki/Category:Giralda" title="Category:Giralda">Giralda</a> in diesem Beispiel) sein. </p>
      <p>In diesem Beispiel ist die Kategorie <a href="/wiki/Category:Views_of_Cathedral_of_Seville_from_the_Giralda" title="Category:Views of Cathedral of Seville from the Giralda">Views of Cathedral of Seville from the Giralda</a> nicht direkt in den Themen- und Ansichtskategorien platziert, sondern in <a href="/wiki/Category:Views_of_the_Cathedral_of_Seville" title="Category:Views of the Cathedral of Seville">Views of the Cathedral of Seville</a> und <a href="/wiki/Category:Views_from_Giralda" title="Category:Views from Giralda">Views from Giralda</a>. Solche Zwischenkategorien sind oft notwendig, um Struktur zu schaffen und <a href="#over-categorization">Über-Kategorisierung</a> zu vermeiden, insbesondere für Ansichten einer Stadt von einem Aussichtspunkt innerhalb der Stadt. Beispielsweise benötigt <a href="/wiki/Category:Views_of_Rome_from_the_Pincio" title="Category:Views of Rome from the Pincio">Views of Rome from the Pincio</a> die Zwischenkategorie <a href="/wiki/Category:Views_of_Rome" title="Category:Views of Rome">Views of Rome</a>, um zu vermeiden, dass sie direkt in <a href="/wiki/Category:Rome" title="Category:Rome">Rome</a> platziert wird, was eine Über-Kategorisierung darstellen würde. </p> 
      <h3><span class="mw-headline" id="Texte">Texte</span></h3> 
      <p>Texte, wie z.B. Scans von Büchern, sollten normalerweise eine Kategorie für jede Version des Scans und jede Ausgabe des Textes haben. So würde ein Buch, das in drei verschiedenen Ausgaben veröffentlicht wird, eine übergeordnete Kategorie für das Buch, drei Unterkategorien für jeden Text und weitere Unterkategorien für den Text als jpeg, DjVu usw. haben, vorausgesetzt, jede Version wurde tatsächlich hochgeladen. Dies ist besonders wichtig für Dateien in anderen Formaten als DjVu und PDF, wo die Kategorie die einzige praktische Möglichkeit ist, die Scans zusammenzuhalten; siehe z.B. <a href="/wiki/Category:The_Chronicles_of_England,_Scotland_and_Ireland,_Holinshed,_1587" title="Category:The Chronicles of England, Scotland and Ireland, Holinshed, 1587">Category:The Chronicles of England, Scotland and Ireland, Holinshed, 1587</a>, die 2857 jpeg-Bilder von Seitenscans enthält. </p> 
      <h3><span id="Gallerien.2C_Bibliotheken.2C_Archive.2C_Museen"></span><span class="mw-headline" id="Gallerien,_Bibliotheken,_Archive,_Museen">Gallerien, Bibliotheken, Archive, Museen</span></h3> 
      <p>Zu Kategorisierungsfragen im Zusammenhang mit Spenden großer Bildermengen von GLAMs (Galerien, Bibliotheken, Archive &amp; Museen) siehe <a href="/wiki/Special:MyLanguage/Commons:Guide_to_batch_uploading#Categories" title="Special:MyLanguage/Commons:Guide to batch uploading">Commons:Guide to batch uploading#Categories</a>. </p> 
      <h2><span id="Kategorisierungs.C3.BCberlauf"></span><span class="mw-headline" id="Kategorisierungsüberlauf">Kategorisierungsüberlauf</span></h2> 
      <p>Gegenwärtig überprüft ein Bot, ob neu hochgeladenen Dateien in aktuellen Kategorien kategorisiert wurden und versucht diejenigen zu kategorisieren, die es nicht sind. Vor dem 17. Juni 2015 war <a href="/wiki/User:CategorizationBot" title="User:CategorizationBot">CategorizationBot</a> für diese Aufgabe zuständig. Ab Juni 2019 sucht <a href="/wiki/User:SteinsplitterBot" title="User:SteinsplitterBot">SteinsplitterBot</a> gelegentlich nach unkategorisierten Dateien. Der Vorgehensweise ist wie folgt: </p> 
      <ol>
       <li>Ein Benutzer lädt eine neue Datei hoch und fügt Kategorien hinzu (oder auch nicht).</li> 
       <li>Ein Bot überprüft, ob die Datei kategorisiert wurde. 
        <ul>
         <li>Die Datei ist bereits kategorisiert =&gt; ok</li> 
         <li>Die Datei ist nicht kategorisiert =&gt; der Bot markiert die Datei mit <a href="/wiki/Category:Media_needing_categories" title="Category:Media needing categories">Category:Media needing categories</a>. Die Dateien von heute befinden sich in <a href="/wiki/Category:Media_needing_categories_as_of_6_June_2021" title="Category:Media needing categories as of 6 June 2021">Category:Media needing categories as of 6 June 2021</a></li>
        </ul></li> 
       <li>Benutzer kategorisieren Dateien weitergehend (z.B. Kategoriediffusion unten)</li>
      </ol> 
      <p><i>Siehe also:</i> <a href="/wiki/User:CategorizationBot#Process" title="User:CategorizationBot">User:CategorizationBot#Process</a>, <a href="/wiki/User:Multichill/Categorization_stats" title="User:Multichill/Categorization stats">Kategorisierungsstatistiken</a> </p>
      <p>Andere, bei manueller Arbeit, sind Kategorisierungs-Workflows möglich: </p> 
      <ul>
       <li>Kategorie füllen: Verwende geeignete Schlüsselwörter in der Suchmaschine, um die Dateien zu finden, die in einer bestimmten Kategorie sein sollten, und lege sie dort ab.</li> 
       <li>Kategorie verbreitern: Gehe zu <a href="/wiki/Category:Categories_requiring_diffusion" title="Category:Categories requiring diffusion">Category:Categories requiring diffusion</a>, wähle eine überfüllte Kategorie, erstelle bei Bedarf geeignete Unterkategorien und verschiebe die Dateien in die Unterkategorien. Helferlein wie Cat-a-lot und Hotcats können dabei helfen.</li>
      </ul> 
      <h2><span id="Mit_.E2.80.9EHIDDENCAT.E2.80.9C_markierte_Kategorien"></span><span class="mw-headline" id="Mit_„HIDDENCAT“_markierte_Kategorien">Mit „HIDDENCAT“ markierte Kategorien</span></h2> 
      <div class="shortcut2" style="">
       <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r433650343"><a href="/wiki/Commons:Shortcuts" title="Commons:Shortcuts">Shortcut</a>
       <br><a href="/wiki/Commons:HIDDENCAT" class="mw-redirect" title="Commons:HIDDENCAT">COM:HIDDENCAT</a>
      </div> 
      <p>Viele nicht-thematische Kategorien sind auf der Kategorieseite mit <code>__HIDDENCAT__</code> oder {{<a href="/wiki/Template:Hiddencat" title="Template:Hiddencat">Hiddencat</a>}} gekennzeichnet. Siehe zum Beispiel <a class="external text" href="https://commons.wikimedia.org/w/index.php?title=Category:PD_NASA&amp;action=edit">Category:PD NASA im Bearbeitungsmodus</a>. </p>
      <p>Während Kategorien generell auf jeder Seite sichtbar sind, sind Kategorien, die mit <code>__HIDDENCAT__</code> gekennzeichnet sind, nur sichtbar: </p> 
      <ul>
       <li>auf dem Bearbeitungsbildschirm: am Ende des Bildschirms, unterhalb des Bearbeitungsfeldes</li> 
       <li>auf Kategorieseiten: 
        <ul>
         <li>auf Unterkategorien zur versteckten Kategorie: an der normalen Stelle, aber in einer separaten Zeile mit kleinerer Schrift und der Beschriftung „Hidden categories.“</li> 
         <li>auf übergeordneten Kategorien: auf die gleiche Weise wie bei anderen Kategorien</li>
        </ul></li> 
       <li>auf Dateibeschreibungsseiten und Galerieseiten: für angemeldete Benutzer, die in ihren <a href="/wiki/Special:Preferences#mw-prefsection-rendering" title="Special:Preferences">Einstellungen, Reiter „Aussehen“,</a> die Option „Versteckte Kategorien anzeigen“ gewählt haben. Dies ist für alle neu registrierten Benutzer aktiviert.</li>
      </ul> 
      <p>Diese Funktion wird im Allgemeinen für vorlagenbasierte Kategorien verwendet, wie z. B. auf Lizenzvorlagen basierende Kategorien. Wenn du z. B. <a href="/wiki/Template:PD-old-100" title="Template:PD-old-100"><span style="font-family:monospace,monospace;">{{PD-old-100}}</span></a> auf einer Dateibeschreibungsseite platzierst, wird die Datei zu <a href="/wiki/Category:Author_died_more_than_100_years_ago_public_domain_images" title="Category:Author died more than 100 years ago public domain images">Category:Author died more than 100 years ago public domain images</a> hinzugefügt, die mit <code>__HIDDENCAT__</code> markiert ist. </p>
      <p>Weitere Details findest du in der <a href="https://meta.wikimedia.org/wiki/Help:Category#Hidden_categories" class="extiw" title="m:Help:Category">Hilfe zu versteckten Kategorien</a> für Mediawiki (die Software, die Commons verwendet). </p> 
      <h2><span id="Vorlagen_f.C3.BCr_Kategorien"></span><span class="mw-headline" id="Vorlagen_für_Kategorien">Vorlagen für Kategorien</span></h2> 
      <p>Einige Vorlagen sind für die Verwendung auf Kategorieseiten konzipiert - siehe <a href="/wiki/Category:Category_namespace_templates" title="Category:Category namespace templates">Category:Category namespace templates</a>. Wenn die Kategorie mit einem Wikidata-Eintrag verlinkt ist, dann verwende: </p> 
      <ul>
       <li><a href="/wiki/Template:Wikidata_Infobox" title="Template:Wikidata Infobox"><span style="font-family:monospace,monospace;">{{Wikidata Infobox}}</span></a>,</li>
      </ul> 
      <p>die eine Zusammenfassung der Informationen des Themas anzeigt, die auf Wikidata verfügbar sind, und außerdem automatisch die Geburt-/Tod-/Namens-/Denkmal-ID-Kategorien hinzufügt. </p>
      <p>Einige der am häufigsten verwendeten sind <a href="/wiki/Category:Category_header_templates" title="Category:Category header templates">Category:Category header templates</a> wie z.B.: </p> 
      <ul>
       <li><a href="/wiki/Template:Category_TOC" class="mw-redirect" title="Template:Category TOC"><span style="font-family:monospace,monospace;">{{Category TOC}}</span></a> und <a href="/wiki/Template:LargeCategoryTOC" title="Template:LargeCategoryTOC"><span style="font-family:monospace,monospace;">{{LargeCategoryTOC}}</span></a> - zur Erstellung eines Inhaltsverzeichnisses für sehr große Kategorien mit vielen Unterkategorien</li> 
       <li><a href="/wiki/Template:ImageTOC" title="Template:ImageTOC"><span style="font-family:monospace,monospace;">{{ImageTOC}}</span></a> und <a href="/wiki/Template:FileCategoryTOC" title="Template:FileCategoryTOC"><span style="font-family:monospace,monospace;">{{FileCategoryTOC}}</span></a> - zur Erstellung eines Inhaltsverzeichnisses für große Kategorien mit vielen Dateien und wenigen Unterkategorien</li> 
       <li><a href="/wiki/Template:Image_template_notice" title="Template:Image template notice"><span style="font-family:monospace,monospace;">{{Image template notice}}</span></a> und <a href="/wiki/Template:Autocat" title="Template:Autocat"><span style="font-family:monospace,monospace;">{{Autocat}}</span></a> - für Wartungskategorien, die von einer bestimmten Vorlage bestückt werden.</li> 
       <li><a href="/wiki/Template:Global_maintenance_category" title="Template:Global maintenance category"><span style="font-family:monospace,monospace;">{{Global maintenance category}}</span></a> und <a href="/wiki/Template:Local_maintenance_category" title="Template:Local maintenance category"><span style="font-family:monospace,monospace;">{{Local maintenance category}}</span></a></li> 
       <li><a href="/wiki/Template:MetaCat" title="Template:MetaCat"><span style="font-family:monospace,monospace;">{{MetaCat}}</span></a> - für Kategorien, die nur andere Kategorien enthalten sollen 
        <ul>
         <li><a href="/wiki/Template:By_country_category" title="Template:By country category"><span style="font-family:monospace,monospace;">{{By country category}}</span></a>, <a href="/wiki/Template:By_language_category" title="Template:By language category"><span style="font-family:monospace,monospace;">{{By language category}}</span></a>, <a href="/wiki/Template:By_century_category" title="Template:By century category"><span style="font-family:monospace,monospace;">{{By century category}}</span></a> - für bestimmte Typen von Metakategorien</li>
        </ul></li> 
       <li><a href="/wiki/Template:CatDiffuse" title="Template:CatDiffuse"><span style="font-family:monospace,monospace;">{{CatDiffuse}}</span></a> (zeitweise) und <a href="/wiki/Template:Categorise" title="Template:Categorise"><span style="font-family:monospace,monospace;">{{Categorise}}</span></a> (permanent) - für große Kategorien, die gewartet werden müssen, um Dateien in Unterkategorien zu verschieben</li> 
       <li><a href="/wiki/Template:Category_tree" title="Template:Category tree"><span style="font-family:monospace,monospace;">{{Category tree}}</span></a></li>
      </ul>

       Further information: <a href="/wiki/Special:MyLanguage/Commons:Tools" title="Special:MyLanguage/Commons:Tools">Commons:Tools</a>

      <ul>
       <li>Helferlein, die in den <a href="/wiki/Special:Preferences#mw-prefsection-gadgets" title="Special:Preferences">Benutzereinstellungen</a> aktiviert sind: 
        <ul>
         <li><i>Cat-a-lot</i>: A tool that helps with moving multiple files between categories or adding categories to search results. <small>[<a href="/wiki/Help:Gadget-Cat-a-lot" title="Help:Gadget-Cat-a-lot">documentation</a>&nbsp;/&nbsp;<a href="/wiki/MediaWiki_talk:Gadget-Cat-a-lot.js" title="MediaWiki talk:Gadget-Cat-a-lot.js">talk</a>]</small>&nbsp;</li> 
         <li><i>HotCat</i>:&nbsp;<sup><abbr title="default gadget – enabled by default">d</abbr></sup> Easily add / remove / change a category on a page, with name suggestions. <small>[<a href="/wiki/Help:Gadget-HotCat" title="Help:Gadget-HotCat">documentation</a>&nbsp;/&nbsp;<a href="/wiki/File:HotCat.png" title="File:HotCat.png">example</a>&nbsp;/&nbsp;<a href="/wiki/MediaWiki_talk:Gadget-HotCat.js" title="MediaWiki talk:Gadget-HotCat.js">talk</a>]</small>&nbsp;</li> 
         <li><i>Gallery Details</i>: Adds a link in the toolbox to display galleries and categories (and <a href="/wiki/Special:NewFiles" title="Special:NewFiles">Newimages</a> and <a href="/wiki/Special:Search" title="Special:Search">Search</a> result pages) with extensive details from file description pages and links to easily mark an image without source, etc. If <i>Pretty log</i> is activated, it also works on <a href="/wiki/Special:Log" title="Special:Log">Log</a> pages. <small>[<a href="/wiki/Help:Gadget-GalleryDetails" title="Help:Gadget-GalleryDetails">documentation</a>&nbsp;/&nbsp;<a href="/wiki/MediaWiki_talk:Gadget-GalleryDetails.js" title="MediaWiki talk:Gadget-GalleryDetails.js">talk</a>]</small>&nbsp;</li> 
         <li>Place categories above content, but below image on file description pages. &nbsp; Ändert die Platzierung von Kategorien auf der Benutzeroberfläche.</li> 
         <li>Add a link to category pages to search for the category name with the option "-incategory". This excludes files already in the category (doesn't work if the category was added by a template). <small>[<a href="/wiki/MediaWiki_talk:Gadget-Searchnotincat.js" title="MediaWiki talk:Gadget-Searchnotincat.js">talk</a>]</small>&nbsp;</li>
        </ul></li>
