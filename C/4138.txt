      <p><a href="/wiki/Help:Template" title="Help:Template">Templates</a> are a very powerful feature of <a href="/wiki/MediaWiki" title="MediaWiki">MediaWiki</a>, but can be confusing to new users, and even experienced users can have difficulty making sense of the more complex one. Templates should therefore be accompanied by <b>documentation</b> to improve usability. </p>
      <p>Template documentation should explain what a template does and how to use it. It should be simple enough that a user without complete knowledge of the intricacies of template syntax – which includes many experienced contributors who focus their attention elsewhere – can use it correctly. This is especially true in the case of very widely-used templates. </p>

      <h2><span class="mw-headline" id="What_to_include">What to include</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Commons:Template_documentation&amp;action=edit&amp;section=1" title="Edit section: What to include">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <p>Documentation should cover: </p> 
      <ul>
       <li>The basic <b>purpose</b> of the template: what it does, and (if it is not immediately obvious) why it needs to be done. If there are other templates with similar names or purposes, it's a good idea to mention those, to reduce the chance of the wrong one being used.</li> 
       <li>Any <b>parameters</b> that the template takes. Explain whether they are numbered or named, whether they are optional (and if so, what the default value is), and what effect they have. If a parameter can take only a limited set of values ("yes" or "no", for example), must be a number, or is constrained in some other way, this should be made clear.</li> 
       <li><b>Usage examples</b>. Specify the exact wikitext that should be used, and the result it produces. The wikitext can be enclosed in &lt;code&gt;...&lt;/code&gt; tags to make it clear and <code>easy to copy, like this</code>. If the template can be used in several different ways (with or without optional parameters, for example), provide a range of examples. A good way to do so is to transclude the template itself into the documentation a few times, with different parameters each time, and list the parameters used in each case.</li> 
       <li><b>Related templates</b>. If the template is one of a series of templates, include links to these – in particular, ensure that every template in the series is linked from each of the others, as this makes navigation easier. (A separate navigation template may be useful for this purpose; see for example <a href="/w/index.php?title=Template:Protection_templates&amp;action=edit&amp;redlink=1" class="new" title="Template:Protection templates (page does not exist)">Template:Protection templates</a>).</li> 
       <li>Appropriate <b><a href="/wiki/Commons:Categories" title="Commons:Categories">categories</a></b> and <b><a href="https://en.wikipedia.org/wiki/Wikipedia:Interwikimedia_link" class="extiw" title="w:Wikipedia:Interwikimedia link">interwiki</a> links</b>, where applicable. Like the documentation, these must be contained within <code>&lt;noinclude&gt;...&lt;/noinclude&gt;</code> tags.</li>
      </ul> 
      <p>Template documentation is often placed in a <a href="https://en.wikipedia.org/wiki/Wikipedia:Subpage" class="extiw" title="w:Wikipedia:Subpage">subpage</a> of the template itself, which is then <a href="https://en.wikipedia.org/wiki/Wikipedia:Transclusion" class="extiw" title="w:Wikipedia:Transclusion">transcluded</a> at the end of the template page. This separates the often complex template code from the documentation, making the documentation easier to edit. It also allows templates to be <a href="/wiki/Commons:Protection_policy" title="Commons:Protection policy">protected</a> where necessary, while allowing anyone to edit the documentation. This method is sometimes referred to as the "template doc page pattern". </p>
      <p>Documentation of any sort on a template page should always be enclosed by <code>&lt;noinclude&gt;...&lt;/noinclude&gt;</code> tags, so that it does not show up when the template is used on another page. Text on the template page itself adds to the amount of text which must be processed when displaying the template, which is <a href="https://en.wikipedia.org/wiki/Wikipedia:Template_limits" class="extiw" title="w:Wikipedia:Template limits">limited</a> for performance reasons. Placing the documentation in a subpage avoids this (MediaWiki developers have <span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:Village_pump_(technical)&amp;diff=prev&amp;oldid=69888944">recommended</a></span> it for this reason). </p> 
      <h2><span class="mw-headline" id="How_to_create_a_documentation_subpage">How to create a documentation subpage</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Commons:Template_documentation&amp;action=edit&amp;section=2" title="Edit section: How to create a documentation subpage">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <p>Documentation subpages should be named and formatted using the following general pattern, for consistency. </p>
      <p>Suppose your template is named <kbd>Template:X</kbd>. Create a subpage with the name <kbd>Template:X/doc</kbd>. Use the following wikitext as a start for the documentation: </p> 
      <dl>
       <dd>
        <code style="display:table; line-height:1.2em;"> &lt;noinclude&gt;{{documentation subpage}}&lt;/noinclude&gt;<br> &lt;!-- EDIT TEMPLATE DOCUMENTATION BELOW THIS LINE --&gt;<br> <br> &lt;includeonly&gt;<br> &lt;!-- ADD CATEGORIES BELOW THIS LINE --&gt;<br> <br> &lt;/includeonly&gt;<br> </code>
       </dd>
      </dl> 
      <p>The top line will display one of two templates – <kbd><a href="/wiki/Template:Documentation" title="Template:Documentation"><span style="font-family:monospace,monospace;">{{Documentation}}</span></a></kbd> or <kbd><a href="/wiki/Template:Documentation_subpage" title="Template:Documentation subpage"><span style="font-family:monospace,monospace;">{{Documentation subpage}}</span></a></kbd> – at the top of the documentation; this makes it clear to the reader that what follows is documentation, and provides a link to edit the /doc subpage. </p>
      <p>Insert the documentation, categories and interwikis as below the appropriate comment lines – leaving the comments in place so that the layout is preserved when the page is edited in future. Ensure that the categories added are similar to <kbd>[[Category:<i>Name</i>|{{PAGENAME}}]]</kbd>, otherwise they will not work. </p>
      <p>Save the subpage, and return to the template itself, <kbd>Template:X</kbd> in this example. Edit the template and append the following at the end of the template code: </p> 
      <dl>
       <dd>
        <code style="display:table; line-height:1.2em;"><font style="background-color: #FFFF66">[--template code--]</font>&lt;noinclude&gt;<br>{{documentation}}<br> &lt;!-- Add categories and interwikis to the /doc subpage, not here! --&gt;<br> &lt;/noinclude&gt;<br> </code>
       </dd>
      </dl> 
      <p>This transcludes <a href="/wiki/Template:Template_doc" class="mw-redirect" title="Template:Template doc">Template:template doc</a> (<a href="/w/index.php?title=Template_talk:Template_doc&amp;action=edit&amp;redlink=1" class="new" title="Template talk:Template doc (page does not exist)">talk</a>, <a href="/wiki/Special:WhatLinksHere/Template:template_doc" title="Special:WhatLinksHere/Template:template doc">backlinks</a>, <a class="external text" href="https://en.wikipedia.org/wiki/Template:template_doc?action=edit">edit</a>) into the template page. </p>
      <p><b>Important</b>: Make sure the opening <code>&lt;noinclude&gt;</code> begins on the same line as the last character of the template code (or text), and <i>not</i> on a new line. Otherwise, extra space will be inserted below the template when it is used, which is usually not wanted. </p>
      <p>If the template is already protected, ask an <a href="/wiki/Commons:Administrators" title="Commons:Administrators">administrator</a> to do this, or request an edit by using <kbd>{{<a href="/wiki/Template:Editprotected" class="mw-redirect" title="Template:Editprotected">editprotected</a>}}</kbd> on the template's talk page. If documentation, categories and interwiki links already exist in a section enclosed with &lt;nowiki&gt; tags, move them into the documentation subpage, as it is best not to have documentation split across two separate pages. </p> 
      <h2><span class="mw-headline" id="Example">Example</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Commons:Template_documentation&amp;action=edit&amp;section=3" title="Edit section: Example">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <p>See the heavily-used <a href="/wiki/Template:Cite_web" title="Template:Cite web">Template:Cite web</a> for an example of detailed template documentation. Note that the template itself is protected, but the documentation subpage, <a href="/wiki/Template:Cite_web/doc" title="Template:Cite web/doc">Template:Cite web/doc</a> is unprotected and can still be edited. </p>
