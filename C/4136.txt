      <h2><span class="mw-headline" id="Usage">Usage</span></h2> 
      <div class="templatebox-usagesample"> 
       <div class="poem"> 
        <p>{{Autoarchive resolved section<br> &nbsp;|age = <br> &nbsp;|archive = <br> &nbsp;|show = <br> &nbsp;|level = <br> &nbsp;|timecompare = <br> &nbsp;|timeout = <br> &nbsp;|width = <br> }} </p> 
       </div> 
      </div> 
      <p><small></small> </p> 
      <div class="templatebox-languagedependant"> 
       <p>SpBot archiving requires the use of either <a href="/wiki/Template:Section_resolved" title="Template:Section resolved">{{Section resolved}}</a> or the <code>timeout</code> parameter of this template (usual archiving like by <a href="/wiki/User:ArchiverBot" title="User:ArchiverBot">ArchiverBot</a>). </p>
       <p>It does not matter where the template <code><span style="white-space:nowrap;">{{Section resolved|…}}</span></code> is placed within the section. It will always refer to the last/previous heading of the section to be archived (see parameter <code>level</code>). </p>
       <p>The template relies on a certain time stamp format, for more see in <a href="#Signature_format">Signature format</a>. </p>
       <p>To quickly use a standard setup to archive your user talk page, see <a href="/wiki/Template:Autoarchive_resolved_section/usertalksetup" title="Template:Autoarchive resolved section/usertalksetup">Template:Autoarchive resolved section/usertalksetup</a> (see own description). </p> 
      </div> 
      <h3><span class="mw-headline" id="Template_parameters">Template parameters</span></h3> 
      <table class="wikitable templatebox-table">
       <tbody>
        <tr>
         <th style="width:15%">Parameter</th>
         <th style="width:65%" colspan="2">Description</th>
         <th style="width:10%">Default</th>
         <th style="width:10%">Status</th>
        </tr>
        <tr>
         <td><code>age</code></td>
         <td colspan="2">The parameter <i><code>age</code></i> defines the required age of the template block <code><span style="white-space:nowrap;">{{Section resolved|…}}</span></code> in days. After this amount of days the section will be archived.</td>
         <td><span class="mw-templatedata-doc-muted" style="color:#777; font-variant:small-caps">empty</span></td>
         <td><b>required</b></td>
        </tr>
        <tr>
         <td><code>archive</code></td>
         <td colspan="2">Path to archive. You can use variables in your <i><code>archive</code></i> parameter as <a href="#Archive_variables">listed below</a>.<br><small>Path inclusion with ASCII apostrophes (') is not necessary, but allowed for historic reasons.</small></td>
         <td><span class="mw-templatedata-doc-muted" style="color:#777; font-variant:small-caps">empty</span></td>
         <td><b>required</b></td>
        </tr>
        <tr>
         <td><code>overview</code></td>

      <h2><span id=".E2.80.9CArchive.E2.80.9D_parameter_variables"></span><span class="mw-headline" id="“Archive”_parameter_variables"><span id="Archive_variables"></span>“Archive” parameter variables</span></h2>

      <h2><span class="mw-headline" id="Signature_format"><span id="Signature_format"></span>Signature format</span></h2> 
      <p>The date and time part of the signature is important for recognition of the timestamp. The default format specified by MediaWiki and automatically created, when <code>~~~~</code> or <code>~~~~~</code> is entered, is sufficient: <i>15:11, 5 June 2021 (UTC)</i>, also with abbreviated month: <i>Jun</i>. This corresponds to the following structure with <a href="https://en.wikipedia.org/wiki/Help:Magic_words#Other_variables_by_type" class="extiw" title="en:Help:Magic words">Mediawiki variables</a>: <code>{{CURRENTTIME}}, {{CURRENTDAY}} {{CURRENTMONTHNAME}} {{CURRENTYEAR}} (UTC)</code> (short month name: <code>{{CURRENTMONTHABBREV}}</code>. When directly entered these variables must be <a href="https://en.wikipedia.org/wiki/Wikipedia:Substitution" class="extiw" title="en:Wikipedia:Substitution">substituted</a> (the according <code>{{LOCAL…}}</code> variants can be used, as well). </p>
      <p>Other formats are not supported. </p>
