      <p>Una <b>categoría</b> es un recurso especial del software <a href="https://en.wikipedia.org/wiki/MediaWiki" class="extiw" title="w:MediaWiki">MediaWiki</a>, una página especial que se usa para agrupar <i>materiales</i> y <i>páginas</i> que estén relacionados entre sí. Su objetivo es mantener organizados los contenidos según un orden lógico de asuntos y en una estructura jerárquica que facilite a los usuarios la localización del material que necesiten. Por ese motivo, todo material de Commons debe pertenecer a una categoría. El nombre de la categoría debería ser suficiente para encontrar el asunto, pero puede incluir una breve descripción inicial para definirla con más precisión. </p>

      <h2><span id="Gu.C3.ADa_r.C3.A1pida"></span><span class="mw-headline" id="Guía_rápida">Guía rápida</span></h2> 
      <p>1. Cómo para encontrar las categorías apropiadas </p> 
      <dl>
       <dd>
        <ul>
         <li>Encuentra categorías con el motor de búsqueda (ver <a href="#consejos_de_Categorización">#consejos de Categorización</a>)</li> 
         <li>O comprobar como se categorizan archivos similares (algunos no se podrán categorizar sólo por similitud)</li> 
         <li>O iniciar desde la categoría del tópico principal (<a href="/wiki/Category:Topics" title="Category:Topics">Categoría:Temas</a>)</li>
        </ul>
       </dd> 
       <dd>
        Empezando desde estas categorías, comprobando los padres o las sub-categorías para encontrar una categoría apropiada. Evita elegir categorías demasiado generales.
       </dd>
      </dl> 
      <p>2. Añadir las categorías al archivo </p> 
      <dl>
       <dd>
        Les selecciona encima el cargar forma
       </dd> 
       <dd>
        O: les añade manualmente por añadir, p. ej. <code>[[Categoría:Foobar]]</code> a la página de descripción del archivo (ve <a href="#Categorizing_páginas">#Categorizing páginas</a>)
       </dd> 
       <dd>
        O: uso HotCat (ve <a href="/wiki/Special:MyLanguage/Help:Gadget-HotCat" title="Special:MyLanguage/Help:Gadget-HotCat">Help:Gadget-HotCat</a>)
       </dd> 
       <dd>
        O: Gato de uso-un-parcela (ve <a href="/wiki/Help:Gadget-Cat-a-lot/es" title="Help:Gadget-Cat-a-lot/es">Ayuda:Gadget-Cat-a-lot</a>)
       </dd>
      </dl>

      <h3><span class="mw-headline" id="Principios">Principios</span></h3> 
      <p>Los principios principales son: </p> 
      <h4><span id="Principio_de_jerarqu.C3.ADa"></span><span class="mw-headline" id="Principio_de_jerarquía">Principio de jerarquía</span></h4> 
      <p>La estructura de categorías es (idealmente) una multi-jerarquía con una única categoría raíz, <a href="/wiki/Category:CommonsRoot" title="Category:CommonsRoot">Category:CommonsRoot</a>. Todas las categorías (salvo la categoría raíz<i>CommonsRoot</i>) deben estar contenidas en otra u otras categorías, y no debería haber ciclos (es decir, que una categoría se contenga a sí misma, directa o indirectamente). </p> 
      <h4><span class="mw-headline" id="Principio_de_modularidad">Principio de modularidad</span></h4> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>The page (file, category) should be put in the most specific category/categories that fit(s) the page (not directly to its parent categories). A category can have more parent categories. A category can combine two (or more) different criteria; such categories are called "compound categories" or "intersection categories". E.g. the root category <a href="/wiki/Category:Churches" title="Category:Churches">Category:Churches</a> and the root category <a href="/wiki/Category:Russia" title="Category:Russia">Category:Russia</a> have a common subcategory <a href="/wiki/Category:Churches_in_Russia" title="Category:Churches in Russia">Churches in Russia</a>. </p> 
      </div> 
      <h4><span class="mw-headline" id="Principio_de_simplicidad">Principio de simplicidad</span></h4> 
      <p>Este principio sugiere no combinar demasiados criterios diferentes. </p> 
      <h4><span class="mw-headline" id="Principio_de_selectividad">Principio de selectividad</span></h4> 
      <p>No deberíamos clasificar en la misma categoría elementos que estén relacionados con diferentes temas. El nombre de la categoría debería ser inequívoco y sin ambigüedades. </p> 
      <h4><span class="mw-headline" id="Principio_de_universalidad">Principio de universalidad</span></h4> 
      <p>Los elementos idénticos tendrían que tener nombres idénticos para todos los países y todos los niveles de categorización. La estructura de categorización tendría que ser tan sistemática y unificada como sea posible y deberían suprimirse los dialectos y la terminología muy local en favor de hacerla lo más universalizada posible. Analogic categorization branches should have analogic structure. </p> 
      <h3><span class="mw-headline" id="Tipos_de_relaciones_reflejadas">Tipos de relaciones reflejadas</span></h3> 
      <p>La estructura de categorías debe reflejar una jerarquía de conceptos, desde el más genérico hasta el más específico. La estructura usa y combina más tipos de relación, e. sol. </p> 
      <ul>
       <li><a href="https://en.wikipedia.org/wiki/en:Hyponymy_and_hypernymy" class="extiw" title="w:en:Hyponymy and hypernymy">Hyponymy</a>: un tipo / tipo / tipo de ... (típicamente en taxonomía biológica)</li> 
       <li><a href="https://en.wikipedia.org/wiki/es:Meronimia" class="extiw" title="w:es:Meronimia">Meronymy</a>: una parte de ..., un miembro de ... (generalmente para división geográfica, edificio / sala, dispositivo / componente, etc.).</li> 
       <li>Atributos: 
        <ul>
         <li>Atributos cualitativos y generales (color, forma, medida, capacidad o incapacidad, nacionalidad, técnica, calidad, premios…)</li> 
         <li>Ubicación: dónde, en…, de… (acontecimiento/de sitio, edificio/de sitio, exposición/de sitio, personas/de sitio, lengua/de país, trabajo/de fuente, fábrica o producto/de país etc.)</li> 
         <li>Cronometrando: cuándo (acontecimiento/de tiempo, tiempo/situación descrita, tiempo de nacimiento, inception o construcción, tiempo de muerte, escombro o terminación etc.)</li>
        </ul></li> 
       <li>Agentive Y relaciones de influencia: (trabajo/de creador, producto/de dispositivo, producto/de compañía, disciplina o profesión/sus temas y plazos, niños/de padre, subordinación, propiedad/de dueño, seguidor/de iniciador, tema/otros temas dedicaron a él o nombrado después de que lo, tema/su duplicado, imitación, representación o símbolo etc etc.)</li> 
       <li>Modificación: original modificado o/modificado/original (evita cíclico estructura) @– rebautizado, reconstruido, repurposed o transformó temas.</li>
      </ul> 
      <h3><span id="Categor.C3.ADas_importantes"></span><span class="mw-headline" id="Categorías_importantes">Categorías importantes</span></h3> 
      <p>Las categorías más altas (las contenidas directamente en CommonsRoot) dividen la estructura de las categorías según el propósito de las otras categorías que contengan: </p> 
      <ul>
       <li><b><a href="/wiki/Category:Image_sources" title="Category:Image sources">Category:Image sources</a></b> - Esta categoría es la raíz común global de los archivos de medios categorizados según el origen o fuente de la que provienen (libros, colecciones, sitios web, etc.)</li> 
       <li><b><a href="/wiki/Category:Copyright_statuses" title="Category:Copyright statuses">Category:Copyright statuses</a></b> - Esta categoría es la raíz común global de los archivos de medios categorizados según su <b>licencia</b> (por la que se libera el <i>copyright</i> o derecho de reproducción). <b>Todos los archivos de medios deberían categorizarse bajo esta categoría</b> con la <a href="/wiki/Commons:Marcas_de_derechos_de_autor" class="mw-redirect" title="Commons:Marcas de derechos de autor">marca de licencia</a> adecuada.</li> 
       <li><b><a href="/wiki/Category:Media_types" title="Category:Media types">Category:Media types</a></b> - Esta categoría es la raíz común global de los archivos de medios categorizados según el <b>tipo de medio</b> (ficheros de imagen fija, vídeo, sonidos, etc.). Frecuentemente se omite este tipo de categorización con las imágenes, ya que la gran mayoría de archivos en Commons son algún tipo de imágenes.</li> 
       <li><b><a href="/wiki/Category:Topics" title="Category:Topics">Category:Topics</a></b> - Esta categoría es la raíz común global de los archivos de medios categorizados según el <b>tema</b> o asunto que ilustran. <b>Todos los archivos de medios deberían categorizarse bajo esta categoría</b> para permitir a otros usuarios encontrarlos al buscar por tema.</li> 
       <li><b><a href="/wiki/Category:User" title="Category:User">Category:User</a></b> - Esta es para categorías que clasifican a usuarios de Commons por características como el lenguaje que hablan. Contiene también la <a href="/wiki/Category:User_galleries" title="Category:User galleries">Category:User galleries</a>, que es para galerías específicas por usuario (es decir, no por tema).</li> 
       <li><b><a href="/wiki/Category:Commons" title="Category:Commons">Category:Commons</a></b> - Esta categoría es la raíz común global de categorizing Commons tareas de mantenimiento y páginas (Commons:-, y Ayuda:-) excepto archivos de medios de comunicación. El tradujo las páginas en cada lengua tendrían que ser categorized bajo sus categorías de lengua, utilizando la "Categoría:Commons-<a href="https://en.wikipedia.org/wiki/ISO_639" class="extiw" title="w:ISO 639">ISO-LENGUA-CÓDIGO</a>" estilo. La estructura de <a href="/wiki/Category:Commons-en" title="Category:Commons-en">Category:Commons-en</a> es la jerarquía de muestra para cada otra lengua sub categoría. No utiliza dos colons en categoría o nombres de página. Ve <a href="/wiki/Category_talk:Commons_batch_uploading" title="Category talk:Commons batch uploading">esta discusión</a> y <a href="/wiki/Special:MyLanguage/Ayuda:Namespaces" class="new" title="Special:MyLanguage/Ayuda:Namespaces (page does not exist)">Ayuda:Namespaces</a>.</li>
      </ul>

      <h2><span id="Uso_de_las_categor.C3.ADas"></span><span class="mw-headline" id="Uso_de_las_categorías">Uso de las categorías</span></h2> 
      <p>Tienes que siempre puesto vuestro carga a categorías y/o páginas de galería según tema, así que vuestras contribuciones pueden ser encontradas y utilizadas por otros. </p>
      <p>No suele ser necesario crear una categoría nueva - antes de hacerlo, asegúrate de estar familiarizado con la estructura de categorías existente, y con los usos y políticas de Commons. Por favor mira si existe un <a href="/wiki/Category:Commons_category_schemes" title="Category:Commons category schemes">esquema de categorías</a> o un <a href="/wiki/Category:Commons_projects" title="Category:Commons projects">proyecto de Commons</a> para el tema que tratas, y sigue las convenciones allí descritas. </p>
      <p><span id="Category_names"></span> </p> 
      <h3><span id="Nombres_de_categor.C3.ADa"></span><span class="mw-headline" id="Nombres_de_categoría">Nombres de categoría</span></h3> 
      <p>Nombres de categoría generalmente tendrían que ser en ingleses (ve <a href="/wiki/Commons:Language_policy/es" title="Commons:Language policy/es">Commons:Política de idiomas</a>). Aun así, hay excepciones como algunos nombres propios, biológicos taxa y nombres para qué el nombre no inglés es más generalmente utilizado en la lengua inglesa (o no hay ninguna evidencia de uso de una versión de lengua inglesa). <a href="/w/index.php?title=Template:Wp-Alfabetos_latinos&amp;action=edit&amp;redlink=1" class="new" title="Template:Wp-Alfabetos latinos (page does not exist)">Template:Wp-Alfabetos latinos</a> están utilizados en la forma original que incluye diacritics y derivó letras, alfabetos no latinos están transcritos al guión latino inglés. Caracteres ingleses básicos (<a href="https://en.wikipedia.org/wiki/ISO/IEC_646" class="extiw" title="w:ISO/IEC 646">ISO/IEC 646</a>) está preferido sobre extensión o variantes nacionales conjuntos de carácter (para caso, estraight' apóstrofos encima 'curly'), donde razonables. </p> 
      <div class="mw-translate-fuzzy"> 
       <p>Los nombres de categorías que se refieran a objetos o personas deberían generalmente ir plural (en inglés): <a href="/wiki/Category:Tools" title="Category:Tools">Category:Tools</a>, <a href="/wiki/Category:Artists" title="Category:Artists">Category:Artists</a>, <a href="/wiki/Category:Lakes" title="Category:Lakes">Category:Lakes</a>, etc, a diferencia de <a href="/wiki/Category:History" title="Category:History">Category:History</a>, <a href="/wiki/Category:Weather" title="Category:Weather">Category:Weather</a>, <a href="/wiki/Category:Music" title="Category:Music">Category:Music</a>. </p> 
      </div> 
      <div class="mw-translate-fuzzy"> 
       <p>Para una discusión general de la función de categorías de MediaWiki, véase la <a href="https://meta.wikimedia.org/wiki/Help:Categories" class="extiw" title="m:Help:Categories">página del manual sobre categorías</a>. </p> 
      </div> 
      <p>Las categorías que agrupan subcategorías por el nombre generalmente tendría que ser nombrado "por nombre" más que "por alfabeto" (p. ej. <a href="/wiki/Category:Ships_by_name" title="Category:Ships by name">Category:Ships by name</a>). </p>
      <p>Nosotros todavía internacionalización de carencia para nombres de categoría, pero este asunto tendría que ser resuelto con cambios apropiados al MediaWiki software (ve <a href="https://phabricator.wikimedia.org/T31928" class="extiw" title="phab:T31928">T31928</a>: ' esqué traducido titula por lengua de usuario en categorías también<i>). Creando intermingled estructuras de categoría en lenguas diferentes sólo cosas de marca peores.</i> </p>
      <p>Para una discusión general de MediaWiki característica de categoría, ver el <a href="https://meta.wikimedia.org/wiki/Help:Categories" class="extiw" title="m:Help:Categories">página manual en categorías</a>. </p> 
      <h3><span id="Categorizaci.C3.B3n_de_p.C3.A1ginas"></span><span class="mw-headline" id="Categorización_de_páginas">Categorización de páginas</span></h3> 
      <p>Para añadir una página (serlo una imagen, una página de galería, o una página de categoría) a una categoría, añadir el código siguiente hasta el final de la página. </p>
      <p><code>[[Categoría:<i>Nombre de categoría</i>]]</code> </p>
      <p>Por ejemplo, si subes un diagrama mostrando la órbita de los cometas, deberías añadir lo siguiente a la página de descripción de la imagen: </p> 
      <pre>[[Category:Diagrams]]
[[Category:Comets]]
</pre> 
      <p>Esto hará que el diagrama aparezca en las categorías de <a href="/wiki/Category:Diagrams" title="Category:Diagrams">diagramas</a> y <a href="/wiki/Category:Comets" title="Category:Comets">Cometas</a>. </p>
      <p>Para saber cómo encontrar buenas categorías a los archivos que subas o a las subcategorías en las que trabajes, lee la sección <i><a href="#Categorizar_tus_archivos">Categorizar tus archivos</a></i> abajo. </p> 
      <h3><span id="Crear_una_nueva_categor.C3.ADa"></span><span class="mw-headline" id="Crear_una_nueva_categoría">Crear una nueva categoría</span></h3> 
      <p>Para crear una nueva categoría, primero añade el enlace a una página como se describe arriba (<i>[[Category:...]]</i>). Después sigue el enlace rojo a esa categoría - entonces puedes editar la página de la categoría como cualquier otra página de una wiki. </p>
      <p>Una página de categoría debería contener la siguiente información (en orden de importancia): </p> 
      <ul>
       <li>Enlaces de categoría que la ubiquen en una o más categorías superiores (<i>[[Category:...]]</i>)</li> 
       <li>Un breve texto descriptivo que explique qué debería haber en esa categoría. El inglés es el idioma preferido para la descripición, y se pueden añadir otros idiomas.</li>
      </ul> 
      <div class="mw-translate-fuzzy"> 
       <ul>
        <li>Interwiki O interlanguage enlaces al artículo o categoría con el mismo tema en Wikipedia por añadir el apropiado sitelinks en el correspondiente Wikidata página. Después de crear la página de categoría, el clic "Añade enlaces" debajo "En Wikipedia" en el fondo del sidebar a la izquierda para añadirles.</li>
       </ul> 
       <p>Si la categoría tendría que ser ordenada según una cuerda diferente que el título de categoría, añadir un {{<code>DEFAULTSORT</code>:}}. Para caso, el título de una categoría sobre una persona no sería la cuerda de clase correcta. Para tales categorías, inserta justo antes de las categorías una línea como con la cuerda de clase correcta. Ve <a href="https://meta.wikimedia.org/wiki/Categories#Sort_key" class="extiw" title="meta:Categories">Categorías</a> para más información. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>See also <a href="#How_to_categorize:_guidance_by_topic">#How to categorize: guidance by topic</a> for guidance on specific classes of category, e.g. categories about <a href="#People">#People</a>. </p> 
      </div> 
      <h3><span id="Cambiar_el_nombre_o_mover_categor.C3.ADas"></span><span class="mw-headline" id="Cambiar_el_nombre_o_mover_categorías">Cambiar el nombre o mover categorías</span></h3> 
      <p>Complacer ver <a href="/wiki/Commons:Rename_a_category/es" title="Commons:Rename a category/es">Commons:Cambiar el nombre de una categoría</a>. </p> 
      <h2><span id="Para_categorizaci.C3.B3n_m.C3.A1s_apropiada"></span><span class="mw-headline" id="Para_categorización_más_apropiada">Para categorización más apropiada</span></h2> 
      <p>Deberías colocar los archivos que subas en categorías y/o páginas de galerías de acuerdo a su tema, para que otros puedan encontrar y utilizar tus contribuciones. </p>
      <p>Ejemplo: Supone que <a href="/wiki/Category:Spheres" title="Category:Spheres">Category:Spheres</a> contiene cuadros únicos de pelotas de cristal. No tienes que añadir <a href="/wiki/Category:Glass" title="Category:Glass">Category:Glass</a> en la página de categoría, según los contenidos actuales, porque puedes tener las esferas hicieron con una variedad grande de materiales. Normalmente, cualquier cuadro que muestra un objeto de vaso sería ya categorizada en <a href="/wiki/Category:Glass" title="Category:Glass">Category:Glass</a> (o en una categoría de su subestructura). Tan, si el <a href="/wiki/Category:Spheres" title="Category:Spheres">Category:Spheres</a> es realmente llenado con cuadros de pelotas del cristal, sea una idea mejor de crear una página de categoría nueva, gusta Categoría:esferas de Vaso o Categoría:pelotas de Cristal, categorizado en <a href="/wiki/Category:Spheres" title="Category:Spheres">Category:Spheres</a> y <a href="/wiki/Category:Glass" title="Category:Glass">Category:Glass</a>. </p>
      <p>En general los archivos sólo deberían estar en la categoría más específica que exista sobre un tema. Por ejemplo los archivos de <a href="/wiki/Category:Paris" title="Category:Paris">Category:Paris</a> no deberían estar también directamente en <a href="/wiki/Category:France" title="Category:France">Category:France</a> (ver <a href="#Sobrecategorización">Sobrecategorización</a> abajo). Si buscas y no encuentras una categoría que se ajuste a tu propósito, puedes crearla - pero antes lee con cuidado la sección sobre <a href="#Uso_de_las_categorías">Uso de las categorías</a>. </p> 
      <div class="mw-translate-fuzzy"> 
       <p>Esto no quiere decir que una imagen pertenezca sólo a una categoría; simplemente significa que las imágenes no deben estar en categoría <i>redundantes</i> no-específicas. Por ejemplo, una imagen de un helicóptero rescatando un oso polar de un iceberg debería estar en <a href="/wiki/Category:Ursus_maritimus" title="Category:Ursus maritimus">Category:Ursus maritimus</a> (oso polar), <a href="/wiki/Category:Icebergs" title="Category:Icebergs">Category:Icebergs</a>, <a href="/wiki/Category:Helicopters" title="Category:Helicopters">Category:Helicopters</a> (helicópteros), y <a href="/wiki/Category:Search_and_Rescue" title="Category:Search and Rescue">Category:Search and Rescue</a> (Búsqueda y rescate). No debería tener el enlace para colgar directamente de <a href="/wiki/Category:Ursidae" title="Category:Ursidae">Category:Ursidae</a> (úrsidos) ni de <a href="/wiki/Category:Aircraft" title="Category:Aircraft">Category:Aircraft</a> (aeronaves). </p> 
      </div> 
      <h3><span id="Consejos_de_categorizaci.C3.B3n"></span><span class="mw-headline" id="Consejos_de_categorización">Consejos de categorización</span></h3> 
      <p>Las categorías (o galerías) que escojas para los archivos que subas deberían contestar al menos a alguna de las siguientes preguntas: </p> 
      <ul>
       <li><b>¿Qué / quién?</b> Qué o a quién muestra el archivo. Por ejemplo <a href="/wiki/Category:Houses" title="Category:Houses">Category:Houses</a> (casas) o <a href="/wiki/Category:Jimmy_Wales" title="Category:Jimmy Wales">Category:Jimmy Wales</a></li> 
       <li><b>¿Dónde?</b> Dónde se tomó la imagen. Obviamente, esto es especialmente importante para imágenes de lugares. Por ejemplo <a href="/wiki/Category:Saxony" title="Category:Saxony">Category:Saxony</a> (Sajonia)</li> 
       <li><b>¿Cuándo?</b> Debería proporcionar contexto histórico, si es relevante. Por ejemplo, <a href="/wiki/Category:World_War_II" title="Category:World War II">Category:World War II</a> (Segunda Guerra Mundial).</li>
      </ul> 
      <div class="mw-translate-fuzzy"> 
       <ul>
        <li><b>¿Quién?</b> Si el artista es bien conocido, tiene sentido categorizar por artista, como <a href="/wiki/Category:Rembrandt" title="Category:Rembrandt">Category:Rembrandt</a>. Puedes también emplear las páginas del espacio de nombres <i>Creator</i> como plantillas para ello (se explicará en <a href="/wiki/Commons:Creator" title="Commons:Creator">Commons:Creator</a>).</li> 
        <li><b>¿Cómo?</b> Debería dar información sobre el tipo de medio, como <a href="/wiki/Category:Audio" title="Category:Audio">Category:Audio</a> (aunque actualmente la categoría <i>Audio</i> no reúne sólo archivos de audio).</li>
       </ul> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>The above questions cover the main aspects of the image to be categorized. For some images it makes sense to use all, for other images only one or two are reasonable. In addition there are several other aspects of the images that can be used to categorize the image: </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <ul>
        <li><b>what source?:</b> information about where the image came from. For example, <a href="/wiki/Category:Images_from_the_German_Federal_Archive" title="Category:Images from the German Federal Archive">Category:Images from the German Federal Archive</a></li> 
        <li><b>what format?:</b> information about the media type if it is unusual. For example, <a href="/wiki/Category:Audio" title="Category:Audio">Category:Audio</a> or <a href="/wiki/Category:Animated_GIF" title="Category:Animated GIF">Category:Animated GIF</a>, <a href="/wiki/Category:SVG" class="mw-disambig" title="Category:SVG">Category:SVG</a></li> 
        <li><b>what software?:</b> information about software used to create the image. For example, <a href="/wiki/Category:Created_with_Hugin" title="Category:Created with Hugin">Category:Created with Hugin</a></li> 
        <li><b>what camera?:</b> information about the camera. For example, <a href="/wiki/Category:Taken_with_Nikon_D80" title="Category:Taken with Nikon D80">Category:Taken with Nikon D80</a></li>
       </ul> 
       <p>This last set is useful and important but should always be done in addition of the main set of criteria. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>Categorization in Wikimedia Commons is more detailed and deep than categorization in Wikipedia projects. Compared to them, Commons has more categories for individual subjects – places, people, organizations, events, terms, etc. Almost every article on a Wikipedia can have a corresponding category on Commons. However, even if there exist more images of an ordinary person or incidental event, it is practical to group them into a special category and categorize that category instead of categorizing all similar images individually to an identical set of parent categories. </p> 
      </div> 
      <h3><span id="Encuentra_una_categor.C3.ADa_apropiada"></span><span class="mw-headline" id="Encuentra_una_categoría_apropiada">Encuentra una categoría apropiada</span></h3> 
      <p>Para encontrar categorías apropiadas para los archivos, deberías navegar por la estructura de categorías empezando por una categoría genérica. Reduce tu búsqueda a subcategorías hasta que encuentres la subcategoría más específica que encaje con el archivo que has subido. Puedes navegar por la estructura de categorías siguiendo enlaces a subcategorías, o expandiendo el árbol de subcategorías picando en los simbolitos <b>[+]</b> ante los nombres de las subcategorías. La sección <i><a href="#Estructura_de_categorías">Estructura de categorías</a></i> de arriba proporciona un punto de inicio. Puedes probar también <b><a rel="nofollow" class="external text" href="http://tools.wikimedia.de/~daniel/WikiSense/CommonSense.php">CommonSense</a></b>, una herramienta diseñada para ayudar con la categorización basada en palabras clave. </p> 
      <h3><span id="Sobrecategorizaci.C3.B3n"></span><span class="mw-headline" id="Sobrecategorización">Sobrecategorización</span></h3> 
      <div class="mw-translate-fuzzy"> 
       <p>Sobrecategorización es lo que ocurre cuando una imagen aparece en varias categorías en la misma línea de un árbol. La regla general es <i>ubica siempre una imagen en las categorías más específicas y no en los niveles superiores</i>. Un ejemplo: </p> 
      </div> 
      <p>Asumimos que las canicas amarillas son esferas de color amarillo. Podemos pensar en la categoría <i>Category:Yellow spheres</i> y en <a href="/wiki/Category:Spheres" title="Category:Spheres">Category:Spheres</a>. Si tenemos que categorizar una imagen que muestra canicas amarillas, lo hacemos en <b>Category:Yellow spheres</b>. Ahora, si también la categorizamos directamente en <a href="/wiki/Category:Spheres" title="Category:Spheres">Category:Spheres</a>, cometemos una sobrecategorización porque ya sabemos que las canicas amarillas son esferas. </p>
      <p>Esto se aplica a la mayoría de imágenes: Como se explica arriba, los archivos de <a href="/wiki/Category:Paris" title="Category:Paris">Category:Paris</a> no deberían estar directamenete en <a href="/wiki/Category:France" title="Category:France">Category:France</a>, los archivos en <a href="/wiki/Category:Albert_Einstein" title="Category:Albert Einstein">Category:Albert Einstein</a> no deberían estar directamente en <a href="/wiki/Category:Physicists_from_Germany" title="Category:Physicists from Germany">Category:Physicists from Germany</a> etcétera. </p> 
      <h4><span id="Por_qu.C3.A9_la_sobrecategorizaci.C3.B3n_constituye_un_problema"></span><span class="mw-headline" id="Por_qué_la_sobrecategorización_constituye_un_problema">Por qué la sobrecategorización constituye un problema</span></h4> 
      <p>A menudo se piensa que cuantas más categorías tenga una imagen, más fácil será encontrarla. Según esta lógica, toda imagen de un hombre debería estar en <a href="/wiki/Category:Men" title="Category:Men">Category:Men</a>, porque aunque un usuario sepa algo más de la persona que está buscando, aparte de que es un hombre, sería capaz de encontrarlo en la categoría general. El resultado es que dicha categoría superior se llena, haciendo necesario pasar por cientos o miles de imágenes, hasta encontrar la que se quiere. Probablemente, el usuario no encontrará la que busca, y lo que es más, los que buscan una fotografía genérica de un hombre para ilustrar el artículo <a href="https://es.wikipedia.org/wiki/Var%C3%B3n" class="extiw" title="es:Varón">es:Varón</a> encontrará que están sepultadas entre estrellas de cine, científicos y políticos. </p>
      <p>En niveles más bajos, el problema es menos grave, ya que la cantidad de imágenes será menor - aunque pueden fácilmente llegar a cientos. Pero sigue habiendo un problema: Volviendo al ejemplo, si un usuario busca una imagen de Einstein y sabe que es un físico, buscará en <a href="/wiki/Category:Physicists" title="Category:Physicists">Category:Physicists</a>. Si encuentra una imagen entre los cientos de esa categoría, aunque no sea la que más le satisface, puede quedarse con ella, en lugar de seguir buscando por la subcategoría <a href="/wiki/Category:Physicists_from_Germany" title="Category:Physicists from Germany">Category:Physicists from Germany</a> hasta llegar a <a href="/wiki/Category:Albert_Einstein" title="Category:Albert Einstein">Category:Albert Einstein</a>, donde puede haber otra mejor. Así, la sobrecategorización ha causado dos problemas, la categoría superior está saturada, y los usuarios pueden dejar de buscar la categoría más relevante, ya que han encontrado una que tiene una imagen más o menos relevante. </p> 
      <h4><span id="Categorizaci.C3.B3n_incorrecta_de_categor.C3.ADas"></span><span class="mw-headline" id="Categorización_incorrecta_de_categorías">Categorización incorrecta de categorías</span></h4> 
      <p>Aunque parezca extraño, la subcategorización es una causa de la sobrecategorización. Esto ocurre cuando una categoría no se categoriza propiamente, haciendo que los usuarios sobrecategoricen una imagen para que encaje en las categorías más relevantes. Por ejemplo, <a href="/wiki/Category:Eiv%C3%B8r_P%C3%A1lsd%C3%B3ttir" title="Category:Eivør Pálsdóttir">Category:Eivør Pálsdóttir</a> se incluyó solo en <a href="/wiki/Category:People_by_name" title="Category:People by name">Category:People by name</a>. Así que si alguien añade imágenes de ella, y sabe quién es, colocará las imágenes en <a href="/wiki/Category:People_of_the_Faroe_Islands" title="Category:People of the Faroe Islands">Category:People of the Faroe Islands</a> y <a href="/wiki/Category:Vocalists" title="Category:Vocalists">Category:Vocalists</a>. Eso es sobrecategorizar; está saturando las categorías superiores al añadir imágenes directamente en ellas. </p>
      <p>Un problema relacionado es la categorización errónea; <a href="/wiki/Category:Notting_Hill" title="Category:Notting Hill">Category:Notting Hill</a> estuvo más de un mes ubicada en <a href="/wiki/Category:London" title="Category:London">Category:London</a>. Eso incitaba a que al añadir una imagen, se colocara directamente en <a href="/wiki/Category:Royal_Borough_of_Kensington_and_Chelsea" title="Category:Royal Borough of Kensington and Chelsea">Category:Royal Borough of Kensington and Chelsea</a>, que es el <i>borough</i> o distrito del Gran Londres en el que se encuentra Notting Hill, al no encontrar la categoría de Notting Hill dentro de Kensington and Chelsea cuando se seguía el árbol de los distritos de Londres. En lugar de eso, cada imagen solo debería estar en las categorías más específicas, y en esas categorías deberían a su vez colocarse las categorías más específicas que las pudieran albergar. </p>
      <p>Cuando encuentres esto, por favor si puedes, categoriza las categorías correctamente. Esto no solo ayudará a evitar la sobrecategorización, sino que hará más fácil moverse por el árbol de categorías. </p> 
      <h4><span id="Excepci.C3.B3n_para_im.C3.A1genes_con_m.C3.A1s_categorizado_temas"></span><span class="mw-headline" id="Excepción_para_imágenes_con_más_categorizado_temas"><span id="OvercatException"></span> Excepción para imágenes con más categorizado temas</span></h4> 
      <p>Un archivo que describe sólo uno el tema pertinente no tendría que ser encima-categorizado. Dónde un archivo describe temas pertinentes adicionales, y los temas adicionales no tienen sus subcategorías propias, la consideración puede ser dada a temporalmente categorizando la imagen en ambas la subcategoría y la categoría de padre. </p>
      <p>Por ejemplo, esta situación podría surgir en el caso de una fotografía de tres políticos, uno de quien es <a href="/wiki/Category:Angela_Merkel" title="Category:Angela Merkel">Angela Merkel</a> (quién tiene su propio Commons categoría), con dos otros políticos que no todavía tener sus categorías propias. Mientras la imagen indudablemente sería categorized en <a href="/wiki/Category:Angela_Merkel" title="Category:Angela Merkel">Category:Angela Merkel</a> o uno de sus subcategorías, típicamente sea considerado para ser encima-categorización a también incluir él en <a href="/wiki/Category:Politicians_of_Germany" title="Category:Politicians of Germany">Category:Politicians of Germany</a>. Usuarios , aun así, probablemente no poder buscar el dos otros políticos en la Merkel categoría. Idealmente, crearíamos subcategorías concretas para el dos otros políticos (dónde warranted), o encontrar otras subcategorías pertinentes (p. ej. <a href="/wiki/Category:Politicians_of_Bavaria" title="Category:Politicians of Bavaria">Category:Politicians of Bavaria</a> o <a href="/wiki/Category:Members_of_the_FDP" title="Category:Members of the FDP">Category:Members of the FDP</a>, etc.), aquello nos habilitaría para evitar encima-categorización. En algunas circunstancias, aun así, podemos necesitar a temporalmente categorize la imagen en <a href="/w/index.php?title=Categor%C3%ADa:Pol%C3%ADticos_de_Alemania&amp;action=edit&amp;redlink=1" class="new" title="Categoría:Políticos de Alemania (page does not exist)">Categoría:Políticos de Alemania</a> dónde otras subcategorías apropiadas no todavía existen. </p> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>Countries may be categorized as part of multiple overlapping categories. For example, <a href="/wiki/Category:India" title="Category:India">Category:India</a> is in <a href="/wiki/Category:Countries_of_South_Asia" title="Category:Countries of South Asia">Category:Countries of South Asia</a> as well as <a href="/wiki/Category:Countries_of_Asia" title="Category:Countries of Asia">Category:Countries of Asia</a>. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>Also user categories are exempted of over-categorization as those are not visible to most viewers, and project users include them for many different purposes like sorting, stats, filling values for userboxes, etc. </p> 
      </div> 
      <h2><span id="C.C3.B3mo_categorizar:_orientaci.C3.B3n_por_tema"></span><span class="mw-headline" id="Cómo_categorizar:_orientación_por_tema">Cómo categorizar: orientación por tema</span></h2> 
      <p>Para algunas categorías, hay guiaje especial encima qué más para ordenar contenido dentro de aquella categoría. Este guiaje puede ser encontrado en un <a href="/w/index.php?title=Categor%C3%ADa:Commons_esquema_de_categor%C3%ADa&amp;action=edit&amp;redlink=1" class="new" title="Categoría:Commons esquema de categoría (page does not exist)">de esquemas de categoría</a> o un <a href="/w/index.php?title=Categor%C3%ADa:Commons_proyecta&amp;action=edit&amp;redlink=1" class="new" title="Categoría:Commons proyecta (page does not exist)">commons proyecto</a> para vuestro tema. hay también algunos categorizing información en esta sección y a veces hay guiaje en la parte superior de la página de la categoría, en la Categoría <a href="https://meta.wikimedia.org/wiki/Namespace" class="extiw" title="meta:Namespace">namespace</a>. Tan, para caso, algún guiaje en categorizing el contenido que describe las personas está al frente de <a href="/wiki/Category:People" title="Category:People">Category:People</a>, y algunos es en la sección <a href="/wiki/Category:People" title="Category:People">Personas</a> abajo. </p> 
      <h3><span class="mw-headline" id="Gente">Gente</span></h3> 
      <p>El contenido que describe las personas tendrían que ser puestas en categorías qué describirles, como <a href="/wiki/Category:Economists_from_the_United_States" title="Category:Economists from the United States">Category:Economists from the United States</a>. El inicio que explora en <a href="/wiki/Category:People" title="Category:People">Category:People</a>. </p>
      <p>Complacer ver <a href="/wiki/Special:MyLanguage/Commons:Category_scheme_People" title="Special:MyLanguage/Commons:Category scheme People">Commons:Category scheme People</a> para detalles encima cómo para nombrar y organizar estas categorías. </p> 
      <h3><span id="Paisajes.2C_vistas_exteriores"></span><span class="mw-headline" id="Paisajes,_vistas_exteriores">Paisajes, vistas exteriores</span></h3> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>Content depicting a given subject from a common vantage point are grouped in <i>Views of Subject from Viewpoint</i> categories such as <a href="/wiki/Category:Views_of_Cathedral_of_Seville_from_the_Giralda" title="Category:Views of Cathedral of Seville from the Giralda">Views of Cathedral of Seville from the Giralda</a>. Such categories should be subcategories of both the subject's category (<a href="/wiki/Category:Cathedral_of_Seville" title="Category:Cathedral of Seville">Cathedral of Seville</a> in this example) and the viewpoint's category (<a href="/wiki/Category:Giralda" title="Category:Giralda">Giralda</a> in this example). </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>In this example, the <a href="/wiki/Category:Views_of_Cathedral_of_Seville_from_the_Giralda" title="Category:Views of Cathedral of Seville from the Giralda">Views of Cathedral of Seville from the Giralda</a> category is not placed directly in the subject and viewpoint categories, but in <a href="/wiki/Category:Views_of_the_Cathedral_of_Seville" title="Category:Views of the Cathedral of Seville">Views of the Cathedral of Seville</a> and <a href="/wiki/Category:Views_from_Giralda" title="Category:Views from Giralda">Views from Giralda</a>. Such intermediate categories are often necessary to create structure and avoid <a href="#over-categorization">over-categorization</a>, particularly for views of a city from a vantage point located within the city. For example, <a href="/wiki/Category:Views_of_Rome_from_the_Pincio" title="Category:Views of Rome from the Pincio">Views of Rome from the Pincio</a> needs the intermediate category <a href="/wiki/Category:Views_of_Rome" title="Category:Views of Rome">Views of Rome</a> to avoid placing it directly in <a href="/wiki/Category:Rome" title="Category:Rome">Rome</a>, which would constitute over-categorization. </p> 
      </div> 
      <h3><span class="mw-headline" id="Textos">Textos</span></h3> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>Texts, such as scans of books, should normally have a category for each version of the scan and each edition of the text. Thus a book published in three separate editions would have a parent category for the book, three subcategories for each text, and further subcategories for the text as a jpeg, a DjVu, etc., assuming each version had actually been uploaded. (Categories would not be created for editions not held on Commons.) This is particularly important for files in formats other than DjVu and PDF, where the category is the only practical means of keeping the scans together; see eg. <a href="/wiki/Category:The_Chronicles_of_England,_Scotland_and_Ireland,_Holinshed,_1587" title="Category:The Chronicles of England, Scotland and Ireland, Holinshed, 1587">Category:The Chronicles of England, Scotland and Ireland, Holinshed, 1587</a> which contains 2857 jpeg images of page scans. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <h3><span class="mw-headline" id="GLAMs">GLAMs</span></h3> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>For categorization issues related to mass content donations from GLAMs (Galleries, Libraries, Archives &amp; Museums), please see <a href="/wiki/Special:MyLanguage/Commons:Guide_to_batch_uploading#Categories" title="Special:MyLanguage/Commons:Guide to batch uploading">Commons:Guide to batch uploading#Categories</a>. </p> 
      </div> 
      <h2><span id="Flujo_de_trabajo_de_categorizaci.C3.B3n"></span><span class="mw-headline" id="Flujo_de_trabajo_de_categorización">Flujo de trabajo de categorización</span></h2> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>Currently, a bot checks if newly uploaded files are categorized in topical categories and attempts to categorize files that are not. Before 17 June 2015, <a href="/wiki/User:CategorizationBot" title="User:CategorizationBot">CategorizationBot</a> was responsible for this job. As of June 2019, <a href="/wiki/User:SteinsplitterBot" title="User:SteinsplitterBot">SteinsplitterBot</a> occasionally checks for uncategorized files. The workflow is the following: </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <ol>
        <li>User uploads a new file and adds categories (or not).</li> 
        <li>A bot checks if the file is categorized. 
         <ul>
          <li>File is already categorized =&gt; ok</li> 
          <li>File is not categorized =&gt; the bot tags the file for <a href="/wiki/Category:Media_needing_categories" title="Category:Media needing categories">Category:Media needing categories</a>. Today's files are in <a href="/wiki/Category:Media_needing_categories_as_of_6_June_2021" title="Category:Media needing categories as of 6 June 2021">Category:Media needing categories as of 6 June 2021</a></li>
         </ul></li> 
        <li>Users categorize files further (e.g. category diffusion below)</li>
       </ol> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p><i>See also:</i> <a href="/wiki/User:CategorizationBot#Process" title="User:CategorizationBot">User:CategorizationBot#Process</a>, <a href="/wiki/User:Multichill/Categorization_stats" title="User:Multichill/Categorization stats">categorization statistics</a> </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>Other, if manual, categorization workflows are possible&nbsp;: </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <ul>
        <li>Category filling: Use appropriate keywords in the search engine to find the files that should be in a given category, and put them there.</li> 
        <li>Category diffusing: Go to <a href="/wiki/Category:Categories_requiring_diffusion" title="Category:Categories requiring diffusion">Category:Categories requiring diffusion</a>, select a crowded category, create appropriate subcategories if needed, and move the files to the subcategories. Gadgets like Cat-a-lot and Hotcats can help.</li>
       </ul> 
      </div>

       <h2><span id="Categories_marked_with_.22HIDDENCAT.22"></span><span class="mw-headline" id="Categories_marked_with_&quot;HIDDENCAT&quot;">Categories marked with "HIDDENCAT"</span></h2> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <div class="shortcut2" style="">
        <style data-mw-deduplicate="TemplateStyles:r433650343">.mw-parser-output .shortcut2{float:right;margin:0.4em;border:1px solid #ccc;background-color:white;padding:0 1em;text-align:center;font-size:x-small;border-radius:4px}</style><a href="/wiki/Commons:Shortcuts" title="Commons:Shortcuts">Shortcut</a>
        <br><a href="/wiki/Commons:HIDDENCAT" class="mw-redirect" title="Commons:HIDDENCAT">COM:HIDDENCAT</a>
       </div> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>Many non-topical categories are marked with <code>__HIDDENCAT__</code> or {{<a href="/wiki/Template:Hiddencat" title="Template:Hiddencat">Hiddencat</a>}} on the category page. For example, see <a class="external text" href="https://commons.wikimedia.org/w/index.php?title=Category:PD_NASA&amp;action=edit">Category:PD NASA in edit mode</a>. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>While categories are generally visible on every page, categories marked <code>__HIDDENCAT__</code> are only visible: </p> 
       <ul>
        <li>on the edit screen: at the end of the screen, below the edit box</li> 
        <li>on category pages: 
         <ul>
          <li>on subcategories to the hidden category: in the normal location, but on a separate line with a smaller typeface and the label "Hidden categories."</li> 
          <li>on parent categories: in the same way as other categories</li>
         </ul></li> 
        <li>on file description pages and gallery pages: for logged-in users who have selected to "Show hidden categories" in their <a href="/wiki/Special:Preferences#mw-prefsection-rendering" title="Special:Preferences">appearance preferences</a>. This is activated for all newly registered users.</li>
       </ul> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>This feature is generally used for template-based categories, such as license tag based categories. For example, placing <a href="/wiki/Template:PD-old-100" title="Template:PD-old-100"><span style="font-family:monospace,monospace;">{{PD-old-100}}</span></a> on a file description page adds the file to <a href="/wiki/Category:Author_died_more_than_100_years_ago_public_domain_images" title="Category:Author died more than 100 years ago public domain images">Category:Author died more than 100 years ago public domain images</a>, which is marked with <code>__HIDDENCAT__</code>. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>For more details, see the <a href="https://meta.wikimedia.org/wiki/Help:Category#Hidden_categories" class="extiw" title="m:Help:Category">help section on hidden categories</a> for Mediawiki (the software that Commons uses). </p> 
      </div>

       <ul>
        <li>Gadgets enabled through the <a href="/wiki/Special:Preferences#mw-prefsection-gadgets" title="Special:Preferences">user preferences</a> 
         <ul>
          <li><i>Cat-a-lot</i>: A tool that helps with moving multiple files between categories or adding categories to search results. <small>[<a href="/wiki/Help:Gadget-Cat-a-lot" title="Help:Gadget-Cat-a-lot">documentation</a>&nbsp;/&nbsp;<a href="/wiki/MediaWiki_talk:Gadget-Cat-a-lot.js" title="MediaWiki talk:Gadget-Cat-a-lot.js">talk</a>]</small>&nbsp;</li> 
          <li><i>HotCat</i>:&nbsp;<sup><abbr title="default gadget – enabled by default">d</abbr></sup> Easily add / remove / change a category on a page, with name suggestions. <small>[<a href="/wiki/Help:Gadget-HotCat" title="Help:Gadget-HotCat">documentation</a>&nbsp;/&nbsp;<a href="/wiki/File:HotCat.png" title="File:HotCat.png">example</a>&nbsp;/&nbsp;<a href="/wiki/MediaWiki_talk:Gadget-HotCat.js" title="MediaWiki talk:Gadget-HotCat.js">talk</a>]</small>&nbsp;</li> 
          <li><i>Gallery Details</i>: Adds a link in the toolbox to display galleries and categories (and <a href="/wiki/Special:NewFiles" title="Special:NewFiles">Newimages</a> and <a href="/wiki/Special:Search" title="Special:Search">Search</a> result pages) with extensive details from file description pages and links to easily mark an image without source, etc. If <i>Pretty log</i> is activated, it also works on <a href="/wiki/Special:Log" title="Special:Log">Log</a> pages. <small>[<a href="/wiki/Help:Gadget-GalleryDetails" title="Help:Gadget-GalleryDetails">documentation</a>&nbsp;/&nbsp;<a href="/wiki/MediaWiki_talk:Gadget-GalleryDetails.js" title="MediaWiki talk:Gadget-GalleryDetails.js">talk</a>]</small>&nbsp;</li> 
          <li>Place categories above content, but below image on file description pages. &nbsp;Modifies the placement of categories on the user interface.</li> 
          <li>Add a link to category pages to search for the category name with the option "-incategory". This excludes files already in the category (doesn't work if the category was added by a template). <small>[<a href="/wiki/MediaWiki_talk:Gadget-Searchnotincat.js" title="MediaWiki talk:Gadget-Searchnotincat.js">talk</a>]</small>&nbsp;</li>
         </ul></li>
