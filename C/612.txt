      <p>Uma <b>categoria</b> é uma funcionalidade do software <a href="https://pt.wikipedia.org/wiki/MediaWiki" class="extiw" title="pt:MediaWiki">MediaWiki</a>, uma página especial que pretende agrupar as páginas e multimédia relacionados. Na prática, isto implica que irá associar um único assunto a uma dada categoria. O nome da categoria deveria ser suficiente claro para adivinhar o assunto, embora algum texto extra possa ser útil para a definir de forma mais precisa. A estrutura da <a href="/wiki/Special:MyLanguage/Commons:Guide_to_layout/Category_pages" title="Special:MyLanguage/Commons:Guide to layout/Category pages">categoria</a> é o modo principal para organizar e encontrar ficheiros na Commons. É essencial que cada ficheiro possa ser encontrado ao navegar pela estrutura da categoria. Para permitir isto, cada ficheiro deve ser colocado diretamente numa categoria. Cada categoria deverá estar ela própria em categorias mais gerais, formando uma estrutura hierárquica. </p>

      <h2><span id="Guia_r.C3.A1pido"></span><span class="mw-headline" id="Guia_rápido">Guia rápido</span></h2> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>1. How to find the appropriate categories </p> 
       <dl>
        <dd>
         <ul>
          <li>Find categories with the search engine (see <a href="#Categorization_tips">#Categorization tips</a>)</li> 
          <li>or check how similar files are categorized (some may not be categorized though)</li> 
          <li>or start from the main topical category (<a href="/wiki/Category:Topics" title="Category:Topics">Category:Topics</a>)</li> 
          <li>Starting from these categories, check their parent or sub-categories to find an appropriate category. Avoid picking too general categories.</li>
         </ul>
        </dd>
       </dl> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>2. Add the categories to the file </p> 
       <dl>
        <dd>
         <ul>
          <li>Select them on the upload form</li> 
          <li>or: add them manually by adding, e.g. <code>[[Category:Foobar]]</code> to the file description page (see <a href="#Categorizing_pages">#Categorizing pages</a>)</li> 
          <li>or: use HotCat (see <a href="/wiki/Special:MyLanguage/Help:Gadget-HotCat" title="Special:MyLanguage/Help:Gadget-HotCat">Help:Gadget-HotCat</a>)</li> 
          <li>or: use Cat-a-lot (see <a href="/wiki/Special:MyLanguage/Help:Gadget-Cat-a-lot" title="Special:MyLanguage/Help:Gadget-Cat-a-lot">Help:Gadget-Cat-a-lot</a>)</li>
         </ul>
        </dd>
       </dl>

      <h2><span id="Estrutura_de_categoriza.C3.A7.C3.A3o_na_Wikimedia_Commons"></span><span class="mw-headline" id="Estrutura_de_categorização_na_Wikimedia_Commons">Estrutura de categorização na Wikimedia Commons</span></h2> 
      <h3><span id="Princ.C3.ADpios"></span><span class="mw-headline" id="Princípios">Princípios</span></h3> 
      <p>Os princípios principais são: </p> 
      <h4><span id="Princ.C3.ADpio_de_hierarquia"></span><span class="mw-headline" id="Princípio_de_hierarquia">Princípio de hierarquia</span></h4> 
      <div class="mw-translate-fuzzy"> 
       <p>A estrutura de categorização é (idealmente) uma <a href="https://en.wikipedia.org/wiki/pt:Grafos_ac%C3%ADclicos_dirigidos" class="extiw" title="w:pt:Grafos acíclicos dirigidos">multi-hierarquia</a>, com apenas uma única categoria raiz <a href="/wiki/Category:CommonsRoot" title="Category:CommonsRoot">Category:CommonsRoot</a>. Todas as categorias (exceto [[:Category:CommonsRoot|"CommonsRoot") deveriam encontrar-se incluídas em pelo menos uma outra categoria. Não deveriam existir ciclos (por exemplo, uma categoria não deveria conter-se a si própria, diretamente ou indiretamente). </p> 
      </div> 
      <h4><span id="Princ.C3.ADpio_de_modularidade"></span><span class="mw-headline" id="Princípio_de_modularidade">Princípio de modularidade</span></h4> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>The page (file, category) should be put in the most specific category/categories that fit(s) the page (not directly to its parent categories). A category can have more parent categories. A category can combine two (or more) different criteria; such categories are called "compound categories" or "intersection categories". E.g. the root category <a href="/wiki/Category:Churches" title="Category:Churches">Category:Churches</a> and the root category <a href="/wiki/Category:Russia" title="Category:Russia">Category:Russia</a> have a common subcategory <a href="/wiki/Category:Churches_in_Russia" title="Category:Churches in Russia">Churches in Russia</a>. </p> 
      </div> 
      <h4><span id="Princ.C3.ADpio_de_simplicidade"></span><span class="mw-headline" id="Princípio_de_simplicidade">Princípio de simplicidade</span></h4> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>This principle suggests not to combine too many different criteria. </p> 
      </div> 
      <h4><span id="Princ.C3.ADpio_da_seletividade"></span><span class="mw-headline" id="Princípio_da_seletividade">Princípio da seletividade</span></h4> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>We should not classify items which are related to different subjects in the same category. The category name should be unambiguous and not homonymous. </p> 
      </div> 
      <h4><span id="Princ.C3.ADpio_da_universalidade"></span><span class="mw-headline" id="Princípio_da_universalidade">Princípio da universalidade</span></h4> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>Identical items should have identical names for all countries and at all levels of categorization. Categorization structure should be as systematical and unified as possible, local dialects and terminology should be supressed in favour of universality if possible. Analogic categorization branches should have analogic structure. </p> 
      </div> 
      <h3><span id="Tipos_de_rela.C3.A7.C3.B5es_refletidas"></span><span class="mw-headline" id="Tipos_de_relações_refletidas">Tipos de relações refletidas</span></h3> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>The category structure should reflect a hierarchy of concepts, from the most generic one down to the very specific. The structure uses and combines more types of relation, e. g. </p> 
       <ul>
        <li><a href="https://en.wikipedia.org/wiki/pt:Hiper%C3%B4nimo_e_hip%C3%B4nimo" class="extiw" title="w:pt:Hiperônimo e hipônimo">Hyponymy</a>: a sort/kind/type of… (typically in biological taxonomy)</li> 
        <li><a href="https://en.wikipedia.org/wiki/pt:Meron%C3%ADmia" class="extiw" title="w:pt:Meronímia">Meronymy</a>: a part of…, a member of… (typically for geographical division, building/room, device/component etc.)</li>
       </ul> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <ul>
        <li>Attributes: 
         <ul>
          <li>Qualitative and general attributes (color, shape, size, ability or disability, nationality, technique, quality, awards…)</li> 
          <li>Location: where, in…, from… (place/event, place/building, place/exhibit, place/people, country/language, source/work, factory or country/product etc.)</li> 
          <li>Timing: when (time/event, time/depicted situation, time of birth, inception or construction, time of death, demolition or termination etc.)</li>
         </ul></li>
       </ul> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <ul>
        <li>Agentive and influence relations: (creator/work, device/product, company/product, discipline or profession/their subjects and terms, parent/children, subordination, owner/property, initiator/follower, subject/other subjects dedicated to it or named after it, subject/its duplicate, imitiation, depiction or symbol etc etc.)</li> 
        <li>Modification: original/modified or modified/original (avoid cyclic structure) – renamed, rebuilt, repurposed or transformed subjects.</li>
       </ul> 
      </div> 
      <h3><span class="mw-headline" id="Categorias_principais">Categorias principais</span></h3> 
      <p>As categorias de topo (as contidas diretamente em CommonsRoot) dividem a estrutura de categorização de acordo com o seu propósito: </p> 
      <ul>
       <li><b><a href="/wiki/Category:Topics" title="Category:Topics">Categoria:Tópicos</a></b> - Esta categoria é a categoria raiz <b>comum global</b> dos ficheiros de multimédia categorizados por <b>TÓPICO</b>. 'TODOS' os ficheiros de multimédia deveriam estar categorizados sob esta categoria para ajudar os outros a encontrá-los por tópico. As categorias correntes não deveriam ser incluídas nos modelos.</li> 
       <li><b><a href="/wiki/Category:Copyright_statuses" title="Category:Copyright statuses">Categoria:Estados de direitos de autor</a></b> - Esta categoria é a categoria raiz <b>comum global</b> dos ficheiros de multimédia categorizados por <b>LICENÇA</b>. 'TODOS' os ficheiros de multimédia deveriam estar categorizados sob esta categoria com uma <a href="/wiki/Commons:Copyright_tags/pt" title="Commons:Copyright tags/pt">marcas de licença</a>. Este tipo de categoria é adicionado ao inclui-lo nos modelos.</li> 
       <li><b><a href="/wiki/Category:Image_sources" title="Category:Image sources">Categoria:Fontes de imagem</a></b> – Esta categoria é a categoria raiz <b>comum global</b> dos ficheiros de multimédia categorizados por <b>FONTE</b>, ou seja, de onde provêm (livros, coleções, <i>sites</i>, etc). Este tipo de categoria é geralmente adicionado pelo modelo.</li> 
       <li><b><a href="/wiki/Category:Media_types" title="Category:Media types">Categoria:Tipos de multimédia</a></b> - Esta categoria é a categoria raiz <b>comum global</b> dos ficheiros de multimédia categorizados por <b>TIPO DE MULTIMÉDIA</b>. Por favor, note que por vezes a categorização é omitida para as imagens, porque constituem a vasta maioria dos ficheiros da Commons.</li> 
       <li><b><a href="/wiki/Category:Commons" title="Category:Commons">Categoria:Commons</a></b> – Esta categoria é a raiz comum global de categorização de páginas de manutenção e de projecto e ajuda (Commons:- e Help:-), exceptuando-se os ficheiros mídia. As páginas traduzidas devem ser categorizadas sob cada categoria da língua correspondente, no molde "Category:Commons-<a href="https://pt.wikipedia.org/wiki/ISO_639" class="extiw" title="pt:ISO 639">CÓDIGO DE LÍNGUAS ISO</a>". A estrutura disposta em <a href="/wiki/Category:Commons-en" title="Category:Commons-en">Category:Commons-en</a> serve de exemplo para a categorização noutras línguas.</li>
      </ul> 
      <dl>
       <dd>
        <dl>
         <dd>
          Existe uma subcategoria <a href="/wiki/Category:Commons_maintenance_content" title="Category:Commons maintenance content">Categoria:Conteúdo de manutenção da Commons</a> que serve pata a manutenção especial de <b>conteúdos comuns globais</b> do Wikimedia Commons e que não é traduzida. <b>TODOS os ficheiros multimédia</b> deverão encontrar-se categorizados sob as seguintes quatro categorias, mas APENAS ficheiros com problemas e em necessidade de serem avaliados deverão encontra-se também na subcategoria <a href="/wiki/Category:Commons_maintenance_content" title="Category:Commons maintenance content">Category:Commons maintenance content</a>.
         </dd>
        </dl>
       </dd>
      </dl> 
      <ul>
       <li><b><a href="/wiki/Category:User" title="Category:User">Categoria:Utilizador</a></b> – serve para categorias que contenham usuários do Commons, divididos por tópicos como, por exemplo, a língua que falam. Também contém a <a href="/wiki/Category:User_galleries" title="Category:User galleries">Categoria:galerias do utilizador</a>, que serve para galerias específicas de usuários (isto é, que não se encaixam num tópico).</li>
      </ul> 
      <h2><span class="mw-headline" id="Como_utilizar_as_categorias">Como utilizar as categorias</span></h2> 
      <p>Você deve colocar sempre os ficheiros que carrega em categorias e/ou galerias de acordo com o tópico, de modo a que as suas contribuições possam ser encontradas e utilizadas por terceiros. </p>
      <p>É raramente necessário criar uma nova categoria (veja no entanto uma excepção em <i><a href="#Pessoas">Pessoas</a></i>, abaixo). Antes de criar uma categoria, assegure-se de que se encontra familiarizado com a estrutura de categorização e com os costumes e políticas do Commons. Por favor verifique se existe um <a href="/w/index.php?title=Category:Commons_category_schemes/pt&amp;action=edit&amp;redlink=1" class="new" title="Category:Commons category schemes/pt (page does not exist)">esquema de categorização</a> ou um <a href="/wiki/Category:Commons_projects" title="Category:Commons projects">projecto</a> relacionado com o seu tópico e siga as convenções descritas nessas páginas. </p>
      <p><span id="Category_names"></span> </p> 
      <h3><span class="mw-headline" id="Nome_da_categoria">Nome da categoria</span></h3> 
      <p>Os nomes de categoria deveriam estar geralmente em inglês (consulte <a href="/wiki/Commons:Language_policy/pt" title="Commons:Language policy/pt">Commons:Política de Idiomas</a>). No entanto, existem exceções, como alguns nomes próprios, táxones biológicos e nomes para os quais o nome que não é o inglês é mais comummente utilizado no idioma inglês (ou não há evidências de utilização de uma versão em inglês). Os <a href="https://en.wikipedia.org/wiki/pt:Lista_de_alfabetos_derivados_do_latino" class="extiw" title="w:pt:Lista de alfabetos derivados do latino">alfabetos latinos</a> são utilizados ??na forma original, incluindo diacríticos e letras derivadas; alfabetos não latinos são transcritos para o <i>script</i> em latim inglês. Os carateres ingleses básicos (<a href="https://en.wikipedia.org/wiki/ISO/IEC_646" class="extiw" title="w:ISO/IEC 646">ISO/IEC 646</a>) são preferidos a variantes nacionais ou conjuntos de carateres de extensão (por exemplo, apóstrofos 'retos' em vez de 'curvos'), sempre que razoável. </p> 
      <ul>
       <li>Tipos ou grupos de objetos ou pessoas deverão geralmente ter nomes na forma de plural: <a href="/wiki/Category:Tools" title="Category:Tools">Category:Tools</a>, <a href="/wiki/Category:Artists" title="Category:Artists">Category:Artists</a>, <a href="/wiki/Category:Lakes" title="Category:Lakes">Category:Lakes</a>, <a href="/wiki/Category:Paintings" title="Category:Paintings">Category:Paintings</a>, <a href="/wiki/Category:Sculptures" title="Category:Sculptures">Category:Sculptures</a>, <a href="/wiki/Category:Popes" title="Category:Popes">Category:Popes</a> etc, e se possível em inglês.</li> 
       <li>Termas gerais ou atividades requerem normalmente um nome na forma singular: <a href="/wiki/Category:History" title="Category:History">Category:History</a>, <a href="/wiki/Category:Weather" title="Category:Weather">Category:Weather</a>, <a href="/wiki/Category:Music" title="Category:Music">Category:Music</a>, <a href="/wiki/Category:Painting" title="Category:Painting">Category:Painting</a>, <a href="/wiki/Category:Sculpture" title="Category:Sculpture">Category:Sculpture</a>, <a href="/wiki/Category:Papacy" title="Category:Papacy">Category:Papacy</a> etc, e se possível em inglês.</li> 
       <li>Um objeto individual em particular (uma pessoas específica, edifício, monumento, obra de arte, organização, evento, etc.) geralmente utiliza um forma singular (mas nem sempre). Os <a href="https://en.wikipedia.org/wiki/pt:Nome_pr%C3%B3prio" class="extiw" title="w:pt:Nome próprio">substantivos próprios</a> que não possuem uma variante em inglês estabelecida não são traduzidos <i>ad hoc</i>, mas utilizam a forma original.</li>
      </ul> 
      <p>Também existe uma <a href="/wiki/Special:MyLanguage/Commons:Naming_categories" title="Special:MyLanguage/Commons:Naming categories">proposta falhada</a> relacionada. </p> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>Categories grouping subcategories by name should generally be named "by name" rather than "by alphabet" (e.g. <a href="/wiki/Category:Ships_by_name" title="Category:Ships by name">Category:Ships by name</a>). </p> 
      </div> 
      <p>Ainda não temos internacionalização dos nomes de categorias, mas este problema deverá ser resolvido com futuras mudanças apropriadas ao software MediaWiki (ver <a href="https://bugzilla.wikimedia.org/show_bug.cgi?id=5638" class="extiw" title="bugzilla:5638">bugzilla:5638</a>). A criação de estruturas entrelaçadas de categorias em diferentes línguas só pioraria o problema. </p>
      <p>Para uma discussão geral da funcionalidade de categorização do MediaWiki, consulte o <a href="https://meta.wikimedia.org/wiki/Help:Categories" class="extiw" title="m:Help:Categories">manual sobre as categorias</a>.<b></b> </p> 
      <h3><span id="Categoriza.C3.A7.C3.A3o_de_p.C3.A1ginas"></span><span class="mw-headline" id="Categorização_de_páginas">Categorização de páginas</span></h3> 
      <p>Para adicionar uma página (seja ela uma imagem, uma galeria ou uma categoria) a uma categoria, adicione o seguinte código no fim da página: </p> 
      <pre>[[Category:Nome da categoria]]
</pre> 
      <p>Por exemplo, se você carregar um diagrama mostrando órbitas de cometas, você poderia acrescentar o seguinte à página de descrição da imagem: </p> 
      <pre>[[Category:Astronomical diagrams]]
[[Category:Comets]]
</pre> 
      <p>Isto fará com que o diagrama apareça nas categorias <a href="/wiki/Category:Astronomical_diagrams" title="Category:Astronomical diagrams">Astronomical diagrams</a> (diagramas de astronomia) e <a href="/wiki/Category:Comets" title="Category:Comets">Comets</a> (cometas). </p>
      <p>Para infomação sobre como encontrar boas categorias para os seus ficheiros e galerias, por favor leia a secção <i><a href="#Categorizando_os_seus_ficheiros">Categorizando os seus ficheiros</a></i> abaixo. </p> 
      <h3><span class="mw-headline" id="Criando_uma_nova_categoria">Criando uma nova categoria</span></h3> 
      <p>Para criar uma nova categoria: </p> 
      <ol>
       <li>Adicione primeiro a ligação à categoria numa página, tal como descrito acima (<i>[[Category:...]]</i>).</li> 
       <li>Siga então a ligação (a vermelho) para essa categoria, clicando no link – você pode agora editar a página da categoria, como qualquer outra página wiki.</li>
      </ol> 
      <p>Uma página de categoria deve conter a seguinte informação (por ordem de importância): </p> 
      <ul>
       <li>Ligações a categorias que a coloquem numa ou mais categorias de hierarquia superior;</li> 
       <li>Um texto descritivo curto que explica o que deve encontrar-se na categoria. O Inglês é a língua preferida, mas outras línguas podem ser acrescentadas (com a predefinição <tt>{{pt|...}}</tt> para descrição em Português, <tt>{{en|...}}</tt> para descrição em Inglês, etc, tal como listado em <a href="/wiki/Special:MyLanguage/Commons:Language_templates" title="Special:MyLanguage/Commons:Language templates">Predefinições para galerias</a>.</li>
      </ul> 
      <div class="mw-translate-fuzzy"> 
       <ul>
        <li>Ligações interwiki para categorias ou páginas sobre o mesmo tópico em Wikipédias (isto é, interwikis <tt>[[pt:...]]</tt> para a página na Wikipédia lusófona, <tt>[[en:...]]</tt> para a página na Wikipédia anglófona, etc.).</li>
       </ul> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>See also <a href="#How_to_categorize:_guidance_by_topic">#How to categorize: guidance by topic</a> for guidance on specific classes of category, e.g. categories about <a href="#People">#People</a>. </p> 
      </div> 
      <h3><span class="mw-headline" id="Renomeando_ou_movendo_categorias">Renomeando ou movendo categorias</span></h3> 
      <p>Por favor, veja <a href="/wiki/Commons:Rename_a_category/pt" title="Commons:Rename a category/pt">Commons:Alteração dos nomes de categorias</a>. </p> 
      <h2><span id="Para_categoriza.C3.A7.C3.A3o_mais_apropriada"></span><span class="mw-headline" id="Para_categorização_mais_apropriada">Para categorização mais apropriada</span></h2> 
      <p>As páginas (incluindo páginas de categorias) são categorizadas de acordo com o seu <b>assunto</b> e não o seu <b>conteúdo</b>, por o conteúdo não ser, em geral, uma característica permanente de uma categoria; em particular, você poderá encontrar momentaneamente conteúdo inadequado numa página de categoria. </p>
      <p>Por exemplo: Imagine que a categoria <a href="/wiki/Category:Spheres" title="Category:Spheres">Category:Spheres</a> (esferas) contém apenas imagens de bolas de cristal. Você não deve adicionar <a href="/wiki/Category:Glass" title="Category:Glass">Category:Glass</a> (vidro) a essa categoria, de acordo com o conteúdo actual, pois você pode ter esferas feitas de diversos materiais. Normalmente, uma imagem mostrando um objecto de vidro deverá estar categorizada em <a href="/wiki/Category:Glass" title="Category:Glass">Category:Glass</a> (ou numa subcategoria desta). Por isso, se a categoria <a href="/wiki/Category:Spheres" title="Category:Spheres">Category:Spheres</a> está cheia de imagens de bolas de cristal, então será melhor ideia criar uma nova categoria, tal como "Category:Glass spheres" (esferas de vidro) ou "Category:Crystal balls" (bolas de cristal), categorizada em <a href="/wiki/Category:Spheres" title="Category:Spheres">Category:Spheres</a> e <a href="/wiki/Category:Glass" title="Category:Glass">Category:Glass</a>. </p>
      <p>Em geral, os ficheiros devem encontrar-se apenas na categoria mais específica que existe para um determinado tópico. Por exemplo, ficheiros na categoria <a href="/wiki/Category:Paris" title="Category:Paris">Category:Paris</a> não devem estar também na categoria <a href="/wiki/Category:France" title="Category:France">Category:France</a> (França) <i>(ver <a href="#Sobre-categorização">sobre-categorização</a> abaixo)</i>. Se você não encontrar uma categoria que sirva o seu interesse, você pode criá-la – mas por favor leia cuidadosamente a secção <a href="#Utilizando_categorias">Utilizando categorias</a>. </p>
      <p>Isto não significa que uma imagem pertença a apenas uma categoria; significa apenas que as imagens não deverão encontrar-se em categorias redundantes ou não específicas. Por exemplo, uma imagem de um urso polar sendo salvo de um icebergue com um helicóptero deve estar em <a href="/wiki/Category:Ursus_maritimus" title="Category:Ursus maritimus">Category:Ursus maritimus</a>, <a href="/wiki/Category:Icebergs" title="Category:Icebergs">Category:Icebergs</a>, <a href="/wiki/Category:Helicopters" title="Category:Helicopters">Category:Helicopters</a> (helicópteros) e <a href="/wiki/Category:Search_and_Rescue" title="Category:Search and Rescue">Category:Search and Rescue</a> (Procura e salvamento). Não deve estar, porém, na categoria <a href="/wiki/Category:Ursidae" title="Category:Ursidae">Category:Ursidae</a> ou na <a href="/wiki/Category:Aircraft" title="Category:Aircraft">Category:Aircraft</a> (transportes aéreos). </p> 
      <h3><span id="Dicas_de_categoriza.C3.A7.C3.A3o"></span><span class="mw-headline" id="Dicas_de_categorização">Dicas de categorização</span></h3> 
      <p>As categorias (ou galerias) que você escolhe para os seus uploads devem responder a pelo menos uma das seguintes questões: </p> 
      <ul>
       <li><b>O quê? / Quem?:</b> O que ou quem é representado no ficheiro? Por exemplo, <a href="/wiki/Category:Houses" title="Category:Houses">Category:Houses</a> (casas) ou <a href="/wiki/Category:Jimmy_Wales" title="Category:Jimmy Wales">Category:Jimmy Wales</a>.</li> 
       <li><b>Onde?:</b> Onde foi feita a imagem? Isto é, obviamente, especialmente importante para imagens de lugares. Por exemplo, <a href="/wiki/Category:Saxony" title="Category:Saxony">Category:Saxony</a> (Saxónia).</li> 
       <li><b>Quando?:</b> Deve ser providenciado algum contexto histórico, se relevante. Um exemplo seria <a href="/wiki/Category:World_War_II" title="Category:World War II">Category:World War II</a> (Segunda Guerra Mundial).</li> 
       <li><b>De quem?:</b> Se o artista é bem conhecido, como <a href="/wiki/Category:Rembrandt" title="Category:Rembrandt">Category:Rembrandt</a>, faz sentido categorizar por artista; você pode também utilizar páginas do domínio <i>Creator</i> ("Criador") como moldes para conseguir esta divisão. Veja <a href="/wiki/Special:MyLanguage/Commons:Creator" title="Special:MyLanguage/Commons:Creator">Commons:Creator</a> para mais detalhes.</li> 
       <li><b>Como?:</b> Deve ser dada alguma informação sobre o tipo de ficheiro, tal como <a href="/wiki/Category:Audio" title="Category:Audio">Category:Audio</a>.</li>
      </ul> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>The above questions cover the main aspects of the image to be categorized. For some images it makes sense to use all, for other images only one or two are reasonable. In addition there are several other aspects of the images that can be used to categorize the image: </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <ul>
        <li><b>what source?:</b> information about where the image came from. For example, <a href="/wiki/Category:Images_from_the_German_Federal_Archive" title="Category:Images from the German Federal Archive">Category:Images from the German Federal Archive</a></li> 
        <li><b>what format?:</b> information about the media type if it is unusual. For example, <a href="/wiki/Category:Audio" title="Category:Audio">Category:Audio</a> or <a href="/wiki/Category:Animated_GIF" title="Category:Animated GIF">Category:Animated GIF</a>, <a href="/wiki/Category:SVG" class="mw-disambig" title="Category:SVG">Category:SVG</a></li> 
        <li><b>what software?:</b> information about software used to create the image. For example, <a href="/wiki/Category:Created_with_Hugin" title="Category:Created with Hugin">Category:Created with Hugin</a></li> 
        <li><b>what camera?:</b> information about the camera. For example, <a href="/wiki/Category:Taken_with_Nikon_D80" title="Category:Taken with Nikon D80">Category:Taken with Nikon D80</a></li>
       </ul> 
       <p>This last set is useful and important but should always be done in addition of the main set of criteria. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>Categorization in Wikimedia Commons is more detailed and deep than categorization in Wikipedia projects. Compared to them, Commons has more categories for individual subjects – places, people, organizations, events, terms, etc. Almost every article on a Wikipedia can have a corresponding category on Commons. However, even if there exist more images of an ordinary person or incidental event, it is practical to group them into a special category and categorize that category instead of categorizing all similar images individually to an identical set of parent categories. </p> 
      </div> 
      <h3><span class="mw-headline" id="Encontrar_uma_categoria_adequada">Encontrar uma categoria adequada</span></h3> 
      <p>Para encontrar categorias adequadas para os seus uploads, você pode navegar a estrutura de categorias, começando numa categoria genérica. Estreite a sua busca a subcategorias até encontrar a categoria mais específica adequada ao seu ficheiro. Você pode navegar pela estrutura de categorias seguindo ligações para subcategorias, ou expandindo a árvore de categorias clicando nos símbolos + que aparecem junto dos nomes de subcategorias. A secção sobre <i><a href="#Estrutura_de_categorias">estrutura de categorias</a></i>, mais abaixo, fornece um ponto de partida e a secção <i><a href="#Como_categorizar:_guia_por_tópico">como categorizar: guia por tópico</a></i> faz a cobertura de mais tópicos. Você pode também tentar a ferramenta <b><a rel="nofollow" class="external text" href="http://tools.wikimedia.de/~daniel/WikiSense/CommonSense.php">CommonSense</a></b>, feita para ajudar na categorização baseando-se nalgumas palavras-chave. </p> 
      <h3><span id="Sobre-categoriza.C3.A7.C3.A3o"></span><span class="mw-headline" id="Sobre-categorização">Sobre-categorização</span></h3> 
      <div class="mw-translate-fuzzy"> 
       <p>A sobre-categorização acontece quando uma imagem é colocada em diversas categorias que pertencem à mesma árvore de categorias. A regra geral é <i>colocar sempre uma imagem nas categorias mais específicas e não nos níveis acima destas.</i> </p> 
      </div> 
      <p>Um exemplo: vamos assumir que esferas amarelas são esferas de cor amarela. Podemos pensar numa categoria <i>Category:Yellow spheres</i> (esferas amarelas) e <a href="/wiki/Category:Spheres" title="Category:Spheres">Category:Spheres</a> (esferas). A imagem a ser categorizada mostra berlindes amarelos. Nós categorizamos o ficheiro em <b>Category:Yellow spheres</b>. Agora, se categorizarmos a imagem em <a href="/wiki/Category:Spheres" title="Category:Spheres">Category:Spheres</a>, isto é <i>sobre-categorização</i>, porque já sabemos que os berlindes amarelos são esferas. </p>
      <p>Isto aplica-se à maioria das imagens; tal como mencionado acima, ficheiros na categoria <a href="/wiki/Category:Paris" title="Category:Paris">Category:Paris</a> não devem estar também na categoria <a href="/wiki/Category:France" title="Category:France">Category:France</a> (França), ficheiros em <a href="/wiki/Category:Albert_Einstein" title="Category:Albert Einstein">Category:Albert Einstein</a> não devem estar em <a href="/wiki/Category:Physicists_from_Germany" title="Category:Physicists from Germany">Category:Physicists from Germany</a> (Físicos da Alemanha), etc. </p> 
      <h4><span id="Porque_a_sobre-categoriza.C3.A7.C3.A3o_.C3.A9_um_problema"></span><span class="mw-headline" id="Porque_a_sobre-categorização_é_um_problema">Porque a sobre-categorização é um problema</span></h4> 
      <p>É muitas vezes assumido que quanto mais categorias uma imagem tiver, mais simples será de a encontrar. Outro exemplo: usando esta lógica, toda a imagem mostrando um homem deveria estar na categoria <a href="/wiki/Category:Men" title="Category:Men">Category:Men</a> (homens), porque mesmo que não se saiba mais nada sobre essa pessoa a não ser que é um homem, ainda assim será possível encontrá-la. O resultado é o preenchimento da categoria superior, tornando-se necessário percorrer centenas (neste caso até, provavelmente milhares) de imagens até encontrar a que você deseja. Provavelmente você não encontrará o que quer; além disso, aqueles que procuram uma imagem genérica de um homem, para ilustrar artigos como <a href="https://pt.wikipedia.org/wiki/Homem" class="extiw" title="pt:Homem">pt:Homem</a>, encontrar-se-ão perdidos no meio de imagens de estrelas de cinema, cientistas e políticos. </p>
      <p>Em níveis inferiores, o problema é menor, já que o número de imagens será também menor – podem ainda ser centenas de ficheiros, no entanto. Sobra no entanto ainda um problema que pode ser ilustrado com um exemplo: Sabendo que Einstein é um físico, poderemos procurá-lo na categoria <a href="/wiki/Category:Physicists" title="Category:Physicists">Category:Physicists</a>, onde encontramos uma imagem entre centenas. Como havia aí uma imagem, contentamo-nos com isso e não procuramos mais, em vez de verificarmos a categoria <a href="/wiki/Category:Physicists_from_Germany" title="Category:Physicists from Germany">Category:Physicists from Germany</a>, que levaria a encontrar a <a href="/wiki/Category:Albert_Einstein" title="Category:Albert Einstein">Category:Albert Einstein</a>, onde potencialmente existirá uma imagem melhor. Assim, a sobre-categorização provocou dois problemas: a categoria de topo está cheia e os usuários não procuram uma categoria mais relevante, por terem chegado a uma que contém uma imagem de interesse. </p> 
      <h4><span id="A_categoriza.C3.A7.C3.A3o_inadequada_de_categorias_causa_sobre-categoriza.C3.A7.C3.A3o"></span><span class="mw-headline" id="A_categorização_inadequada_de_categorias_causa_sobre-categorização">A categorização inadequada de categorias causa sobre-categorização</span></h4> 
      <p>Por muito estranho que possa soar, um défice de categorização pode levar a uma sobre-categorização. Isto acontece quando uma categoria não se encontra devidamente categorizada, levando os usuários a sobre-categorizar uma imagem para a incluir nas categorias relevantes. Por exemplo: a categoria <a href="/wiki/Category:Eiv%C3%B8r_P%C3%A1lsd%C3%B3ttir" title="Category:Eivør Pálsdóttir">Category:Eivør Pálsdóttir</a> encontrava-se categorizada apenas em <a href="/w/index.php?title=Category:People_by_alphabet&amp;action=edit&amp;redlink=1" class="new" title="Category:People by alphabet (page does not exist)">Category:People by alphabet</a> (Pessoas por alfabeto). Por isso, se adicionarmos à imagem a categoria <a href="/wiki/Category:People_of_the_Faroe_Islands" title="Category:People of the Faroe Islands">Category:People of the Faroe Islands</a> (Pessoas das Ilhas Faroe) e <a href="/wiki/Category:Vocalists" title="Category:Vocalists">Category:Vocalists</a> (Vocalistas) por sabermos quem ela é, estaremos a adicionar inutilmente a imagem a categorias de topo, causando sobre-categorização. A <a href="/wiki/Category:Eiv%C3%B8r_P%C3%A1lsd%C3%B3ttir" title="Category:Eivør Pálsdóttir">Category:Eivør Pálsdóttir</a> deveria estar ela própria nestas duas categorias. </p>
      <p>Um problema relacionado: a categoria <a href="/wiki/Category:Notting_Hill" title="Category:Notting Hill">Category:Notting Hill</a> encontrou-se por mais de um mês na <a href="/wiki/Category:London" title="Category:London">Category:London</a> (Londres). Ao adicionar uma imagem, seria tentador adicionar à categoria <a href="/w/index.php?title=Category:Kensington_and_Chelsea&amp;action=edit&amp;redlink=1" class="new" title="Category:Kensington and Chelsea (page does not exist)">Category:Kensington and Chelsea</a>, que é onde se encontra Notting Hill. Em vez disto, cada imagem deve estar nas categorias mais específicas possível e estas também devem estar nas categorias mais específicas possível. </p>
      <p>Quando encontrar esta situação, por favor recategorize as categorias de forma adequada, se souber fazê-lo. Isso ajuda não só a evitar sobre-categorização, como também facilita a navegação pela árvore de categorias. </p> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <h4><span class="mw-headline" id="Exception_for_images_with_more_categorized_subjects"><span id="OvercatException"></span> Exception for images with more categorized subjects</span></h4> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>A file that depicts only one relevant subject should not be over-categorized. Where a file depicts additional relevant subjects, and the additional subjects do not have their own subcategories, consideration can be given to temporarily categorizing the image in both the subcategory and the parent category. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>For example, this situation might arise in the case of a photograph of three politicians, one of whom is <a href="/wiki/Category:Angela_Merkel" title="Category:Angela Merkel">Angela Merkel</a> (who has her own Commons category), with two other politicians who do not yet have their own categories. While the image would undoubtedly be categorized in <a href="/wiki/Category:Angela_Merkel" title="Category:Angela Merkel">Category:Angela Merkel</a> or one of its subcategories, it would typically be considered to be over-categorization to also include it in <a href="/wiki/Category:Politicians_of_Germany" title="Category:Politicians of Germany">Category:Politicians of Germany</a>. Users would, however, be unlikely to search for the two other politicians in the Merkel category. Ideally, we would create specific subcategories for the two other politicians (where warranted), or find other relevant subcategories (e.g. <a href="/wiki/Category:Politicians_of_Bavaria" title="Category:Politicians of Bavaria">Category:Politicians of Bavaria</a> or <a href="/wiki/Category:Members_of_the_FDP" title="Category:Members of the FDP">Category:Members of the FDP</a>, etc.), that would enable us to avoid over-categorization. In some circumstances, however, we may need to temporarily categorize the image in <a href="/wiki/Category:Politicians_of_Germany" title="Category:Politicians of Germany">Category:Politicians of Germany</a> where other appropriate subcategories do not yet exist. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>Countries may be categorized as part of multiple overlapping categories. For example, <a href="/wiki/Category:India" title="Category:India">Category:India</a> is in <a href="/wiki/Category:Countries_of_South_Asia" title="Category:Countries of South Asia">Category:Countries of South Asia</a> as well as <a href="/wiki/Category:Countries_of_Asia" title="Category:Countries of Asia">Category:Countries of Asia</a>. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>Also user categories are exempted of over-categorization as those are not visible to most viewers, and project users include them for many different purposes like sorting, stats, filling values for userboxes, etc. </p> 
      </div> 
      <h2><span id="Como_categorizar:_guia_por_t.C3.B3pico"></span><span class="mw-headline" id="Como_categorizar:_guia_por_tópico">Como categorizar: guia por tópico</span></h2> 
      <p>Para algumas categorias, existem guias específicos de como distribuir melhor o conteúdo dentro de dentro dessas categorias. Alguns desses guias encontram-se nesta secção. Existe também, por vezes, instruções no topo da página da categoria, no domínio Category. Assim, por exemplo, algumas instruções para categorizar pessoas encontram-se no topo de <a href="/wiki/Category:People" title="Category:People">Category:People</a> e algumas na secção <b><a href="#Pessoas">Pessoas</a></b> abaixo. </p> 
      <h3><span class="mw-headline" id="Pessoas">Pessoas</span></h3> 
      <p>O conteúdo mostrando pessoas pode ser colocado em categorias e/ou galerias que as descrevam, tais como <a href="/wiki/Category:Economists_from_the_United_States" title="Category:Economists from the United States">Category:Economists from the United States</a> (Economistas dos Estados Unidos). Comece por explorar <a href="/wiki/Category:People" title="Category:People">Category:People</a>. </p>
      <p>Poderá não existir muito conteúdo de algumas pessoas notáveis, ou alguns nomes não muito conhecidos, fazendo com que haja a necessidade de criar uma categoria ou uma galeria usando o seu nome, tal como <a href="/wiki/Category:Albert_Einstein" title="Category:Albert Einstein">Category:Albert Einstein</a> (este é um caso em que você poderá criar frequentemente categorias, isto é, de cada vez que carrega uma imagem de uma pessoa notável). Utilize nomes comuns e ortografia em Inglês para a categoria ou galeria; é uma pena sermos forçados a utilizar o Inglês para todos os nomes do mundo, mas, como já notado acima, a utilização de categorias em Inglês no Commons é feita por uma questão de limitação técnica. </p>
      <p>Assim que exista uma categoria ou galeria com o nome de uma pessoa, então todas as categorias que descrevam essa pessoa são aplicáveis a todas as imagens dessa pessoa. Tais categorias devem por isso ser aplicadas à categoria ou galeria da pessoa e não a imagens individuais dessa pessoa. cada imagem deverá ter uma ligação à categoria ou galeria de cada pessoa representada e possivelmente categorias que distingam uma imagem de determinada pessoa de outras imagens semelhantes. </p>
      <p>Todas as categorias e galerias com o nome de uma pessoa devem estar incluídas na categoria <a href="/w/index.php?title=Category:People_by_alphabet&amp;action=edit&amp;redlink=1" class="new" title="Category:People by alphabet (page does not exist)">category:People by alphabet</a> (pessoas por alfabeto) e distribuídas de acordo com o seu nome de família: [[Category:People by alphabet|Nome de família, nome próprio]]. <b>Não</b> categorize, no entanto, imagens dentro desta categoria! Existem tantas imagens de pessoas notáveis que a sua inclusão directa iria sobrecarregar a categoria. A limitação da categoria <i>Category:People by alphabet</i> a páginas de categorias e galerias reduz significativamente o número de entradas. </p>
      <p>Please see <a href="/wiki/Special:MyLanguage/Commons:Category_scheme_People" title="Special:MyLanguage/Commons:Category scheme People">Commons:Category scheme People</a> for details on how to name and organize these categories. </p> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <h3><span id="Landscapes.2C_outdoor_views"></span><span class="mw-headline" id="Landscapes,_outdoor_views">Landscapes, outdoor views</span></h3> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>Content depicting a given subject from a common vantage point are grouped in <i>Views of Subject from Viewpoint</i> categories such as <a href="/wiki/Category:Views_of_Cathedral_of_Seville_from_the_Giralda" title="Category:Views of Cathedral of Seville from the Giralda">Views of Cathedral of Seville from the Giralda</a>. Such categories should be subcategories of both the subject's category (<a href="/wiki/Category:Cathedral_of_Seville" title="Category:Cathedral of Seville">Cathedral of Seville</a> in this example) and the viewpoint's category (<a href="/wiki/Category:Giralda" title="Category:Giralda">Giralda</a> in this example). </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>In this example, the <a href="/wiki/Category:Views_of_Cathedral_of_Seville_from_the_Giralda" title="Category:Views of Cathedral of Seville from the Giralda">Views of Cathedral of Seville from the Giralda</a> category is not placed directly in the subject and viewpoint categories, but in <a href="/wiki/Category:Views_of_the_Cathedral_of_Seville" title="Category:Views of the Cathedral of Seville">Views of the Cathedral of Seville</a> and <a href="/wiki/Category:Views_from_Giralda" title="Category:Views from Giralda">Views from Giralda</a>. Such intermediate categories are often necessary to create structure and avoid <a href="#over-categorization">over-categorization</a>, particularly for views of a city from a vantage point located within the city. For example, <a href="/wiki/Category:Views_of_Rome_from_the_Pincio" title="Category:Views of Rome from the Pincio">Views of Rome from the Pincio</a> needs the intermediate category <a href="/wiki/Category:Views_of_Rome" title="Category:Views of Rome">Views of Rome</a> to avoid placing it directly in <a href="/wiki/Category:Rome" title="Category:Rome">Rome</a>, which would constitute over-categorization. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <h3><span class="mw-headline" id="Texts">Texts</span></h3> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>Texts, such as scans of books, should normally have a category for each version of the scan and each edition of the text. Thus a book published in three separate editions would have a parent category for the book, three subcategories for each text, and further subcategories for the text as a jpeg, a DjVu, etc., assuming each version had actually been uploaded. (Categories would not be created for editions not held on Commons.) This is particularly important for files in formats other than DjVu and PDF, where the category is the only practical means of keeping the scans together; see eg. <a href="/wiki/Category:The_Chronicles_of_England,_Scotland_and_Ireland,_Holinshed,_1587" title="Category:The Chronicles of England, Scotland and Ireland, Holinshed, 1587">Category:The Chronicles of England, Scotland and Ireland, Holinshed, 1587</a> which contains 2857 jpeg images of page scans. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <h3><span class="mw-headline" id="GLAMs">GLAMs</span></h3> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>For categorization issues related to mass content donations from GLAMs (Galleries, Libraries, Archives &amp; Museums), please see <a href="/wiki/Special:MyLanguage/Commons:Guide_to_batch_uploading#Categories" title="Special:MyLanguage/Commons:Guide to batch uploading">Commons:Guide to batch uploading#Categories</a>. </p>

      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <h2><span class="mw-headline" id="Categorization_workflow">Categorization workflow</span></h2> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>Currently, a bot checks if newly uploaded files are categorized in topical categories and attempts to categorize files that are not. Before 17 June 2015, <a href="/wiki/User:CategorizationBot" title="User:CategorizationBot">CategorizationBot</a> was responsible for this job. As of June 2019, <a href="/wiki/User:SteinsplitterBot" title="User:SteinsplitterBot">SteinsplitterBot</a> occasionally checks for uncategorized files. The workflow is the following: </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <ol>
        <li>User uploads a new file and adds categories (or not).</li> 
        <li>A bot checks if the file is categorized. 
         <ul>
          <li>File is already categorized =&gt; ok</li> 
          <li>File is not categorized =&gt; the bot tags the file for <a href="/wiki/Category:Media_needing_categories" title="Category:Media needing categories">Category:Media needing categories</a>. Today's files are in <a href="/wiki/Category:Media_needing_categories_as_of_6_June_2021" title="Category:Media needing categories as of 6 June 2021">Category:Media needing categories as of 6 June 2021</a></li>
         </ul></li> 
        <li>Users categorize files further (e.g. category diffusion below)</li>
       </ol> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p><i>See also:</i> <a href="/wiki/User:CategorizationBot#Process" title="User:CategorizationBot">User:CategorizationBot#Process</a>, <a href="/wiki/User:Multichill/Categorization_stats" title="User:Multichill/Categorization stats">categorization statistics</a> </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>Other, if manual, categorization workflows are possible&nbsp;: </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <ul>
        <li>Category filling: Use appropriate keywords in the search engine to find the files that should be in a given category, and put them there.</li> 
        <li>Category diffusing: Go to <a href="/wiki/Category:Categories_requiring_diffusion" title="Category:Categories requiring diffusion">Category:Categories requiring diffusion</a>, select a crowded category, create appropriate subcategories if needed, and move the files to the subcategories. Gadgets like Cat-a-lot and Hotcats can help.</li>
       </ul> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <h2><span id="Categories_marked_with_.22HIDDENCAT.22"></span><span class="mw-headline" id="Categories_marked_with_&quot;HIDDENCAT&quot;">Categories marked with "HIDDENCAT"</span></h2> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <div class="shortcut2" style="">
        <style data-mw-deduplicate="TemplateStyles:r433650343">.mw-parser-output .shortcut2{float:right;margin:0.4em;border:1px solid #ccc;background-color:white;padding:0 1em;text-align:center;font-size:x-small;border-radius:4px}</style><a href="/wiki/Commons:Shortcuts" title="Commons:Shortcuts">Shortcut</a>
        <br><a href="/wiki/Commons:HIDDENCAT" class="mw-redirect" title="Commons:HIDDENCAT">COM:HIDDENCAT</a>
       </div> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>Many non-topical categories are marked with <code>__HIDDENCAT__</code> or {{<a href="/wiki/Template:Hiddencat" title="Template:Hiddencat">Hiddencat</a>}} on the category page. For example, see <a class="external text" href="https://commons.wikimedia.org/w/index.php?title=Category:PD_NASA&amp;action=edit">Category:PD NASA in edit mode</a>. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>While categories are generally visible on every page, categories marked <code>__HIDDENCAT__</code> are only visible: </p> 
       <ul>
        <li>on the edit screen: at the end of the screen, below the edit box</li> 
        <li>on category pages: 
         <ul>
          <li>on subcategories to the hidden category: in the normal location, but on a separate line with a smaller typeface and the label "Hidden category."</li> 
          <li>on parent categories: in the same way as other categories</li>
         </ul></li> 
        <li>on file description pages and gallery pages: for logged-in users who have selected to "Show hidden categories" in their <a href="/wiki/Special:Preferences#mw-prefsection-rendering" title="Special:Preferences">appearance preferences</a>. This is activated for all newly registered users.</li>
       </ul> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>This feature is generally used for template-based categories, such as license tag based categories. For example, placing <a href="/wiki/Template:PD-old-100" title="Template:PD-old-100"><span style="font-family:monospace,monospace;">{{PD-old-100}}</span></a> on a file description page adds the file to <a href="/wiki/Category:Author_died_more_than_100_years_ago_public_domain_images" title="Category:Author died more than 100 years ago public domain images">Category:Author died more than 100 years ago public domain images</a>, which is marked with <code>__HIDDENCAT__</code>. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>For more details, see the <a href="https://meta.wikimedia.org/wiki/Help:Category#Hidden_categories" class="extiw" title="m:Help:Category">help section on hidden categories</a> for Mediawiki (the software that Commons uses). </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <h2><span class="mw-headline" id="Templates_for_categories">Templates for categories</span></h2> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>Some templates are designed for use on category pages - see <a href="/wiki/Category:Category_namespace_templates" title="Category:Category namespace templates">Category:Category namespace templates</a>. If the category is linked to a Wikidata entry, then you can use: </p> 
       <ul>
        <li><a href="/wiki/Template:Wikidata_Infobox" title="Template:Wikidata Infobox"><span style="font-family:monospace,monospace;">{{Wikidata Infobox}}</span></a></li>
       </ul> 
       <p>which displays a summary of the topic's information that is available on Wikidata, and also auto-adds birth/death/name/monument ID categories. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>Some of the more commonly used ones are <a href="/wiki/Category:Category_header_templates" title="Category:Category header templates">Category:Category header templates</a> such as: </p> 
       <ul>

       <h2><span class="mw-headline" id="Tools">Tools</span></h2> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <style data-mw-deduplicate="TemplateStyles:r428405392">.mw-parser-output .hatnote{font-style:italic}.mw-parser-output .hatnote i,.mw-parser-output .hatnote.noitalic{font-style:normal}.mw-parser-output div.hatnote{padding-left:1.6em;margin-bottom:0.5em}.mw-parser-output div.hatnote+div.hatnote{margin-top:-0.5em}@media print{.mw-parser-output .hatnote{display:none}}</style> 
       <div role="note" class="hatnote navigation-not-searchable">
        Further information: <a href="/wiki/Special:MyLanguage/Commons:Tools" title="Special:MyLanguage/Commons:Tools">Commons:Tools</a> 
       </div> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <ul>
        <li>Gadgets enabled through the <a href="/wiki/Special:Preferences#mw-prefsection-gadgets" title="Special:Preferences">user preferences</a> 
         <ul>
          <li><i>Cat-a-lot</i>: A tool that helps with moving multiple files between categories or adding categories to search results. <small>[<a href="/wiki/Help:Gadget-Cat-a-lot" title="Help:Gadget-Cat-a-lot">documentation</a>&nbsp;/&nbsp;<a href="/wiki/MediaWiki_talk:Gadget-Cat-a-lot.js" title="MediaWiki talk:Gadget-Cat-a-lot.js">talk</a>]</small>&nbsp;</li> 
          <li><i>HotCat</i>:&nbsp;<sup><abbr title="default gadget – enabled by default">d</abbr></sup> Easily add / remove / change a category on a page, with name suggestions. <small>[<a href="/wiki/Help:Gadget-HotCat" title="Help:Gadget-HotCat">documentation</a>&nbsp;/&nbsp;<a href="/wiki/File:HotCat.png" title="File:HotCat.png">example</a>&nbsp;/&nbsp;<a href="/wiki/MediaWiki_talk:Gadget-HotCat.js" title="MediaWiki talk:Gadget-HotCat.js">talk</a>]</small>&nbsp;</li> 
          <li><i>Gallery Details</i>: Adds a link in the toolbox to display galleries and categories (and <a href="/wiki/Special:NewFiles" title="Special:NewFiles">Newimages</a> and <a href="/wiki/Special:Search" title="Special:Search">Search</a> result pages) with extensive details from file description pages and links to easily mark an image without source, etc. If <i>Pretty log</i> is activated, it also works on <a href="/wiki/Special:Log" title="Special:Log">Log</a> pages. <small>[<a href="/wiki/Help:Gadget-GalleryDetails" title="Help:Gadget-GalleryDetails">documentation</a>&nbsp;/&nbsp;<a href="/wiki/MediaWiki_talk:Gadget-GalleryDetails.js" title="MediaWiki talk:Gadget-GalleryDetails.js">talk</a>]</small>&nbsp;</li> 
          <li>Place categories above content, but below image on file description pages. &nbsp;Modifies the placement of categories on the user interface.</li> 
          <li>Add a link to category pages to search for the category name with the option "-incategory". This excludes files already in the category (doesn't work if the category was added by a template). <small>[<a href="/wiki/MediaWiki_talk:Gadget-Searchnotincat.js" title="MediaWiki talk:Gadget-Searchnotincat.js">talk</a>]</small>&nbsp;</li>
         </ul></li>
       </ul> 
      </div>
