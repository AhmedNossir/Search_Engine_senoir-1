       <p>GitHub CLI brings GitHub to your terminal. It reduces context switching, helps you focus, and enables you to more easily script and create your own workflows. Earlier this year, we <a href="https://github.blog/2020-02-12-supercharge-your-command-line-experience-github-cli-is-now-in-beta/">announced the beta of GitHub CLI</a>. Since we released the beta, users have created over 250,000 pull requests, performed over 350,000 merges, and created over 20,000 issues with GitHub CLI. Weve received so much thoughtful feedback, and today GitHub CLI is out of beta and available to download on <a href="https://cli.github.com/">Windows, macOS, and Linux</a>.</p> 
       <p>With GitHub CLI 1.0, you can:</p> 
       <ul> 
        <li>Run your entire GitHub workflow from the terminal, from issues through releases</li> 
        <li>Call the GitHub API to script nearly any action, and set a custom alias for any command</li> 
        <li>Connect to GitHub Enterprise Server in addition to GitHub.com</li> 
       </ul> 
       <h2 id="from-issue-to-release">From issue to release<a href="#from-issue-to-release" class="heading-link" aria-label="From issue to release" data-anchorjs-icon="#" style="padding-left: 0.375em;"></a></h2> 
       <p>Use GitHub CLI for your entire GitHub workflow.</p> 
       <ul> 
        <li>Clone the repository you want to work with using <code>gh repo clone owner/repo</code>.</li> 
        <li>Find the next thing you need to work on with <code>gh issue status</code> or <code>gh issue list --assignee billygriffin</code>.</li> 
       </ul> 
       <p><img loading="lazy" class="aligncenter size-full wp-image-54361" src="https://github.blog/wp-content/uploads/2020/09/gh-issue-list-on-github-command-line-terminal-app.png?resize=1024%2C273" alt="Screenshot of GitHub CLI running 'gh issue list'" width="1024" height="273" srcset="https://github.blog/wp-content/uploads/2020/09/gh-issue-list-on-github-command-line-terminal-app.png?resize=1024%2C273?w=1412 1412w, https://github.blog/wp-content/uploads/2020/09/gh-issue-list-on-github-command-line-terminal-app.png?resize=1024%2C273?w=300 300w, https://github.blog/wp-content/uploads/2020/09/gh-issue-list-on-github-command-line-terminal-app.png?resize=1024%2C273?w=768 768w, https://github.blog/wp-content/uploads/2020/09/gh-issue-list-on-github-command-line-terminal-app.png?resize=1024%2C273?w=1024 1024w" sizes="(max-width: 1000px) 100vw, 1000px" data-recalc-dims="1"></p> 
       <ul> 
        <li>When youve finished adding that feature or fixing that bug, use <code>gh pr create</code> to create your pull request on GitHub.</li> 
       </ul> 
       <p><img loading="lazy" class="aligncenter size-full wp-image-54360" src="https://github.blog/wp-content/uploads/2020/09/gh-pr-create-on-github-command-line-terminal-app.png?resize=1024%2C360" alt="Screenshot of GitHub CLI running 'gh pr create'" width="1024" height="360" srcset="https://github.blog/wp-content/uploads/2020/09/gh-pr-create-on-github-command-line-terminal-app.png?resize=1024%2C360?w=1412 1412w, https://github.blog/wp-content/uploads/2020/09/gh-pr-create-on-github-command-line-terminal-app.png?resize=1024%2C360?w=300 300w, https://github.blog/wp-content/uploads/2020/09/gh-pr-create-on-github-command-line-terminal-app.png?resize=1024%2C360?w=768 768w, https://github.blog/wp-content/uploads/2020/09/gh-pr-create-on-github-command-line-terminal-app.png?resize=1024%2C360?w=1024 1024w" sizes="(max-width: 1000px) 100vw, 1000px" data-recalc-dims="1"></p> 
       <ul> 
        <li>And your teammate can check out your pull request using <code>gh pr checkout 1337</code>, view the diff with <code>gh pr diff</code>, and even provide a lightweight review using <code>gh pr review</code>.</li> 
       </ul> 
       <p><img loading="lazy" class="aligncenter size-full wp-image-54359" src="https://github.blog/wp-content/uploads/2020/09/gh-pr-checkout-on-github-command-line-terminal-app.png?resize=1024%2C476" alt="Screenshot of GitHub CLI running 'gh pr checkout 23' and 'gh pr diff'" width="1024" height="476" srcset="https://github.blog/wp-content/uploads/2020/09/gh-pr-checkout-on-github-command-line-terminal-app.png?resize=1024%2C476?w=1412 1412w, https://github.blog/wp-content/uploads/2020/09/gh-pr-checkout-on-github-command-line-terminal-app.png?resize=1024%2C476?w=300 300w, https://github.blog/wp-content/uploads/2020/09/gh-pr-checkout-on-github-command-line-terminal-app.png?resize=1024%2C476?w=768 768w, https://github.blog/wp-content/uploads/2020/09/gh-pr-checkout-on-github-command-line-terminal-app.png?resize=1024%2C476?w=1024 1024w" sizes="(max-width: 1000px) 100vw, 1000px" data-recalc-dims="1"></p> 
       <ul> 
        <li>After the pull request is approved, you can make sure all your tests are passing with <code>gh pr checks</code>, and then go ahead and merge it right from your terminal with <code>gh pr merge</code>. GitHub CLI will even offer to delete your branch locally and on GitHub.com after the merge.</li> 
       </ul> 
       <p><img loading="lazy" class="aligncenter size-full wp-image-54358" src="https://github.blog/wp-content/uploads/2020/09/gh-pr-checks-on-github-command-line-terminal-app.png?resize=1024%2C534" alt="Screenshot of GitHub CLI running 'gh pr checks' and 'gh pr merge'" width="1024" height="534" srcset="https://github.blog/wp-content/uploads/2020/09/gh-pr-checks-on-github-command-line-terminal-app.png?resize=1024%2C534?w=1412 1412w, https://github.blog/wp-content/uploads/2020/09/gh-pr-checks-on-github-command-line-terminal-app.png?resize=1024%2C534?w=300 300w, https://github.blog/wp-content/uploads/2020/09/gh-pr-checks-on-github-command-line-terminal-app.png?resize=1024%2C534?w=768 768w, https://github.blog/wp-content/uploads/2020/09/gh-pr-checks-on-github-command-line-terminal-app.png?resize=1024%2C534?w=1024 1024w" sizes="(max-width: 1000px) 100vw, 1000px" data-recalc-dims="1"></p> 
       <ul> 
        <li>And when youre ready to cut your next release, just use <code>gh release create [tag name]</code> and make your creation available to the world without ever leaving your command line!</li> 
       </ul> 
       <h2 id="make-github-cli-your-own-with-aliases-and-gh-api">Make GitHub CLI your own with aliases and <code>gh api</code><a href="#make-github-cli-your-own-with-aliases-and-gh-api" class="heading-link" aria-label="Make GitHub CLI your own with aliases and <code>gh api</code>" data-anchorjs-icon="#" style="padding-left: 0.375em;"></a></h2> 
       <p>GitHub CLI now allows you to create aliases for any command using <code>gh alias set</code>. And with the powerful <code>gh api</code> allowing you to access the GitHub API directly, theres no limit to what you can do with <code>gh</code>. Commands are also easily composable.</p> 
       <p>Heres a tip: to share your aliases with a gist, you can use <code>gh alias list | gh gist create</code>. If you think of a cool way to work with aliases using GitHub CLI, tweet at us @github!</p> 
       <h2 id="github-cli-is-available-for-github-enterprise-server">GitHub CLI is available for GitHub Enterprise Server<a href="#github-cli-is-available-for-github-enterprise-server" class="heading-link" aria-label="GitHub CLI is available for GitHub Enterprise Server" data-anchorjs-icon="#" style="padding-left: 0.375em;"></a></h2> 
       <p>Finally, you can use GitHub CLI with repositories hosted on GitHub Enterprise Server 2.20+. This has been the most frequent request since we announced the beta, and were excited that more and more people using GHES will be able to also use GitHub CLI.</p>
