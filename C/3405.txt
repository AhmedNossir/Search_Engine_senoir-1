      <h2><span class="mw-headline" id="Overview">Overview</span></h2> 
      <p>The <b>What links here</b> facility lists the pages on the same site which link to a given page, either the pages in all namespaces, or in a specified one. To see this information, choose the <i>What links here</i> link while looking at any page. The pages linking to and/or embedding the given page are listed. The list is sorted by page id, i.e., by date of creation of the page. </p>
      <p>This info comes from the <a href="/wiki/Special:MyLanguage/Manual:pagelinks_table" title="Special:MyLanguage/Manual:pagelinks table">pagelinks</a><span style="display:none"><a href="/wiki/Manual:Pagelinks_table" title="Manual:Pagelinks table"> </a></span> table and the <a href="/wiki/Special:MyLanguage/Manual:templatelinks_table" title="Special:MyLanguage/Manual:templatelinks table">templatelinks</a><span style="display:none"><a href="/wiki/Manual:Templatelinks_table" title="Manual:Templatelinks table"> </a></span> table. The pages embedding the given page are marked with the content of <a href="/wiki/MediaWiki:Istemplate" title="MediaWiki:Istemplate">MediaWiki:Istemplate</a> ("transclusion" or "inclusion"); for these pages it is not shown whether they also <i>link</i> to the given page. This can be seen with <a href="https://meta.wikimedia.org/wiki/Special:MyLanguage/Query#backlinks" class="extiw" title="m:Special:MyLanguage/Query">m:Query#backlinks</a>. </p>
      <p>The list of links to an article is useful in a number of ways: </p> 
      <ul>
       <li>It gives a very rough indication of how popular a page is. Pages with many links are likely to be viewed often and should therefore be of the very best quality. Pages with few or no links may not be very popular.</li> 
       <li>Where the subject material of an article is unclear, the list of articles linking to it might provide useful context. For instance when presented with a stub about <i>John Smith</i> that gives only his date of birth and death, viewing the list of links to the article might reveal that he won a gold medal in the Olympics.</li> 
       <li>Facilitates proper <a href="https://en.wikipedia.org/wiki/Wikipedia:Disambiguation" class="extiw" title="w:Wikipedia:Disambiguation">disambiguation</a>.</li>
      </ul> 
      <p>This facility works also for a page that does not exist (there may be links to it, which makes it extra useful to create it). The <i>What links here</i> button is also on the edit page on which one arrives when following a broken link. Thus, for example, if film articles are linked to the actors, one can find the films an actor has played in even if there is no article about the actor. </p>
      <p>To invoke a <i>What links here</i> list directly, use e.g. </p> 
      <pre>[[w:Special:Whatlinkshere/John Smith|w:Special:Whatlinkshere/John Smith]]
</pre> 
      <p>resulting in <a href="https://en.wikipedia.org/wiki/Special:Whatlinkshere/John_Smith" class="extiw" title="w:Special:Whatlinkshere/John Smith">w:Special:Whatlinkshere/John_Smith</a>. </p> 
      <h2><span class="mw-headline" id="Limitations_and_workarounds">Limitations and workarounds</span></h2> 
      <p>Not listed are: </p> 
      <ul>
       <li><a href="/wiki/Special:MyLanguage/Help:Link#Subpage_feature" title="Special:MyLanguage/Help:Link">subpages</a> that just have an <i>automatic</i> link to a page</li> 
       <li><a href="/wiki/Special:MyLanguage/Help:Edit_summary" title="Special:MyLanguage/Help:Edit summary">links in edit summaries</a></li>
      </ul> 
      <p>Also listed are: </p> 
      <ul>
       <li>pages with #ifexist applied to the given page (see <span><a href="https://phabricator.wikimedia.org/T14019" class="extiw" title="phabricator:T14019">bug&nbsp;12019</a></span>)</li>
      </ul> 
      <p>In the case of links to sections or other anchors, the precise target is not shown. Also, "What links here" cannot list the backlinks of a specific section/anchor only.<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">[1]</a></sup> These features are provided by <a href="/wiki/Special:MyLanguage/Help:Linksearch" title="Special:MyLanguage/Help:Linksearch">Linksearch</a> after adding a "hidden external link". </p>
      <p>Alternatives: </p>
      <p>If this is desired, the section can be split off. The content can still be shown on the original page, by transcluding the new page. Alternatively, or in preparaton of splitting off, instead of creating a link to a section one can create a link to a page redirecting to the section. The list of backlinks of the full page is now sorted by exact target, though not listing the names of the target sections but the names of the redirect pages. This also makes it easier to keep the links working when changing the name of a section: only the target of the redirect page has to changed, not the source pages of the links. Note that the redirect message is at the top of the whole page, so typically not visible at the target position. This may be an advantage (the disadvantage of the redirect message cluttering the target is minimal) or a disadvantage. To see the name of the page without scrolling up, note that the browser usually shows it in the window title at the very top, while the address bar shows the name of the redirect page with the name of the section on the target page. See also <a href="https://meta.wikimedia.org/wiki/Special:MyLanguage/Help:Section_linking_and_redirects" class="extiw" title="m:Special:MyLanguage/Help:Section linking and redirects">Help:Section linking and redirects</a> (this link itself is also an example of what has been described). </p>
      <p>Also note that if a page's links change due to a change in the template, the backlinks for that page are <a href="https://meta.wikimedia.org/wiki/Special:MyLanguage/Help:Job_queue#Updating_links_tables_when_a_template_changes" class="extiw" title="m:Special:MyLanguage/Help:Job queue">not updated immediately, but via the job queue</a>. </p> 
      <h2><span class="mw-headline" id="Redirects">Redirects</span></h2> 
      <p>The backlinks feature shows which backlinks are redirects. </p>
      <p>The backlinks of the redirect are also shown indented under the redirect, and if they include a redirect, the backlinks of that also (not more). </p>
      <p>This makes it a useful tool for finding double redirects, which do not work, and, except in special applications (see below) can better be replaced by redirects to the final target. </p>
      <p>For listing just the redirects to e.g. <a href="https://en.wikipedia.org/wiki/Wikipedia:Be_bold" class="extiw" title="w:Wikipedia:Be bold">w:Wikipedia:Be bold</a>, use <a class="external free" href="https://en.wikipedia.org/w/api.php?action=query&amp;list=backlinks&amp;bltitle=Wikipedia:Be%20bold&amp;blfilterredir=redirects&amp;bllimit=500">https://en.wikipedia.org/w/api.php?action=query&amp;list=backlinks&amp;bltitle=Wikipedia:Be%20bold&amp;blfilterredir=redirects&amp;bllimit=500</a> or <a class="external free" href="https://tools.wmflabs.org/redirectviews/?project=en.wikipedia.org&amp;platform=all-access&amp;agent=user&amp;range=latest-20&amp;sort=views&amp;direction=1&amp;view=list&amp;page=Wikipedia:Be_bold">https://tools.wmflabs.org/redirectviews/?project=en.wikipedia.org&amp;platform=all-access&amp;agent=user&amp;range=latest-20&amp;sort=views&amp;direction=1&amp;view=list&amp;page=Wikipedia:Be_bold</a> </p> 
      <h2><span class="mw-headline" id="Cases_of_inclusion">Cases of inclusion</span></h2> 
      <p>Whether there is a link from A to C is relevant for backlinks as well as Related changes. If C does not exist, only backlinks are applicable. </p> 
      <ul>
       <li>Inclusion is listed as such.</li> 
       <li>A link from A to C counts as such even if the link is not explicit in the wikitext of A, but due to the inclusion of B, which links to C.</li> 
       <li>If A calls B and there is a link in B to C within noinclude tags then B links to C, but A does not.</li> 
       <li>If A calls B and there is a link in B to C within includeonly tags then A links to C, but B does not.</li> 
       <li>If A calls B and B links to a page C that depends on a variable or parameter, then: 
        <ul>
         <li>The question whether A links to C depends on the value of the variable on page A, or the parameter with which A calls B.</li> 
         <li>The question whether B links to C depends on the value of the variable on page B; in the case of a parameter, considered from page B, the page may e.g. not call a valid page name (because it calls one with a name with braces), so B does not link to any valid C, or a name depending on the default value of the parameter.</li>
        </ul></li>
      </ul> 
      <p>Thus a link from A to C may be due to the inclusion of B, even when B, considered by itself, does not link to C. Thus, to find out what templates cause pages to link to C, it is not sufficient to check for templates in the backlinks of C. </p>
      <p>If some pages link to C due to the inclusion of B, it is more convenient if B is in the list of backlinks of C. One can assure this by putting an appropriate example of usage of B on page B, inside noinclude tags (B includes B, but without recursion). </p>
      <p>Otherwise a page like B, which causes pages which include B to link to C, can only be found by looking for pages which directly or indirectly are included in these backlink pages of C. </p>
      <p>Examples: </p> 
      <ul>
       <li><a href="https://meta.wikimedia.org/wiki/Template:H:f" class="extiw" title="m:Template:H:f">Template:H:f</a> calls Template:h:f {{NAMESPACE}}. Considered by itself that is <a href="https://meta.wikimedia.org/wiki/Template:H:f_Template" class="extiw" title="m:Template:H:f Template">Template:H:f Template</a> <a class="external text" href="https://meta.wikimedia.org/w/index.php?title=Special:Recentchangeslinked&amp;target=Template:H:f&amp;days=3000&amp;limit=500">Related chages of Template:H:f</a>, <a class="external text" href="https://meta.wikimedia.org/w/index.php?title=Special:Whatlinkshere&amp;target=Template:H:f_Template">What links to Template:H:f Template</a></li> 
       <li>This page <b>Help:What links here</b> links through <a href="https://meta.wikimedia.org/wiki/Template:H:f" class="extiw" title="m:Template:H:f">Template:H:f</a> to <a href="https://meta.wikimedia.org/wiki/Template:H:f_Help" class="extiw" title="m:Template:H:f Help">Template:H:f Help</a> <a class="external text" href="https://meta.wikimedia.org/w/index.php?title=Special:Recentchangeslinked&amp;target=Help:What%20links%20here&amp;days=3000&amp;limit=500">Related changes of Help:What links here</a>, <a class="external text" href="https://meta.wikimedia.org/w/index.php?title=Special:Whatlinkshere&amp;target=Template%3AH%3Af_Help">What links to Template:H:f Help</a>. This page is therefore a backlink of the latter, but Template:H:f is not listed.</li>
      </ul> 
      <p>Even without a template name depending on a variable or parameter, if page A links to page C due to the inclusion of template B1 which redirects to B2, which links to C, A is in the list of backlinks of C but B1 is not. </p>
      <p>Summarizing, for the purpose of backlinks and Related changes, A links to C if there is, in that order, a chain of template calls, a single redirect, template calls, a single redirect, etc., template calls, and a chain of redirects. </p>
      <p>If a page is conditionally embedded through #if, #ifeq, #ifexist, #ifexpr, or #switch, it is counted as embedded, even if the condition is not fulfilled </p>
      <p>If on page A the name of an embedded page is time-dependent, e.g. {{abc{{CURRENTDOW}}}}, currently giving <a href="/w/index.php?title=Template:Abc0&amp;action=edit&amp;redlink=1" class="new" title="Template:Abc0 (page does not exist)">Template:Abc0</a>, the "What links here" list of one of the possibly embedded pages shows page A: the current one, or, depending on caching issues, perhaps an older one. </p>
      <p>Thus, although the same functionality is obtained with {{#switch:{{CURRENTDOW}}|0={{abc0}}|1={{abc1}}|..}}, these methods differ in whether the seven templates all count as embedded, or just one. </p>
      <p>In the case of an image, using it in a page is listed in the third section of the image description page, but not in "What links here" of that page </p> 
      <h2><span class="mw-headline" id="User_styles">User styles</span></h2>
      <p> The page body has selector body.mw-special-Whatlinkshere, so we can e.g. use the CSS </p>
      <div class="mw-highlight mw-highlight-lang-css mw-content-ltr" dir="ltr">
       <pre><span></span><span class="nt">body</span><span class="p">.</span><span class="nc">mw-special-Whatlinkshere</span> <span class="nt">ul</span> <span class="p">{</span> <span class="k">list-style</span><span class="p">:</span> <span class="kc">decimal</span> <span class="p">}</span>
</pre>
      </div>
      <p> to number the backlinks. </p>
      <h2><span id="Order.3B_query"></span><span class="mw-headline" id="Order;_query">Order; query</span></h2> 
      <p>"What links here" sorts by page id, i.e., by date of creation of the page. </p>
      <p>Comparison with <a href="https://meta.wikimedia.org/wiki/Special:MyLanguage/Query#backlinks" class="extiw" title="m:Special:MyLanguage/Query">m:Query#backlinks</a> and <a href="https://meta.wikimedia.org/wiki/Special:MyLanguage/Query#embeddedin" class="extiw" title="m:Special:MyLanguage/Query">m:Query#embeddedin</a>: </p>
      <p>Compare <a href="/wiki/Special:WhatLinksHere/Template:T" title="Special:WhatLinksHere/Template:T">Special:Whatlinkshere/Template:T</a> with <a class="external free" href="https://meta.wikimedia.org/w/api.php?what=backlinks%7Cembeddedin&amp;titles=template:t&amp;bllimit=500&amp;eilimit=500&amp;blfilter=all&amp;eifilter=all">https://meta.wikimedia.org/w/api.php?what=backlinks%7Cembeddedin&amp;titles=template:t&amp;bllimit=500&amp;eilimit=500&amp;blfilter=all&amp;eifilter=all</a> </p>
      <p>Differences demonstrated: </p> 
      <ul>
       <li>Query sorts by function first (backlinks and inclusions), then by <a href="/wiki/Special:MyLanguage/page_table" title="Special:MyLanguage/page table">page id</a>, "what links here" sorts by page id.</li> 
       <li>Query distinguishes between B being only included in A, and B also being linked from A.</li> 
       <li>Query does not indicate which backlinks are redirects, and does not include <i>their</i> backlinks.</li> 
       <li>Query provides page ids.</li>
      </ul> 
      <p>Another difference: </p> 
      <ul>
       <li>Query allows restriction to a specified namespace.</li>
      </ul> 
      <p>{{#dpl:linksto = <i>pagename</i>}} using <a href="/wiki/Special:MyLanguage/Extension:DynamicPageList" title="Special:MyLanguage/Extension:DynamicPageList">Extension:DynamicPageList</a><span style="display:none"><a href="/wiki/Extension:DynamicPageList" class="mw-disambig" title="Extension:DynamicPageList"> </a></span>, produces backlinks sorted in alphabetic order. Other sorting options are also available. </p> 
      <h2><span class="mw-headline" id="In-page_list_of_backlinks">In-page list of backlinks</span></h2> 
      <p>You can display all the links to a page directly within a page by embedding the page "Special:Whatlinkshere"; see <a href="/wiki/Special:MyLanguage/Help:Transclusion" title="Special:MyLanguage/Help:Transclusion">Help:Transclusion</a><span style="display:none"><a href="/wiki/Help:Transclusion" title="Help:Transclusion"> </a></span>. </p>
      <p>This can also be done using any of the <a href="/wiki/Special:MyLanguage/Extension:DynamicPageList" title="Special:MyLanguage/Extension:DynamicPageList">Extension:DynamicPageList</a><span style="display:none"><a href="/wiki/Extension:DynamicPageList" class="mw-disambig" title="Extension:DynamicPageList"> </a></span> extensions. </p>
      <p>Two extensions, <a href="/wiki/Special:MyLanguage/Extension:Semantic_MediaWiki" title="Special:MyLanguage/Extension:Semantic MediaWiki">Extension:Semantic MediaWiki</a><span style="display:none"><a href="/wiki/Extension:Semantic_MediaWiki" title="Extension:Semantic MediaWiki"> </a></span> and <a href="/wiki/Special:MyLanguage/Extension:Cargo" title="Special:MyLanguage/Extension:Cargo">Extension:Cargo</a><span style="display:none"><a href="/wiki/Extension:Cargo" title="Extension:Cargo"> </a></span>, can be used to make more detailed lists of incoming links, grouped by the specific relationship between the two pages, but only for relationships that are encoded via those extensions. </p> 
      <h2><span class="mw-headline" id="Finding_external_links_to_a_page">Finding external links to a page</span></h2> 
      <p>For a given source wiki we can find the external links to any URL with <a href="/wiki/Special:LinkSearch" title="Special:LinkSearch">Special:Linksearch</a> (if installed). Thus, in particular, we can find e.g. pages on the English Wikipedia having an external link to this page or even to an old version, an edit or history page, etc. (provided that the parameter "title" comes first): <a class="external autonumber" href="https://en.wikipedia.org/w/index.php?title=Special%3ALinksearch&amp;target=meta.wikimedia.org%2Fwiki%2FHelp%3AWhat_links_here">[1]</a> <a class="external autonumber" href="https://en.wikipedia.org/w/index.php?title=Special%3ALinksearch&amp;target=meta.wikimedia.org/w/index.php?title=Help:What_links_here">[2]</a>. </p>
      <p>There is no such feature for interwiki links. This may be a reason to prefer external link style. </p>
