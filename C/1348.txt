      <div id="article-contents" class="article-grid-body"> 
       <p>You can search for issues and pull requests globally across all of GitHub, or search for issues and pull requests within a particular organization. For more information, see "<a href="/en/articles/about-searching-on-github">About searching on GitHub</a>."</p> 
       <div class="extended-markdown tip border rounded-1 mb-4 p-3 color-border-info color-bg-info f5"> 
        <p><strong>Tips:</strong></p> 
        <ul> 
         <li> <p>For a list of search syntaxes that you can add to any search qualifier to further improve your results, see "<a href="/en/articles/understanding-the-search-syntax">Understanding the search syntax</a>".</p> </li> 
         <li> <p>Use quotations around multi-word search terms. For example, if you want to search for issues with the label "In progress," you'd search for <code>label:"in progress"</code>. Search is not case sensitive.</p> </li> 
         <li> <p>You can focus your cursor on the search bar above the issue or pull request list with a keyboard shortcut. For more information, see "<a href="/en/articles/keyboard-shortcuts/#issue-and-pull-request-lists">Keyboard shortcuts</a>."</p> </li>
        </ul>
       </div> 
       <h3 id="search-only-issues-or-pull-requests"><a href="#search-only-issues-or-pull-requests">Search only issues or pull requests</a></h3> 
       <p>By default, GitHub search will return both issues and pull requests. However, you can restrict search results to just issues or pull requests using the <code>type</code> or <code>is</code> qualifier.</p> 
       <table>
        <thead>
         <tr>
          <th>Qualifier</th>
          <th>Example</th>
         </tr>
        </thead>
        <tbody>
         <tr>
          <td><code>type:pr</code></td>
          <td><a href="https://github.com/search?q=cat+type%3Apr&amp;type=Issues"><strong>cat type:pr</strong></a> matches pull requests with the word "cat."</td>
         </tr>
         <tr>
          <td><code>type:issue</code></td>
          <td><a href="https://github.com/search?q=github+commenter%3Adefunkt+type%3Aissue&amp;type=Issues"><strong>github commenter:defunkt type:issue</strong></a> matches issues that contain the word "github," and have a comment by @defunkt.</td>
         </tr>
         <tr>
          <td><code>is:pr</code></td>
          <td><a href="https://github.com/search?utf8=%E2%9C%93&amp;q=event+is%3Apr&amp;type="><strong>event is:pr</strong></a> matches pull requests with the word "event."</td>
         </tr>
         <tr>
          <td><code>is:issue</code></td>
          <td><a href="https://github.com/search?utf8=%E2%9C%93&amp;q=is%3Aissue+label%3Abug+is%3Aclosed&amp;type="><strong>is:issue label:bug is:closed</strong></a> matches closed issues with the label "bug."</td>
         </tr>
        </tbody>
       </table> 
       <h3 id="search-by-the-title-body-or-comments"><a href="#search-by-the-title-body-or-comments">Search by the title, body, or comments</a></h3> 
       <p>With the <code>in</code> qualifier you can restrict your search to the title, body, comments, or any combination of these. When you omit this qualifier, the title, body, and comments are all searched.</p> 
       <table>
        <thead>
         <tr>
          <th>Qualifier</th>
          <th>Example</th>
         </tr>
        </thead>
        <tbody>
         <tr>
          <td><code>in:title</code></td>
          <td><a href="https://github.com/search?q=warning+in%3Atitle&amp;type=Issues"><strong>warning in:title</strong></a> matches issues with "warning" in their title.</td>
         </tr>
         <tr>
          <td><code>in:body</code></td>
          <td><a href="https://github.com/search?q=error+in%3Atitle%2Cbody&amp;type=Issues"><strong>error in:title,body</strong></a> matches issues with "error" in their title or body.</td>
         </tr>
         <tr>
          <td><code>in:comments</code></td>
          <td><a href="https://github.com/search?q=shipit+in%3Acomment&amp;type=Issues"><strong>shipit in:comments</strong></a> matches issues mentioning "shipit" in their comments.</td>
         </tr>

       <h3 id="search-by-when-an-issue-or-pull-request-was-created-or-last-updated"><a href="#search-by-when-an-issue-or-pull-request-was-created-or-last-updated">Search by when an issue or pull request was created or last updated</a></h3> 
       <p>You can filter issues based on times of creation, or when they were last updated. For issue creation, you can use the <code>created</code> qualifier; to find out when an issue was last updated, you'll want to use the <code>updated</code> qualifier.</p> 
       <p>Both take a date as a parameter. Date formatting must follow the <a href="http://en.wikipedia.org/wiki/ISO_8601">ISO8601</a> standard, which is <code>YYYY-MM-DD</code> (year-month-day). You can also add optional time information <code>THH:MM:SS+00:00</code> after the date, to search by the hour, minute, and second. That's <code>T</code>, followed by <code>HH:MM:SS</code> (hour-minutes-seconds), and a UTC offset (<code>+00:00</code>).</p> 
       <p>When you search for a date, you can use greater than, less than, and range qualifiers to further filter results. For more information, see "<a href="/en/github/searching-for-information-on-github/understanding-the-search-syntax">Understanding the search syntax</a>."</p> 
       <table>
        <thead>
         <tr>
          <th>Qualifier</th>
          <th>Example</th>

         <tr>
          <td><code>created:<em>YYYY-MM-DD</em></code></td>
          <td><a href="https://github.com/search?q=language%3Ac%23+created%3A%3C2011-01-01+state%3Aopen&amp;type=Issues"><strong>language:c# created:&lt;2011-01-01 state:open</strong></a> matches open issues that were created before 2011 in repositories written in C#.</td>
