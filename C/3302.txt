      <p><b><a href="/wiki/Wikipedia:Protection_policy#pc1" title="Wikipedia:Protection policy">Pending changes protection</a></b> is a tool used to suppress <a href="/wiki/Wikipedia:Vandalism" title="Wikipedia:Vandalism">vandalism</a> and certain other recurrent nuisances on Wikipedia while allowing a <a href="/wiki/Wikipedia:Assume_good_faith" title="Wikipedia:Assume good faith">good-faith</a> user to submit an edit for review. Intended for infrequently edited articles that are experiencing high levels of such troublesome edits from new or unregistered users, pending changes protection can be used as an alternative to semi-protection and full protection to allow unregistered and new users to edit pages, while keeping the edits hidden to most readers until they are accepted by a reviewer. There are relatively few articles on Wikipedia with this type of protection. </p>
      <p>When a page under pending changes protection is edited by an <a href="/wiki/Wikipedia:IP" class="mw-redirect" title="Wikipedia:IP">unregistered</a> (also called IP) editor or a <a href="/wiki/Wikipedia:User_access_levels#New_users" title="Wikipedia:User access levels">new user</a>, the edit is not directly visible to the majority of Wikipedia readers, until it is reviewed and accepted by an editor with the <a href="/wiki/Wikipedia:User_access_levels#Reviewer" title="Wikipedia:User access levels">reviewer right</a>. </p>
      <p>Pending changes are visible in the page history, where they are marked as pending review. The latest accepted revision is displayed to the general public, while logged-in users see the latest revision of the page, with all changes applied. When editors who are not reviewers make changes to an article with unreviewed pending changes, their edits are also marked as pending and are not visible to most readers. </p>
      <p>Both logged-in users and <a href="/wiki/Wikipedia:Welcome_unregistered_editing" title="Wikipedia:Welcome unregistered editing">unregistered users</a> who click the "edit this page" tab edit the latest version as usual. If there are pending changes awaiting review, there will be a dropdown box next to the article title, pointing to the pending changes. </p>
      <p>Pending changes may be used to protect articles against persistent vandalism, violations of the <a href="/wiki/Wikipedia:BLP" class="mw-redirect" title="Wikipedia:BLP">biographies of living persons</a> policy, and copyright violations. </p>

      <h2><span class="mw-headline" id="Applying_pending_changes_protection">Applying pending changes protection</span></h2> 
      <table class="plainlinks ombox ombox-notice" role="presentation">
       <tbody>
        <tr>
         <td class="mbox-image"><img alt="" src="//upload.wikimedia.org/wikipedia/en/thumb/1/1d/Information_icon4.svg/40px-Information_icon4.svg.png" decoding="async" width="40" height="40" srcset="//upload.wikimedia.org/wikipedia/en/thumb/1/1d/Information_icon4.svg/60px-Information_icon4.svg.png 1.5x, //upload.wikimedia.org/wikipedia/en/thumb/1/1d/Information_icon4.svg/80px-Information_icon4.svg.png 2x" data-file-width="620" data-file-height="620"></td>

       </tbody>

      <p>Administrators may apply pending changes protection to pages that are subject to heavy and persistent <a href="/wiki/Wikipedia:Vandalism" title="Wikipedia:Vandalism">vandalism</a>, violations of the <a href="/wiki/Wikipedia:BLP" class="mw-redirect" title="Wikipedia:BLP">biographies of living persons policy</a>, or insertion of content that violates copyright. Pending changes protection should not be used as a preemptive measure against violations that have not yet occurred, nor should it be used to privilege registered users over unregistered users in content disputes. Pending changes protection should not be used on articles with a very high edit rate, even if they meet the aforementioned criteria. Instead semi-protection should be considered. </p>
      <p>In addition, administrators may apply temporary pending changes protection on pages that are subject to significant but temporary vandalism or disruption (for example, due to media attention) when blocking individual users is not a feasible option. As with other forms of protection, the time frame of the protection should be proportional to the problem. Indefinite PC protection should only be used in cases of severe long-term disruption. </p>
      <p>Like semi-protection, PC protection should <b>never</b> be used in genuine content disputes, where there is a risk of placing a particular group of editors at a disadvantage. </p>
      <p>Editors without administrator privileges can <a href="/wiki/Wikipedia:Requests_for_page_protection" title="Wikipedia:Requests for page protection">request page protection</a> if the above criteria are met. Removal of pending changes protection can be requested of any administrator, or at <a href="/wiki/Wikipedia:RFPP#Current_requests_for_unprotection" class="mw-redirect" title="Wikipedia:RFPP">requests for unprotection</a>. </p> 
      <h2><span class="mw-headline" id="Reviewing_pending_edits">Reviewing pending edits</span></h2> 
      <table class="plainlinks ombox ombox-notice" role="presentation">
       <tbody>
        <tr>
         <td class="mbox-image"><img alt="" src="//upload.wikimedia.org/wikipedia/en/thumb/1/1d/Information_icon4.svg/40px-Information_icon4.svg.png" decoding="async" width="40" height="40" srcset="//upload.wikimedia.org/wikipedia/en/thumb/1/1d/Information_icon4.svg/60px-Information_icon4.svg.png 1.5x, //upload.wikimedia.org/wikipedia/en/thumb/1/1d/Information_icon4.svg/80px-Information_icon4.svg.png 2x" data-file-width="620" data-file-height="620"></td>

      <p>The process of reviewing is intended as a quick check to ensure edits don't contain: </p> 
      <ul>
       <li><a href="/wiki/Wikipedia:VAND" class="mw-redirect" title="Wikipedia:VAND">vandalism</a></li> 
       <li>violations of the <a href="/wiki/Wikipedia:BLP" class="mw-redirect" title="Wikipedia:BLP">policy on living people</a></li> 
       <li>copyright violations</li> 
       <li>other obviously inappropriate content</li>
      </ul> 
      <p>Reviewers are sufficiently experienced users who are granted the ability to accept other users' edits. Reviewers have a similar level of trust to <a href="/wiki/Wikipedia:Rollback_feature" class="mw-redirect" title="Wikipedia:Rollback feature">rollbackers</a>; all administrators have the reviewer right. Potential reviewers should recognize vandalism, be familiar with basic content policies such as the policy on living people, and have a reasonable level of experience editing Wikipedia. Reading the <a href="/wiki/Wikipedia:RG" class="mw-redirect" title="Wikipedia:RG">reviewing guideline</a>, where the reviewing process and expectations for a reviewer are detailed, is recommended. </p>
      <p>Reviewers and administrators will see a pink watchlist banner on their watchlist whenever there is a pending edit needing review. If a reviewer or administrator wishes to disable it, they can paste <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">#mw-fr-watchlist-pending-notice {display: none}</code> to <a href="/wiki/Special:MyPage/common.css" title="Special:MyPage/common.css">their common.css</a>. </p>
      <p>Acceptance of an edit by a reviewer is not an endorsement of the edit. It merely indicates that the edit has been checked for obvious problems as listed above. </p>
      <p>Reviewer rights are granted upon request at <a href="/wiki/Wikipedia:Requests_for_permissions" title="Wikipedia:Requests for permissions">Wikipedia:Requests for permissions</a>. While any administrator has the technical ability to remove the reviewer permission, removal should occur only as the result of consensus from a discussion or when an editor requests the removal of their own permission. Discussion regarding removal of the reviewer permission should normally occur at the <a href="/wiki/Wikipedia:AN" class="mw-redirect" title="Wikipedia:AN">Administrators' noticeboard</a>. Discussion with the involved editor and/or a request for a second opinion at the <a href="/wiki/Wikipedia_talk:PC" class="mw-redirect" title="Wikipedia talk:PC">Pending changes talk page</a> is recommended before formally requesting removal. </p>
      <p>Reviewing of pending changes should be resolved within reasonable time limits (at most a few hours). Backlog management should be coordinated at a community level. The backlog can be viewed at <a href="/wiki/Special:PendingChanges" title="Special:PendingChanges">Special:PendingChanges</a>. As of January 2013, edits are rarely unreviewed for more than two to three hours and the backlog is frequently empty. </p> 
      <h2><span class="mw-headline" id="Pending_changes_adds_highlighting_that_is_lost_when_disabled">Pending changes adds highlighting that is lost when disabled</span></h2> 
      <p>In the edit history, accepted revisions are highlighted, which improves readability. Additionally, visible tags are applied to indicate why particular edits were accepted ("automatically accepted"/"accepted by [Username]"). As of September 2018<sup class="plainlinks noexcerpt noprint asof-tag update" style="display:none;"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Pending_changes&amp;action=edit">[update]</a></sup>, this highlighting is still permanently lost for past changes on a given page whenever the pending changes setting is disabled.<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">[1]</a></sup> When pending changes are enabled again, the highlighting will only be applied to <i>newer</i> changes. Therefore, it is a good choice to <b>leave pending changes enabled</b> when other protections are applied.<sup id="cite_ref-2" class="reference"><a href="#cite_note-2">[2]</a></sup> </p>

      <h2><span class="mw-headline" id="Frequently_asked_questions">Frequently asked questions</span></h2> 
      <dl>
       <dt>
        If an established user edits an article with unreviewed pending changes, is the new version automatically accepted?
       </dt> 
       <dd>
        No. If the user is a <a href="/wiki/Wikipedia:Reviewing_pending_changes" title="Wikipedia:Reviewing pending changes">reviewer</a> (that is, the user has been granted the "reviewer" permission), they will be prompted to review and accept any unreviewed pending changes. If the user is not a reviewer, the edit will also be marked as "pending review". (Reviewers can <a href="/wiki/Wikipedia:Pending_changes/Testing" title="Wikipedia:Pending changes/Testing">test this</a> by unaccepting the current version of a page under pending changes and then trying to edit.) An exception to this is when a user reverts a pending edit to the latest accepted revision: in this case the revert is automatically accepted.
       </dd> 
       <dt>
        What happens if several IP edits to an article under pending changes result in a <a href="/wiki/Wikipedia:NULL" class="mw-redirect" title="Wikipedia:NULL">null edit</a>? (For example, an IP makes an edit, then another IP <a href="/wiki/Wikipedia:UNDO" class="mw-redirect" title="Wikipedia:UNDO">undoes</a> it.)
       </dt> 
       <dd>
        If they were all made by a single IP, the new version is automatically accepted. If different users edited, the new version is not accepted (to prevent potential abuse).
       </dd> 
       <dt>
        On which <a href="/wiki/Wikipedia:Namespace" title="Wikipedia:Namespace">kinds of pages</a> can pending changes be used?
       </dt> 
       <dd>
        At first, it was determined by consensus that pending changes could be used only on articles, subject to the <a href="/wiki/Wikipedia:PP" class="mw-redirect" title="Wikipedia:PP">protection policy</a>, and on <a href="/wiki/Wikipedia:Pending_changes/Testing" title="Wikipedia:Pending changes/Testing">test pages</a> in project space. A later request for comment found it permissible to use pending changes beyond articles; however, it is restricted by the software to the main and project namespaces, and no request to allow other namespaces was made. It is not technically possible for talk pages to be placed on pending changes.
       </dd> 
       <dt>
        Wasn't pending changes protection dropped?
       </dt> 
       <dd>
        Yes and no. Pending changes protection was deployed on a trial basis in 2010. In 2011, pending changes protection was dropped as a mechanism for protecting pages, until a consensus agreement on its deployment was reached. There have been a series of discussions on using the feature and it was put back into service on December 1, 2012. Since then only pending changes level 1, affecting the edits of new and unregistered users, is being used. As of January 2017 there has been consensus to drop pending changes level 2, and as a result only level 1 is now used.
       </dd> 
       <dt>
        How can you tell if a page has pending changes protection?
       </dt> 
       <dd>
        Protected pages are normally marked with a small padlock symbol in the top corner depending on its level of protection. Also, there will be a drop-down box next to the article title, pointing to the pending changes, if there are any.
       </dd>
      </dl>

The portion for creating a semi-automated review tool was withdrawn from the RfC as not needing consensus, and the RfC was later <a href="/wiki/Wikipedia:SNOW" class="mw-redirect" title="Wikipedia:SNOW">snow-closed</a> with consensus against all remaining proposed changes.</li>
