      <p><span id="Must_be_a_media_file"></span> </p> 
      <h3><span class="mw-headline" id="Devu_esti_amaskomunikila_dosiero">Devu esti amaskomunikila dosiero</span></h3> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>Wikimedia Commons hosts only media files such as photographs, scanned images, diagrams, animations, audio (e.g. music and spoken dialogue) and video clips, along with any associated metadata. Explanatory and other text is permitted on the file page only to the extent to which it advances <a href="#commons-aims">Commons' aim</a> and is not <a href="#Excluded_educational_content">excluded educational content</a>. </p> 
      </div> 
      <p>La jena ne estas konsideritaj amaskomunikiloj dosierojn, kaj ne estu gastigita tie: </p> 
      <ul>
       <li>Komputilaj programoj en ajna formato inkludante duuma plenumeblajn dosierojn kaj kruda fontkodo listigo. Fonto-kodo eble kie rilatajn formo dividas de la priskriba a? metadatenojn, ekz grafika dosiero povas inkluzivi kiel priskriba teksto la kodo uzata por krei ?in.</li>
      </ul> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <ul>
        <li>Files which are representative merely of raw text, e.g. ASCII files, raw source code listings as mentioned above, etc.</li>
       </ul> 
      </div> 
      <p><span id="Must_be_an_allowable_free_file_format"></span> </p> 
      <h3><span class="mw-headline" id="Devu_esti_randon_libera_dosierformaton">Devu esti randon libera dosierformaton</span></h3> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>Only free file formats are allowed. Proprietary formats such as Microsoft <a href="https://en.wikipedia.org/wiki/eo:Microsoft_Word" class="extiw" title="w:eo:Microsoft Word">Word</a> and <a href="https://en.wikipedia.org/wiki/en:Microsoft_Excel" class="extiw" title="w:en:Microsoft Excel">Excel</a> documents, <a href="https://en.wikipedia.org/wiki/en:Advanced_Audio_Coding" class="extiw" title="w:en:Advanced Audio Coding">AAC</a>, <a href="https://en.wikipedia.org/wiki/en:Windows_Media_Audio" class="extiw" title="w:en:Windows Media Audio">WMA</a>, <a href="https://en.wikipedia.org/wiki/High_Efficiency_Video_Coding" class="extiw" title="w:High Efficiency Video Coding">H.265</a>, <a href="https://en.wikipedia.org/wiki/High_Efficiency_Image_File_Format" class="extiw" title="w:High Efficiency Image File Format">HEIC</a>, <a href="https://en.wikipedia.org/wiki/en:Flash_Video" class="extiw" title="w:en:Flash Video">Flash</a> and the like are not permitted. More generally, any file that requires an unfree program or patent-encumbered codec in order to access its content is prohibited. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>A list of currently allowed file formats can be found at <a href="/wiki/Special:MyLanguage/Commons:Project_scope/Allowable_file_types" title="Special:MyLanguage/Commons:Project scope/Allowable file types">Commons:Project scope/Allowable file types</a>. </p> 
      </div> 
      <p><span id="Must_be_freely_licensed_or_public_domain"></span><span id="FREEORPD"></span> </p> 
      <h3><span id="Devas_esti_libere_licencitaj_a.C5.AD_publika.C4.B5o"></span><span class="mw-headline" id="Devas_esti_libere_licencitaj_a?_publika?o">Devas esti libere licencitaj a? publika?o</span></h3> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>Wikimedia Commons accepts only <a href="https://en.wikipedia.org/wiki/eo:Libera_verko" class="extiw" title="w:eo:Libera verko">free content</a>, in other words files that are either freely licensed or which are in the <a href="https://en.wikipedia.org/wiki/eo:Publika_hava%C4%B5o" class="extiw" title="w:eo:Publika hava?o">public domain</a>. A file is considered public domain if either all copyright has expired or if the copyright owner(s) has voluntarily placed the content of the file into the public domain by irrevocably renouncing all copyright. A file which is ineligible for copyright protection is also considered public domain. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>Any file hosted here must normally be freely licensed or public domain according to both the law of the United States and according to the law of the source country, if different: see <a href="/wiki/Commons:Licensing/eo" title="Commons:Licensing/eo">Commons:Licencigo</a>. </p> 
      </div>

      <p><span id="Evidence"></span> </p> 
      <h4><span class="mw-headline" id="Evidenteco">Evidenteco</span></h4> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>In all cases the uploader must provide appropriate evidence to demonstrate either that the file is in the public domain or that the copyright owner has released it under a suitable licence. Typically that requires at least that the source of the file be specified,<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">[1]</a></sup> along with the original source where the file is a <a href="/wiki/Commons:Derivative_works/eo" title="Commons:Derivative works/eo">derivative work</a>. Also, the creator or copyright owner should be identified, if known or reasonably ascertainable. If there is any question, evidence may need to be supplied that the copyright owner has indeed released the file under the given licence. </p>
       <p>Where the file is a photograph which shows an identifiable person, the subject's consent may be required as described at <a href="/wiki/Commons:Photographs_of_identifiable_people/eo" title="Commons:Photographs of identifiable people/eo">Commons:Photographs of identifiable people</a>. </p>
       <p>In all cases, the burden of proof lies on the uploader or other person arguing for the file to be retained to demonstrate that as far as can reasonably be determined: </p> 
       <ul>
        <li>the file is in the public domain or is properly licensed, and</li> 
        <li>that any required consent has been obtained.</li>
       </ul> 
       <p>In the case of an old file, date and country of publication can help determine if the file is in the public domain due to age. </p>

      <p><span id="PRP"></span><span id="Precautionary_principle"></span> </p> 
      <h4><span id="Anta.C5.ADgarda_principo"></span><span class="mw-headline" id="Anta?garda_principo">Anta?garda principo</span></h4> 
      <p>Commons' users aim to build and maintain in good faith a repository of media files which to the best of our knowledge are free or freely-licensed. <b>The precautionary principle</b> is that where there is <i>significant</i> doubt about the freedom of a particular file, it should be deleted. </p>
      <p>Also, arguments that amount to "we can get away with it", such as the following, are against Commons' aims: </p> 
      <ol>
       <li>"<i>The copyright owner will not bother to sue or cannot afford to.</i>"</li> 
       <li>"<i>The copyright owner will never find out.</i>"</li> 
       <li>"<i>The copyright owner will not mind/should be pleased that we have disseminated their work.</i>"</li> 
       <li>"<i>Nobody knows who the copyright owner is, so it really doesn’t matter.</i>"</li> 
       <li>"<i>The file is obviously common property. It can be found all over the internet and nobody has complained.</i>"</li>
      </ol>

      <p>Ni tenu multaj alta kvalito bildoj de specio-identigita birdoj, kaj ne ekzistas realisma uzo educativo por malgranda, neklaraj, malbone formita ekrankopion de anonima kaj neidentigeblaj birdo. Kompreneble, ?iam estas loko por alia klerecrilate distinga bildo, ekzemple ilustrante iu aspekto de birdo konduton kiu ni ne aktuale kovras, e? se la bildo estas eble ne de la plej alta kvalito. </p>
      <p>Povas foje esti argumento por retener multnombraj bildoj kiuj estas (de pedagogia vidpunkto) tute simila, pro vario kaj disponibilidad de elekto, sed estas nenia celo en nia retprovizanton multaj esence identa malbonkvalitaj bildoj kiuj ne havas realisma eduka valoro. </p> 
      <div class="mw-translate-fuzzy"> 
       <p>Novaj edukaj dosierojn de escepta kvalito estas ?iam bonvena, kaj la posta al?utas de tiaj dosieroj povas principe repagi anta?e neuzitaj malbonkvalitaj dosierojn klerecrilate redunda. Tamen, kiel indikite supre, dosiero kiu uzas bonfide sur Vikimedia projekto estas ?iam konsiderita edukaj, tiom malri?a kvalito dosieron kiu restas en uzo ne riskos deletion e? se pli bona kvalito dosiero kovrante la sama temo poste igas havebla. </p> 
      </div> 
      <p>Novaj kaj ekzistantaj dosieroj de malri?aj a? mediocres kvalito povas a? ne esti realisme utila por eduka celo depende kion ili ilustras kaj kion aliaj dosieroj ni havas de la sama temo. Kie temo estas maloftaj kaj / a? malfacile kapti, e? malri?a kvalito dosiero estu de signifa eduka valoro, speciale se Komunejo havas tre malmultaj a? neniu simila dosierojn jam. Aliflanke, malbona a? mediocres dosierojn de komuna kaj facile kapti temoj povas havi neniun realisman eduka valoro, speciale se Komuna?o jam gastigas multajn similajn a? pli bona kvalito ekzemploj. </p>
      <p>Bildo kvalito estas nur unu el la faktoroj kiuj povas limigi la eduka utilecon de dosiero. Aliaj limigaj faktoroj povas inkluzivi malalta rezolucio kaj hard-al-forigi filigranoj. </p>
      <p><span id="PDF_and_DjVu_formats"></span><span id="PDF"></span><span id="DjVu"></span> </p> 
      <h3><span class="mw-headline" id="PDF_kaj_DjVu_formatojn">PDF kaj DjVu formatojn</span></h3> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>Although PDF and DjVu file formats are permitted, they are expected to be used only in appropriate cases. There should be some reason for the choice of format that is in line with <a href="#commons-aims">Commons' aims</a>. All of the above rules apply as well, of course. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>An admin considering whether to delete a PDF or DjVu file may have to exercise judgement as to whether the chosen format is in line with Commons' aims, and the context and intent (if known) may be relevant. For example, while a published university thesis in PDF format may be OK, a user-created original-research article that is making use of Commons as a free web-host may not be. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>Files that might realistically be useful to one or more other Wikimedia projects, e.g. Wikisource or Wikibooks, should be kept; deletions should not be based on the sole ground that the file would be better hosted on one of those other projects. Any media file that is realistically useful to or is within the scope of even one other Wikimedia Foundation project can be hosted here. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>Remember that Wikisource often needs PDF or DjVu files in order to proofread or create source texts: Therefore, scans of suitable editions of notable public domain works are <i>almost always within scope</i> for this reason. That said, remember that editorial decisions involved in preparing a text from several sources may result in a new copyright, so the editions used <i>must</i> be out of copyright themselves. </p> 
      </div> 
      <p><span id="Allowable_reasons_for_PDF_and_DjVu_formats"></span> </p> 
      <h4><span class="mw-headline" id="Permesita_kialoj_por_PDF_kaj_DjVu_formatojn">Permesita kialoj por PDF kaj DjVu formatojn</span></h4> 
      <ul>
       <li>La formato estis selektita por oportuneco de presado.</li> 
       <li>La formato estis selektita por ebligi vidi a? presi de specialaj tiparoj (ekz dokumentoj en progresinta matematiko a? lingvistiko).</li> 
       <li>La dosiero estus ene de la medio de alia projekto de Vikimedia Fonda?o, se ?i devis esti al?utita en la sama formato al tiu projekto, ekzemple: 
        <ul>
         <li>PDF a? DjVu dosiero de eldonitaj kaj peer-reviziitaj laboro estus en medion sur Vikifontoj kaj estas sekve anka? en medion sur Komunejo. Ekzemploj de en-medion dokumentoj inkludas eldonitaj libroj (sed ne vanteco-eldonejoj), peer-reviziis akademiaj paperoj ktp, universitata tezo kaj paroladojn.</li> 
         <li>La dosiero estas escaneado de dokumento de historiaj a? aliaj eksteraj signifon (ekz skanu de ekzistanta kopirajto-libera a? licencita librojn, raportojn, ?urnalojn ktp).</li> 
         <li>La dosiero estas uzebla kiel fiksa, kontrolebla fonto dokumento, por ekz Vikifontoj a? Vikilibroj (ta?ga fijación de enhavo).</li> 
         <li>La formato elektita provizas teknikajn avanta?ojn al almena? unu alia Vikimedia Fonda?o projekto.</li>
        </ul></li>
      </ul> 
      <p><span id="Non-allowable_reasons_for_PDF_and_DjVu_formats"></span> </p> 
      <h4><span class="mw-headline" id="Ne-permesita_kialoj_por_PDF_kaj_DjVu_formatojn">Ne-permesita kialoj por PDF kaj DjVu formatojn</span></h4> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <ul>
        <li>The format has apparently been selected by the author or uploader to prevent or to discourage the creation of derivative works contrary to <a href="#commons-aims">Commons' aims</a>. Such inappropriate fixation of content would include a self-published vanity article or a self-created image that the author has uploaded in PDF format in an effort to discourage others from creating derivatives.</li> 
        <li>The content is essentially raw text; such files are not considered media files. Note that scans of existing books, reports, newspapers etc of historic or other external significance are not excluded on this ground, even if they contain no images.</li> 
        <li>The content would be prohibited under another section; for example, promotional material is outside Commons' scope regardless of the file format.</li>
       </ul> 
      </div> 
      <p><span id="Scope_part_2:_Pages,_galleries_and_categories"></span><span id="Scope_part_2"></span><span id="galleries"></span> </p>

      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <h2><span class="mw-headline" id="A_word_on_some_areas_of_particular_concern">A word on some areas of particular concern</span></h2> 
      </div> 
      <p><span id="Censorship"></span><span id="CENSOR"></span> </p> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <h3><span class="mw-headline" id="Censorship">Censorship</span></h3> 
      </div> 
      <div class="shortcut2" style="">
       <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r433650343"><a href="/wiki/Commons:Shortcuts" title="Commons:Shortcuts">Shortcuts</a>
       <br><a href="/wiki/Commons:CENSOR" class="mw-redirect" title="Commons:CENSOR">COM:CENSOR</a>
       <br><a href="/wiki/Commons:CENSORSHIP" class="mw-redirect" title="Commons:CENSORSHIP">COM:CENSORSHIP</a>
       <br><a href="/wiki/Commons:OMGAPENIS" class="mw-redirect" title="Commons:OMGAPENIS">COM:OMGAPENIS</a>
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>Files and other materials which are not lawful for Commons to host on its servers in Virginia will be deleted immediately upon being identified as unlawful (including copyright violations), even if the material otherwise falls within Commons scope, as set out above. However, Commons is not censored, and legitimately includes content which some users may consider objectionable or offensive. The policy of <i>"Commons is not censored"</i> means that a lawfully-hosted file, which falls within Commons' definitions of scope, will not be deleted solely on the grounds that it may not be "child-friendly" or that it may cause offence to you or others, for moral, personal, religious, social, or other reasons. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>The counterpoint to this, is that the statement <i>"Commons is not censored"</i> is not a valid argument for keeping a file that falls outside Commons' defined scope, as set out above. Photographs of nudity including male and female genitalia are sometimes uploaded for non-educational motives, and such images are not exempt from the requirement to comply with the rules of Commons' scope. If the images are of demonstrably inferior quality, or add nothing educationally distinct to the stock of such images we hold already, they may fail the test of being realistically useful for an educational purpose. </p> 
      </div> 
      <div lang="en" dir="ltr" class="mw-content-ltr"> 
       <p>A balance has to be struck between accepting useful media files with legitimate educational content that some may find offensive, and not allowing Commons to be used as a general-purpose media-hosting service (like Flickr, Photobucket, YouTube, etc.), without regard for the project's stated goals. The purpose of Commons is to serve as a media repository, a reliable resource of useful, open source media content; organized and comprehensive in coverage (with accurate file descriptions/information), educational, and intended both for use by Wikimedia projects, and as a public service freely accessible to everyone. </p> 
      </div>
