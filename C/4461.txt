      <h3><span class="mw-headline" id="Bing_Maps">Bing Maps</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Commons:%E3%82%B8%E3%82%AA%E3%82%B3%E3%83%BC%E3%83%87%E3%82%A3%E3%83%B3%E3%82%B0&amp;action=edit&amp;section=7" title="Edit section: Bing Maps">edit</a><span class="mw-editsection-bracket">]</span></span></h3> 
      <p>Right click where the exact location is and you have the coordinates. Paste into the Location template. </p> 
      <h3><span class="mw-headline" id="OpenStreetMap">OpenStreetMap</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Commons:%E3%82%B8%E3%82%AA%E3%82%B3%E3%83%BC%E3%83%87%E3%82%A3%E3%83%B3%E3%82%B0&amp;action=edit&amp;section=8" title="Edit section: OpenStreetMap">edit</a><span class="mw-editsection-bracket">]</span></span></h3> 
      <ol>
       <li>Open a new browser tab/window and load <a rel="nofollow" class="external text" href="https://www.openstreetmap.org/">https://www.openstreetmap.org/</a>.</li> 
       <li>Search for the correct location; you probably want to zoom in to make this easier.</li> 
       <li>Right-click on the exact location the media was recorded. A little pop-up menu opens there. From the pop-up menu, select <i>Show address</i>.</li>
      </ol> 
      <h3><span class="mw-headline" id="Google_Maps">Google Maps</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Commons:%E3%82%B8%E3%82%AA%E3%82%B3%E3%83%BC%E3%83%87%E3%82%A3%E3%83%B3%E3%82%B0&amp;action=edit&amp;section=9" title="Edit section: Google Maps">edit</a><span class="mw-editsection-bracket">]</span></span></h3>

       <li>Search for the correct location (zoom in, Satellite mode is suggested).</li> 
       <li>Right-click on the exact location the media was recorded and select <i>Center map here</i></li> 
       <li>Click the link icon in the left sidebar</li> 
       <li>Copy the link in the top box (do not click <i>short url</i>)</li> 
       <li>Use the <a href="/wiki/User:Dschwen/Coordinate_conversion_helper" title="User:Dschwen/Coordinate conversion helper">Coordinate conversion helper</a> to insert the coordinates on the image page (add a heading such as "N","SE","NNW")

         </dd>

      <h3><span class="mw-headline" id="Google_Earth">Google Earth</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Commons:%E3%82%B8%E3%82%AA%E3%82%B3%E3%83%BC%E3%83%87%E3%82%A3%E3%83%B3%E3%82%B0&amp;action=edit&amp;section=10" title="Edit section: Google Earth">edit</a><span class="mw-editsection-bracket">]</span></span></h3> 
      <ul>
       <li>If you have <a href="https://en.wikipedia.org/wiki/Google_Earth" class="extiw" title="w:Google Earth">Google Earth</a>, a tool for geotagging can be installed: <a rel="nofollow" class="external text" href="http://www.alder-digital.de/wiki/images/Ald-Hjl-Koord-en.kmz"><b>Ald-Hjl-Koord-en.kmz</b></a>. A crosshair will appear in the middle of the Google Earth screen. Place the crosshair exactly over the camera location by moving the view, and click on it. The location template can be copied from the window that opens. (Please deactivate the tool if unused.)</li> 
       <li>If you have coordinates in a <a href="https://en.wikipedia.org/wiki/Keyhole_Markup_Language" class="extiw" title="w:Keyhole Markup Language">KML</a> file (such as created in Google Earth and exported with "Save As" or "Copy", or aided by <a href="https://en.wikipedia.org/wiki/Picasa" class="extiw" title="w:Picasa">Picasa</a>), the <b><a class="external text" href="https://tools.wmflabs.org/geocommons/kmlconvert.php">kmlconvert</a></b> tool can be used to convert the coordinates to Commons templates. This method is suitable for geocoding a series of images.</li>
      </ul>

      <h2><span class="mw-headline" id="Adding_a_location_template">Adding a location template</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Commons:%E3%82%B8%E3%82%AA%E3%82%B3%E3%83%BC%E3%83%87%E3%82%A3%E3%83%B3%E3%82%B0&amp;action=edit&amp;section=11" title="Edit section: Adding a location template">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <p>The use of the following templates is encouraged: </p> 
      <ul>
       <li><a href="/wiki/Template:Location" title="Template:Location"><span style="font-family:monospace,monospace;">{{Location}}</span></a> for camera locations</li> 
       <li><a href="/wiki/Template:Object_location" title="Template:Object location"><span style="font-family:monospace,monospace;">{{Object location}}</span></a> same idea for <i>object</i> instead of <i>camera</i> locations</li>
      </ul> 
      <p><b>Do not simply invent your own new templates or copy geocoding templates from other wikis without discussion. Doing so can break the automation that reads and uses the geocoding. Commons geocoding needs are also not the same as those of other projects.</b> Note that you also should not create your own wrappers around <a href="/wiki/Template:Location" title="Template:Location"><span style="font-family:monospace,monospace;">{{Location}}</span></a> or <a href="/wiki/Template:Object_location" title="Template:Object location"><span style="font-family:monospace,monospace;">{{Object location}}</span></a>; this will break the automation as well. Just call one of the templates directly from the image description page. </p>
      <p>Simply add <span style="white-space:nowrap;"><code>{{Location|lat|long}}</code></span> to the image page, filling in the lat and long from the procedure above. If the image page has an <span style="white-space:nowrap;"><code>{{Information}}</code></span> template, or similar, the <span style="white-space:nowrap;"><code>{{Location}}</code></span> template should come immediately after it. For example, the result for <span style="white-space:nowrap;"><code>{{Location|37.51136|-77.602615}}</code></span> will look like this: </p>

      <h3><span class="mw-headline" id="Parameters">Parameters</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Commons:%E3%82%B8%E3%82%AA%E3%82%B3%E3%83%BC%E3%83%87%E3%82%A3%E3%83%B3%E3%82%B0&amp;action=edit&amp;section=12" title="Edit section: Parameters">edit</a><span class="mw-editsection-bracket">]</span></span></h3> 
      <p>Wikipedia geocoding often uses <a href="https://en.wikipedia.org/wiki/Wikipedia:WikiProject_Geographical_coordinates#Parameters" class="extiw" title="w:Wikipedia:WikiProject Geographical coordinates">parameters</a> after the coordinates to give more information about the location, but some of them are not useful here. </p> 
      <ul>
       <li><i>type</i> and <i>scale</i> - The defaults are <code style="font-size:smaller">type:landmark_scale:5000</code>. See <a href="#Manual_procedures">above</a> for other values of <i>scale</i>, and the corresponding initial <abbr title="Open Street Map">OSM</abbr> zoom when opened via <a href="https://iw.toolforge.org/geohack" class="extiw" title="toollabs:geohack">GeoHack</a>. An explicitly given <i>scale</i> trumps an implicit <i>scale</i> value associated with an also given <i>type</i>. <small style="font-size:85%;">TBD: list of <i>type</i> values and their corresponding <i>scale</i> defaults for commons, dewiki, and enwiki for comparison.</small></li> 
       <li><i>dim</i> - The <i><abbr title="dimension">dim</abbr></i> in meters can be used for bigger areas in {{<a href="/wiki/Template:Object_location" title="Template:Object location">object location</a>}}, e.g., <code style="font-size:smaller">type:mountain_dim:2000_scale:10000_region:PL-MA</code>. <small style="font-size:85%;">TBD: document the effect (if any) of <i>dim</i>.</small></li> 
       <li><i>region</i> - If given the region <samp class="plainlinks" style="font:small monospace,monospace;"><a rel="nofollow" class="external text" href="https://tools.ietf.org/html/rfc2119#section-1">MUST</a></samp> be an <span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Special%3APrefixIndex&amp;prefix=ISO+3166-2%3A&amp;namespace=0&amp;hideredirects=1&amp;stripprefix=1">ISO 3166-2</a></span> code, e.g., US-CA for <a href="https://en.wikipedia.org/wiki/California" class="extiw" title="en:California">California</a>. The <i>region</i> can be used for the <a href="https://iw.toolforge.org/geohack" class="extiw" title="toollabs:geohack">GeoHack</a> selection of regional map services.</li> 
       <li><i>heading</i> - Used to indicate the direction the camera was pointing at. It is given as degree values <i>0-360</i> (clockwise with north as 0) or a compass point abbreviation as defined in <a href="https://en.wikipedia.org/wiki/Boxing_the_compass" class="extiw" title="w:Boxing the compass">w:Boxing the compass</a>. (Example: <i>heading:NW</i>) Compass overlay can be displayed on Google Maps or in Google Earth to help find the correct heading, and the <a rel="nofollow" class="external text" href="http://tools.freeside.sk/geolocator/geolocator.html">GeoLocator</a> tool allows interactive picking of correct heading on a map as part of metadata authoring.</li>
      </ul> 
      <h3><span class="mw-headline" id="Precision">Precision</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Commons:%E3%82%B8%E3%82%AA%E3%82%B3%E3%83%BC%E3%83%87%E3%82%A3%E3%83%B3%E3%82%B0&amp;action=edit&amp;section=13" title="Edit section: Precision">edit</a><span class="mw-editsection-bracket">]</span></span></h3> 
      <p>At the moment the named parameter <code style="font-size:smaller">prec=1000</code> (example) in <a href="/wiki/Template:Location" title="Template:Location"><span style="font-family:monospace,monospace;">{{Location}}</span></a> or <a href="/wiki/Template:Object_location" title="Template:Object location"><span style="font-family:monospace,monospace;">{{Object location}}</span></a> is only used to truncate the displayed coordinates on the page (category, file, or gallery.) The <a href="https://iw.toolforge.org/geohack" class="extiw" title="toollabs:geohack">GeoHack</a> map service still gets the full coordinates as specified without indication of the desired precision. The effect depends on the position (see below) and on the display style (decimal <samp>ddd.nnnnnn</samp> vs. <samp>ddd°mm?ss.nnn?</samp>), e.g., <code style="font-size:smaller">prec=1000</code> removes the <samp>ss.nnn?</samp> (seconds) display everywhere on earth, while <code style="font-size:smaller">prec=100</code> only removes <samp>.nnn</samp> (decimal fractions of seconds.) </p>
      <p>The linear measurement of a longitudinal angle varies according to latitude. If we are aiming to a precision of about 0.5–1&nbsp;m, then it is appropriate to use decimal degrees with 5 decimal places. For the minute/second format, seconds with 1 decimal place will similarly give precision of about 1.5–3 meters of real earth surface. More decimal places have not sense because available maps and navigations are not more precise. </p>

      <h2><span class="mw-headline" id="Use_of_the_information">Use of the information</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Commons:%E3%82%B8%E3%82%AA%E3%82%B3%E3%83%BC%E3%83%87%E3%82%A3%E3%83%B3%E3%82%B0&amp;action=edit&amp;section=15" title="Edit section: Use of the information">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <p>Clicking on the coordinates on an image page will lead to a list of various services that can show the location. It is also possible to browse Commons images on a map: </p> 
      <ul>
       <li>Geocoded images from Commons plotted in OSM per <a class="external text" href="https://tools.wmflabs.org/wikimap/?wp=false">WikiMap tool</a> (lat/lon/zoom parameters or <a class="external text" href="https://tools.wmflabs.org/wikimap/?cat=Skyglow_in_New_York_City">whole cat</a> possible).</li> 
       <li>A mini world map with thumbnails of geocoded images can be found from <a href="//www.mediawiki.org/wiki/Toolserver:WikiMiniAtlas" class="extiw" title="tswiki:WikiMiniAtlas">WikiMiniAtlas on Commons</a>.</li> 
       <li>Some geocoded images from Commons plotted in <a class="external text" href="https://tools.wmflabs.org/wiwosm/osm-on-ol/commons-on-osm.php?zoom=1&amp;lat=0&amp;lon=0">Open Street Map</a> (images tagged with <a href="/wiki/Template:Object_location" title="Template:Object location">{{Object location}}</a> are not displayed).</li> 
       <li><s>Some geocoded images from Commons plotted in <a rel="nofollow" class="external text" href="https://maps.google.com/maps?q=http:%2F%2Ftoolserver.org%2F~para%2FGeoCommons%2FGeoCommons-simple.kml&amp;ie=UTF8&amp;om=1&amp;ll=35.46067,15.46875&amp;spn=90.926037,111.445313&amp;z=3">Google Maps</a> (images tagged with <a href="/wiki/Template:Object_location" title="Template:Object location">{{Object location}}</a> are not displayed).</s> – this service is no longer available</li> 
       <li>KML-file of some geocoded images from Commons for use in Google Earth: <a class="external text" href="https://tools.wmflabs.org/geocommons/">GeoCommons</a> (images tagged with <a href="/wiki/Template:Object_location" title="Template:Object location">{{Object location}}</a> are not displayed).</li> 
       <li>In the "gadget" tab of <a href="/wiki/Special:Preferences" title="Special:Preferences">Special:Preferences</a>, there is an option "View category on Google maps". If enabled, this adds a link "map" to all categories. This shows the location of geocoded images in the category on Google maps.</li> 
       <li>Adding <a href="/wiki/Template:Geogroup" title="Template:Geogroup"><span style="font-family:monospace,monospace;">{{Geogroup}}</span></a> to a category page adds links to view the images in the category with various maps.</li>
      </ul> 
      <p>Warnings: </p> 
      <ul>
       <li>Services displaying the tagged images (categories, articles etc.) in the map have frequent link failures. Try to click a little later.</li> 
       <li>Indexing of the database is often delayed. A new image can appear in the map almost immediately, or after several days or even several weeks or months.</li> 
       <li>The number of image-icons displayed in the linked maps is very limited, i.e. the map can display only some of the geocoded images from the displayed area. To display more images of a specific place, you can zoom in or shift to a less crowded area.</li> 
       <li>At OpenStreetmap, icons with identical or nearby coordinates cover one another; only the top one is available to be clicked.</li>
      </ul>
