      <p>Az <a href="/wiki/Commons:Administrators/hu" title="Commons:Administrators/hu">adminisztrátorok</a> képesek lapok és fájlok levédésére a lap szerkesztésének vagy átnevezésének korlátozása érdekében. Ez a védelem lehet határozatlan idej?, vagy lejárhat egy meghatározott id? után. </p>
      <p>Ezt a lehet?séget általában csak <a href="/wiki/Commons:Vandalism/hu" title="Commons:Vandalism/hu">vandalizmus</a> és <a href="https://en.wikipedia.org/wiki/hu:Wikip%C3%A9dia:Szerkeszt%C3%A9si_h%C3%A1bor%C3%BA" class="extiw" title="w:hu:Wikipédia:Szerkesztési háború">szerkesztési háborúk</a> ellen használják, és a védelem általában ideiglenes. Néhány kivétel ez alól a sok helyen használt sablonok és az archivált szerkeszt?ijog-kérelmek. </p>
      <p>A <a href="/wiki/Special:Log/protect" title="Special:Log/protect">lapvédelmi naplóban</a> látható, hogy ki, milyen lapot, milyen indoklással védett le. A jelenleg védett lapok listája a <a href="/wiki/Special:ProtectedPages" title="Special:ProtectedPages">Special:Protectedpages</a> lapon látható. A nem létez?, létrehozás ellen védett lapok listája a <a href="/wiki/Special:ProtectedTitles" title="Special:ProtectedTitles">Special:Protectedtitles</a> lapon található. </p>
      <p>A teljesen védett lapokat meg kell jelölni egy megfelel? <a href="/wiki/Category:Protection_templates" title="Category:Protection templates">védelemsablonnal</a> mások tájékoztatása és a lap <a href="/wiki/Category:Protected" title="Category:Protected">Category:Protected</a> alá való automatikus bekategorizálása érdekében. A félig védett lapok általában nincsenek felsablonozva. </p>

      <h2><span id="A_v.C3.A9delem_t.C3.ADpusai"></span><span class="mw-headline" id="A_védelem_típusai">A védelem típusai</span></h2> 
      <p><span id="Types_of_protection"></span> </p> 
      <h3><span id="Teljes_v.C3.A9delem"></span><span class="mw-headline" id="Teljes_védelem">Teljes védelem</span></h3> 
      <p>Full protection disables editing for everyone except administrators. Fully protected media files cannot be overwritten by new uploads except by administrators. Pages may be full protected <i>indefinitely</i> if they are heavily used images or frequently transcluded templates to prevent vandalism. Indefinite full protection may also be used for pages which should not be modified for copyright or legal reasons, such as <a href="/wiki/Commons:GNU_Free_Documentation_License" title="Commons:GNU Free Documentation License">Commons:GNU Free Documentation License</a>. </p>
      <p>Pages experiencing edit warring as the result of a dispute may be <i>temporarily</i> protected. During this time, administrators should only make significant changes if there is consensus to do so. Any proposed changes should be discussed on the talk page. Such requests may, if they are immediate and uncontroversial, be accompanied by the <a href="/wiki/Template:Editprotected" class="mw-redirect" title="Template:Editprotected"><span style="font-family:monospace,monospace;">{{Editprotected}}</span></a> tag to attract administrators' attention. </p>

      <h3><span id="R.C3.A9szleges_v.C3.A9delem"></span><span class="mw-headline" id="Részleges_védelem">Részleges védelem</span></h3> 
      <div class="floatright">
       <a href="/wiki/File:Semi-protection-shackle.svg" class="image"><img alt="Semi-protection-shackle.svg" src="https://upload.wikimedia.org/wikipedia/commons/thumb/1/1b/Semi-protection-shackle.svg/50px-Semi-protection-shackle.svg.png" decoding="async" width="50" height="50" srcset="https://upload.wikimedia.org/wikipedia/commons/thumb/1/1b/Semi-protection-shackle.svg/75px-Semi-protection-shackle.svg.png 1.5x, https://upload.wikimedia.org/wikipedia/commons/thumb/1/1b/Semi-protection-shackle.svg/100px-Semi-protection-shackle.svg.png 2x" data-file-width="512" data-file-height="512"></a>

      <p>Semi-protection disables editing for anonymous users and registered accounts less than four days old. This is the most common type of protection and is commonly used to prevent IP vandalism. </p> 
      <h3><span id=".C3.81tnevez.C3.A9s_elleni_v.C3.A9delem"></span><span class="mw-headline" id="Átnevezés_elleni_védelem">Átnevezés elleni védelem</span></h3> 
      <div class="floatright">
       <a href="/wiki/File:Move-protection-shackle.svg" class="image"><img alt="Move-protection-shackle.svg" src="https://upload.wikimedia.org/wikipedia/commons/thumb/4/44/Move-protection-shackle.svg/50px-Move-protection-shackle.svg.png" decoding="async" width="50" height="50" srcset="https://upload.wikimedia.org/wikipedia/commons/thumb/4/44/Move-protection-shackle.svg/75px-Move-protection-shackle.svg.png 1.5x, https://upload.wikimedia.org/wikipedia/commons/thumb/4/44/Move-protection-shackle.svg/100px-Move-protection-shackle.svg.png 2x" data-file-width="512" data-file-height="512"></a>

      <p>Move protection protects the page solely from being moved. By default, fully protected pages are also move protected. </p> 
      <h3><span id="Kaszk.C3.A1dolt_v.C3.A9delem"></span><span class="mw-headline" id="Kaszkádolt_védelem">Kaszkádolt védelem</span></h3> 
      <div class="floatright">
       <a href="/wiki/File:Cascade-protection-shackle-double-chain-link.svg" class="image"><img alt="Cascade-protection-shackle-double-chain-link.svg" src="https://upload.wikimedia.org/wikipedia/commons/thumb/8/84/Cascade-protection-shackle-double-chain-link.svg/50px-Cascade-protection-shackle-double-chain-link.svg.png" decoding="async" width="50" height="50" srcset="https://upload.wikimedia.org/wikipedia/commons/thumb/8/84/Cascade-protection-shackle-double-chain-link.svg/75px-Cascade-protection-shackle-double-chain-link.svg.png 1.5x, https://upload.wikimedia.org/wikipedia/commons/thumb/8/84/Cascade-protection-shackle-double-chain-link.svg/100px-Cascade-protection-shackle-double-chain-link.svg.png 2x" data-file-width="512" data-file-height="512"></a>
      </div> 
      <p>Cascading protection fully protects all pages <a href="https://en.wikipedia.org/wiki/en:Help:Transclusion" class="extiw" title="w:en:Help:Transclusion">transcluded</a> on the protected page, including any images and their file description page, and any templates. Due to the widespread and powerful nature of cascading protection, its usage should be kept to a minimum. On Commons, cascading protection should only be used in rare situations where there's a consensus to do so. </p>
      <p>In the past, it wasn't possible to protect nonexistent pages from being created. As a workaround, these pages <a href="/wiki/Commons:Protected_against_recreation" title="Commons:Protected against recreation">were transcluded on a page that had cascading protection</a>. It is now possible to protect pages using MediaWiki's built-in protection of nonexistent pages, which is applied in the same manner as ordinary protection. However, the old method has not been abandoned completely. </p>
      <p>Cascading semiprotection is <a href="https://phabricator.wikimedia.org/T10796" class="extiw" title="phab:T10796">disabled</a>, because it enabled non-sysops to fully protect pages by transcluding them. </p> 
      <h3><span id="Felt.C3.B6lt.C3.A9s_elleni_v.C3.A9delem"></span><span class="mw-headline" id="Feltöltés_elleni_védelem">Feltöltés elleni védelem</span></h3> 
      <div class="floatright">
       <a href="/wiki/File:Upload-protection-shackle.svg" class="image"><img alt="Upload-protection-shackle.svg" src="https://upload.wikimedia.org/wikipedia/commons/thumb/d/d7/Upload-protection-shackle.svg/50px-Upload-protection-shackle.svg.png" decoding="async" width="50" height="50" srcset="https://upload.wikimedia.org/wikipedia/commons/thumb/d/d7/Upload-protection-shackle.svg/75px-Upload-protection-shackle.svg.png 1.5x, https://upload.wikimedia.org/wikipedia/commons/thumb/d/d7/Upload-protection-shackle.svg/100px-Upload-protection-shackle.svg.png 2x" data-file-width="512" data-file-height="512"></a>
      </div> 
      <p>Upload protection might be used to prevent overwriting of files that are either heavily used across Wikimedia projects (e.g. template icons) or used in a dangerous location (e.g. wiki's main pages) in order to prevent vandalism. These protections might be indefinitely or temporary, for example only as long as a file is on a wiki's main page. For “heavily used” files <code>[upload=sysop]</code> <i>and</i> (to prevent malicious or accidental moves) <code>[move=sysop]</code> should be used. <code>[edit=sysop]</code> is not needed (and not useful since it, for example, does not allow category changes) if just the file's contents are important (e.g. if only very few views of the file page can be expected as it happens in non-linked template use). </p> 
      <h2><span id="Lapok_lev.C3.A9d.C3.A9s.C3.A9nek_.C3.A9s_a_v.C3.A9delem_felold.C3.A1s.C3.A1nak_k.C3.A9r.C3.A9se"></span><span class="mw-headline" id="Lapok_levédésének_és_a_védelem_feloldásának_kérése">Lapok levédésének és a védelem feloldásának kérése</span></h2> 
      <dl>
       <dt>
        To request protection
       </dt> 
       <dd></dd>
      </dl> 
      <ul>
       <li>Ask an administrator or make a request on <a href="/wiki/Commons:Administrators%27_noticeboard/Blocks_and_protections" title="Commons:Administrators' noticeboard/Blocks and protections">Commons:Administrators' noticeboard/Blocks and protections</a>.</li>
      </ul> 
      <dl>
       <dt>
        To request unprotection
       </dt> 
       <dd></dd>
      </dl> 
      <ul>
       <li>First, ask the administrator who protected the page.</li> 
       <li>If they are unresponsive, ask a different administrator or ask at <a href="/wiki/Commons:Administrators%27_noticeboard/Blocks_and_protections" title="Commons:Administrators' noticeboard/Blocks and protections">Commons:Administrators' noticeboard/Blocks and protections</a>.</li>
      </ul>
